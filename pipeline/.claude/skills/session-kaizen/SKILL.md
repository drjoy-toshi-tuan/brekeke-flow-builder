# session-kaizen — セッション改善サイクル（防ぐが勝ち）

> **方針**: エラーは1回だけ。2回目は無い。
> セッションのチャット内容は揮発する。**学びは repo に着地して初めて資産になる。**
> 途中は普通に作業してよい（トークン節約）。守るのは **冒頭宣言** と **末尾収穫** の2点だけ。

## 使い方

- **セッション冒頭**: `/session-kaizen` を呼ぶ → 下記マップを読み、今日のセッションが
  どのカテゴリに触れそうかを 1〜2 行で宣言する（例:「今日は P7 実機。カテゴリ 3/4 が出やすい」）。
- **セッション末（または大きな区切り）**: 「収穫チェック」を実行し、学びを分類 → 着地 → commit/PR。

## 7つの改善カテゴリと着地先マップ

| # | カテゴリ | 着地先（優先順） |
|---|---|---|
| 1 | **specs の変更** | `docs/specs/`（正本）→ `tools/spec_template/`（テンプレ）→ `schemas/qa_validator.py`（機械チェック化） |
| 2 | **実データの追加** | `tools/keyword_master.json` / `tools/normalize_dictionary.json` / 施設別 `keyword_custom.json` |
| 3 | **AmiVoice ケース追加** | `docs/amivoice/misrecognition_log.csv`（誤認識ペア）/ `docs/amivoice/base_keywords.yaml` |
| 4 | **事故・不具合ケース** | `docs/incidents/`（台帳・ID付与）→ **必ず機械ゲート化を検討**（validator ルール / verify ツール / auto_fixer） |
| 5 | **新しい設定方法** | `Sheet_Settings` キー追加（raw_to_spec）/ `docs/specs/env_*.txt` / `docs/brekeke/` ガイド |
| 6 | **UX・ロジック改善** | FAQ モード等のブロック型オプション（csv_to_yaml）/ `docs/ai/skills/SKILL_*.md` テンプレ / TTS 文言ガイド |
| 7 | **品質・顧客満足** | `modules/` 新部品（決定論置換）/ score_gate 基準 / P6・P7 テストケース拡充 |

## 学びの着地優先度（強い順）

```
① 機械ゲート（validator / verify / auto_fixer）… 人が忘れても機械が防ぐ ← 最優先
② データ（辞書 / テンプレ / 部品 spec）      … 機械が使う、人は記入だけ
③ Skill / SKILL.md                           … Claude が毎回読む手順
④ CLAUDE.md / ガイド文書                     … 最弱（参考情報）
```
**下の階層で済ませた学びは、可能な限り上の階層へ昇格させる。**
（例: 恵佑会札幌 260629 = 「気を付ける」で終わらせず `verify_test_bivr.py` の自動ゲートに昇格）

## 収穫チェック（セッション末に実行）

1. このセッションで**手で直したものは何か？** → 2回目が起きない着地先はどこか（上表で分類）
2. **ユーザーに2回以上説明した/聞いたことは何か？** → ガイド・テンプレに不足
3. **想定外の挙動・事故はあったか？** → `docs/incidents/` に ID 付きで記録 + 機械ゲート化の提案
4. **新しい実データ（キーワード・誤認識・科名・FAQ）を見たか？** → カテゴリ 2/3 へ追記
5. 提案を**ユーザーに提示して承認を得てから** commit（保護ゾーンは PR）。着地しない学びは無かったものになる

## incidents 台帳の書式

`docs/incidents/{YYMMDD}_{短い名前}.md`:
```markdown
# INC-{YYMMDD}-{連番}: タイトル
- 症状 / 影響:
- 原因:
- 対策（着地先と階層①〜④）:
- 再発防止ゲート: （無ければ「なし」と明記し、理由を書く）
```
対策コード・SKILL 側にも `INC-ID` を逆参照で書く（例: verify_test_bivr.py ⇔ INC-260629-1）。

## やらないこと
- セッション中の常時記録（トークン浪費）— 冒頭宣言と末尾収穫だけ
- チャットにだけ残る「口約束の改善」— 着地しない提案は成果と見なさない
- 保護ゾーンへの直コミット — 提案 → ユーザー承認 → PR
