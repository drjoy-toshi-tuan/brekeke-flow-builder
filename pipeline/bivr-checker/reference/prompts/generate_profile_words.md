# profile_words 生成プロンプトテンプレート

**用途**: 診療科リストを入力すると、STTモジュール用の `profile_words` を生成する。
**出力形式**: `.bivr` の `profile_words` パラメータに直接貼り付け可能なテキスト形式（`表記 よみ` の1行1エントリ）。

---

## プロンプト（LLMに貼り付けて使用）

````
以下の診療科リストをもとに、AmiVoice STTモジュール用の profile_words を生成してください。

## 入力（診療科リスト）

```
{診療科リストをここに貼る}
例:
内科
外科
整形外科
皮膚科
眼科
耳鼻科
```

## 生成ルール

### ルール1: 基本エントリ
各診療科について以下を生成する。
- `診療科名（漢字） よみがな` — 正式名称
- `省略形（漢字） よみがな` — よく使われる短縮形（例: 消化器内科 → 内科）
- `読み仮名のみ よみがな` — 頭落ち対策（認識結果が仮名のみになるケース）

### ルール2: フィラー前置きパターン（14種 × キーワード）
各診療科の主要キーワード（よみがな）に対して、以下14種のフィラーを前置きしたパターンを生成する。

標準フィラー14種:
  あ / あー / あの / あのー / え / えー / えっと / えーと / ん / んー / はい / ま / まー / そうですね

フォーマット: `診療科名 フィラー+よみがな`

例（整形外科）:
```
整形外科 あせいけいげか
整形外科 あーせいけいげか
整形外科 あのせいけいげか
整形外科 あのーせいけいげか
整形外科 えせいけいげか
整形外科 えーせいけいげか
整形外科 えっとせいけいげか
整形外科 えーとせいけいげか
整形外科 んせいけいげか
整形外科 んーせいけいげか
整形外科 はいせいけいげか
整形外科 ませいけいげか
整形外科 まーせいけいげか
整形外科 そうですねせいけいげか
```

### ルール3: 語尾バリエーション（主要パターン）
各診療科の基本キーワードに対して、以下の語尾を付加したパターンを生成する。
ただし全語尾は不要。よみがなの後に自然につながるもの5〜8種に絞ること。

主要語尾（参考）: です / で / になります / なんですが / なんです / ですけど / お願いします / をお願いします

例（整形外科）:
```
整形外科 せいけいげかです
整形外科 せいけいげかで
整形外科 せいけいげかになります
整形外科 せいけいげかなんですが
整形外科 せいけいげかをおねがいします
```

### ルール4: 頭落ちパターン
音声認識では発話の先頭1〜2音が欠落することが多い。各診療科の読みの先頭1音節・2音節を削除したパターンを追加する。

頭落ちの優先度:
- 3音節以上の科名は必須
- 「し」「ょ」「ゅ」などの拗音・促音を含む先頭は特に落ちやすい

例:
```
消化器内科 ょうかきないか    # 「しょ」→「ょ」
消化器内科 うかきないか      # 「しょう」→「う」（2音節落ち）
循環器内科 ゅんかんきないか  # 「じゅ」→「ゅ」
整形外科   えいけいげか      # 「せ」→「え」
```

### ルール5: 類義語・言い換え表現
患者が正式名でなく別の表現で言う可能性がある場合は追加する。

例:
```
皮膚科    はだのびょうき
眼科      めのびょうき
耳鼻科    はなのびょうき
耳鼻科    みみのびょうき
内科      ねつがある
内科      かぜ
小児科    こどものびょうき
```

### ルール6: 出力フォーマット
- 1行1エントリ: `表記 よみ`（半角スペース区切り）
- 診療科ごとにコメント行 `# === 科名 ===` で区切る
- フィラー・語尾・頭落ちの順番でまとめて出力する
- コメント行はbivr埋め込み時に削除する（生成時は可読性のため残してOK）

## 注意事項
- フィラーは原則として診療科の**よみがな**（仮名）に前置きすること（漢字表記には前置きしない）
- 省略形（例: 整形 せいけい）にもフィラー前置きパターンを追加すると精度が上がる
- 生成後は `profile_words_diagnosis.txt` の汎用テンプレートと照合し、重複を除いたうえで追加・上書きする
- 診療科対象外の科名（電話案内の対象外）も、まず「認識させてから振り分ける」ために辞書登録する
````

---

## 使い方

1. 上記プロンプトの `{診療科リストをここに貼る}` を実際の科名に置き換える
2. Claude / GPT-4 等に投入する
3. 出力を `reference/dictionaries/profile_words_diagnosis.txt` の `施設固有追加エリア` に追記する
4. `.bivr` 修正時、該当STTモジュールの `profile_words` パラメータに貼り付ける
