  → 各時間分岐 ──→ 冒頭アナウンス(AI受付) / 冒頭アナウンス2(代表転送) / 時間外アナウンス
  → 冒頭アナウンス → openAI_営業時間内Flag → 確認_問い合わせ内容(TTS)
  → 入力_問い合わせ内容(STT) → openAI_確認_問い合わせ内容

          → 成功 → 移動_電話番号聴取(Jump to Flow)
              → 成功 → Flag_終話 → 終話アナウンス → 終話_成功(Disconnect)
              → 失敗 → 代表転送アナウンス → 転送_代表電話
          → 失敗 → 代表転送アナウンス → 転送_代表電話
      → リトライ → リトライ_確認_問い合わせ内容
          → Retry → 入力_問い合わせ内容
          → No more → 営業時間分岐(ContextMatchRouter)
              → 営業時間内 → 代表転送アナウンス → 転送_代表電話

      → 転送失敗 → Flag_かけ直し → かけ直しアナウンス → 終話_かけ直し案内(Disconnect)

## 4. 設計上の特徴

### 4.1 独自の時間帯判定（acceptance_times を使わず Script で実装）

他のpairが `drjoy^External Integration$acceptance_times` を使うのに対し、本フローは **Scriptモジュールで完全独自実装**。

- `script_曜日分岐`: 外部祝日API（`holidays-jp.github.io`）をコールして祝日判定
- `script_平日時間分岐`・`script_土曜日時間分岐`・`script_祝日時間分岐`: 各曜日の時間帯を判定し「AI受付/代表転送/時間外」の3ルートに振り分け

> **他pairとの相違点**: acceptance_times は「受付可/不可」の2値だが、本フローは「AI受付/代表転送/時間外」の3値を曜日・時間帯で細かく制御している。

### 4.2 openAI_営業時間内Flagの設計

`冒頭アナウンス`（AI受付時間帯確定後） → `openAI_営業時間内Flag` → `確認_問い合わせ内容`

- このOpenAIモジュールは「常に`営業時間内`を出力する」固定エンジン
- 実際の時間帯判定はscriptで完結しており、このOpenAIは冗長に見える
- おそらく **コンテキストに「営業時間内フラグ」を記録する**用途（contextName: isAfterHours への書き込み）

### 4.3 電話番号の二重確認設計

電話番号確認は以下の2パスを持つ：

1. **着信番号が取得できた場合**: SSML生成（数字読み上げ）→ 音声復唱 → 肯定否定確認（DTMF+STT）
2. **手動入力が必要な場合**: Phone Normalization → Re-confirmation 復唱 → DTMF確認 → STT肯定否定

### 4.4 SSML音声生成（script_ssml）

JavaScriptで電話番号を人間が聞き取りやすい形式に変換：

- 携帯11桁: `090` + `-` + 4桁 + `-` + 4桁 形式でbreak要素付き
- 固定10桁: 先頭桁数で区切りを自動判定
- 各数字の後に50ms間隔、区切りで「の」挿入
- 読み上げ速度90%、末尾に600msブレーク

### 4.5 サブフロー構成（シンプル）

本フローは最小限の3フロー構成：
- メインフロー: 時間帯判定 + 問い合わせ聴取
- 氏名サブフロー
- 電話番号サブフロー

診療科聴取・生年月日聴取・診察券番号聴取などは**一切なし**。「問い合わせ内容 + 氏名 + 電話番号」の3つのみ聴取するシンプル設計。

## 5. profile_words の実態

### 入力_患者_氏名（STT）

**空白**。氏名はOpenAIの正規化プロンプトに委ねる設計。

### 入力_問い合わせ内容（STT）

**空白**。フリーテキストのため辞書は不要。

### 入力_患者_電話番号確認（DTMF+STT）

**超詳細な肯定否定辞書（600行超）**。他pairと比較して最も充実した辞書設定。

構成:
- 標準フィラー14種 × 「はい」「そうです」「いいえ」「違います」等の各肯定否定語
- 語尾バリエーション30種以上（ね/さ/か/が/の/に/や/でして/ですけれども等）
- 頭落ちパターン（「あうです」「がいます」「い」「あい」等）

### 入力_復唱_患者_電話番号（STT）

同じく充実した肯定否定辞書（600行超）。

## 6. OpenAI プロンプト詳細

### openAI_確認_問い合わせ内容

- **module**: 入力_問い合わせ内容
- **contextName**: inquiry
- **役割**: 問い合わせ内容を簡潔な一行テキストに要約・正規化

**主なルール**:
- STT誤認識の補正指示あり（例: 「小児科」→「仁科」誤認識のパターン）
- 個人情報（氏名・生年月日・電話番号）は除外して要約
- Few-Shot例示（5件程度）あり
- 出力: 簡潔な一行テキスト または `NO_RESULT`

**next分岐**:
- `^TIMEOUT$` → リトライ
- `^ERROR$` → リトライ
- `^NO_RESULT$` → リトライ
- `^.+$` → 移動_氏名（サブフロー）

### openAI_患者_電話番号確認

- **module**: 入力_患者_電話番号確認
- **役割**: DTMF入力または STT入力から肯定/否定を判定

**主なルール**:
- DTMF優先: 「1」→肯定、「2」→否定
- STT入力の断片からも判定（「あー」→肯定、「ちが」→否定等）
- functionCall形式でintentを出力

**next分岐**:
- `^肯定$` → script_電話番号（返却）
- `^否定$` → 患者_電話番号（再入力）
- `^NO_RESULT$` → リトライ
- `^ERROR$` / `^TIMEOUT$` → リトライ

### openAI_営業時間内Flag

- **役割**: 入力内容に関わらず常に「営業時間内」を出力（固定エンジン）
- プロンプトインジェクション防御の実装例として参考になる

### OpenAI_復唱_患者_電話番号

- **module**: 入力_復唱_患者_電話番号
- **役割**: 復唱確認時の肯定/否定判定（STT入力版）

## 8. 注目ポイント・他pairとの比較

### 本pairで初登場の設計要素

1. **Script主導の曜日・時間帯判定**: acceptance_times 非使用。外部祝日APIをScript内でコールしてJavaScriptで全て制御
2. **SSML音声生成スクリプト**: 電話番号の自然な日本語読み上げをJavaScriptで実装（break/speed制御）
3. **シンプル3聴取設計**: 問い合わせ内容・氏名・電話番号のみ。予約詳細聴取なし（問い合わせ内容はフリーテキストで受けてオペレーターが折り返す設計）
4. **時間帯によるAI/スタッフ切替**: 同じ電話番号でも時間帯でAI電話とスタッフ対応を切り替える設計

### 設計上の注意点

1. **Script禁止パターンに近い実装**: `$runner.getModuleResult()` 等を多用 → CLAUDE.md Section 4「Script禁止事項」参照。実際に動作しているので施設固有の例外として蓄積
2. **acceptance_times 未使用**: 独自Script実装のため `FLOW-002`（acceptance_times なし）に近い状態だが、意図的な代替実装
3. **時間外でも問い合わせ聴取**: 時間外アナウンス後に `確認_問い合わせ内容` に遷移する設計（時間外でも折り返し用に情報を収集する意図）
