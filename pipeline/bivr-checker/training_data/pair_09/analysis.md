### 4.3 注目すべき実装パターン

1. **ContextMatchRouter の多用**: 用件分岐（1=初診,2=再診,3=変更,4=キャンセル,5=確認）、紹介状分岐（1=無し→自己負担,2=有り→診療科）、用件分岐後の共通処理分岐、終話分岐をContextMatchRouterで制御
2. **Custom Jump to Flow**: 氏名・生年月日・電話番号・用件はサブフローとしてCustom Jump to Flowで呼び出し。戻り先は次のノードに自動遷移
3. **completeflag_SMSflag**: 各終話前にsaveCompletionFlag2dbで完了フラグ＋SMS送信フラグを保存（1=代表案内, 2=初診, 3=診療科不明, 4=再診, 5=変更, 6=キャンセル, 7=確認）
4. **リトライパターン**: 全STT入力にSpeech Retry Counter（通常3回）を配置。失敗時はフォールバック先に遷移
5. **openAI判定**: ほぼ全ての分岐判定にopenAI generate_by_OpenAIを使用。YES/NO判定、意図分類、日付正規化、診療科抽出など多目的

## 5. フロー全体構造サマリ

```
[冒頭待ち時間] → [コンテキスト設定] → [冒頭TTS]
    ↓
[当日確認TTS] → [STT] → [openAI_当日確認]
    ├── 対象(当日) → [終話_代表案内] → [completeflag_1] → [Disconnect]
    └── 対象外(明日以降) → [用件サブフロー]
                              ↓
    ┌── 診察予約 → [確認_受診歴] → openAI判定
    │     ├── 初診 → script(context=1) → メイン戻り
    │     └── 再診 → script(context=2) → メイン戻り
    ├── 予約変更   → script(context=3) → メイン戻り
    ├── 予約キャンセル → script(context=4) → メイン戻り
    └── 予約確認   → script(context=5) → メイン戻り
                              ↓
[用件(Custom Jump)] ← 用件別分岐
    ├── 初診予約 → [紹介状有無] → [復唱] → [紹介状分岐]
```
