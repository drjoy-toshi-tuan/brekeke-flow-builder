# profile_words 生成エージェント

## 役割

`scripts/pw_analyzer.py` の出力（分析結果JSON）を受け取り、各STT/DTMFモジュールの `params.profile_words` を生成・調整する。

---

## 処理フロー

```
1. pw_analyzer.py の出力JSONを読み込む
2. status が "empty" または "insufficient" のモジュールを対象にする
3. status が "excessive" のモジュールも対象にする（語数を100-200語に調整）
4. 各モジュールについて:
   a. input_type に応じた参照辞書を読み込む（存在すれば）
   b. preceding_tts からベースキーワードを抽出
   c. 展開ルールに従って profile_words を生成
   d. 目標語数（100-200語）に収まるよう調整
   e. フローJSON の params.profile_words を更新
5. 修正済みJSONを保存
```

---

## 出力フォーマット

```
単語 よみ
単語 よみ
...
```

- 区切り: 半角スペース1つ（タブ不可、スペース2つ以上不可）
- よみ: 全てひらがな（長音「ー」は許容）
- 左側の「単語」は正規化された同一文字列に統一（同じ意味の語は同じ表記にする）
- 同じよみで別の単語は登録しない
- **数字は必ず半角（1, 2, 3 ...）で記述する。漢数字（一, 二, 三）や全角数字（１, ２, ３）は使用禁止。** AmiVoiceは半角数字のみ正しく認識する

---

## 生成戦略: 「制御された展開」

教師データ11ペア（pair_01〜pair_11）の横断分析に基づく。

### フィラーTOP6（教師データ使用頻度順）

```
あ / え / えー / あの / はい / ま
```

> **「まー」は使わない。** 教師データ14種のフィラー中、唯一の未使用フィラー。

### 語尾TOP8（教師データ使用頻度順）

```
です / で / なんですが / になります / ね / さ / でして / か
```

### 頭切れパターン

先頭1〜2モーラが欠落した形を登録する。

- 2モーラ以上のキーワードが対象
- 頭切れの結果1文字のみになる場合はスキップ
- 教師データの87%のモジュールで使用されている必須パターン

例:
| 本来の発話 | 頭切れ1モーラ | 頭切れ2モーラ |
|---|---|---|
| よやく | やく | く（1文字→スキップ） |
| へんこう | んこう | こう |
| しょうわ | ょうわ | うわ |
| きゃんせる | ゃんせる | んせる |

### 展開ルール（1キーワードあたり）

```
1. 基本形: キーワードそのまま → 1語
2. フィラー+キーワード: 6種 → 6語
3. キーワード+語尾: 8種 → 8語
4. 頭切れ: 先頭1-2モーラ欠落 → 2-3語
合計: 約17-18語/キーワード
```

### 目標語数

**100〜200語/モジュール**（教師データ中央値138語に準拠）

- 100語未満: insufficient（キーワードまたは展開パターンを追加）
- 200語超過: excessive（低頻度の語尾パターンを削除して調整）
- freetext は例外で50語以下でもOK

## 入力種別別 生成ガイドライン

### 1. 用件分類（yoken）

**ベースキーワード**: 選択肢名（予約/変更/キャンセル/確認/その他）+ 各選択肢の言い回し

言い回し例:
- 予約 → 予約したい/受診したい/取りたい/お願いしたい/診てほしい
- 変更 → 予約変更/日程変更/日にちを変えたい/ずらしたい
- キャンセル → 取消/やめたい/行けなくなった
- 確認 → 聞きたい/わからない

**展開**: フィラーTOP6 + 語尾TOP8 + 頭切れ（全パターン適用）

### 2. 肯定否定（yes_no）

**参照辞書**: `reference/dictionaries/profile_words_yes_no.txt`（存在する場合はベースとして読み込む）

**肯定系キーワード**:
```
はい / ええ / うん / そうです / あってます / 大丈夫 / お願いします / よろしいです
```

**否定系キーワード**:
```
いいえ / いや / 違います / 違う / 間違い
```

**頭切れ必須パターン**（教師データで確認済み、特に重要）:
```
はい → あい / い
そうです → おうです / うです
いいえ → いえ
違います → がいます
大丈夫です → いじょうぶです
お願いします → ねがいします
あってます → ってます
```

### 3. 生年月日（dob）

**参照辞書**: `reference/dictionaries/profile_words_dob.txt`（存在する場合はベースとして読み込む）

**元号（必須）**:
```
令和 れいわ
平成 へいせい
昭和 しょうわ
大正 たいしょう
```

**元号の頭切れ・誤認識パターン（必須）**:
```
昭和 きょうわ
昭和 きょうは
令和 えいわ
平成 えいせい
```

**月名**: 一月〜十二月の読みを個別登録（いちがつ/にがつ/.../じゅうにがつ）

**日付特殊読み（必須）**:
```
一日 ついたち
二日 ふつか
三日 みっか
四日 よっか
八日 ようか
十日 とおか
二十日 はつか
```

**数字読み**: いち〜じゅう + にじゅう/さんじゅう の各読み

### 4. 氏名（name）

**参照辞書**: `reference/dictionaries/profile_words_name.txt`（存在する場合はベースとして読み込む）

**フィラー率: 15%に制限**（名前を言う時にフィラーは少ない）
- フィラー6種のうち2種程度に絞る（「あ」「え」のみ）

**フレーズ系**:
```
私は わたしは
名前は なまえは
〜と申します ともうします
〜です です
```

**注意**: 頻出苗字はOpenAI正規化に委ねるため辞書登録は最小限。施設所在地の地域特有の苗字があれば追加する程度。

### 5. 診療科（department）

**情報源**: PDFまたはフローの `saveContextModel2DB` の `rangeValues` から診療科名を取得

**各科名について登録するもの**:
- 正式名称（消化器外科 しょうかきげか）
- 省略形（消化器 しょうかき / 外科 げか）
- 類義語があれば追加

**頭切れ必須**:
```
消化器外科 → ょうかきげか
整形外科 → えいけいげか
泌尿器科 → にょうきか
耳鼻咽喉科 → びいんこうか
```

**フィラー + 語尾**: 標準の展開ルールを適用

### 6. 電話番号（phone）

**DTMFがメインのため辞書は最小限（50語以下でもOK）**

数字読みのみ:
```
一 いち / 二 に / 三 さん / 四 よん / 四 し / 五 ご / 六 ろく / 七 なな / 七 しち / 八 はち / 九 きゅう / 九 く / 零 ぜろ / 零 れい
```

### 7. 健診コース（kenshin_course）

**情報源**: PDFまたはフローの情報からコース名を取得

**各コース名について登録するもの**:
- 正式名称（人間ドック にんげんどっく）
- 略称（ドック どっく）
- 通称（協会けんぽ きょうかいけんぽ → けんぽ けんぽ）

**展開**: フィラーTOP6 + 語尾TOP8 + 頭切れ（全パターン適用）

**注意**: VFBは健診コース名を一切生成しないため、必ず手動で全コース名を登録する。

### 8. 日付（date）

**生年月日辞書と共通部分あり**: 元号 + 月名 + 日付特殊読み

**追加で登録する相対日付**:
```
来週 らいしゅう
再来週 さらいしゅう
来月 らいげつ
明日 あした / あす
明後日 あさって
今週 こんしゅう
今月 こんげつ
```

**曜日**:
```
月曜日 げつようび / 火曜日 かようび / 水曜日 すいようび / 木曜日 もくようび / 金曜日 きんようび / 土曜日 どようび / 日曜日 にちようび
```

### 9. 自由テキスト（freetext）

**最小限の辞書（50語以下でもOK）**

- ドメイン固有語（施設名、専門用語）があれば追加
- 汎用フィラーとよく使われる接続詞程度でよい
- 無理に100語まで膨らませない

## 品質チェックリスト（生成後セルフチェック）

生成した profile_words は、保存前に以下7項目を全てチェックする。1つでも不合格なら修正してから保存する。

1. **区切り文字**: 各行が「単語（半角スペース1つ）よみ」の形式か
2. **よみの文字種**: よみが全てひらがな（+ 長音「ー」）か。カタカナ・漢字・英字が混入していないか
3. **単語の統一**: 同じ意味の語の左側（単語）が統一されているか（例: 「予約 よやく」と「予約 あよやく」で左側が同じ「予約」）
4. **語数範囲**: 合計100〜200語の範囲か（freetext/phone は50語以下でも可）
5. **フィラーTOP6**: 「あ/え/えー/あの/はい/ま」が含まれるか（freetext/phone以外）
6. **頭切れ**: 2モーラ以上のキーワードに頭切れパターンが含まれるか
7. **「まー」不在**: 「まー」がフィラーとして含まれていないか（教師データで未使用）
