### 移管方針
- Gen1「箱」→ Gen3 9 ブロック型へ変換。opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「お電話ありがとうございます。井上病院の発熱外来、AI電話です」。
- **用件分岐なし**（全件＝発熱外来当日予約一本道）。よって用件 hearing / OpenAI 用件分類は不要。
- 主流れ: opening → API_DRJOY → 入電時間判別（平日 9〜18 内のみ通過、その外は時間外で切断）→ GET_EMPTY_FRAME → CHECK_EMPTY_FRAME（HAS_EMPTY_FRAME 数値で在庫判定）→ 注意事項 TTS → 受診歴有無（高校生未満ゲート + 本人確認 + 当院初診確認）→ 診察券番号（受診歴ありルート）→ 患者氏名 → 生年月日 → 性別確認 → 連絡先 → 完了フラグ → BOOKING_AI → 終話。
- **空き枠在庫 Script ブロック が必須**: GET_EMPTY_FRAME（在庫取得）/ CHECK_EMPTY_FRAME（数値判定）/ NOT_EMPTY_FRAME（枠なしで切断）/ BOOKING_AI（最終登録）の 4 つは Dr.JOY 連携 Script として `script` ブロックで実装（generate_by_OpenAI 不要）。CHECK_EMPTY_FRAME は HAS_EMPTY_FRAME context の 数値=0 で「予約枠終了」終話、>0 で正規ルート。
- **System overload 救済**: `BOOKING_AI` 失敗時の「予約を承ることができませんでした。再度電話を掛けなおしてください」リカバリ termination を 1 本用意（main.yaml の System overload 箱）。
- 入電時間判別.yaml は date_of_call_classifier ブロック（addCurrentDate: Yes 自動付与対象）。受付時間外時は「本日の受付時間は終了いたしました。受付時間は、平日10:00〜12:00となります」TTS で切断。

### 施設固有の特殊ルール
- **受付時間**: **平日 10:00〜12:00**（午前のみ短時間枠）。元 yaml の「(平日)時間の分岐」は 9 時前〜18 時で組まれているが、案内 TTS と挨拶冒頭は **10:00〜12:00** が正。director は元 yaml の時刻条件ではなく **「電話切断へ」TTS 文言** を時間表記の出典とする。
- **代表電話案内 TTS**: 「お電話が遠いようで、回答が確認できませんでした。恐れ入りますが、代表電話へご連絡下さい」（具体番号案内なし、「代表電話」のみ）。聞き取れず終話 4 種（患者氏名 / 生年月日 / 診察券番号 / 連絡先番号）すべてここに合流。
- **受診歴有無サブフロー（高校生未満ゲート）**: 冒頭で「本窓口は、高校生未満の場合、ご予約を承ることができません。診察を希望される方が高校生未満の場合はお電話をお切りください。もう一度お聞きしたい方は、『もう一度』とお話ください」アナウンス→「もう一度」発話で再生。**判定 hearing なし、案内のみ**で次へ進む。
- **本人確認 hearing**: 「診察を受けるご本人様からのお電話ですか？」Yes/No 受付（双方とも当院受診確認に進むため、PatientName subject 切替の伏線情報のみ）。続いて「当院のご受診は、初めてですか？」Yes（初診）→ END_戻る_1 / No（受診歴あり）→ END_戻る で main 側 CMR 分岐。**初診なら診察券スキップで氏名→生年月日へ、受診歴ありなら診察券→次へ→氏名→生年月日**。
- **性別確認サブフロー**: 連絡先聞取の **直前** に「性別をお伺いいたします。男性、もしくは女性でお答えください」hearing。発熱外来特有（他フローには無い）。エンティティ「性別」(男性/女性 + STT 救済語多数) で生命/オンナ/オス/メス/弾性/助成/異常部 等の音響的曖昧語を OpenAI 正規化辞書で吸収。
- **注意事項 TTS**: CHECK_EMPTY_FRAME 通過後・受診歴有無 hearing 前に「このお電話は、途中で電話を切ると予約確定できませんのでご注意ください」を必ず再生（announcement ブロック 1 つで構成）。
- **完了挨拶**: 「ご予約の申し込みを受付ました。予約についてのご連絡を、ショートメッセージ、またはお電話にてご連絡いたします。それでは、失礼いたします」（折り返し電話/SMS の二段案内、携帯/固定で文言分岐させない）。
- **DTMF なし**: 全 hearing 発話のみ（数字入力ボタン操作は無し）。

### director が見落としやすいポイント
- 冒頭 6 箱（API_DRJOY_CONTEXT_SETTINGS / 挨拶 / API_DRJOY / 入電時間判断へ / 入電時間判別 / GET_EMPTY_FRAME）は Gen3 で opening + date_of_call_classifier + 在庫 Script ブロックに整理。**接続詞_NOOP 箱（入電時間判断へ）はセリフ none 振り分けのみなので独立ブロック不要、前段の next 書き換えで吸収**。
- セリフ none 振り分けのみの「箱」（次へ / 次へ_1 / 次に / 連絡先聴取へ / 完了フラグ / 診察券有り / 診察券無し / 各種チェックポイント 5 種 / API_DRJOY 系 2 種）は独立ブロック不要 → 前段の next 書き換えに吸収。「次へ」「次へ_1」「次に」「連絡先聴取へ」の TTS は次段サブフローの冒頭 announcement にマージ。
- 「次へ」「次へ_1」「次に」「連絡先聴取へ」の TTS（「それでは、」「続いて、」「次に」「ありがとうございます。最後に、」）は **直後のサブフローの冒頭 hearing TTS にマージ**して短縮。独立 announcement ブロック化禁止。
- **用件 hearing 不要**: 発熱外来単独ダイヤルなので用件分類段階を作らない。湘南藤沢/八尾/福岡徳洲会型の用件 4〜5 分岐構造をコピペで持ち込まないこと。
- **PatientName 1 枠**: 受診者本人 = 入電者前提（本人確認 hearing で Yes/No 取るが分岐先は同じ）→ subject="patient" 1 枠で完結。受診者氏名/入電者氏名の二段聴取は無し。
- **HAS_EMPTY_FRAME context 命名**: scaffold が context_match_router で参照する場合は `^0$` を「枠なし」、`^[1-9][0-9]*$` または other を「枠あり」として `4 層責任モデル CMR ^0$ other` 形式で記述。
- 連絡先聞取り.yaml の電話番号区切り `市外局番5桁/4桁/3桁/2桁` は Gen3 phonetype subflow で吸収 → ノート不要。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**（丸数字 ①②③ や全角括弧禁止）。`連絡先番号聞き取れず チェックポイント` 等の全角スペース込み命名は ASCII 化（例: `END_phone_failed`）。
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**。CMR conditions 末尾は必ず `match: "other"` + next 明示。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。Gen1 動的タグ `<? $... ?>` → Gen3 `<% var %>` に変換（本フローでは未使用）。
