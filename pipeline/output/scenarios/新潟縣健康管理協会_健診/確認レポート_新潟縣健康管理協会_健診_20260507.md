### 抽出した要件サマリー

- **施設名固定の冒頭アナウンス**: 「お電話ありがとうございます。新潟縣健康管理協会の、予約専用、AI電話です。」3 施設用にあった東新潟/西新潟/長岡の DTMF 施設選択は **削除**（受診票確認後に登録番号へ直行）
- **用件 5 分岐**: 健診予約 / 日時変更 / キャンセル / 企業問合せ / 問い合わせ。echo_back（用件の復唱）あり
- **健診予約ルート**: 受診票確認 → ありの場合は登録番号 → 受診希望日 → 希望時間 → 氏名/生年月日/電話番号 → 終話。なしの場合は WEB 予約案内終話
- **日時変更ルート**: 予約日 → 受診希望日 → 希望時間 → 氏名/生年月日/電話番号 → 終話
- **キャンセルルート**: 予約日 → 氏名/生年月日/電話番号 → キャンセル受付終話（時間判定スキップ）
- **企業問合せルート**: 内容 → 企業名・担当者名 → 企業連絡先電話番号 → 終話
- **問い合わせ（その他）ルート**: 内容 → 氏名/生年月日/電話番号 → 終話
- **終話の時間判定**: 聴取完了後に `date_of_call_classifier` を再実行し、営業時間内 (「内容を確認の上、担当者よりご連絡いたします」) / 営業時間外 (「当施設の営業日に担当者より、ご連絡いたします」) でアナウンス文言を切替（キャンセル除く）
- **smsFlag 割当**: 健診予約・日時変更 = `1` (SMS 送信あり) / それ以外 = `-1`（CLAUDE.md 状態フラグ規則に準拠）
- **折り返し確認サブフロー**: Gen1 では 12 経路の中継箱が存在するが、Gen3 では各 hearing の retry_failure を `end_failure` に統一（scaffold が `END_聴取失敗` を自動配線）。明示的な折り返し確認 hearing は NON-BLOCKER N-2 として記録
- **当日予約 10:30 締切ロジック**: Gen1 受診希望日聞き取り内に存在。Gen3 では `script` ブロック（`script_template: custom`）として配置可能だが、本設計書では BLOCKER 化せず NON-BLOCKER N-1 として記載
- **AmiVoice 辞書**: entity_classification の R1〜R3 ルールから抽出（用件 79 surface / 受診票 持っている・持ってない / 折返し電話 / 肯定否定）
