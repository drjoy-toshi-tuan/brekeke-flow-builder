### 移管方針
- 既存ノート `gen1_新潟県けんこう財団_健診.md`（3施設用）の **設計をベースに、新潟縣固有差分のみ**当ノートに記述。共通部分（用件 5 分岐、SMS 送信判定、折り返し確認、当日予約 10:30 締切、終話パターン、コンテキストフィールド、DTMF 化方針、AmiVoice 例外）は 3施設用ノート参照
- 冒頭文言を「新潟縣健康管理協会の、予約専用、AI電話です。」固定に置換。施設選択 DTMF（東新潟/西新潟/長岡）は **削除**
- flow_dir 名は launch_parallel regex 制約から「新潟縣健康管理協会_健診」を採用（漢字「縣」は許容範囲。半角スペース・丸括弧などのみ禁止）
- 主流れ: opening → API_DRJOY → 入電曜日時間判断（acceptance_times）→ お盆連休判定 → 用件聴取（通常 or 連休中）→ 用件分岐 → 各ルート → 氏名 / 生年月日 / 連絡先 → 完了フラグ → 入電曜日時間判断（再判定）→ 営業時間内/外 別の完了アナウンス → 終話
- 完了フラグ後に**もう一度 acceptance_times を通す**理由は、聴取中に営業時間が切り替わった場合の終話文言切替（営業時間内 → 「内容を確認の上、担当者よりご連絡」 / 営業時間外 → 「当施設の営業日に担当者より、ご連絡」）

### 施設固有の特殊ルール
- **施設名 = 「新潟縣健康管理協会」固定**: 3施設用が施設選択 DTMF で 3 つに分岐するのに対し、新潟縣用は施設選択ステップ自体が削除されている。施設別 holidays / 施設別代表番号の分岐は不要
- **GW ハードコード無し**: 長岡用の `GWアナウンス` `5/1時間分岐` `通常営業へ` チェーンに相当する箱が `挨拶（連休中）` 1 つのみ。GW については通常 acceptance_times 判定 + holidays 配列に集約（director デフォルト）
- **お盆連休のみハードコード (`挨拶（連休中）` の「8月16日土曜日以降にご連絡ください」)**: 復帰日 8/16 をハードコード。Gen3 では holidays 配列に集約、復帰日アナウンスは director デフォルトに含めない（休日アナウンスのみ）
- **連休中専用 完了 TTS が 4 種類**: `連休中：健診予約` `連休中：予約変更` `連休中：健診予約_1` `連休中：予約変更_1` の 4 箱が同一文言「予約の確定は、当施設の営業日に担当者より、ご連絡いたします」→ Gen3 では 1 本に圧縮可能
- **時間分岐ハードコード閾値**: 営業時間内/外の判定は **入電曜日時間判断.yaml に閉じている**（main.yaml に `時間で分岐` 箱が無い、長岡用との差分）→ acceptance_times の判定結果のみで分岐し、開始/終了時刻の閾値は yaml 内部から director が抽出する
- **代表電話番号は customer_doc 確認**: main.yaml には電話番号 TTS が一切無く、SMS / 折り返しのみで完結。代表番号アナウンスが必要かは director が customer_doc から判断（無ければ termination は番号案内なしで OK）
- **「(保留)折返し方法確認.yaml」は dead code**: main.yaml `（保留）折返し方法確認` 箱が存在するが `SMS送信へ` 直結で実質 SMS のみ → Gen3 では削除
- **用件聞き取り（連休中）.yaml と通常版**: 連休中は「予約・変更・キャンセル・問い合わせ・企業問合せの 5 つから」連休中限定文言の可能性 → director が yaml の中身確認して prompt 切替（or 1 本に統合）
- **折り返し確認 NG 中継 12 系統**: 希望日 / 氏名 / 電話番号 / 予約日 / 内容 / 用件 / 希望時間 / 生年月日 / 受診票有無 / 登録番号 / 折返し方法 / **担当名** の 12 経路。長岡用との差分は「担当名NG」が追加されている = 企業ルートで企業担当者名聴取失敗時の経路。Gen3 では全て折り返し確認 1 本に集約

### director が見落としやすいポイント
- 3施設用にあった `施設選択（東新潟/西新潟/長岡）` subflow は **新潟縣用には存在しない** → 受診票確認 → 登録番号 → 受診希望日 へストレートに進む
- `分岐：連休中` `分岐：連休中_1` `分岐：連休中_2` `分岐：連休中_3` の 4 箱は **時間内/外 × 健診/変更** の 2x2 マトリクスで連休中フラグを立てる中継 → Gen3 では context に `holiday_flag=true` を持って単一 termination 経由
- `用件の分岐_1` `用件の分岐_2` の 2 段分岐: _1 は受診希望日聴取後、_2 は予約日聴取後（日時変更→受診希望日、企業問合せ→企業担当者名、デフォルト→氏名）。長岡用と完全同一（CMR で吸収）
- `折り返し確認へ(*NG)` 12 箱は **すべて `折り返し確認` 直結**で実質 1 本に集約可能（director デフォルト）
- 完了フラグは **聴取完了後の acceptance_times 再判定** + 営業時間内/外で完了 TTS を切り替える起点 → Gen3 では date_of_call_classifier 前段 OpenAI（addCurrentDate: Yes scaffold 自動付与）を 1 段
- `企業：連絡先電話番号聞き取り.yaml` は企業問合せルートのみ通る → 個人 4 ルートでは通常の連絡先聴取 subflow を使用、混在しない
- 漢字「縣」は施設正式名 → flow_dir / customer_doc / TTS では保持。step 名 / module 名は ASCII または「縣」を含む安全な日本語で揃える（scaffold が `jump_xxx` rename しても元 step 名は表示用に保持される）
- Gen1 動的タグ `<? $予約日の読み ?>` 等は echo_back に集約済み → `<% var %>` 構文への置換チェック
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字禁止
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**
- **CMR conditions は必ず最後に `match: "other"` + next 明示**
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
