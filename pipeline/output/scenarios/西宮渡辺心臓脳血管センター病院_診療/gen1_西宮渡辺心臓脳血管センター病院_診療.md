### 移管方針
- Gen1「箱」構造 → Gen3 9 ブロック型に変換。`挨拶` → opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「お電話ありがとうございます。西宮渡辺心臓脳・血管センターの、予約専用 AI 電話です」。
- `時間外チェック` サブシナリオ（営業前チェック → 祝日チェック → 平日/土曜時間分岐）→ date_of_call_classifier ブロック（addCurrentDate: Yes 自動付与対象）。`営業前` 状態（2/3 8:30 〜運用開始）専用文言は date_of_call_classifier の特殊ケースで吸収。受付時間外は `時間外案内`「翌営業日に折り返し電話、もしくは SMS にてご連絡」案内 → 用件聞き取りに合流（時間内と同じルート、終話メッセージで時間外を区別）。
- 用件は **5 分類**: `予約変更` `予約キャンセル` `確認` `再診` `新規`。Pattern A（用件 OpenAI 直後分岐 + 復唱、4 復唱）。
- 用件確定後の主分岐（メイン `用件を判断`）:
  - 再診 / 予約変更 / 予約キャンセル / 確認 → `新規予約・予約変更・キャンセル・定期受診・確認へ` → 診療科聞き取り
  - 新規 → 紹介状の有無確認サブフロー → 診療科聞き取り（**新規のみ紹介状ルート**）
- 診療科確定後 → `用件の確認` で 4 用件を再判定:
  - 予約変更/予約キャンセル → `次へ`（「では次に、」） → 予約日の聞き取り → 内容確認 → キャンセルチェック → 変更キャンセル分岐 → 用件の確認_1
  - 確認 → `確認`（「では次に確認されたい内容を」）→ 内容確認
  - 新規（チェック）→ `次へ_1`（「それでは」）→ 患者氏名聞き取り
  - 再診 → `続いて`（「続いて」）→ 診察券番号聞き取り → 患者氏名聞き取り
- 個人情報聴取（共通）: 患者氏名 → 生年月日 → 連絡先携帯聞き取り → 完了フラグ → 用件を判断_1（用件別終話）。
- 終話: 用件 × 携帯有無で **5 分岐**（再診/新規=「予約の申し込みを受付いたしました」+ 携帯あり/なし、予約変更=「予約の変更を受付いたしました」+ 携帯あり/なし、確認=「予約の確認を受付いたしました」+ 携帯あり/なし、予約キャンセル=「ご予約のキャンセルを承りました」+ 携帯あり/なし、それぞれ時間外時は専用文言）。Gen3 では context_match_router で集約。

### 施設固有の特殊ルール
- **代表電話 0798-36-1880**: 救急案内 / 聞き取れず終話 で「0798-36-1880 へご連絡ください」を **3 回繰り返し**（聞き取れず終話）+ **2 回繰り返し**（救急案内）+ **3 回繰り返し**（未対応終話・当日予約は有人）。
- **当日予約は AI 不可**: `当日予約は有人`「当日のご予約に関しては、こちらの AI 電話では受付できません」案内で代表電話に誘導 → `当日チェックNG` → 終話。**ただしどの分岐から到達するかが yaml 上で明示されておらず**、director は元資料 yaml を熟読すること（西宮渡辺診療には「タイトル」箱があるが本フロー main.yaml には無い → 予約日聞き取り or 用件聞き取りサブフロー側で当日判定する想定）。
- **緊急/救急**: 診療科聞き取りサブフロー内で `緊急救急` エンティティ（救急 / 緊急）→ メインの `救急案内`「大変申し訳ございません。この電話では受付しておりません」案内 → 代表電話誘導 → 終話。
- **AI 非対応診療科**: フットケア外来 / 外来リハビリ / 心血管センター / 健診 / 予防接種 / セカンドオピニオン外来 → `未対応終話`「電話番号が分かる方は、このままお電話をお切りください。代表電話 0798-36-1880」3 回繰り返し → 終話。
- **わからないエンティティ**: 「分からない」発話 → 診療科サブフロー内 `わからないセット` → 通常ルート（後で担当者から聞き取り）。緊急救急/未対応とは別扱い。
- **2/3 8:30 〜運用開始**: `営業前`「2/3　8:30 〜利用開始となります」 → END_切断_1（切断待ち_1）。**運用開始日の暫定文言**なので、Gen3 では基本除外し、director は date_of_call_classifier のスケジュール設定で吸収（または時限的アナウンスとして patch_box で管理）。
- **携帯有無による SMS 終話文言**: `携帯電話番号チェック` → 携帯あり → 時間外チェック → SMS あり/なし、携帯なし → 時間外チェック_1 → SMS なし/なし_1。**携帯あり/なし × 時間内/時間外** の 4 通りで終話文言を出し分け。
- **3 回失敗時**: 用件 / 診療科 / 紹介状 / 氏名 各サブで `count 聞き直し` → 3 回不可 → `電話切断へ` → `聞き取れず終話` → 「西宮渡辺心臓脳・血管センター 代表番号までおかけなおしください」案内（3 回繰り返し）→ 終話。

### director が見落としやすいポイント
- 最初の `API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `API_DRJOY` → `テスト用` → `時間外チェック_2`（サブシナリオ呼び出し）→ `受付時間内` → `用件聞き取り` の 7 箱は Gen3 では opening 冒頭処理 + date_of_call_classifier に吸収。`API_DRJOY_CONTEXT_SETTINGS` `API_DRJOY` は明示ブロック化不要。
- 「セリフ: none」かつ振り分けのみの「箱」（`次へ` `次へ_1` `新規チェック` `キャンセルチェック` `携帯電話番号チェック` `時間外チェック / _1` `用件の確認 / _1` `用件を判断 / _1` `診察券有り` `完了フラグ` `当日チェックNG` `電話切断へ` `変更_1` `キャンセル_1` `変更キャンセル確認できず` `用件聞き取れず` `氏名聞き取れず` `紹介状有無聞き取れず` `診療科聞き取れず` `生年月日聞き取り_1` `テストルート` `時間外案内しないテスト` 等）→ Gen3 で独立ブロック不要、前段 next 書き換え or context_match_router 吸収。
- `次へ`（「では次に、」）/ `次へ_1`（「それでは」）/ `続いて`（「続いて」）/ `生年月日聞き取り_1`（「続いて」）/ `連絡先電話番号聞き取りへ`（「最後に、ご連絡先のお電話番号をお伺いします」）/ `変更`（「では次に変更の理由を」）/ `キャンセル`（「では次にキャンセルの理由を」）/ `確認`（「では次に確認されたい内容を」）/ `新規予約・予約変更・キャンセル・定期受診・確認へ`（「それでは、」）等の **短い繋ぎ TTS は次段 hearing/subflow ブロックの導入文言にマージ**するか announcement で個別配置するかは director 判断。
- 用件 OpenAI 出力 enum: `予約変更 / 予約キャンセル / 確認 / 再診 / 新規` の 5 値（西宮渡辺診療の 4 値 + 通院確認とは構造が違う）。
- `用件の確認_1` は **予約変更/予約キャンセル のみ通る再判定**（変更キャンセル分岐サブフローからの戻り値判定）→ Gen3 では context_match_router で吸収。
- 紹介状の有無確認サブフローは **「新規」用件のみ通る**（`用件を判断` で `新規 → 紹介状の有無確認 → 新規予約・予約変更・キャンセル・定期受診・確認へ`）→ Gen3 では subflow を「新規」用件でのみ呼び出し。
- 内容確認（変更/キャンセル/確認）は同一サブフロー `内容確認.yaml` を 3 用件で共有 → Gen3 では用件 context を渡しつつ subflow 1 個で処理。
- `テスト用` `テストルート` `時間外案内しないテスト` `TEST通話用（時間外無視）` 系箱は **削除**（テスト入口）。
- `予約日の聞き取り.yaml` は和暦判定（大正/昭和/平成/令和）+ DTMF 4 桁を含む可能性あり。Gen3 では generate_by_OpenAI の output_format=datetime + DTMF 並走で吸収。和暦は **生年月日聞き取りサブフロー側のみで使う**点に注意（director 混同しやすい）。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字（①②③⑩等）禁止。
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**（scaffold 付与 prefix）を使う。
- **CMR conditions は必ず最後に `match: "other"` + next 明示**。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。Gen1 動的タグ `<? $var ?>` → `<% var %>` に変換。
- 患者氏名は PatientName サブフロー 1 枠（patient 1 名のみ、入電者=受診者前提）。
- entity_classification.md / entity_stt_patch.json / entity_openai_synonyms.md にエンティティは自動分類済み。director は転記不要、scaffold/properties が自動取り込み。
