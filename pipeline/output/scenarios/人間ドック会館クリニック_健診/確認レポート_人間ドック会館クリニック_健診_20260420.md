- **作業種別**: Gen2→Gen3 移管

- **シナリオ名**: 健診

### 抽出した要件サマリー

- **冒頭**: 非通知拒否（status=2/smsFlag=0）→ catch-all で通常冒頭アナウンスへ。受付時間分岐なし（source 記載なし）
- **用件聴取（classification, Pattern A + 復唱あり）**: enum=予約/変更/キャンセル/問い合わせ。DTMF+音声入力（固定 DTMF digit=1 は Gen3 で DTMF_AmiVoice モジュールに代替）
- **個人/企業分岐（companyCheck, Pattern A / Yes-No）**: 個人=氏名+生年月日、企業=企業名+担当者名（担当者名は `patientName` に格納）
- **電話番号聴取**: 着信=携帯→着信番号で復唱 Yes/No、着信=固定→手動聴取+復唱。個人情報サブフロー.bivr を完全コピー
- **用件別分岐（Pattern B / ContextMatchRouter）**: classification 値で 4 ルートに分岐。合流なく各ルートが独立終話
- **各ルートの聴取項目**:
  - 予約: 健保組合 → 希望コース → 追加オプション → 受診希望日 → その他 → 終話(smsFlag=1)
  - 変更: 予約日 → 変更内容 → その他 → 終話(smsFlag=2)
  - キャンセル: 予約日 → その他 → 終話(smsFlag=3)
  - 問い合わせ: 問合せ内容 → その他 → 終話(smsFlag=4)
- **終話ステータス**: 予約/変更/キャンセル/問い合わせ=status=1、非通知=status=2、聴取失敗=status=2
- **サブフロー化**: 氏名・生年月日・電話番号のみ（Rule 9）。企業名・担当者名は本体 hearing ブロック
