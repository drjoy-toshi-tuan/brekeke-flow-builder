## 検出事項

### C-001: 受診希望日2 の出力値と条件値の不一致
- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_受診希望日2
- **問題**: 
  - 設計書（scenario_flow / 受診希望日2 / line 402-404）では：
    ```yaml
    conditions:
      - match: "なし"
        next: 終話分岐_新規予約
    ```
    つまり、**プロンプトが「なし」を出力することを期待**しています。
  
  - 一方、プロンプト（prompts_*.md / 受診希望日2 / line 1024-1028）では：
    ```
    出力仕様（厳守）
    - yyyy-MM-dd 00:00:00
    - 希望なし
    - 分からない
    - NO_RESULT
    ```
    つまり、**プロンプトは「希望なし」を出力するよう記載**されています。
  
  - フロー制御エンジンが JSON の conditions.match 値と照合する際、「希望なし」という出力値は「なし」とマッチしないため、分岐判定が失敗する。
  - 結果として、第二希望が無い場合に正しく「終話分岐_新規予約」へ分岐されず、後続の「受診希望日3」へ進むか、エラーハンドリングが発動する可能性がある。

- **設計書の notes での記述**:
  - line 400: 「「なし／特にない／希望なし」等もしくは preferredDate2='' なら...」
  - これはユーザーの多様な言い方を例示していますが、conditions の match値は単一の値「なし」に統一されています。

- **修正方針**: 
  - **推奨案**: プロンプト出力値を「希望なし」→「なし」に変更する。理由：
    1. 「変更希望日有無」プロンプトが「あり」「なし」を正確に使用している（一貫性）
    2. 設計書の conditions と完全に一致する
    3. JSON フロー制御が正常に動作することを保証
  
  - 修正箇所: prompts_*.md の line 1026（「Output（厳守）」セクション）
    ```
    変更前:
    - 希望なし
    
    変更後:
    - なし
    ```
