### 抽出した要件サマリー

- **冒頭**: 挨拶「お電話ありがとうございます。八尾徳洲会総合病院の、予約専用、AI 電話です」 + 時間判断（受付時間: 月〜金 8:00-16:00 / 土 8:00-11:00 / 日祝休み / 12/31-1/4 年末年始休止）
- **用件分岐**: 4 用件（予約変更 / 予約キャンセル / 予約確認 / 歯科口腔外科）+ 健診センター + 予約以外 の計 6 ラベル。echo_back あり
- **診療科**: 分岐なし診療科群（内科・外科・整形外科・眼科 等 60+ 科）+ 検査名（CT/MRI/大腸ファイバー 等 30+）+ 歯科口腔外科 + 健診センター + 病児保育診察 + わからない の振り分け。後段 ContextMatchRouter で分岐（Pattern B）
- **予約変更ルート**: 診療科 → 予約日（現在）→ 予約希望日 → 変更詳細 → 変更理由 → 個人情報サブフロー連鎖 → 終話
- **予約キャンセルルート**: 診療科 → 予約日（現在）→ キャンセル理由 → 個人情報サブフロー連鎖 → 専用終話 (キャンセル特殊文言)
- **予約確認ルート**: 診療科 → 個人情報サブフロー連鎖 → 終話
- **歯科口腔外科ルート**: DateOfCall Classifier 3 段（8:30 / 12:30 / 13:30 境界）→ 8:30 前 = 案内終話、8:30-12:30 = 歯科転送、12:30-13:30 = 案内終話、13:30 以降 = 歯科転送
- **健診センタールート**: DateOfCall Classifier 1 段（11:00 境界）→ 11 時前 = 代表 + 健診直通 2 番号案内終話、11 時以降 = 健診転送
- **病児保育診察**: 「病児保育診察は代表電話へ」固定 TTS → 代表電話 5 回繰り返し → 終話
- **予約以外**: 「予約専用ダイヤル」案内 → 終話
- **個人情報サブフロー**: 氏名 / 生年月日 / 診察券番号 / 電話番号 の 4 本独立サブフロー。Custom Jump to Flow で連鎖
- **終話電話番号確認**: 元 yaml では入電番号取得済みなら案内のみ、未取得なら聴取。Gen3 では電話番号サブフローに集約（incoming-classifier ベース）
- **DTMF 受付**: 用件 1=変更 / 2=キャンセル / 3=確認 / 4=歯科口腔外科。予約日は 4 桁数字（# / * + 月日）
