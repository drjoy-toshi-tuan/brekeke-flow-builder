### C-001: OpenAI_現在の予約日_変更用 — 年補完ロジック矛盾
- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_現在の予約日_変更用
- **問題**:
  
  STEP3（年の補完）のアルゴリズムと、入出力例の実行結果が矛盾しています。
  
  **基準日**: 2026年4月27日、**入力**: 「3月15日」の場合
  
  - **プロンプト STEP3-1**: 「基準日の月が1月〜9月の場合 → 基準日の年を採用」
  - **基準日の月**: 4月 = 1〜9月の範囲
  - **期待される出力**: 2026-03-15 00:00:00 ✓
  - **実際の入出力例**: 2027-03-15 00:00:00 ✗
  
  同時に、注釈セクションに「基準日との前後関係に関わらず発話どおりに当年補完。基準日が10〜12月の場合のみ翌年補完を適用する」と記載されていますが、これは STEP3 のロジックと矛盾します。
  
  - STEP3 のロジック: 基準日が10〜12月の場合、かつ抽出月が1〜9月の場合のみ翌年補完
  - 注釈: 基準日が10〜12月の場合のみ翌年補完を適用
  
  （注釈が正しいのであれば、基準日が1〜9月の場合は年補完対象外となります）

- **修正方針**:
  1. STEP3 のアルゴリズムと入出力例を統一する
  2. 注釈の「当年補完」の定義を明確にする
  3. 基準日が1〜9月・10〜12月の場合それぞれのロジックを具体例で実装する
  
  **推奨修正**:
  ```
  【STEP3：年の補完】
  以下のルールで年（YYYY）を決定する：
  
  1. 入力に4桁の年（2020〜2030の範囲）が含まれている場合 → その年を採用
  2. 4桁の年が含まれていない場合：
     - 抽出した月が基準日の月以降の場合 → 基準日の年を採用
     - 抽出した月が基準日の月より前の場合 → 基準日の年 + 1（翌年）
  3. 「来年」「再来年」「去年」「昨年」等の相対年表現は採用せず NO_RESULT
  ```
  
  また、入出力例を正しい動作に修正してください。

### W-002: OpenAI_用件確認 — 複合意図への耐性確認
- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_用件確認
- **問題**:
  
  「キャンセルして新規予約もしたい」「予約の変更とキャンセルを同時に」といった複合意図の入力に対して、
  プロンプトの判定ロジックが明確に記載されていません。
  
  STEP2（語句一致判定）では「複数の選択肢に一致した場合は上から優先」と記載されていますが、
  例えば「キャンセルしたいけど新しく予約もしたい」という入力では、
  「キャンセル」と「予約」の両方が一致する可能性があります。
  
  - 「キャンセル」の判定語句: キャンセル・やめたい・中止・など
  - 「予約」の判定語句: 予約・予約したい・新規予約・など
  
  この場合、「上から優先」で「キャンセル」を返すのか、NO_RESULT を返すのかが不明確です。

- **修正方針**:
  
  複合意図の入力に対して、以下のいずれかの処理を明示してください：
  
  1. **単一判定（現在の仕様の延長）**: 複数に一致する場合は「上から優先」で最初に一致した項目を採用
  2. **NO_RESULT に統一**: 複数に一致する場合は曖昧性が高いため NO_RESULT を返す
  3. **順序付け**: 「予約」「変更」「キャンセル」「問い合わせ」の優先度を明確にする
  
  推奨：複合意図は「上から優先」で対応することを入出力例で示してください。
