### 移管方針
- 冒頭は opening + 冒頭_アナウンス の2ブロック1セット。冒頭アナウンス文言は元資料原文（「お電話ありがとうございます。 蘇生会総合病院、診察予約専用、AI電話です。3診療日以内のお問い合わせの方や救急の方は代表電話へおかけなおしください。」）をそのまま TTS に流し、入力待ち無しで患者名サブフローへ接続。
- **非通知着信は最初に弾く**：incoming-classifier 相当で anonymous / unknown / 空文字を判定 → END_非通知（専用アナウンス、endpoint=冒頭切断）で即切断。通常ルートへ進ませない。
- 連絡先電話番号は Date of Call Classifier ではなく **incoming_phone_number 型による分岐**。携帯（070/080/090/060）→「今おかけいただいている番号でよろしいですか？」の Yes/No 確認、それ以外（固定電話等）→ 直接 DTMF 付き手動聴取。Yes 肯定時は additionalPhoneNumber に Incoming_phone_number をそのまま保存。
- 氏名／DOB／電話番号／診察券番号は **サブフロー経由**（Jump to Flow）。患者氏名は PatientName サブフロー1本。診察券の「わからない」スキップ、8桁復唱、電話番号の9/10/11桁判定・復唱・再聴取などの定型部品はサブフロー側に閉じ込める。メインにインライン展開禁止。
- ヒアリング順序は **患者名 → 旧姓確認 → 診察券番号 → 生年月日 → 連絡先電話番号 → 診療科 → 用件**。旧姓確認は Gen2 固有の hearing ブロックを挟む（former に格納、スキップ不可）。
- 診療科は DEPARTMENT 正規化 → **対象外診療科分岐**（救急科/麻酔科/三叉神経痛/ロービジョン外来/手術センター/血液浄化センター/脳血管内治療センター/内視鏡センター/放射線科）→ 代表電話案内（075-621-3101）で終話、**健康増進センター**は別電話（050-1724-0765）で終話。context_match_router で一括判定（Pattern B）が素直。checkpoint=2 / status=2。
- 用件は「予約／変更／キャンセル／確認」の4分岐（Pattern A: OpenAI 直後分岐）。デフォルトは「確認」にフォールバック（『上記以外の用件：質問事項もそのまま質問として格納する。FAQで回答しない』→ free に自由文格納）。
- **歯科口腔外科の変更ルートだけ残薬確認をスキップ**する分岐（変更_歯科判定）が必要。context_match_router で clinicalDepartment が「歯科」含むかを判定し、含む→最後の問い合わせ直行、含まない→残薬確認。
- 終話パターン：END_汎用受付（予約/変更/確認: 「4診療日以内にショートメッセージ、または折り返し電話」）／END_キャンセル（「必要がある場合のみ折り返し」）／END_代表電話案内_診療科対象外／END_健康増進センター案内／END_代表電話案内_エラー（リトライ超過）／END_非通知 の6系統を termination_patterns に定義。
- RAG は元資料に記載なし → 不要。FAQ 質問は代表電話（075-621-3101）案内で逃がす運用（『プロンプト（本FAQ）に記載がない質問』）。

### 施設固有の特殊ルール
- **単語置換ルール**：「印刷/定期/新薬/再診/最新/禁忌」→「診察」、「薬/お薬」→「予約」、「緊急薬」→「新規予約」。用件ブロックの prompt 側で対処（prompter 担当）。
- **受診歴→classification 書き換え**：用件「予約」後の受診歴確認で Yes→"新規"、No→"再診" に classification を上書きする。enum に「予約」「新規」「再診」「変更」「キャンセル」「確認」全部あり。
- **症状ブロックが2種類**：新規=症状①「現在の症状をお話しください。」／再診=症状②「前回と同じ症状でご受診ですか？」。どちらも disease に格納、ブロックは分ける。
- **checkpoint は段階的に更新**：初期 0 → 各用件の最終手前で 1（医師指定/症状②/残薬確認/キャンセル理由/確認内容の完了時）→ 対象外診療科 2。status / smsFlag は checkpoint=1→1、=2→2、=0→0 の写しで function description どおり。
- **復唱は診察券と電話番号のみ**：診察券 `<speak type="digits" breakc="300ms">{}</speak>ですね。`、電話番号 `<speak type="telephone" breakc="300ms">{}</speak>ですね。`。**SSML 禁止ルールの例外**として元資料どおり保持（director 判断でプロパティ側に残す）。それ以外の項目は復唱禁止。
- **患者名カタカナ補正**：カタカナにならない場合「うまく聞き取りができませんでした、ゆっくり話すとAIが聞き取れません。普通のスピードではっきりと発話してください。」で再聴取。PatientName サブフロー側のリトライ処理。
- **生年月日の詳細正規化ルール**：2桁年の元号確認（00-07→令和/平成、08-31→西暦/昭和/平成、32-63→西暦/昭和、64-99→西暦）、3桁年（900-999→19XX、100-899→再ヒア）、Year(NOW)-110 未満は再ヒア、「今日は/そうわ/社長は/希少は」→昭和。DOB サブフロー prompter に反映必須。
- **診療科リストに「ペイン外来」が function enum 列挙に含まれるが yaml 分岐に出てこない**：director 判断で対象外扱いにしない（通常受付）。

### director が見落としやすいポイント
- 『**予約日と予約希望日は別なので、予約日のヒアリングがなかった場合に予約希望日の中身を予約日に入れてはいけない**』。変更ルートで reservationDate（DATE 型）と DesiredreservationDate（TEXT フリーワード）を別項目として聴取。フォーマット違いに注意。
- 『10月以降で1月から9月の日付を言われたときは、翌年にする』の年度判定ルール。予約日正規化プロンプトに必須。
- 『**入電者が一度に複数の必要情報を発話した場合、すでに伝えられた内容については絶対に復唱・確認を行わず、不足している項目のみを一つずつ質問する**』が最重要共通ルール。全 generate_by_OpenAI プロンプトに反映。
- 『**聴取した内容について、復唱をしてはいけない**』（電話番号・診察券以外）。特に DOB・患者名・予約日の復唱禁止を全プロンプトに明記。
- 『**checkpoint / smsFlag / status は内部フラグ**。AI は絶対に発話しない』。TTS 混入防止。
- 患者名は**苗字のみ発話時のみ**「下のお名前もお伺いしてもよろしいですか？」で再質問を許可（他項目の再質問は一切禁止）。PatientName サブフロー側で対応。
- 『"おっしゃってください"ではなく"お話ください"』の口調統一。ただし元資料の個別アナウンスには「おっしゃってください」が残っている箇所あり（患者名・旧姓・診察券・DOB・予約希望時期・キャンセル理由）。**原文どおり残す**（director 判断で口調統一しない）。
- 代表電話 **075-621-3101** は対象外診療科・エラーリトライ超過の両方で3回繰り返しアナウンスする文言（元資料原文）をそのまま TTS に載せる。
- キャンセル終話文は「必要がある場合のみ、担当者から折り返し電話、もしくはショートメール」で、汎用終話の「4診療日以内にショートメッセージ、または折り返し電話」とは異なる。終話2パターンを混同しない。
