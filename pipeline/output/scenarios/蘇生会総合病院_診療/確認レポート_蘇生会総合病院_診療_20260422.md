- **非通知は冒頭で即終話**（専用アナウンス: 186を付けて掛け直し）
- 個人情報サブフロー化: 氏名聴取／診察券番号聴取／生年月日聴取／電話番号聴取 の4本
  - **旧姓確認は hearing ブロック**（Gen2固有、former に格納、サブフロー化しない）
- ヒアリング順序: 患者名 → 旧姓確認 → 診察券番号 → 生年月日 → 連絡先電話番号 → 診療科 → 用件
- 診療科: 37 科（ペイン外来含む）を enum で DEPARTMENT 格納 → `context_match_router` で受付可否判定（Pattern B）
  - 対象外9科（救急科／麻酔科／三叉神経痛／ロービジョン外来／手術センター／血液浄化センター／脳血管内治療センター／内視鏡センター／放射線科）→ 代表電話案内（075-621-3101）で終話
  - 健康増進センター → 専用電話案内（050-1724-0765）で終話
  - それ以外 → 用件聴取へ
- 用件: 4分岐（予約／変更／キャンセル／確認）Pattern A。デフォルトは「確認」にフォールバック
- 予約ルート: 受診歴確認 → 新規（紹介状→症状①→医師指定）／再診（症状②）。Pattern A で history により classification を "新規" / "再診" に上書き
- 変更ルート: 予約日 → 予約希望時期 → **歯科判定**（歯科口腔外科のみ残薬確認スキップ）→ 残薬確認
- キャンセルルート: 予約日 → キャンセル理由
- 確認ルート: 確認内容
- 合流: 「最後の問い合わせ」聴取 → classification がキャンセル → END_キャンセル、それ以外 → END_汎用受付
- 終話6系統: END_汎用受付／END_キャンセル／END_代表電話案内_診療科対象外／END_健康増進センター案内／END_代表電話案内_エラー／END_非通知
- RAG/FAQ: 元資料に記載なし → 不使用（代表電話案内で逃がす運用）

| N-2 | MISSING | office_id | `"TODO_要確認"` | Dr.JOY管理画面で確認。フローJSON生成には不要だが、Brekeke設定時に必要 |
| N-3 | DECISION | END_キャンセルの smsFlag | `sms_flag: "1"`（送信あり、文言: 「必要がある場合のみショートメール」と明記されているため送信前提） | 元資料の文言は「必要がある場合のみ、担当者から折り返し電話、もしくはショートメール」。SMS送信自体はあるが Dr.JOY 側で「必要がある場合のみ送信」の運用制御を行う前提で `1` に設定。送信なしに統一したい場合は `-1` に変更 |
| N-4 | DECISION | END_代表電話案内_エラー の status | `status: "3"`（聴取失敗） | Gen2 function では status=0/1/2 の3値だが、Gen3 の許可値は 1/2/3/6/7/8/9。リトライ上限到達は Gen3 では `3`（聴取失敗）が近い。Gen2 の checkpoint=0 時 status=0 は Gen3 では使えないため `3` を採用 |
| N-5 | DECISION | 診療科デフォルトの no_result 処理 | `no_result_default: "登録なし"` | モジュール選定ガイド 2.5 に基づき DEPARTMENT 聴取項目は必ず `登録なし` を設定。Gen2 プロンプトに「近しい診療科をレコメンド」とあるため、プロンプト側では類似科レコメンドを優先、最終的に no_result のときのみ登録なしに落ちる |
| N-6 | DECISION | ペイン外来の扱い | 通常ルート（対象外分岐に含めない） | 移管ノートの判断通り。元資料の yaml 分岐・プロンプト本文のどちらにも対象外記述なし。function enum のみに存在するため通常受付として扱う |

| # | 判断箇所 | 選択肢 | 採用した判断 | 根拠 |

| 1 | 用件分岐のパターン | Pattern A（OpenAI直後分岐）／Pattern B（context_match_router） | **Pattern A** | 用件 → 4択分岐で OpenAI の出力ラベルが分岐先を一意に決定する。後段処理を挟む必要なし |
| 2 | 診療科分岐のパターン | Pattern A／Pattern B | **Pattern B**（context_match_router） | 37 科の enum を受けた後、対象外9科／健康増進センター／通常 の判定を実施する必要があるため。「対象外」という集合的分岐は context_match_router（match で科名含有判定）が素直 |
| 3 | 受診歴確認後の classification 書き換え | プロンプト内で実装／後続 saveContext2DB で実装 | **Pattern A + saveContext2DB** | 受診歴 hearing を enum で「はい」「いいえ」に分岐し、それぞれのルート先頭に classification を書き換える saveContext2DB を配置。scaffold_generator が自動で生成する条件分岐パスを利用 |
| 4 | 歯科口腔外科の変更ルート | プロンプトで判定／context_match_router | **context_match_router** | 既に clinicalDepartment に保存済みの値を参照する分岐。Pattern B が素直 |
| 5 | 旧姓確認の配置 | サブフロー／メインフローの hearing | **メインフローの hearing** | Gen2 固有の項目で、他施設で再利用しない。hearing（text）でそのまま聴取 |
| 6 | 最後の問い合わせ後の終話分岐 | Pattern A（OpenAIで分岐）／Pattern B（context_match_router） | **Pattern B** | 最後の問い合わせは text 格納（自由発話）で分岐に使わない。分岐条件は classification の値（既に保存済み）なので context_match_router が適切 |
