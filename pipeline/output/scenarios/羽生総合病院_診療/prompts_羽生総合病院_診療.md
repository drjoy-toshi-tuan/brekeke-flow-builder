# 判定アルゴリズム

## 【STEP1：入力正規化】
1. 前後の空白削除、改行・タブ削除

---

## 【STEP2：数字モード判定（DTMF / 数字発話）】
正規化後入力に、以下のいずれかが**部分マッチで含まれる**場合に判定する。
※ 「に」「さん」「し」「いち」「よん」など、単独では助詞・敬称・動詞語尾と区別がつかないモーラ単位の読みは**判定対象外**（誤判定防止のため、完結した番号読みフレーズ「いちばん／にばん／さんばん／よんばん」「N番」のみを採用する）。
※ 番号読みフレーズの前後に「お願いします」「です」「で」等の自然言語が付いていても判定する。

- 1 / 1番 / いちばん → 予約
- 2 / 2番 / にばん → 変更
- 3 / 3番 / さんばん → キャンセル
- 4 / 4番 / よんばん → 確認

**部分マッチ方針**: 入力全体が一致する必要はない。「1番でお願いします」「にばんで」「3でいきます」のように番号トークンを含む発話を判定する。ただし、上記リストに**完全一致するトークン**（半角数字単独、「N番」、「いちばん／にばん／さんばん／よんばん」）が含まれる場合のみ採用し、「いち」「に」「さん」「し」「よん」「ばん〇」のような断片マッチは禁止。
含まれない場合は STEP3 へ進む。

---

※自然言語の助詞・接続詞・敬語（「です」「ます」「ね」「お願いします」「と思います」 等）が前後にあっても判定する

入力全体が日付パターンに完全一致する必要はなく、自然言語の助詞・接続詞・敬語（「です」「ます」「に」「の」「は」「ね」「予約は」「予約日は」「希望日は」 等）が前後にあっても、日付パターンが含まれていれば抽出する。

- (A) 「YYYY年M月D日」形式 — `\d{4}年\d{1,2}月\d{1,2}日` を含む（前後ノイズ可）
  - 例: 「2026年5月20日」「2026年5月20日です」「予約は2026年5月20日に」
- (B) 「M月D日」形式 — `\d{1,2}月` の直後または近接位置に `\d{1,2}日` を含む（前後ノイズ可）
  - 例: 「5月7日」「5月7日です」「5月の7日」「5月の15日」「9月30日」「5月だいたい15日」「5月くらい15日」
  - 「月」と「日」の間に挟まり得るノイズ語句は次のいずれかに限定（完全網羅・他の語は不可）：
    - `の` / `くらい` / `ぐらい` / `だいたい` / `たぶん` / `あたり` / `ごろ` / `頃` / `に` / `は` / 半角空白
  - ノイズ語句が挟まる場合、両端の数字を月（左側）・日（右側）として抽出する。月と日の間に上記以外の文字（例: 数字、別の月日トークン）が挟まる場合は (B) 不採用とし、他パターン (A)(C)(D) または NO_RESULT を検討する。
- (C) 「MMDD」4桁数字形式 — 他の数字に隣接しない 4 桁数字 `(?<![0-9])\d{4}(?![0-9])` を含む
  - 例: 「0520」「0520です」「次は0520でお願いします」
- (D) 「YYYYMMDD」8桁数字形式 — 他の数字に隣接しない 8 桁数字を含む
  - 例: 「20260520」「20260520です」
- (E) 上記いずれにも該当しない、または相対表現（明日/明後日/来週/来月 等）→ NO_RESULT

DTMF入力の場合は (C) または (D) の純粋な数字列が来る前提。半角数字のみで構成される入力は、4 桁または 8 桁のみ有効。それ以外の桁数（3 桁、5 桁、6 桁、7 桁、9 桁以上）は NO_RESULT。

## 【STEP3：年の補完】
- (A) (D) は発話に年が含まれているのでそのまま採用。
- (B) (C) は年が省略されているため、システム基準日を使って補完する：
  - 抽出した M月D日（または MM月DD日）が、基準日の月日**と同じか以降**であれば、基準日と同じ年を採用。
  - 抽出した月日が基準日より過去であれば、基準日の翌年を採用。

## 【STEP4：有効範囲チェック】
抽出・年補完した日付が以下の条件を**すべて**満たさない場合は NO_RESULT：
- カレンダー上に実在する日付であること（うるう年考慮、月の日数を超えない）
- システム基準日と同じか、基準日より未来であること（過去日は採用しない）

## 【STEP5：出力整形】
抽出・検証された日付を「yyyy-MM-dd 00:00:00」形式で出力する。
月と日は必ず2桁（ゼロ埋め）。時刻は固定で `00:00:00`。
