### 施設固有の特殊ルール
- **用件 4 分類**: `予約を取る / 変更 / キャンセル / 確認`（番号「1〜4」DTMF 互換語含む）。Gen1 では「再診→新規」「初診/新規→紹介状確認サブフロー」分岐があり、**新規（=初診）のみ紹介状有無を確認**。Gen3 では `用件_classifier` → 新規分岐後に subflow `紹介状確認` を挟む（hearing yesno → context: 紹介状あり/なし、後段で発話分岐は無し、Dr.JOY 側で参照）。
- **診療科の 3 段分岐**: 
  - L1: 「分岐あり診療科」`呼吸器 / 甲状腺 / 小児科` のいずれかに当たれば L2 に進む。
  - L2: 呼吸器 → 内科/外科、甲状腺 → 糖尿内分泌/外科、小児科 → 診察/リハビリテーション科 を再聴取して確定。
  - 並行して「分岐なし診療科」「健診」「面会」「わからない」「腎臓内科（予約不要）」「予約対応なし（病理/麻酔）」「代表案内（救急・歯科口腔・小児科非予約・透析・心理・鍼灸・放射線・緩和・外来化学療法・胃婁交換・面会）」を **CMR で並列分岐**。
  - 中黒（`・`）正規化: `放射線画像診断科・放射線治療科` `糖尿・内分泌` `リウマチ・膠原病内科` は OpenAI 出力の中黒揺れを script で吸収（読み上げ用には中黒入り、context 値はスラッシュ無し ASCII safe へ）。
- **当日予約は NG**: 入電時間判断後に必ず「当日希望ですか？」を hearing_yesno（template: `echo_back_yesno`）→ Yes なら `代表電話 048-562-3000` 案内 → call_transfer は使わず announcement + termination。
- **携帯電話番号チェック**: 連絡先聴取の終端で「入電番号が携帯か否か」を script 判定し、用件に応じて分岐（キャンセル/変更/再診/新規/確認 × 携帯あり/なし）→ 受付完了アナウンス文言が 2 系統（電話のみ／電話＋SMS）。**director 判断**: SMS の有無で文言を出し分けるか、施設要望に確認の上で電話＋SMS 統一に倒すかは元資料に明示無し。要 CS 確認。
- **予約日聞取り**: 元資料は「現在予約されている日」を聞いている（変更/キャンセル/確認用）。新規/再診の希望日聞取りは含まれていない（予約は人間が後追い）。Gen3 でも **予約日 hearing は確認/変更/キャンセル系のみ**、新規・再診は氏名/DOB/電話番号のみで終話。
- **生年月日**: 和暦/西暦両対応 + DTMF 8 桁 + 復唱形式の出し分け（和暦/西暦）。Gen3 の生年月日サブフローテンプレートで吸収。
- **エンティティ件数**: STT 辞書 644 件 / OpenAI synonyms 138 件。診療科 391 件は `additional_words`、template 流用は `hearing_shinryoka_basic / hearing_yesno_common / hearing_yoken_common / hearing_unknown / echo_back_yesno / hearing_phone_number / hearing_kenshin_course_basic` を使う。

### director が見落としやすいポイント
- main.yaml の `携帯電話番号チェック_0〜4` は **5 つそれぞれ用件分岐後に置かれる別ブロック**。Gen3 では「携帯判定 script を 1 つだけ作って、用件 context と組み合わせて termination_patterns で 10 通りの完了文言にマッピング」が現実的。**director 判断**: 文言バリエーションを縮約してよいか CS 要確認。
- `紹介状確認` は END_戻る しか持たず、**結果を Dr.JOY に渡す context（紹介状あり/なし）の保存先が未明示**。Gen1 では呼出元の用件.yaml が context を見ずにそのまま終話に流している。Gen3 では `context_intro_letter = yes/no` を save2db 必須。
- `非対応診療科` 分岐の `健診` だけ別アナウンス（健康管理センター 048-562-5555）。他 11 件は代表電話 048-562-3000 共通。**面会** は Gen1 では `予約対応なし` に倒している（13 surfaces 全て OpenAI、STT は無い）。
- `年末年始日付分岐` は受付完了系の終話 4 種（変更/キャンセル/再診/新規/予約確認）の **直前** に挟まる。announcement の文言を date_of_call で出し分け、終話チェーンは共通 1 本に合流。
- `用件再設定へ`「失礼しました、」/`再度聞き取りへ`「うまく聞き取りが出来ませんでした。再度、」等の**繋ぎ TTS は director デフォルト 12 項目に含まれているので省略可**。
- `当日予約聞き取れず` `聴取失敗：終話へ` の長文アナウンス（代表電話 + 受付時間案内）は **施設固有のリテラル**、`tts_g` で全文ベタ書き必須（テンプレ流用しない）。
- ASCII のみ、丸数字禁止。CMR の reference_module で subflow を指す場合は `jump_{step名}` 形式。CMR conditions 末尾に `match: "other"` 必須。
