### C-001: 用件_確認での DTMF キーワード追加による誤判定リスク
- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_用件_確認 (STEP2: 数字モード判定)
- **問題の詳細**:  
  設計書（step_details）では用件確認の DTMF キーワードが以下のように定義：
  ```
  予約: 「1」「いちばん」
  変更: 「2」「にばん」
  キャンセル: 「3」「さんばん」
  確認: 「4」「よんばん」
  ```
  
  一方、プロンプットでは以下のように拡張：
  ```
  予約: 「1」「1番」「いち」「いちばん」「ばん1」
  変更: 「2」「2番」「に」「にばん」「ばん2」  ← 「に」が単独追加
  キャンセル: 「3」「3番」「さん」「さんばん」「ばん3」  ← 「さん」が単独追加
  確認: 「4」「4番」「よん」「よんばん」「し」「ばん4」  ← 「し」が単独追加
  ```
  
  **具体的なリスク**:
  - 「に」: 「に関する」「に〜」で始まる文に含まれるため、本来「変更」ではない発話が「変更」と誤判定される可能性
    - 例: 「ニキビ治療の予約」→ 「に」を含む → 「変更」と誤判定
  
  - 「さん」: 敬称や「さんまい」など一般的な単語に含まれる
    - 例: 「田中さんの代理で電話しました」→ 「さん」を含む → 「キャンセル」と誤判定
  
  - 「し」: 「します」「した」「試す」など日常的な動詞に含まれる（最大リスク）
    - 例: 「診療科を試しに変更したいんですが」→ 「し」を含む → 「確認」と誤判定

- **設計書との齟齬**: 
  - 設計書では「1番」「いち」等の拡張に言及なし
  - 特に「に」「さん」「し」の単独キーワードは設計書に明記されていない
  - これらは DTMF キーワードの読み方から派生したと推定されるが、自然言語処理では極めて危険

- **修正方針**: 
  設計書と一致させ、以下のいずれかの修正を実施：
  1. 設計書に合わせてプロンプトを修正（推奨）: DTMF キーワードは「数字」「○番」「よみ+番」形式のみに限定
  2. または、設計書側で「に」「さん」「し」単独キーワードの追加を明示的に承認
