### 移管方針
- 愛仁会健診センターと **同一構造**。Gen3 移管も同じテンプレートを流用、文言と代表番号と土曜営業日のみ差し替え:
  - `API_DRJOY_CONTEXT_SETTINGS` / `API_DRJOY` / `挨拶` → opening + 冒頭_アナウンス。冒頭文言「お電話ありがとうございます。カーム尼崎健診プラザの、予約専用 AI 電話です。」
  - `入電日時判断` → acceptance_times（曜日・時刻判定、**土曜は第 2・第 4 のみ営業の特殊ルール**）+ Script（祝日/年末年始 + 第 2・第 4 土曜判定）
  - `用件聴取.yaml` → hearing（**DTMF 2 値分岐**: 1=新規予約 / 2=予約以外）+ echo_back 2 本（用件の復唱：健康診断 / 変更）→ Pattern A（max_dtmf_length: 1、音声入力なし）
  - 新規ルート (DTMF=1): 用件の分岐 → 受診希望日聴取 → 完了フラグ → 終話挨拶（「3 営業日以内に担当者から折り返し電話、もしくはショートメールにてご連絡」）
  - 予約以外ルート (DTMF=2): 用件の分岐_1 → 携帯チェック → 携帯あり: WEB 予約希望確認（DTMF 1=WEB / 2=TEL）→ WEB 選択時 `WEB予約フォーム案内`（SMS 送付）、TEL 選択時 受診者名 → タイトル → 生年月日 → それでは → 連絡先 → 用件の分岐（**OpenAI で「予約以外（その他）」判定**）→ その他なら携帯チェック → WEB 予約案内、それ以外は受診者名 → 現在の予約日聴取 → 完了フラグ → 終話挨拶
  - **携帯なし**: WEB 予約案内スキップ → 直接受診者名聴取
  - 氏名 / 生年月日 / 連絡先 / 電話番号区切り → subflow ブロックで Gen3 標準サンプル JSON 完コピ。氏名は受診者名（担当者名）の 1 枠 → PatientName 1 枠に充当
  - 受診希望日聴取 / 現在の予約日聴取 → datetime hearing（OpenAI 正規化、addCurrentDate: Yes scaffold 自動付与）

### 施設固有の特殊ルール
- **用件入口は DTMF 2 値のみ**: 愛仁会と同じく「1: 予約を取る / 2: 予約の変更・キャンセル等」の DTMF 分岐。音声入力は受付けない。Gen3 でも DTMF 専用 hearing 1 本（OpenAI 正規化なし、scaffold の DTMF input ブロックで吸収）
- **土曜は第 2・第 4 のみ営業**: 受付時間 TTS「日祝日、年末年始を除く、月-金 9:00-15:00、第 2、第 4 土曜日は 9:00-12:00。その他土曜日は受け付けておりませんのでご注意下さい」を保持。**Gen3 では acceptance_times の通常曜日判定では吸収できない** → Script（第 N 土曜判定: `Math.ceil(day / 7)` で 2 / 4 か判定）を opening 直後に配置するか、time_def カスタム式 `Sat: [09:00-12:00 only when (week_of_month in [2,4])]` で表現するか director 判断
- **代表電話番号**: 06-6430-1315（カーム尼崎健診プラザ直通）。「祝日を除く、月-金 8:45-17:00、土 8:45-12:00 に直通へかけ直し」と案内 → 各 CP 系（希望日 / 氏名 / 生年月日 / キャンセル日 / 用件 / 連絡先）のリトライ上限失敗時に集約
- **時間外案内**: 「日祝日・年末年始除く、月-金 9:00-15:00、第 2・第 4 土曜のみ 9:00-12:00、その他土曜は休業」が本番受付時間（director 判断）。代表電話案内の「8:45-17:00 / 土 8:45-12:00」と差分があるが、運用実態は時間外案内の値を優先（土曜不定休のため）
- **新規ルートは受診希望日のみ聴取**: 健診種類（人間ドック / 協会けんぽ / その他）の聴取は無し → 担当者折り返しに任せる
- **WEB 予約希望確認は携帯ありのみ**: `携帯チェック` で携帯あり判定された場合のみ「WEB は 1、電話は 2」を聞く。携帯なしは無条件で電話聴取に進む
- **DTMF 入力 LoopCount 5 回**: WEB 予約希望確認の DTMF で 5 回失敗 → `入力確認できないので TEL へ` → 通常電話聴取に切り替え
- **転送無し / Blind Transfer 無し**: 担当者直結転送は無し → 全ルート最終段で番号案内型 termination（CP 系）or 完了フラグ → 終話挨拶（折り返し約束）
- **「予約以外」エンティティの同義語**: 愛仁会と同じく「アクセス、お見舞い、くすり、会計、行き方、伺い、質問、処方箋、書類、症状、証明書、診断書、病棟、副作用、文書、聞きたい」等 22 件。連絡先聴取後の `用件の分岐` で OpenAI 判定 → 該当 → WEB 予約案内（SMS）に誘導
- **時間外フォールバックは `時間外案内` 直行**: 愛仁会では `タイトル_1`（「ご予約以外の方、お電話で予約の申し込みをご希望の方は、」）を経由してから時間外案内へ流れるが、**カームは `時間外へ` 箱（セリフ「ご予約以外の方、お電話で予約の申し込みをご希望の方は、」）→ `時間外案内` 直行で迂回が短い**（director 注意、姉妹フローとの差分）

### director が見落としやすいポイント
- 冒頭の `API_DRJOY_CONTEXT_SETTINGS` と `API_DRJOY` は Gen3 opening 標準処理で吸収 → 明示ブロック化不要
- 「タイトル」（セリフ「それでは次に、」）「それでは」（セリフ「それでは、」）等の繋ぎ TTS は次段ブロック先頭 TTS にマージ
- 「セリフ: none」かつ振り分けのみの箱（`用件の分岐` / `_1`、`完了フラグ`、`携帯チェック` / `_1`、`時間外切断へ`、`入電時間判断へ 開発中はシナリオへ繋ぐ`、`連絡先聞き取れずCP`、`希望日聞き取れず ＣＰ`、`氏名聞き取れず ＣＰ`、`生年月日聞き取れず ＣＰ`、`キャンセル日聞き取れず CP`、`用件聞き取れず CP`、`LoopCount`）は独立ブロック不要 → 前段 next 書き換えか CMR の一部として吸収
- `入電時間判断へ 開発中はシナリオへ繋ぐ` 箱は **開発時の暫定迂回箱の名残** → Gen3 では削除して直接 acceptance_times へ
- 「CP 〜」プレフィックス箱は Compliance Path 系のリトライ上限失敗 → Gen3 では各 hearing/subflow の `error_jump` 先 termination（代表電話 06-6430-1315 案内）に集約
- 用件聴取サブフローの「end_用件取得できず」（DTMF 3 回失敗）は CP 用件聞き取れずの `代表電話へ` に合流
- 連絡先聴取後の `用件の分岐` は **OpenAI で「予約以外（その他）」を判定する唯一の OpenAI 分類点** → エンティティ「予約以外」の同義語列挙が肝
- 第 N 土曜判定は **acceptance_times の標準仕様で表現困難** → Script ブロック追加 or time_def 拡張のいずれかを director が選択（同パッケージの愛仁会には無い独自要件）
- 「<? $連絡先電話番号（ハイフン区切あり） ?>」「<? $入電番号（ハイフン区切りあり） ?>」等の Gen1 動的タグは Gen3 では `<% var %>` 構文へ
- 受診者名サブフローは「受診者または担当者」と聴取するが Gen3 では PatientName 1 枠に充当（受診者と担当者の分離設計は無し）
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字等の環境依存文字は禁止
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`** を使う
- **CMR conditions は必ず最後に `match: "other"` + next 明示**
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
