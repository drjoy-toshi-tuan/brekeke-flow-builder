### W-003: 診療科_初診 STEP2 と STEP3 の冗長性  <prompter>

- **修正担当**: prompter
- **対象モジュール**: OpenAI_診療科_初診（L583-909）
- **重大度**: Warning
- **問題**:
  - STEP2 完全一致判定（L583）で全43診療科に対応する
  - その直後の STEP3 マッピング（L588-909）でも同一診療科が複数回登場
  - 例：「内科」は STEP2 で完全一致で対応し、同時に STEP3 で も「内科」→「内科」のマッピング（L898-903）があり冗長
  - 過度な冗長性により、OpenAI の CPU 処理負荷が増加する可能性あり（ただし機能的には問題なし）

- **修正方針**:
  1. STEP3 マッピングは、STEP2 で確定できない **略称・類義語・異表記** に限定
  2. 完全一致名（例：「内科」「外科」「眼科」等の医学用語そのもの）は STEP3 から削除
  3. または、STEP3 を「STEP2 の完全一致で判定できなかった場合のみ実行」と明記
