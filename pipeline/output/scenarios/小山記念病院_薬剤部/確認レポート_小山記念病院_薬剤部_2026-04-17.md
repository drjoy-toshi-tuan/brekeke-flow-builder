- **作業種別**: Gen2→Gen3 移管

- **シナリオ名**: 薬剤部

### 抽出した要件サマリー

- 発信者は薬局、受信者は小山記念病院 薬剤部（一般的な患者→病院フローと異なる B2B 型フロー）
- 用件は 3 区分: 疑義照会 / 報告 / その他問合せ
- 共通聴取: 薬局名（E1） → 用件（C） → 担当者名（T1） → 患者 ID（T2） → 患者名（T3）
- 疑義照会ルート: → 診療科 → 疑義内容 → Z 判定
- 報告ルート: → 診療科 → 報告内容 → 折返有無 → Z 判定
- その他問合せルート: → 問合せ内容 → Z 判定
- Gen2 固有機能:
  - 着信番号と薬局 NDJSON 77件の全桁完全一致 → clinicalDepartment2 自動格納、E1 スキップ
  - 非通知着信時は Fend（連絡先電話番号）聴取 → Z2 終話
- 全体方針: 復唱・聞き返し禁止（§1.1）、創作・補完禁止（§9）

## 4. director の判断ログ

| # | 判断箇所 | 選択肢 | 採用した判断 | 根拠 |
|---|---|---|---|---|
| 1 | 着信番号→薬局名自動特定（Gen2 §4） | A: Script モジュールで NDJSON 突合を自作 / B: 機能廃止し常時 E1 聴取 | **B（廃止）** | Script は「最後の手段」（モジュール選定ガイド 3.8）。77件の NDJSON 突合は複雑で、E1 聴取による UX 劣化は1ステップ追加のみで許容範囲と判断 |
