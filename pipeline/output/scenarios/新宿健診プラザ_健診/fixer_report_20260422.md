**W-001: OpenAI_登録番号聴取_新規 の出力値表現のゆらぎ** <prompter>

**重大度**: Warning

**対象モジュール**: OpenAI_登録番号聴取_新規

**問題**: 

設計書 `step_details` での `output_values` の記述：

```yaml
output_values:
  - "わからない"
  - "default"
  - "NO_RESULT"
```

一方、プロンプト MD での出力仕様：

```markdown
# 出力仕様（厳守）
以下のいずれか1語のみを出力すること：

- わからない
- 10桁の半角数字そのもの（例: 0123456789）
- NO_RESULT
```

**齟齬の詳細**:
- 設計書の「`default` ラベル」は、実装上は「10桁数字の raw 値」を表す
- プロンプト MD は「10桁の半角数字そのもの」と明記している
- JSON/conditions では「`match: "default"`」となっているため、**分岐上は問題なし**
- しかし **プロンプト側の出力仕様に「数字の具体例」を挙げている** ため、冗長性あり

**修正方針**:

プロンプト MD の出力仕様を以下のように修正：

```markdown
# 出力仕様（厳守）
以下のいずれか1語のみを出力すること：

- わからない
- 10桁の半角数字（例: 0123456789） ← 変更なし（既に正確）
  ※ default ラベルに対応。保有する 10 桁登録番号をそのまま出力すること
- NO_RESULT
```

**注**: 既存の記述で実装上は問題ないが、設計書表現との **統一性向上** のため推奨。
