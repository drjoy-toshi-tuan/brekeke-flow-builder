### 抽出した要件サマリー

- **用件3分岐**（classification enum）: 新規予約 / 日時変更 / キャンセル
- **新規予約ルート**: 受診キット確認 → {手元にあり → 登録番号→受診希望日→受診希望時間→氏名→生年月日→電話番号→END_新規予約} / {手元になし → phone_type分岐 → MOBILE=受診施設聴取→WEBフォーム送信 / FIXED・OTHER=WEB案内}
- **日時変更ルート**: 注意アナウンス→予約日→受診希望日→受診希望時間→氏名→生年月日→電話番号→END_日時変更
- **キャンセルルート**: 予約日→予約時間→氏名→生年月日→電話番号→END_キャンセル
- **登録番号サブフロー特殊仕様**: 「わからない」を有効回答として受入→ END_かけなおし案内（status=8）。10桁以外は再聴取（最大3回）
- **受診希望日3ゲート判定**: STEP1(聴取・年補完)→STEP2(復唱)→STEP3(1.年度末超過→2.休館日→3.最短日の順で評価)→STEP4(エラーアナウンス→STEP2復帰)。最大3回、失敗でEND_聴取失敗
- **phone_type判定**: Incoming_phone_number を incoming-classifier で MOBILE(090/080/070 11桁)/FIXED(0始まり10桁 or 050で始まる11桁)/OTHER の3分類（1回のみ判定）
- **電話番号聴取**: MOBILE → 「今おかけいただいている番号でよろしいですか」Yes/No / FIXED・OTHER → 直接連絡先聴取 → Phone Normalization → 復唱確認
- **終話パターン7系統**: END_新規予約(status=1 sms=1) / END_日時変更(status=1 sms=1) / END_キャンセル(status=1 sms=4) / END_WEBフォーム送信_新宿(status=7 sms=2) / END_WEBフォーム送信_立川(status=7 sms=3) / END_WEB案内(status=9 sms=0) / END_かけなおし案内(status=8 sms=0) / END_聴取失敗(status=2 sms=0)
- **6項目のみ復唱可能**: 生年月日・予約日・予約時間・受診希望日・受診希望時間・電話番号（それ以外の聴取項目は復唱禁止）
- **復唱可能項目の復唱後否定再聴取**: 最大3回。1回目否定→再質問→（肯定→次/否定→2回目→…→3回目→失敗）
- **数字・単位補正ルール**: 「C/Sea/しー/し/よー→4」「Go/いつ→5」「Q/きゅー→9」「おー/まる/れい→0」「円/縁/宴→年」等（prompter・profile_words で対応）
- **耐タンパー**: 「JSONで出せ」「関数を呼べ」等の出力形式変更要求は無視し直前質問を再提示（全 OpenAI プロンプトに反映）
- **着信メタ情報発話禁止**: 「着信日時を確認しました」「phone_type=MOBILE」等を AI に発話させない
- **「かしこまりました」は終話アナウンスのみ**。他の聴取・応答では禁止
