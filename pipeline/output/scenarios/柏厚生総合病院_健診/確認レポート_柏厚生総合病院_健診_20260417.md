### 抽出した要件サマリー

- **4ルート分岐**: 予約 / 予約変更 / キャンセル / その他問い合わせ（Gen2 yaml の「共通_用件聴取」から）
- **ルート別聴取項目**:
  - 予約: 受診内容(course) → オプション(option) → 健康保険組合名(insurance) → 受診希望日(Preferred_date)
  - 予約変更: 現在の受診内容(Checkup_details) → 現在の予約日(reservationDate) → 変更希望内容(reason)
  - キャンセル: 現在の受診内容(Checkup_details) → 現在の予約日(reservationDate)
  - その他問い合わせ: 問い合わせ内容(reason) + RAG 折り返しループ
- **共通後続**: 氏名(patientName) → 生年月日(patientDateOfBirth) → 連絡先電話番号(additionalPhoneNumber) → 終話
- **復唱許可**: 用件・生年月日・連絡先電話番号 のみ（氏名は復唱絶対禁止）
- **着信分類**: 非通知(anonymous/unknown/空白) → END_非通知 / それ以外 → 本フロー
- **携帯/固定分岐**: 電話番号聴取サブフロー内で処理（着信番号が 090/080/070/060 なら提案復唱、それ以外なら直接聴取）
- **サブフロー分割型**（Gen2→Gen3 移管デフォルト）: 氏名 / 生年月日 / 電話番号 / RAG検索 を個別サブフローへ
- **RAG pattern 3**: その他ルート内 + 全終話前 の2箇所に RAG 挿入
- **数字認識ルール**: 「4/四/し/よん」→4、「5/五/ご/ごお/ごー」→5 等の正規化
- **日付補完ルール**: 月日のみ発話時に年度補完、10月以降で1〜9月発話なら翌年

## 4. directorの判断ログ

| # | 判断箇所 | 選択肢 | 採用した判断 | 根拠 |
|---|---|---|---|---|
| 1 | `Preferred_date`（受診希望日）の output_format | datetime / text | **text** | Gen2 function で `display_type: "TEXT"`、元資料で「フリーワードで言われたまま文字起こし」と明記。曖昧表現（「来月上旬」等）を許容するため datetime の YYYY-MM-DD HH:MM 制約に適合しない |
| 2 | `reservationDate`（現在の予約日）の output_format | datetime / text | **datetime** | Gen2 function で `display_type: "DATE"`、元資料で「yyyy-MM-dd 00:00:00」形式を明記 |
| 3 | 用件確認の STT 種別 | DTMF_AmiVoice / AmiVoice_STT | **DTMF_AmiVoice** | Gen2 は voice-only だが、director ガイド I-6「Gen2 の分岐選択肢に 1 から順番を振り notes に DTMF 対応を記載」に従い DTMF 併用とする |
| 4 | `status` 値のマッピング | Gen2 の 0/1/2/8 をそのまま保持 / Gen3 許可値（1/2/3/6）へ変換 | **Gen3 許可値へ変換** | director 指針で Gen2 の 0/8 は禁止値。0（冒頭切断・途中切断）→ 2（代表案内・案内不可）、8（非通知）→ 2 に置換。1（通話完了）・2（代表案内）はそのまま |
