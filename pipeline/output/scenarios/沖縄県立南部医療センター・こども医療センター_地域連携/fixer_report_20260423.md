### W-003: STEP4 部分一致ルールの優先順位が曖昧（重大度: Warning）

- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_診療科 - STEP4
- **問題**:
  
  プロンプト Line 485-487：
  ```
  ## 【STEP4：部分一致判定】
  STEP2・STEP3で一致しなかった場合、STEP2・STEP3で挙げた診療科キーワードの部分一致で再評価する。
  ※複数一致した場合は最も長く一致したものを優先
  ```
  
  「最も長く一致したもの」という基準は必要ですが、複数の診療科が**同じ長さ**で部分一致した場合の優先順序が不明です：
  
  ```
  例1) ユーザー: 「脳」
       候補: 脳神経内科, 脳神経外科, 脳卒中センター（いずれも「脳」1字で同一長）
       → どれを優先？
  
  例2) ユーザー: 「小児脳」
       候補: 小児脳神経外科（4字）, 小児脳神経（3字）
       → 「最も長く」なら小児脳神経外科（正しい）
       
  例3) ユーザー: 「心臓」
       候補: 心臓血管外科（2字＝「心臓」）, 成人先天性心疾患外来（1字＝「心」）
       → 同一長は存在しないが、STEP3 で心臓血管外科が既に列挙されていれば STEP4 に進まない
  ```
  
  設計書では詳細ロジックが記載されておらず、プロンプトの記述も抽象的です。

- **修正方針**:
  1. STEP3 で網羅される診療科キーワード数・長さの分布を検証
  2. STEP4 が本当に必要な判定ケースを列挙
  3. 複数一致時の優先順位を以下いずれかで明確化：
     - a. リスト上の先着順（STEP3 マッピング順）
     - b. 部分一致文字列の長さで降順ソート（現在の説明）
     - c. 医学的信頼度順（例：汎用名「心臓外科」より専門名「心臓血管外科」を優先）
  4. 修正後のロジックをプロンプトに明記
