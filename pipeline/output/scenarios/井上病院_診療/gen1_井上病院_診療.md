### 移管方針
- Gen1「箱」 → Gen3 scenario_flow へ変換。opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「お電話ありがとうございます。井上病院の、予約専用AI電話です」。
- 主流れ: opening → 入電日時間判断（date_of_call_classifier、addCurrentDate=Yes 自動）→ 受付時間外なら **電話切断 TTS で切断（年末年始独立ブロック作らない）** → 経路判断サブフロー（**WEB / 電話の 2 値分岐**）→ ID 聞き取り → 患者氏名 → 生年月日 → **WEB / 電話で分岐_1** → 電話系: 問い合わせ内容聴取 → 連絡先聴取 → 受付完了連絡 → 終話／WEB 系: 連絡先聴取 → 受付完了連絡：WEB → 終話。
- **二段分岐 (Pattern B = ContextMatchRouter)**: 経路判断サブフロー戻り値 (`WEB` / `電話`) を **2 箇所**（生年月日後、連絡先聴取前）で参照。1 度目は「電話=問い合わせ内容聴取へ進む / WEB=問い合わせスキップで連絡先聴取へ」、2 度目（完了フラグ→分岐）は「電話=受付完了連絡（TTS A）/ WEB=受付完了連絡：WEB（TTS B、URL SMS 送信案内）」。同一 context を再利用するので `subject` 命名を統一して CMR で 2 箇所参照する。
- サブフロー化対象: 経路判断 / ID 聞き取り（=診察券番号、subject="patient_id"）/ 患者氏名 / 生年月日 / 問い合わせ内容聴取（自由文 hearing）/ 連絡先聴取（+ 電話番号区切りは phonetype subflow 内で吸収）。**メインへインライン展開禁止**。
- エンティティ仕分け（entity_classification.md 参照）: hearing_yesno_common / hearing_unknown / hearing_phone_number / hearing_yoken_common / echo_back_yesno の use_template 19 件で大半吸収。OpenAI 正規化辞書は entity_openai_synonyms.md に分離済み（30 surfaces）。STT 辞書追加語は entity_stt_patch.json。

### 施設固有の特殊ルール
- **受付時間**: **祝日を除く 月〜金 08:30〜16:00**。受付時間外 TTS は「申し訳ございません。ただいまの時間は受付時間外となっております。受付時間は、祝日を除く、月曜日から金曜日、朝8時30分から16時までです。恐れ入りますが、受付時間内におかけなおしください」（main.yaml「電話切断へ」より）。**Dr.JOY 側 acceptance_times に集約、フローには独立 announcement 作らない**。
- **WEB 予約 / 電話予約の二系統**: 経路判断サブフロー冒頭 hearing「ショートメールを受信できる携帯電話をお持ちの方は、WEB受付が可能です」→ 「WEB受付をご希望の場合は『はい』、固定電話や、このまま電話での受付をご希望の場合は『いいえ』とお答えください」（経路判断.yaml「案内」「ヒアリング」より）。Yes/No 判定だが結果値は `WEB` / `電話` の 2 値を context 化（CMR 再利用のため）。
- **WEB 系の受付完了 TTS**: 「受付はまだ完了しておりませんので、お電話終了後にショートメールで入力用URLをお送りいたしますので依頼内容を入力していただき、申し込みを完了させてください」（main.yaml「受付完了連絡：WEB」より）。**SMS の URL 送信は Dr.JOY 側責任**、フロー側は文言を出すだけ。
- **電話系の受付完了 TTS**: 「お問い合わせを受け付けました。後ほど担当者からショートメール、もしくは折り返し電話にてご連絡いたします」（main.yaml「受付完了連絡」より）。折り返し連絡前提。
- **3 回失敗時の代表案内 TTS**: 「予約制ではないので、基本的に当日受診が可能です。診療時間や休診情報等に関しては代表番号にてご確認ください」（main.yaml「代表案内」より）。診察券 / 氏名 / 連絡先 / 問い合わせ内容のいずれかで 3 回聞き取れず時は **全部この単一 TTS に合流して終話**（具体電話番号なし、「代表番号」のみ案内）。
- **問い合わせ内容聴取（自由文 hearing）**: 「お問い合わせ内容をお伺いします。"8月1日の内科予約を8月8日以降に変更してほしい"、など、具体的にお話ください」（問い合わせ内容聴取.yaml）。**用件分類 OpenAI ではなく自由テキスト保存（SKILL_E）**。例文に日付が含まれるが addCurrentDate 不要（OpenAI で日付正規化しないため）。
- **END_受付時間説明**（main.yaml 終了ブロック）: 「代表電話の受付時間は、年末年始及び祝日を除く月曜から金曜の、朝9時から17時までです。整形外科と眼科は、受付時間が一部異なり月曜から金曜の朝9時30分から12時30分となります」— **AI 受付の 8:30〜16:00 とは別**（代表電話の案内）。本フローのどこから呼ばれるかは元資料に明示されていない（孤児箱の可能性）→ **director 判断**（不要なら scenario_flow から除外）。
- **DTMF 併用**: 診察券番号は発話 / DTMF（8 桁）両対応（ID聞き取り.yaml「ダイヤルプッシュ案内」）。生年月日も DTMF 8 桁対応（生年月日聴取.yaml「ダイヤルプッシュ案内」）。電話番号も DTMF 対応。**hearing.dtmf_enabled=true を全 3 箇所で立てる**。
- **PatientName 1 枠・subject="patient"**: 「患者さんのお名前を、フルネームでお答えください」（患者氏名聞き取り.yaml）。受診者本人前提、入電者氏名の二段聴取なし。

### director が見落としやすいポイント
- **「テスト用」箱（main.yaml）**は **本番投入時は削除**。「渡部」「千葉」発話で動作テスト用の分岐を残しているが、本番フローでは入電日時間判断へ直結（main.yaml「入電時間判断へ」→「テスト用」のテスト分岐は不要）。
- **「完了フラグ」箱はセリフ none 振り分けのみ**だが、直後の「分岐」（WEB/電話 CMR 再利用）への中継として **CMR 1 ブロックに統合**（独立化禁止）。
- **電話番号区切りサブフロー**は Gen3 phonetype subflow（フリー0120 / 0800 / 携帯 / IP / 固定 2〜5 桁市外局番） で吸収。**メインから直接呼ばない**、連絡先聴取サブフロー内部で完結させる。
- **生年月日和暦/西暦の細かい年号判定（大正/昭和/平成/令和元年）**は元 yaml に詳細ロジックあるが、Gen3 では OpenAI 日付正規化（YYYY-MM-DD 固定出力）に **完全置換**。和暦/西暦の発話差し戻しロジックはコピペ禁止。
- **「分かりません」発話で診察券 END_診察券無し**: ID聞き取り.yaml「わからないセット」→「END_診察券無し」。**診察券なし系統でもフロー打ち切りせず氏名→生年月日へ進む**（main.yaml では ID 聞き取り次は無条件で患者氏名聞き取り）。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**（全角括弧 / 丸数字禁止）。`診察券番号確認（初回）` 等は ASCII 化推奨。
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**。CMR conditions 末尾は必ず `match: "other"` + next 明示。WEB/電話分岐は `^WEB$ / ^電話$ / other` 3 分岐ではなく **`^WEB$ / other`（other=電話）** の 2 分岐で十分（4 層責任モデル「`^0$ other` 排他形式」と同じ流儀）。
