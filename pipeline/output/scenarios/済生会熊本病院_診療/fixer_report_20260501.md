### W-001: 存在しない日付（2月30日・4月31日等）のセーフガード欠落

- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_現在の予約日_変更、OpenAI_現在の予約日_キャンセル（共通）
- **重大度**: Warning
- **問題**: 

  パターンE（8桁 YYYYMMDD）の検証ロジックで、以下のチェックしかない：
  ```
  月の値が 1〜12 以外、または日の値が 1〜31 以外の場合は NO_RESULT
  ```
  
  このため、**2月30日（20260230）、4月31日（20260431）** のような存在しない日付が、形式チェックを通過して `YYYY-MM-DD` 形式で出力される可能性がある。
  
  設計書 `no_result_condition: "日付として解釈不能な入力"` に違反する。

- **修正方針**:

  STEP3の各パターン（特にパターンB, D, E）で、以下の日付妥当性チェックを追加：
  
  ```
  【月別の日数検証】
  - 1月・3月・5月・7月・8月・10月・12月: 1-31
  - 4月・6月・9月・11月: 1-30
  - 2月: 1-28（閏年は1-29。YYYYMMDD で明示的に与える場合のみ判定、M月D日形式では不要）
  
  上記範囲外の日付 → NO_RESULT
  ```
  
  または、より簡潔に：
  ```
  【2月の日数制限】
  - 2月30日以上 → NO_RESULT
  【4月・6月・9月・11月の日数制限】
  - 31日 → NO_RESULT
  ```
  
  LLM の推論力に頼るのではなく、プロンプト内でルール化すること。

- **優先度**: 低〜中。実運用で「2月30日」を発話するユーザーはほぼゼロだが、スクリプト化・自動テストで検出可能性があるため、修正推奨。
