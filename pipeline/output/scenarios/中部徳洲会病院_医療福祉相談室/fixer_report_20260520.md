### W-001: NO_RESULT条件の曖昧性（「〇〇のみで下の名前が判別できない」）
- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_大城聞き取り / OpenAI_下地聞き取り / OpenAI_山城聞き取り
- **問題**: 
  - 大城聞き取り L218-223: 「『大城』のみで下の名前が判別できない入力」
  - 下地聞き取り L310-316: 「『下地』のみで下の名前が判別できない入力」
  - 山城聞き取り L409-415: 「『山城』のみで下の名前・職種が判別できない入力」
  
  これら表現は曖昧で、OpenAI の推測判定に依存。例えば：
  - ユーザー: 「大城です」→ 「大城」を含む + 下の名前なし → NO_RESULT か、リトライか？
  - ユーザー: 「大城さんでお願いします」→ 「大城」を含む + 「さん」を敬語扱い → NO_RESULT か？
  
  LLM が時々「大城」への確信度に基づいて判定揺らぎが発生する可能性

- **修正方針**:
  ```
  【STEP3：それ以外】
  以下は NO_RESULT：
  
  - 無音
  - フィラーのみ（例：えー あのー のみ）
  - 『大城』だけが含まれ、かつ以下のいずれも含まれない場合：
    * 実 / みのる / ミノル / OOSHIRO MINORU
    * 尚幸 / なおゆき / ナオユキ / OOSHIRO NAOYUKI
  - 意味不明
  - 上記いずれのキーワードにもマッチしない入力
  ```
  
  同様に下地・山城も**明示的な除外リスト**として記載

### W-003: 出力値の複数単語防止が曖昧
- **修正担当**: prompter  <prompter>
- **対象モジュール**: 全プロンプト
- **問題**: 出力仕様で「**1語のみ**を出力すること」と記載（全モジュール L17/145/242/335）だが、定義が曖昧
  - 現在の表現では「1語」の意味が、言語学的には複数解釈可能
    - 「大城実」は1語か2語か？
    - スペース区切り「大城 山城」は1語か2語か？
  - OpenAI が複数値を返すリスク（例: 「大城 下地」「大城実, 大城尚幸」）
  
- **修正方針**:
  ```
  # 出力仕様（厳守）
  以下のいずれか**1つのみ**を出力すること（改行・スペース・記号なし）：
  
  - 大城
  - 下地
  - 山城
  - わからない
  - NO_RESULT
  
  【禁止】以下は絶対に出力しないこと：
  - 複数値（「大城 下地」「大城/下地」など）
  - 説明付き（「大城です」「NO_RESULT（理由：...）」）
  - 句読点・括弧（「大城。」「(大城)」）
  ```
