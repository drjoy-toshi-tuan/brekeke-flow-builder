### 移管方針
- Gen1「箱」構造 → Gen3 9 ブロック型に変換。八尾と違い**新規予約フロー（紹介状有無 → 診療科 → 地域連携 / 通常進行）あり**。湘南藤沢の診療科 × 用件マトリクスのような複雑分岐は持たないが、**「本日受診の確認」が用件聴取の前段に入る**点が徳洲会内で札幌固有。
- 冒頭は opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「札幌徳洲会病院の、予約専用、AI 電話です」（湘南藤沢のような「当日不可」案内は無し）。
- 用件分岐は **Pattern A**（用件聴取 OpenAI 直後に 5 分岐: 新規予約 / 再診予約 / 予約変更 / 予約キャンセル / 予約確認）。各分岐に echo_back（「診察のご予約でよろしいですか?」「ご予約の変更でよろしいですか?」「ご予約のキャンセルでよろしいですか?」「ご予約の確認でよろしいですか?」）を直配置。**用件は echo_back 必須**。新規/再診は用件 yaml で「現在、通院中の診療科のご予約ですか?」Yes/No で派生する 2 段構造（director 判断、Gen3 で 5 分岐に圧縮するか湘南藤沢方式の 6 分岐構造を維持するか）。
- **本日確認サブフロー**は用件聴取の **直前**（挨拶後 → API_DRJOY → 時間外設定 → 本日確認 → 用件）。Yes なら「本日受診」用 termination（代表電話案内） / No なら通常用件聴取に進む。**hearing ブロック 1 段で展開（subflow 化までは不要）が director 判断**。
- 用件別フロー:
  - 新規予約: 紹介状有無 → 診療科聞き取り → 地域連携室 / 通常進行 / 健診NG / 内視鏡NG → 氏名 → 生年月日 → 電話 → 終話
  - 再診予約: 診療科聴取（再診）→ リハビリNG / 内視鏡NG / 健診NG / 通常 → 診察券 → 氏名 → 生年月日 → 電話 → 終話
  - 予約変更: 診療科 → 用件確認 → 予約日 → 理由 → 予約希望日 → 氏名 → 生年月日 → 電話 → 終話
  - 予約キャンセル: 診療科 → 用件確認 → 予約日 → 理由 → 氏名 → 生年月日 → 電話 → 終話
  - 予約確認: 問い合わせ内容（フリー入力）→ 氏名 → 生年月日 → 電話 → 終話
- サブフロー（患者氏名 / 生年月日 / 電話番号 / 診察券）は Jump to Flow 経由。
- 時間外設定.yaml（祝日チェック → 12/30 営業判定 → 平日/曜日判定 → 9-17 時の時間判定）→ date_of_call_classifier ブロック。**addCurrentDate: Yes 自動付与対象**。
- entity_classification.md / entity_stt_patch.json / entity_openai_synonyms.md は scaffold/properties が自動取り込み。director は転記不要。

### 施設固有の特殊ルール
- **受付時間**: 月〜金 9:00〜17:00（歯科・口腔外科のみ 13:30〜17:00）。土日祝・年末年始・12/31〜1/3 は時間外。**12/30 は営業判定の特異日**（祝日チェックの後で「12/30 なら平日時間判定」分岐を持つ、director 判断）。時間外案内は「土曜日曜祝日、年末年始を除く、月曜日から金曜日の 9 時から 17 時、となっております」。
- **代表電話番号**: `011-890-1110`（病院代表、対象外科・内視鏡・健診・側弯症等の案内ルート、聞き取れず終話用）
- **代表電話番号（終話案内用）**: `011-890-1622`（リハビリ転送失敗時のかけ直し案内専用、文言「お時間を空けておかけ直しください。電話番号は 011-890-1622 です」）
- **新規予約の紹介状フロー**:
  - 紹介状あり Yes → 診療科聞き取り → 通常初診進行（**選定療養費 2200 円の事前案内なし**、紹介状あり前提で進行）
  - 紹介状なし → 「初診時選定療養費として 2200 円を全額自己負担いただいております」固定 TTS → 診療科聞き取り → 通常初診進行
  - **地域連携室案内**: 整形外科 / 糖尿病内科 / 脳神経外科 の 3 科のみ「紹介元の病院から直接、当院、医療連携室にご連絡をお願い」固定 TTS → 終話。**初診紹介状あり/なし両方とも対象**。
  - **健診/内視鏡 → 各専用案内**: 健診→「健康診断は健診担当が承ります。代表番号にかけて健診センターへ」固定 TTS。内視鏡→「内視鏡検査の予約は内視鏡担当が承ります。代表番号にかけて内視鏡室へ」固定 TTS。両方とも代表電話 0 1 1-890-1110 を読み上げて終話。
- **再診予約の診療科 NG 分岐**:
  - リハビリテーション科 → **別シナリオ「リハビリ科へ転送」**（リハビリ室への直接転送、転送失敗時は終話案内 0 1 1-890-1622）
  - 内視鏡 → 「内視鏡検査の予約変更・キャンセルは内視鏡担当が承ります」案内 → 代表電話案内
  - 健診 → 「健診担当へ」案内 → 代表電話案内
  - その他 → 通常再診進行（診察券聞き取り → 氏名等）
- **産婦人科 × ワクチン分岐**: 産婦人科を聞き取った場合のみ、「ワクチン」発話を拾うと「受診前に必ずホームページのワクチン注意事項をご確認ください」案内 → 通常完了に戻る（`産婦人科判断` `ワクチン案内` 箱）。**それ以外の診療科でワクチン発話があった場合は無視**。
- **予約日 DTMF 4 桁機能**: 予約日確認.yaml にシャープ/アスタ + 4 桁数字受付（過去日なら +1 年で確定）。Gen3 で SKILL_C 自由テキスト + 後段 datetime 正規化で吸収するか、別途 dtmf 並走させるかは **director 判断**。
- **予約変更/キャンセル時の「理由聞き取り」**: 変更ルートとキャンセルルートの両方で「理由をお伺いいたします」フリー入力 1 段が入る。**SKILL_E 自由テキスト hearing 1 ブロック**（用件で TTS は変わらない）。
- **予約変更時の「予約希望日」**: 予約日（現在の予約日）→ 理由 → 予約希望日（変更先）の 3 段聴取。希望日は「決まっていない」エンティティで未定回答対応。**Gen3 で hearing ブロック 2 本（予約日 + 予約希望日）+ 理由 hearing 1 本の 3 段構成**。
- **予約確認の「問い合わせ内容」**: 用件 = 予約確認の場合のみ、「お問い合わせしたい内容をお話ください」フリー入力 → 氏名等の通常聴取に進む。**変更/キャンセルの理由聞き取りとは別の hearing ブロック**（TTS が違う）。
- **携帯/固定で終話 TTS 分岐**: 5 用件すべてで携帯（=携帯電話取得あり）/固定 で終話 TTS 別建て。**携帯時のみ「終話後、予約に関する SMS が送信されますのでご確認をお願いいたします」を併記**。固定時は「3 診療日以内に担当者から折り返し電話にてご連絡」のみ。`携帯電話番号チェック / _1〜_4` 箱 5 個（5 用件分）が同じパターンで `予約変更の流れ` `予約変更の流れ_1` 等の 2 系統に分岐。**予約確認のみ「終話後、予約内容確認の SMS」と文言が別**。
- **聞き取れず終話**: 「うまく聞き取ることができませんでした。代表番号 0 1 1-890-1110 までおかけなおしください。受付時間は月〜金 13 時〜17 時です。なお、歯科・口腔外科については受付時間が 13:30〜17:00 となります」固定 TTS。**受付時間案内が代表電話案内 TTS（13-17 時）と通常時間外案内（9-17 時）で異なる**点に注意（director 注意）。
- **DTMF 受付**: 用件聴取（1=診察予約 / 2=変更 / 3=確認 / 4=キャンセル）と Yes/No 確認（はい/いいえ系）。予約日聴取（シャープ/アスタ + 4 桁）。
- **復唱（echo_back）対象**: 用件（5 種）/ 診療科 / 予約日 / 紹介状有無 Yes/No / 本日確認 Yes/No。氏名・生年月日・電話・診察券はサブフロー内で完結。

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `API_DRJOY` → `施設名` → `施設名セット` → `時間外設定` → `本日確認` の 7 箱は Gen3 では opening 冒頭に吸収（`本日確認` のみ独立 hearing ブロック化）。明示ブロック化不要。
- 「セリフ: none」かつ振り分けのみの「箱」（`次へ / _1〜_3`、`用件の確認 / _1 / _2`、`用件を判断 / _1`、`携帯電話番号チェック / _1〜_4`、`完了フラグ`、`診察券無し / 診察券有り`、`変更・キャンセル・定期受診へ / _1`、`診療科分岐`、`産婦人科判断`、`産婦人科で確認済み`、`電話切断へ / _1`、`終話へ（紹介状CP？）`、`変更ルート / キャンセルルート`、`氏名聞き取れず`、`電話番号聞き取れず`、`本日受診代表案内`、`診療科の聞き取りへ（予約変更・キャンセル）`、`診療科の聞き取りへ（新規予約）`、`新規セット / 再診セット` 等多数）は Gen3 で独立ブロック不要 → 前段の next 書き換え or context_match_router の一部として吸収。
- 用件 yaml の **「現在、通院中の診療科のご予約ですか?」Yes/No 派生**（`セットした用件の分岐_1` `再診セット / 新規セット`）は、診察予約用件に対して「初診（新患）か再診か」を確定する仕様。**Gen3 では用件 5 分岐に圧縮するか、診察予約 → Yes/No → 新規/再診 の 2 段にするかは director 判断**。
- `判定: エンティティ含む` で参照される値（紹介状あり/なし、肯定/否定、地域連携案内診療科、予約案内診療科、決まっていない 等）→ entity_classification.md / entity_stt_patch.json / entity_openai_synonyms.md にすべて自動分類済み。director は転記不要。
- **`祝日チェック` は元資料で「サブシナリオ呼び出し」扱い** → カレンダー Script モジュールが必要（docs/brekeke/モジュール選定ガイド_v2.md §3.8 参照）。addCurrentDate: Yes 自動付与の対象。**12/30 営業判定** も同 Script の中で扱うか、後段 CMR で扱うかは director 判断。
- **リハビリ科への転送は「サブシナリオ：リハビリ科へ転送」**（main yaml では空のサブシナリオ呼び出し扱い、内部実装が無い）→ Gen3 では call_transfer ブロックで実装、転送失敗時の終話 TTS（0 1 1-890-1622 のかけ直し案内）を別ルートで設定。
- **予約日確認.yaml の「再診以外復唱しない」分岐**: 再診以外（新規予約・変更・キャンセル）では予約日復唱 TTS をスキップする仕様。再診のみ「<? $予約日の読み ?>、でよろしいですか?」を発話。Gen3 では context（用件）で分岐する CMR で吸収（director 判断、または用件別 hearing で予約日聴取を分けて echo_back 有無を切り替え）。
- 元資料の `<? $診療科の読み ?>` `<? $予約日の読み ?>` `<? $連絡先電話番号（ハイフン区切あり） ?>` 等の Gen1 構文は **Gen3 では `<% var %>` に変換**。
- **本日確認.yaml の「紹介状あり / 紹介状なし」名称が誤解を招く**: 箱名は「紹介状」だが実際は「本日受診: Yes（=紹介状ありルート）/ No（=紹介状なしルート）」と命名されている（元資料の歴史的経緯）。**実際の挙動は本日 Yes/No 分岐**で、END 名と次の遷移先（END_本日 → 本日代表案内 / END_本日以外 → 通常用件聴取）が示す通り。director は箱名ではなく「セリフ」と「次の箱」で判断する。
- **再診の診療科 NG リスト（リハビリ / 内視鏡 / 健診）と新規の診療科 NG リスト（地域連携 3 科 / 健診 / 内視鏡）は重複しない**: 整形外科は新規=地域連携 NG、再診=通常進行という非対称分岐。**新規ルートと再診ルートで CMR 分岐を別々に書く必要あり**（共通化禁止、director 注意）。
- ContextMatchRouter.reference_module で **subflow を参照する場合は scaffold が付ける `jump_{step 名}` prefix を使う**（subflow ブロックの step 名そのままはダメ）。湘南藤沢徳洲会で取り違えが発生したので、札幌でも同じミスをしないよう注意。
- **CMR conditions は必ず最後に `match: "other"` + next 明示**（catch-all）。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字（①②③⑩等）等の環境依存文字は禁止。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。
