### 移管方針
- Gen1「箱」→ Gen3 9 ブロック型へ変換。opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「お電話ありがとうございます。ユアクリニックお茶の水のAI電話です」（移転告知なし、ユアクリニック秋葉原と異なる）。
- **用件分岐なし**（全件＝問い合わせ預かりのみ、ユアクリニック秋葉原と同じ 3 質問構造）。OpenAI 用件分類 / 診療科聴取 / 予約日聴取 / 診察券聴取 / 生年月日聴取は無し。
- 主流れ: opening + 挨拶 → 時間判断（祝日チェック → 2026 年 GW 日付分岐 → 通常曜日/平日/土曜時間分岐 → AI 電話 / 代表電話転送 / 時間外アナの 3 系統終話）→ AI ※時間外アナ有 announcement → 冒頭発話「全部で3つ質問をさせていただきます」→ Q1: 問い合わせ内容聴取（自由テキスト）→ Q2: 患者名聴取（PatientName 1 枠）→ Q3: 連絡先聴取 → 終話「ありがとうございます。質問は以上です。内容確認後、院内スタッフから土・日、祝日を除く二日以内にショートメッセージもしくはお電話にてご連絡いたします」。
- **時間判断は date_of_call_classifier ブロック**（時間判断.yaml + 3 回聞き取り失敗救済 時間判別.yaml。addCurrentDate: Yes 自動付与）。**祝日チェック → 日付判断（2026 年 GW = 4/29〜5/6 を個別 case 化）→ 曜日分岐（通常）→ 平日/土曜時間分岐 → 時間外/AI/代表電話転送の 3 終話**。
- **代表電話転送（Blind Transfer）必須**: 時間内の特定時間帯は AI ではなく人スタッフへブラインド転送（`院内代表転送` 箱経由）。Gen3 では Call Transfer ブロック（`transfer_destination` は properties でマネージャー入力）。
- 3 質問すべて「3 回聞き取り失敗救済 時間判別」サブシナリオへフォールバック → 院内転送：聞取り失敗（「申し訳ございません、うまく認識ができませんでした。代表電話に転送いたしますのでそのままお待ちください」）→ Blind Transfer。

### 施設固有の特殊ルール
- **受付時間**: 平日 8:30〜19:00 / 土曜 8:30〜17:00 を細分化（時間判断.yaml 参照）。**平日**: 8:30 前=時間外 / 8:30〜11:00=代表転送 / 11:00〜15:00=AI / 15:00〜17:00=代表転送 / 17:00〜19:00=AI。**土曜**: 8:30 前=時間外 / 8:30〜11:00=AI / 11:00〜13:30=代表転送 / 13:30〜15:00=AI / 15:00〜17:00=代表転送。**祝日**: 8:30 前=時間外 / 8:30〜11:00=AI / 11:00〜13:00=代表転送 / 13:00 以降=時間外。
- **2026 年 GW 特別日付**: `日付判断2026年GW` 箱で 4/29 (水)・4/30 (木)・5/1・5/2・5/3 (日)・5/4 (月)・5/5 (火)・5/6 (水) を個別 case 化。**4/29 = 8:30〜16:00 代表転送のみ / 5/1, 5/2 = 通常曜日扱い / 5/3 = 全日時間外 / 5/4, 5/6 = 8:30〜13:00 代表転送のみ / 5/5 = 8:30〜16:00 代表転送のみ**。**この特別日付は 2026 年限定**で 2027 年以降は properties 側で年次更新（director ノート: addCurrentDate=Yes で吸収できない実日付ロジックのため、Script ブロック or properties マネージャー更新枠を残す）。
- **代表電話番号**: 元 yaml に明示なし（`院内代表転送` サブシナリオが Blind Transfer 呼び出し前提）。Gen3 では `call_transfer` ブロックの `transfer_destination` を properties で別途設定（director は記述せず、マネージャー入力枠を残す）。
- **時間外案内 TTS**: 「ただいまのお時間は診療時間外です。救急の方は東京都医療機関案内サービス、ひまわり03-5272-0303におかけください。当院へお問い合わせの方は、この後にご用件をお伺いいたします。内容確認後、院内スタッフから翌営業日終了までにショートメッセージもしくはお電話にてご連絡いたします」→ 冒頭発話「全部で3つ質問をさせていただきます」→ Q1 へ進行（**時間外でも 3 質問受付を継続**）。
- **代表電話転送 TTS**: 「只今のお時間は、スタッフへお繋ぎいたします。代表電話に転送いたしますのでそのままお待ちください」→ Blind Transfer。
- **混み合い時 TTS**: Blind Transfer 失敗時「只今お電話が込み合っております。恐れ入りますが、お時間を空けて再度おかけなおしください」→ 終話。
- **3 つの質問のつなぎ TTS**: Q1 = 「まずは1つ目の質問です」/ Q2 = 「かしこまりました。続いて、二つ目の質問です」（入力待ち=あり）/ Q3 = 「それでは、最後の質問です。ご連絡先のお電話番号をお伺いします」。
- **問い合わせ内容聴取**: フリーテキスト 1 本（output_format=text）。「発話あり/なし」「ご用件あり/なし」「5 秒無音」の 3 種で聴取内容チェック → 3 回失敗で END_戻る：聞取りできず（「うまく聞き取りができなかったため、後ほど担当者にて確認をいたします」）。
- **DTMF なし**（全 hearing 発話のみ）。

### director が見落としやすいポイント
- 冒頭 3 箱（API_DRJOY_CONTEXT_SETTINGS / API_DRJOY / 挨拶）は Gen3 で opening + 挨拶 announcement に整理。**API_DRJOY が挨拶の前にある**順番はユアクリニック秋葉原と共通。
- セリフ none 振り分けのみの「箱」（API_DRJOY 系 2 種、通話完了、転送失敗、混み合っている旨で切断、AI ※時間外アナ有、冒頭発話、時間判断）は独立ブロック不要 → 前段の next 書き換えに吸収。冒頭発話「全部で3つ質問をさせていただきます」TTS は **Q1 冒頭にマージ**して短縮。
- **用件分類なし**: 問い合わせ預かり 1 本。湘南藤沢/八尾/福岡徳洲会/井上健診型の用件 3〜5 分岐構造をコピペで持ち込まないこと。
- **PatientName 1 枠**: 患者本人 = 入電者前提（subject="patient"、受診者氏名/入電者氏名の二段聴取は無し）。**ただし小児科継承先**なので「患者 = 入電者の子」の可能性が高い → patient_name エンティティの説明文で「お子様の場合は保護者からお子様のお名前を」を補足（director ノート対応）。
- **2026 年 GW 特別日付は year-specific**: addCurrentDate=Yes で当日年月日を context に乗せても、4/29〜5/6 の case 値は 2026 年固定。**2027 年以降は properties で更新する運用**を移管ノート備考に残し、マネージャー確認枠を作る。
- **時間判断 yaml の END_テスト用 4/28〜5/7 等は dead code 扱い** → ノートに反映しない。
- **Blind Transfer ブロック必須**: 院内代表転送（時間内）/ 院内転送：聞取り失敗（3 回失敗時）はすべて `call_transfer` ブロック。`transfer_destination` プロパティは properties 側でマネージャーが入力。
- **「3 回聞き取り失敗救済 時間判別」サブシナリオの位置付け**: ユアクリニック秋葉原と同じ。Gen3 では date_of_call_classifier の context を使い回せばサブフロー化不要（director 判断）。
- 連絡先聴取.yaml の電話番号区切り `市外局番5桁/4桁/3桁/2桁` は Gen3 phonetype subflow で吸収 → ノート不要。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**（全角コロン `：` 全角チルダ `〜` 全角括弧 `（）` 全角スペース 全角コロン 半角小文字 `Q1` 等の混在禁止、`END_AI ※時間外アナ有` の `※` も除去、`平日の時間分岐（パータンA）` の typo 「パータン」も整える）。
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**。CMR conditions 末尾は必ず `match: "other"` + next 明示。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。Gen1 動的タグ `<? $... ?>` → Gen3 `<% var %>`（本フローは静的 TTS のみ）。
