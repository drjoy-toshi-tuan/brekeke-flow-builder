### OpenAI_日程確認
- **パターン**: はい/いいえ判定型（SKILL_B）
- **入力元**: 入力_日程確認
- **入力元STTタイプ**: AmiVoice STT（音声のみ）
- **出力ラベル**: はい / いいえ / NO_RESULT
- **Context**: 「お電話ありがとうございます。水府病院の、予約専用、AI電話です。まず初めに、今回のお問い合わせは、本日または翌診療日の予約に関する内容ですか？はい、そうです、または、いいえ、ちがいます、でお答えください。」
- **特記事項**: Gen2のHF-TOPガードから「肺」「拝」「配」「杯」「灰」「ハイ」等のSTT誤変換パターンをYES判定に流用。「CEA」→いいえ もGen2から流用。

### OpenAI_種類
- **パターン**: 分類型 3択（SKILL_A + DTMF）
- **入力元**: 入力_種類
- **入力元STTタイプ**: DTMF AmiVoice STT Input（max_dtmf_length=1）
- **出力ラベル**: 診療検査 / 人間ドック / その他 / NO_RESULT
- **Context**: 「かしこまりました。お問い合わせ内容を、次の 3 つの、いずれかでお話ください。1.診療または検査、2.人間ドック、3.その他、それでは、お話ください。」
- **特記事項**: DTMF 1=診療検査, 2=人間ドック, 3=その他。再質問なし（NO_RESULTもsaveContext_種類_登録なしへ）。

### OpenAI_用件
- **パターン**: 分類型 4択（SKILL_A + DTMF）
- **入力元**: 入力_用件
- **入力元STTタイプ**: DTMF AmiVoice STT Input（max_dtmf_length=1）
- **出力ラベル**: 予約 / 変更 / キャンセル / 確認 / NO_RESULT
- **Context**: 「ご用件を、次の 4 つのうちの、いずれかでお話ください。1.予約を取る、2.変更、3.キャンセル、4.確認、それでは、お話ください。」
- **特記事項**: DTMF 1=予約, 2=変更, 3=キャンセル, 4=確認。STT誤変換: 「薬」→予約、「契約」→予約（Gen2設計書記載の既知パターン）。

### OpenAI_受診歴

### OpenAI_診療科_予約

- WARNINGのP-2（profile_words未設定）はgeneratorの責務。医療機関名・予約希望日・予約日・確認内容のSTTモジュールが対象。

  - 診療科キーワードマッピング（喘息→内科、甲状腺→外科、骨折→整形外科等）
  - 用件のSTT誤変換パターン（薬→予約、契約→予約）
