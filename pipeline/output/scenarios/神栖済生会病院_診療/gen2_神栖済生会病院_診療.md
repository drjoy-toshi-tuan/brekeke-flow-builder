## Gen2→Gen3 移管ノート

### フロー概要

- n8n なし。標準的な Gen2 診療フロー。
- 受付制限: 冒頭で「健診・リハビリに関するお問い合わせは代表電話へおかけ直しください」と案内するのみ。時刻チェック（営業時間判定）は Gen2 に存在しない。
  - **移管方針**: director に判断を委ねる。営業時間外ガードが必要であれば Date of Call Classifier を追加。現状の customer_doc どおりなら不要。
- 用件分類: 新規 / 再診 / 変更 / キャンセル / 確認（その他問い合わせ）の4系統。`classification` に格納される値は 新規 / 再診 / 変更 / キャンセル / 確認 / ""。
- 受診歴で 新規→新規、否定→再診 に `classification` を上書き。
- 冒頭で `phone_type` 判定。withheld（非通知）は即時切断（status=9）。mobile は「今おかけの番号でよいか」確認のショートカットあり、否定時のみ固定電話として聴取。
- 診療科トリガー（インフルエンザ/コロナ/予防接種/皮膚科）は各用件フローの診療科聴取直後に分岐し、それぞれ専用終話アナウンスへ流す。
- 変更 / キャンセルでは `clinicalDepartment2` を必ず聴取し、空文字以外が入れば status=8（複数診療科あり）に上書き。空文字なら status 維持。
- 変更 / キャンセルでは `reservationDate`（現在の予約日）が {TODAY} より前の場合、「予約日過去受付不可」アナウンス→希望日再聴取へ戻す（終話はしない）。

**今回の改訂で追加された制御（2026/04/12）**:
- 変更 / キャンセルフローで `clinicalDepartment2` に正式名称 or "不明" が格納された瞬間に status=8 を強制。後続の `desiredreservationDate` / `reason` 格納で status=1 に降格してはならない。
- 「肺」と認識された発話は、はい/いいえ判定ブロックすべてで「はい」と同義に扱う（生年月日確認 / 診察券番号確認 / 電話番号携帯確認 / 電話番号固定確認 / 用件確認 / 受診歴 / 紹介状確認 / 予約日過去確認）。
  - **Gen3 移管方針**: generate_by_OpenAI の Yes/No プロンプトに「肺」「せいかい」「あっています」「そう」「そうです」「そうですね」「はい持っています」を肯定同義語として明示。否定同義語は「いいえ」「いえ」「いや」「ちがう」「ちゃう」「ちがいます」「ふせいかい」「いいえ持っています」。

### 特記事項

1. **1回聴取禁止 / 再質問は合計2回まで / 空文字格納**: 再質問は最大1回（初回＋再聴取1回＝計2回）。1回聞き直しても有効回答が得られなければ該当パラメータに空文字 `""` を格納し、「かしこまりました。」と発話して次ステップへ進む。空文字格納も「データ格納成功」としてステータス判定トリガーになる。
2. **再確認必須項目（4項目）**: 生年月日 / 診察券番号 / 電話番号(固定) / 用件。専用フォーマット文言で確認。
3. **再確認厳禁項目**: 患者名 / 電話番号(携帯) / 受診歴 / 紹介状確認 / 診療科 / 予約希望日 / 診療科2 / 現在の予約日 / 理由 / 確認内容。generate_by_OpenAI モジュールのプロンプトで復唱・「〜ですね」「〜を承りました」を禁止する旨を明記。
4. **「肺」＝「はい」の救済（今回追加）**: はい/いいえを聞く全ブロックで「肺」を肯定同義語として扱う。prompter は Yes/No テンプレート（SKILL_A〜E_*.md）に同義語リスト「はい／そう／そうです／そうですね／あっています／せいかい／肺／はい持っています」を埋め込む。否定同義語は「いいえ／いえ／いや／ちがう／ちゃう／ちがいます／ふせいかい／いいえ持っています」。
5. **複数診療科 status=8（今回強化）**: 変更 / キャンセル フローの `clinicalDepartment2` 格納時、正式名称 or "不明" が入れば status=8 を確定。以降の `desiredreservationDate` / `reason` 格納で status=1 に降格してはならない。Gen3 では ContextMatchRouter で `clinicalDepartment2 != ""` を判定して save2db で status=8 固定、その後の希望日/理由格納時の save2db では smsFlag=1 のみ更新し status には触れない設計にする（memory: feedback_save2db_shared_per_block を参照、1ブロック1save2dbで contextName 管理）。
6. **SSML 禁止（Gen3 ルール）**: customer_doc の `<dtmf/>` `<dtmf digit="8"/>` `<break time="150ms"/>` `<break time="200ms"/>` `<speak type="telephone" breakc="200ms">` `<speak type="digits" breakc="50ms">` はすべて **Gen3 では削除**。DTMF ガイダンスは tts_g の通常文言に置き換え、電話番号/診察券番号の桁区切り読み上げは tts_g の読み方（半角スペース挿入）で再現する。Gen2 のルール「SSML絶対保持」は Gen3 には継承しない。prompter はこの方針を generate_by_OpenAI の各プロンプトに明記しないこと（prompter はプロンプト記述のみ、memory: feedback_prompter_scope）。
7. **キーワード同義語（診療科分岐トリガー）**: 診療科聴取の ContextMatchRouter 分岐は以下のキーワードを見る。
   - インフルエンザワクチン系: 「インフル」「インフルエンザ」「インフルワクチン」「インフルエンザ予防接種」「インフル予防接種」
   - コロナウイルスワクチン系: 「コロナ」「コロナウイルス」「コロナワクチン」「コロナウイルスワクチン」「コロナ予防接種」「コロナウイルス予防接種」
   - 予防接種（汎用）: 「ワクチン」「予防接種」「接種」（インフル/コロナのいずれにも一致しない場合のみ）
   - 皮膚科: 「皮膚科」「皮膚」「肌」
8. **用件分類キーワード**:
   - 予約（新規）: 「予約」「お薬」「よやく」「しんきよやく」「新規」「しんき」「鍼灸薬」「しんきゅうやく」「通訳」「ようやく」

   - キャンセル: 「やめる」「取り消し」「取り消す」「取消」「やめたい」「経験」「建設」「キャンセル」

9. **数字認識ルール**: 「よん」「し」→"4"、「ご」「ごお」「ごー」→"5"。「for」「C」「号」「go」等の誤変換を禁じる。generate_by_OpenAI の数値正規化プロンプトに記載。
10. **診察券番号 8桁バリデーション**: 「わからない」「持っていない」発話時は桁数チェック免除・再確認スキップ・そのまま次ステップへ。8桁以外が入力された場合のみ1回再聴取→空文字格納。
11. **電話番号 桁数バリデーション**: mobile/ip050 は11桁、landline は10桁。不一致は1回再聴取→空文字格納→status=1、smsFlag=0 のまま次へ。携帯ショートカット（入電番号でよいか）で肯定された場合は incoming_phone_number を additionalPhoneNumber にコピー、smsFlag=1。
12. **現在の予約日 過去日制御**: 抽出後 {TODAY} 以前でも **まず再確認（復唱）を行う**（過去日確定前に間違い訂正の機会を与える）。肯定後に [予約日過去受付不可] アナウンスを発話し、予約希望日ステップに戻す（終話はしない）。Gen3 では Date of Call Classifier 的な日付比較ブロック＋ContextMatchRouter で実装。
13. **年補完ルール**: 現在の予約日（reservationDate）にのみ適用。±6ヶ月以内は当年、6ヶ月超なら過去日は翌年扱い（予約受付文脈では原則 future 補完）。相対表現（今日／明日／あさって／〇日後／〇週間後）は {TODAY} ベースで即確定、追加補完なし。予約希望日（desiredreservationDate）は自由テキストで格納（「連休明け」等もそのまま）。
14. **理由 / 確認内容 はユーザーへ復唱厳禁**: `reason` `details` は内部記録用。AI から「お大事に」などの同情・意見・質問を発話しない。終話挨拶の中でも内容に言及しない。prompter は自由テキスト抽出テンプレート（SKILL_E_*.md）ベースで、「抽出のみ・発話で触れない」を明示。
15. **聴取項目順序（シーケンシャル）**: STEP1〜STEP5（共通）→ STEP6 用件分岐。STEP5 までは用件に関係なく全件聴取を完了させる。
16. **皮膚科受付終了案内**: 2026年3月末で皮膚科診療終了。診療科 / 診療科2 いずれでも「皮膚科」「皮膚」「肌」が含まれたら [皮膚科案内] 終話へ。endpoint=通話完了, status=0 維持, smsFlag=0。
17. **URL / ヒアリングシートは無視** (memory: feedback_gen2_customer_doc_priority)。yaml セクション＋function セクションを一次情報、プロンプトセクションは二次情報として扱う。
18. **TTS形式**: `{tts_g:...}` 全小文字。`{tts_ai:...}` / `{TTS_AI:...}` は使わない。SSML タグは削除済み前提で gen_properties.py へ入力。
