### 移管方針
- Gen1「箱」構造 → Gen3 9 ブロック型に変換。`挨拶` → opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「セントラルクリニック・伊島です」（**注: 元資料の挨拶文言が「セントラルクリニック・伊島」となっており、施設名は「岡山中央病院」だが法人内のクリニック名で挨拶している模様**。director は元資料 main.yaml 行 22 を確認の上、岡山中央病院の正式名称に差し替えるか元のまま維持するかを判断する）。
- `時間外切断` サブシナリオ → date_of_call_classifier ブロック（addCurrentDate: Yes 自動付与対象）。受付時間外は `時間外`「ただいまの時間帯は、AI 電話の対応時間外です」案内 → 終話。
- 用件は **3 分類**: `診察の予約について` `人間ドックについて` `その他ご相談`。Pattern A（用件 OpenAI 直後分岐 + 復唱、3 復唱）。
- 用件確定後の主分岐（メイン `用件を判断`）:
  - 診察の予約について → `診療科聴取へ` → 診療科聞き取り → 希望日聞き取り（フリー聴収）→ 連絡事項確認 → 完了フラグ → 終話アナウンス
  - 人間ドックについて → `人間ドック案内`「健康増進センターにて承ります」 → 健康センター番号案内 0570-001789（注: yaml では 086-252-3222、director は要確認）→ 3 回繰り返し → 終話
  - その他ご相談 → `その他ご相談`（「それでは、」）→ お問い合わせ内容を → 問い合わせ内容（自由テキスト相談）→ 連絡事項確認 → 完了フラグ → 終話アナウンス

### 施設固有の特殊ルール
- **挨拶文言が「セントラルクリニック・伊島」**: 元資料 main.yaml 行 22 で `セリフ: セントラルクリニック・伊島です。` となっており、岡山中央病院本体の名乗りではない。**法人グループ内の系列クリニック扱いで通話を受けている可能性**あり → director は顧客（岡山中央病院）に正式な挨拶文言を確認するか、元資料のまま「セントラルクリニック・伊島です」で進めるかの判断が必要（**N-* で確認項目として明示**）。
- **健康増進センター案内 086-252-3222**: 人間ドックについて → `健康センター番号案内`「健康増進センターの電話番号は、086-252-3222 です」 → **3 回繰り返し**（3 回ループ）→ 終話。元資料の `0570-001789` は箱名のみで、セリフ実体は 086-252-3222 → director は両者の対応関係を確認。
- **3 用件のみ受付（予約変更/キャンセル/確認は無し）**: 西宮渡辺・医誠会・中部徳洲会等の標準予約フローと異なり、本フローは 3 用件しか受けない。**変更/キャンセル/確認の発話は「診察の予約について」エンティティ内の値リストに統合済み**（用件聴取 yaml 行 21: `予約のキャンセル / 予約のキャンセル / 予約変更 / 予約を変更 / 薬のキャンセル` 等が `診察の予約について` 値配下に含まれる）。Gen3 ではこの仕様を維持するか分割するかは director 判断。**現行の合意は合体運用**（director は変更/キャンセル/確認も「診察の予約について」 1 系統で扱い、後段の自由テキストで詳細を担当者に伝える）。
- **その他ご相談ルートの自由テキスト聴取**: `問い合わせ内容` サブフロー（ファイル `問い合わせ内容.yaml`）+ `連絡事項確認` サブフローで自由テキスト相談を受け付ける。Gen3 では SKILL_E（自由テキスト）相当で実装。
- **聞き取れず終話**: `聞き取れず終話`「お電話が遠いようで、回答が確認できませんでした。恐れ入りますが、おかけなおしください」 → 終話。代表電話案内なし（短い文言で離脱）。
- **タイトル_2「混み合っている」案内**: `タイトル_2`「只今電話が混み合っております。恐れ入りますが、おかけ直しください」 → 終話。**どの分岐から到達するかが yaml 上で明示されていない**（dead code 候補）。director は元資料を確認、未到達なら除外、到達があれば termination_patterns に保持。
- **Brekeke 連携**: `API_DRJOY` `API_DRJOY_CONTEXT_SETTINGS` は標準処理（opening 冒頭に吸収）。

### director が見落としやすいポイント
- 最初の `API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `API_DRJOY` → `テストルート` → `時間外切断` → `連絡先` の 6 箱は Gen3 では opening 冒頭処理 + date_of_call_classifier + announcement「AI 電話にてご用件を承り、担当者よりお電話いたしますので、予めご了承ください」に吸収。
- 「セリフ: none」かつ振り分けのみの「箱」（`完了フラグ` `用件NGCP` `電話番号NGCP` `タイトル_1` `テストルート` `用件を判断` `セットした用件の分岐 / _1` `用件を確認` `count上記以外 / _1` `count再質問` `count聞き直し` `用件のセット` `用件再設定へ` `否定` `3 回ループ` `繰り返します。` 等）→ Gen3 で独立ブロック不要、前段 next 書き換え or context_match_router 吸収。
- `生年月日聴収へ`（「続いて、」）/ `連絡先聴収へ`（「続いて、」）/ `診療科聴取へ`（「それでは、」）/ `タイトル`（「それでは、」）/ `お問い合わせ内容を`（「お問い合わせ内容を」）/ `その他ご相談`（「それでは、」）等の **短い繋ぎ TTS は次段 hearing/subflow ブロックの導入文言にマージ**。
- **聴取順序**: 連絡先（announcement）→ 患者氏名 → 連絡先聞き取り → 生年月日 → 診察券番号確認 → 用件聴取 → 用件分岐 → （診察予約 or その他ご相談ルート、または健康増進センター案内）。**個人情報聴取が用件聴取の前**に来る点が他施設と異なる（西宮渡辺・医誠会等は用件 → 個人情報の順）。director は元資料の順序を維持すること。
- 用件聴取サブフロー内エンティティ `診察の予約について` の値リストに **予約変更/キャンセル/確認関連の発話が混在**（位置・一・印刷 等の誤認識補正含む）。Gen3 では entity_classification.md に分類済みのため director は転記不要だが、**「印刷」「位置」「一」「いんさつ」「インサツ」が誤認識補正用のセーフリストとして残されている**ため、prompter プロンプト内で「同音異義語の積極的補正」を明示することで OpenAI 側でカバー（SKILL_A 系の正規化プロンプト）。
- 「もしもし」エンティティはなし（北里大・多摩総合と異なる）。`用件聴取.yaml` 内の `もしもし` カスタム entity は空配列 → 無視 OK。
- `テストルート` 箱は **削除**（テスト用入口）。
- `タイトル_2` 箱の到達経路を確認 → 未到達なら削除。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字（①②③⑩等）禁止。
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**（scaffold 付与 prefix）を使う。
- **CMR conditions は必ず最後に `match: "other"` + next 明示**。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。Gen1 動的タグ `<? $var ?>` → `<% var %>` に変換。
- 患者氏名は PatientName サブフロー 1 枠（patient 1 名のみ、入電者=受診者前提）。
- entity_classification.md / entity_stt_patch.json / entity_openai_synonyms.md にエンティティは自動分類済み。director は転記不要、scaffold/properties が自動取り込み。
