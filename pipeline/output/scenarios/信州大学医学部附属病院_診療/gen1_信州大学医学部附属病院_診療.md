### 移管方針
- Gen1「箱」→ Gen3 9 ブロック型へ変換。opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「お電話ありがとうございます。信州大学医学部付属病院の、予約専用、AI 電話です」+ 「12/25追加アナウンス」箱の「このお電話は約3分〜5分で完了します。」を冒頭_アナウンスに合流。
- **用件分岐は Pattern A**（用件聞き取り OpenAI 直後に 4 分岐: 新規 / 予約変更 / 予約キャンセル / 予約確認 + DTMF 1〜4 並走）。echo_back（診察予約復唱 / 予約変更復唱 / キャンセル復唱 / 予約確認復唱）を各分岐直後に配置。
- 元資料には用件聞き取りが **2 系統**（`用件.yaml`「診察の予約を取る/変更する/キャンセルする/確認する」と `用件未聴取.yaml`「予約を取る/変更する/キャンセルする/確認する」）あり。短縮文言の `用件未聴取.yaml` は **電話番号判別で渡部・清水以外（=時間判断経由）に使うショートバージョン**。Gen3 では同一 hearing ブロック 1 本に統合し、TTS 文言は本番採用版（`用件.yaml`）を採用（director 判断、両 yaml の差分は文言のみ）。
- **CT/MRI 検査予約は別レーン**: 用件聞き取りで「CT」「MRI」発話を検出 → 「CT ですね/MRI ですね」復唱 → 通常用件分岐に戻す（CTMRI フラグを context に保持）。診療科聞き取りで再度 CT/MRI に分岐するため、director は CT/MRI context を **用件層と診療科層の両方** で扱う必要あり。
- **診療科聞き取りはハイラルキ二段**: ① 親エンティティ（呼吸器 / 消化器 / 脳神経 / 産婦人科 / 内分泌 + 分岐なし診療科）→ ② 親が「分岐あり」だった場合のみ追加 hearing で内科/外科（または産科/婦人科/妊孕科）を聞き直し → ③ 確認復唱（`診療科の読み`）。Gen3 では **親hearing → context_match_router → 子hearing（5 種類 × 必要時のみ）→ 確認 echo_back** で再構成（director 判断）。子 hearing が 5 種類あるが scenario_flow ではどれも近い構造のため `subflow ブロック` 化せず `hearing` ブロックを 5 つ並べる方針推奨。
- **入電時間判断（時間判断.yaml）**: 12/30〜1/5 を年末年始休診として個別判定 + 平日 8:30〜16:30 + 祝日チェック（サブシナリオ呼び出し）→ date_of_call_classifier ブロックに集約。**addCurrentDate: Yes 自動付与対象**。
- **電話番号判別**は AI 受付電話番号自体の判別（`渡部` `清水`）→ 該当時のみ用件聞き取りへ直行、それ以外は時間判断経由。Gen3 では incoming-classifier 側で吸収（main JSON 入口の Pattern2 判定で扱う、director 判断）。
- サブフロー（患者氏名 / 生年月日 / 電話番号 / 診察券）は Jump to Flow 経由でサンプル JSON 完コピ。
- 自由テキスト hearing は **理由_症状.yaml の「症状確認」/ 確認内容の聞き取り.yaml の「予約確認内容/変更理由/キャンセル理由」/ 紹介元医療機関名.yaml** の 3 種類。output_format=text、SKILL_C 自由テキスト、5 秒無言で打ち切り。

### 施設固有の特殊ルール
- **受付時間**: 平日（月〜金）8:30〜16:30、年末年始 12/30〜1/5 休診（時間判断.yaml の `稼働日： 4/10` 箱で個別日を逐次比較）。祝日例外は `祝日チェック` サブシナリオに委譲され、Gen1 でも詳細未明示 → director デフォルト適用、acceptance_times に集約。
- **稼働開始日 4/10**: `稼働前` 箱に「この番号は、4月10日、朝8時半より利用開始となります」TTS あり。**移管時点（5/1）で稼働済みのためこのアナウンスは Gen3 で再現不要**（director 判断、設計書からは削除）。
- **代表電話案内（聞き取り不可・診療科 NG・予約以外）**: 0570-00-3010、月〜金 8:30〜17:00、土日祝休。失敗系 termination_pattern に集約。
- **紹介受診重点医療機関**: 紹介状なしの場合、初診時選定療養費 7,700 円の自己負担案内 TTS。**頭痛外来のみ別文言**で「初診時選定療養費 7,700 円」案内のみ（紹介状なし → そのまま受付可）。それ以外の診療科で紹介状なし → 代表案内へ分岐（main.yaml 中の `頭痛外来以外` 箱の TTS「申し訳ございません、当院は紹介受診重点医療機関に指定されており、…まずは地域の医療機関にご相談ください」）。
- **クローズ診療科 5 種（AI 受付不可）**: 血液内科 / 精神科 / 子どものこころ診療部 / 海外渡航者ワクチン外来 / 移植外科。それぞれ専用 TTS で別電話番号案内 → 代表案内 termination。
  - 血液内科: 「紹介元医療機関から外来予約センターへ」案内（電話番号なし）
  - 精神科: 0263-37-2846、平日 15:00〜16:30
  - 子どものこころ診療部: 0263-37-3381、平日 15:00〜16:30
  - 海外渡航者ワクチン外来: 0263-37-3475、平日 13:00〜16:00（**ただし用件「新規予約」かつ受診歴ありの場合のみ通常フロー**: 受診歴.yaml で初診/再診を判定 → 再診なら `海外渡航者ワクチン外来判断` で入電時刻判断 → 時間内なら 0263-37-3475 へ Blind Transfer、時間外なら代表案内）
  - 移植外科: 0263-37-2930、平日 8:30〜17:00
- **海外渡航者ワクチン外来の専用転送**: 受診歴ありの再診のみ受付、時間内（13:00〜16:00）なら専用番号 0263-37-3475 へ Blind Transfer、転送失敗時は別 TTS（`転送不可`/`転送不可_1`「申し訳ございませんが、担当部署が混み合っているため…」+ 0263-37-3475 を 2 回繰り返し）。**director デフォルト 8（転送失敗 1 段階）に該当**するが、TTS 文言は施設固有のため明示が必要。
- **CT/MRI 検査予約**: 用件聞き取り段階で CT/MRI 発話検出 → CT/MRI フラグを context 保持 → 診療科聞き取り「通院されている診療科をおっしゃってください」（受診済み前提）に流す。脳神経外科 + CT/MRI の組合せのみ「END_戻る：脳神経外科」専用出口あり（脳神経外科は新規予約不可だが CT/MRI 検査予約は受付）。
- **新規予約の追加質問**: ① 当院受診初めてかどうか（受診歴.yaml）→ 初診/再診で出口分け。② 産科のみ「出産予定日」聴取（出産予定日.yaml は **出産予定日 + 妊娠週数の二段聴取**、わからない場合は「わからない」発話可）。③ 一部診療科でペースメーカー有無確認（ペースメーカー.yaml は CT/MRI 検査時のみ前段に「<? $CTMRI ?>検査を受診される方へ」TTS で起動）。
- **予約変更の追加質問**: ① 予約日聴取 → ② 変更希望日聴取（変更希望日.yaml）→ ③ 残薬確認（残薬確認.yaml は **3 値分岐**: 1=足りる / 2=足りない / 3=処方されない、DTMF 並走）→ ④ 変更理由（理由_症状.yaml の「変更理由確認」、ただし「2.体調変化のため」を選んだ時のみ追加で症状フリーテキスト聴取）。
- **予約キャンセルの追加質問**: ① 予約日聴取 → ② キャンセル理由（理由_症状.yaml の「キャンセル理由確認」、こちらも「体調変化」選択時のみ症状フリーテキスト追加）。
- **予約確認の追加質問**: 予約確認内容を自由テキストで聴取（「次回の予約日時を教えて欲しい」等）。
- **3 回失敗時**: 各サブで `count再質問` 3 回到達 → `END_聞き取れず` → main の `失敗` 箱（代表案内 0570-00-3010）→ `代表案内フラグ` → `終話`「それでは失礼いたします」。Gen3 では termination_patterns に集約。
- **予約日聴取は DTMF 4 桁並走可**: 月+日の 4 桁 DTMF を受付、過去日付なら +1 年で確定する自動補正ロジックあり（director デフォルト適用、特殊ロジックは Gen3 標準の subflow が吸収する想定）。
- **救済語/同音異義救済**: 「Suica → 麻酔科蘇生科（ペインクリニック）」「BB/GB/DB → 耳鼻咽喉科頭頚部外科」「白内障/緑内障 → 眼科」「カレー → 糖尿病内分泌代謝内科」「物忘れ/認知症 → 脳神経」「狂犬 → 海外渡航者ワクチン外来」「シンゲ → 心臓血管外科」等多数。**entity_classification.md / entity_stt_patch.json に全件あり**、director は転記不要。
- **DTMF 受付（用件聞き取り）**: 1=新規予約 / 2=変更 / 3=キャンセル / 4=確認。冒頭ガイダンスで明示しないが裏で受付（`用件.yaml` の DTMF 分岐）。
- **復唱（echo_back）対象**: 用件 4 種 + 診療科のみ。氏名/生年月日/電話番号/診察券はサブフロー内で完結。

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `12/25追加アナウンス` → `API_DRJOY` の 4 箱は Gen3 では opening 冒頭処理に吸収。明示ブロック化不要。
- 「セリフ: none」かつ振り分けのみの「箱」（`携帯電話番号チェック`、`完了フラグ`、`用件を判断 / _1 / _2 / _3`、`分岐`、`電話番号判別`、`時間外ノードへ：開発時は用件接続`、`代表案内フラグ`、`通話完了`、`産科のみ分岐`、`海外渡航者ワクチン外来判断`、`血液内科・海外渡航者ワクチン外来除外`、`診察券`、`用件NG`、`診療科NG`、`分岐なし診療科セット`、`分岐あり診療科セット`、各 `読み取得` サブシナリオ呼び出し系、各 `セット：xxx`）は Gen3 で独立ブロック不要 → 前段の next 書き換え or context_match_router の一部として吸収。
- **「予約変更」と「予約キャンセル」が 3 経路で出現**: ① main の `用件を判断 / _1 / _2`、② 残薬確認後の `用件を判断_3`、③ `確認内容の聞き取り.yaml` の「用件の確認」/「用件を判断」、④ `理由_症状.yaml` の「セットした用件の分岐」。**Gen3 では用件 context を一度確定したら以降は context_match_router で参照する**ことで重複聴取を避ける（北里と同じ方針）。
- **`携帯電話番号チェック` 箱の意味**: AI 受付電話番号判別で「携帯電話取得あり」場合（incoming-classifier で携帯番号と判定された通話）→ `用件を判断_1` で **再診/初診の選択肢が並ぶ用件確認**（再診ありの想定 = 折り返し用通話の可能性）/ なし → `用件を判断_2`（標準用件）。Gen3 では incoming-classifier 側で携帯/固定を分け、用件 hearing は 1 本に統合可能（director 判断、Gen3 ではこの分岐を維持するか統合するか要検討）。
- **`用件を判断_3` は変更/キャンセルのみが入口**: 変更希望日聴取後の理由分岐の手前で「変更/キャンセル」を再確認している。Gen3 では context 参照で機械的に分岐 → 独立ブロック不要。
- **`分岐` 箱の「用件無」分岐**は OpenAI が用件分類できなかった場合のフォールバック → `用件未聴取` サブシナリオ（短縮版用件聴取）→ 用件再分類。Gen3 では用件 hearing の retry_count + 用件 NG 分岐で吸収。
- **診療科確認復唱で「分岐あり/分岐なし」発話が再度出た場合**は `再聴取`（失礼しました）→ 該当エンティティをセットし直して再聴取（聞き直し）。**「肯定否定」（はい/いいえ等の混在発話）は `読み替え` 箱経由**で診療科名を Gen1 内部表記に正規化（呼吸器内科/感染症内科/アレルギー内科 → 「呼吸器感染症アレルギー内科」、糖尿病・内分泌代謝内科、乳腺・内分泌外科 等）。**Gen3 では診療科 context は OpenAI 出力の正規化済み診療科名 1 個に統一**し、「呼吸器感染症アレルギー内科」のような括り表記は OpenAI 正規化辞書で吸収（director 判断、entity_openai_synonyms.md 参照）。
- **新規予約 → 産婦人科 → 産科 → 出産予定日聴取** の三段ネスト。出産予定日聴取は **二段（出産予定日 + 妊娠週数）**であり、聞けなければ「わからない」で次へ進む寛容モード。Gen3 では subflow ブロック 1 本にまとめるか、hearing 2 本連結か director 判断（PatientName と同じ考え方で subflow 推奨）。
- **`電話番号区切り.yaml` は読み上げ用補助サブシナリオ**（電話番号を桁数別に「090-1234-5678」等にハイフン区切り）→ Gen3 では TTS 動的値構文 `<% var %>` の readable 形式で吸収（director 判断、別ブロック化不要、北里と同方針）。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**（丸数字 ①②③⑩等は使用禁止）。元資料に出口記号は無いが、`END_紹介状無：頭痛外来` `END_紹介状無：戻る` `END_紹介状有：戻る` のようなコロン記法はそのまま使わず、`END_紹介状なし_頭痛外来` 等の安全名に置換。
- **ContextMatchRouter.reference_module は subflow を参照する場合 `jump_{step名}`**（scaffold が付ける prefix）を使う。subflow ブロックの step 名そのもの（例: `電話番号聴取`）ではダメ。
- **CMR conditions は必ず最後に `match: "other"` + next 明示**。`TODO_other_target` プレースホルダーを残さない。
- **転送先は 1 系統のみ**（海外渡航者ワクチン外来の 0263-37-3475、Blind Transfer）。新規予約や予約変更/キャンセルは AI で完結し、Dr.JOY 側で 3 営業日以内に折り返し連絡する設計。**「専用番号転送」サブシナリオは 0263-37-3475 専用**（main.yaml の `転送不可` TTS が同番号を案内）。
- **終話 TTS の集約**: `終話`「それでは失礼いたします」/ `失敗`（代表案内 0570-00-3010、月〜金 8:30〜17:00）/ `頭痛外来以外`（紹介受診重点医療機関 案内）/ `稼働前`（4/10 まで利用不可、移管後は不要）/ `（未設定）1/6以降時間外` / `時間外_1`（受付時間外案内）→ Gen3 では携帯/固定の TTS 分割（director デフォルト 9）+ termination_patterns 集約。
- **TTS は `{tts_g:...}` 小文字のみ、SSML 禁止**。Gen1 変数記法 `<? $診療科の読み ?>` `<? $予約日の読み ?>` `<? $CTMRI ?>` は Gen3 では `<% var %>` に変換。
- 患者氏名は PatientName サブフロー 1 枠に充当（patient 1 名のみ、入電者 = 受診者前提の設計）。
