- **問題**:
  - OpenAI_出産予定日聴取（プロンプト 856-862行）の「出力仕様」セクションで：
    ```
    - 日付が抽出できた場合: `YYYY-MM-DD` 形式で出力
    - 「わからない」系の発話: `わからない` のみ出力
    - 上記以外: `NO_RESULT`
    ```

  - OpenAI_予約日聴取（プロンプト 1340-1345行）でも同様

    - 予約日聴取: output_format: "datetime", output_labels: [] （空）

  - 実装では OpenAI が直接日付パースを担当し、「YYYY-MM-DD」テキストを出力することは正しい

- **リスク**:
  - フロー JSON の hear ing モジュールが datetime 型の場合、validation が異なる可能性
  - 「わからない」「NO_RESULT」の出力がどうフロー側で処理されるか不明確

- **修正方針**:
  - プロンプトの「出力仕様」セクションで「YYYY-MM-DD 形式のみを出力する」と明確化
  - または「わからない」「NO_RESULT」の扱いを設計書 notes と統一する（skip フォールバック時の動作を明記）

### W-002: 日付型プロンプトのオウム返しリスク（YYYY-MM-DD正規化ロジック）

- **修正担当**: prompter  <prompter>
- **対象モジュール**:
  - OpenAI_出産予定日聴取
  - OpenAI_予約日聴取

- **問題**:
  - STEP3/STEP4 で日付パース・正規化のロジックが複雑であり、複数の表現形式（月日表現、和暦、漢数字、相対日付等）を処理している

    - 入力「12月25日」+ 基準日2026-05-01 → 出力「2026-12-25」（同年）
    - 入力「わかりません」→ 出力「NO_RESULT」
