### 判定した作業種別

- **作業種別**: Gen2 → Gen3 移管
- **施設名**: 恵佑会札幌病院
- **シナリオ名**: 診療（歯科口腔外科予約申し込み）
- **対象フロー**: `恵佑会札幌病院$診療`（メインフロー）+ 4サブフロー（診察券番号/氏名/生年月日/電話番号）
- **判定根拠**: 入力資料が `docs/migration/gen2_*.md` 形式で、#yaml + #function + #プロンプト の Gen2 本体プロンプト構造を保持。移管ノートにも「Gen2→Gen3」が明示

### 抽出した要件サマリー

- **最上段分岐（Pattern A）**: 冒頭アナウンス直後に「申し込み方法確認」（ネット=1 / 電話=2）を配置。リトライ1回のみで、2回目失敗は END_代表案内_申し込み方法（011-863-2105 を3回連呼、status=0）
- **SMS用フロー**: ネット選択時は Incoming_phone_type（MOBILE/FIXED/OTHER）で3分岐。MOBILE は着信番号確認Yes/No、FIXED/OTHERは携帯番号手入力→復唱。END_SMS送信案内（status=8, smsFlag=3）
- **通常フロー用件4分岐**: 予約 / 変更 / キャンセル / その他問い合わせ
  - 予約: 受診歴（初診=紹介状有無→医師名 or 希望医師 / 再診=予約希望日_再診）
  - 変更: 予約日→予約希望日
  - キャンセル: 予約日→キャンセル理由
  - その他: 内容確認
- **共通ヒアリング**: 診察券番号→氏名→生年月日→連絡先電話番号（すべてサブフロー）
- **終話パターン**: 4用件×2種別（MOBILE/FIXED）= 8 END + END_受付不可診療科 + END_SMS送信案内 + END_代表案内_申し込み方法 = 11パターン
- **受付不可診療科**: 歯科口腔外科以外の18科（総合内科〜婦人科）キーワードを全ヒアリング項目で監視、検出即 END_受付不可診療科（status=2）
- **status/smsFlag enum**: Gen2 function に合わせ status=0/1/2/8、smsFlag=0/1/2/3 の4値ずつ
- **特殊ルール**: 診察券番号の3ケース分岐、患者名の苗字のみ再質問、医師名の強制進行、生年月日の年判定＋復唱＋専用エラー文言、電話番号の桁数×prefix両方向チェック（10桁/11桁 × 090/080/070/050）、SSMLタグ保持（<speak type=\"telephone\">, <dtmf digit=...>, <dtmf2/>, <break>, <sub alias>, <prosody>）
