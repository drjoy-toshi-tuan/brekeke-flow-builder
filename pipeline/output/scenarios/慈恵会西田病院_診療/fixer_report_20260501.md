### C-001: 医師名逆引きロジックの矛盾  
**修正担当**: prompter  <prompter>

**対象モジュール**: OpenAI_診療科聴取_再診（lines 432-437）/ OpenAI_診療科聴取_既存（lines 900-905）

**問題**: 
設計書の step_details（line 816）では：
```
- input: "医師名（奥田/加隈/吉田/三股/山口/柴田/小田/森/清水/増田/中川/島袋/波津久/武内/木許）の発話"
  output: "OpenAI が医師名から該当診療科を逆引きして返却（不明時は NO_RESULT）"
```

と明記されています。つまり「医師名 → 診療科を返出」すべきです。

しかし、プロンプムの STEP4（lines 432-437）では：
```
## 【STEP4：医師名による逆引き判定】
入力に以下の医師名のいずれかのみ（診療科名のキーワードを含まず）が含まれる場合は、
現時点では医師名と診療科の対応表が確立されていないため、
安全側に倒し NO_RESULT を返す。
```

と記載されており、医師名のみの場合は **NO_RESULT を返す** ように設計されています。

設計書コメント（line 1298-1299）でも「波津久先生 → 消化器外科 等の逆引きが正しく動くか要確認」と記載されており、逆引き機能を期待しています。

**修正方針**:
以下のいずれかの選択が必要です：
1. **医師名と診療科の対応表を整備**して、プロンプムを修正し「医師名 → 診療科を返出」する
2. **設計書の step_details を修正**して、「医師名のみ場合は NO_RESULT」に統一する

現在はプロンプむと設計書が矛盾しており、フロー動作が不確定です。

### W-001: 予約日聴取_既存の出力フォーマット表記の混在  
**修正担当**: prompter  <prompter>

**対象モジュール**: OpenAI_予約日聴取_既存（lines 1007-1147）

**問題**: 
設計書の step_details（line 915）では：
```
- input: "音声「4月15日」「来週の火曜」等 → YYYY-MM-DD HH:MM 正規化"
  output: "datetime（年は推論、比較日より前なら +1 年）"
```

と表記されており、フォーマットに「HH:MM（時刻）」が含まれています。

一方、プロンプむの出力仕様（line 1037）では：
```
- 日付が抽出できた場合: `yyyy/mm/dd` 形式で出力（時刻は出力しない、ゼロ埋め必須）
```

と明記されており、**時刻は出力しない** となっています。

同時に設計書の「#」区切り使用（line 1037）とプロンプむの「/」区切り使用との間にも表記ゆれがあります（ただし、実装上は "/" で統一するべき）。

**修正方針**:
- 設計書の step_details（line 915）の「HH:MM」を削除して「YYYY-MM-DD」に統一する
- またはプロンプむの出力仕様を「YYYY-MM-DD HH:MM」に変更する（ただし、DTMF 4桁からは時刻が抽出不可なため、これは現実的でない）

**推奨**: 設計書を「時刻は出力しない」に統一する方が合理的。
