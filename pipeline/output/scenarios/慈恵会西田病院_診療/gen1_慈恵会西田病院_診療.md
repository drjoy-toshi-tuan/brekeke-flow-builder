### 移管方針
- Gen1「箱」構造 → Gen3 9 ブロック型に変換。`挨拶`(「お電話ありがとうございます。西田病院の、予約専用AI電話です」)→ opening + 冒頭_アナウンス。
- `時間判断` サブシナリオ → date_of_call_classifier ブロック。**24h 稼働ベース**で、休診期間として **GW 期間(4/30 17時以降〜5/7 7:59 まで)のみ案内**して切断、それ以外は全時間帯営業中扱い。営業時間の時間帯判定は元資料に無い(director 判断: 平日土曜の営業時間を customer_doc で確認、無ければ 24h で実装)。
- 用件分岐は **Pattern A**(用件 OpenAI 直後に 4 分岐: 再診 / 予約変更 / 予約キャンセル / 代表案内)。各分岐に echo_back を直配置。**初診はエンティティとして拾うが、復唱なしで直接 END_初診代表案内**へ。
- 用件確定後の **「複数診療科 or ひとつ」分岐**(`診療科数確認.yaml`)あり。**変更 / キャンセル時のみ**確認、再診時は単一診療科で固定。複数の場合 `複数診療科名聴取（フリー）` で全診療科をフリー発話で取得 → メイン進行、ひとつの場合は `診療科聴取（医師名）` の通常 hearing。**Pattern B 併用**。
- サブフロー(患者名 / 生年月日 / 電話番号、診察券は無し)は Jump to Flow 経由。**診察券番号サブフローは元資料に無い**。
- `連絡先聴取.yaml` 内に **入電番号確認 → 別番号入力 → 携帯番号有無確認**の 3 段ロジックあり。Gen3 では PhoneNumber サブフローテンプレで完結。
- 用件別フロー:
  - 再診: 予約候補日フリー → 病院への追加連絡事項 → 生年月日 → 患者名 → 連絡先 → 終話
  - 変更: 予約日(現在)→ 病院への追加連絡事項 → 生年月日 → 患者名 → 連絡先 → 終話
  - キャンセル: 予約日 → キャンセル理由聴取 → 病院への追加連絡事項 → 生年月日 → 患者名 → 連絡先 → 終話
  - 代表案内: 即時「AI 電話では対応できません」→ 番号案内 → 切断

### 施設固有の特殊ルール
- **初診 NG**: 用件聞き取りで「初診 / 健康診断 / 紹介状」エンティティ → `END_初診代表案内` → 「初めて受診される方は、代表電話番号へおかけなおし下さい」案内 → 切断。**健康診断・紹介状の発話は初診扱いに集約**。
- **代表電話**: 0972-22-0180。3 回聞き取れず時 / 予約以外 / 初診案内ですべて 3 回繰り返し読み上げ。
- **受付時間案内文言**: 「日曜祝日、年末年始を除く、月曜から金曜、7時30分〜17時、土曜は7時30分〜11時30分」。
- **GW 休診期間**: 4/30 17 時以降〜5/7 朝 8 時までは「休診期間中のため、5月7日、木曜日、朝8時以降におかけください」案内 → 切断。**毎年メンテナンスが必要なリテラル日付**で、Gen3 では Script モジュールで個別ハンドリング。
- **DTMF(4 桁)受付**: 予約日聴取で「シャープ / アスタ + 月 2 桁 + 日 2 桁」を受け付ける。比較日より前なら +1 年で確定。
- **3 回失敗時**: 用件・診療科・予約日・氏名・生年月日・連絡先 各サブで count(5 / 8 / 9 以上で escalate)→ 3 回不可で `END_聞き取れず` → 番号案内 3 回繰り返し → 切断。
- **診療科 二段聴取**: 「消化器」発話時のみ「消化器内科 / 消化器外科」の二段聴取。**他科は単発で確定**。
- **医師名フリー聴取**: 診療科確認で否定 →「先生名」エンティティ含む発話なら **医師名フリー聴取サブシナリオ → 消化器外科判定**で診療科を逆引き。
- **予約候補日フリー聴取(再診ルートのみ)**: 「予約を希望される候補日を、教えてください。当日のご予約をご希望の場合は、代表電話へおかけなおし下さい」→ 自由テキスト hearing。**「当日予約 NG」案内が文言内に既に含まれているので独立 NG 分岐は不要**。
- **病院への追加連絡事項聴取**: キャンセル時のみ実行。
- **複数診療科名聴取(フリー)** の TTS は `<? $classification ?>` で「変更 / キャンセル」を動的差し込み。Gen3 では `<% var %>` に変換、context 名は `用件` で OK。
- **復唱(echo_back)対象**: 用件 3 種(再診 / 変更 / キャンセル)+ 診療科 + 医師名 + 分岐あり診療科 + 予約日 のみ。

### director が見落としやすいポイント
- 冒頭の `API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `API_DRJOY` → `時間判断` → `用件聴取へ`(セリフ「それでは、」)の 5 箱は Gen3 では opening 冒頭処理に吸収。
- セリフ none 振り分けのみの「箱」が大量(用件を判断 / _1 / _2 / _3、診療科数を判断、用件のセット、用件再設定へ、初診のセット、代表案内セット、セットした用件の分岐 / _1、smsflag1（再診）、smsflag2（変更）、電話切断へ、各種NG 9 種 等多数)→ 独立ブロック不要、前段の next 書き換え or context_match_router の一部として吸収。
- 短い繋ぎ TTS(`次へ`「ありがとうございます。では次に、」/ `氏名聴取へ`「続いて、」/ `終話へ`「かしこまりました。」/ `生年月日聴取へ`「つぎに、」/ `連絡先電話番号聞き取りへ`「最後に、」/ `病院への連絡事項聴取へ`「続いて、」/ `複数診療科聴取へ`「次に、」/ `診療科聴取完了`「では、次に、」/ `予約日聴取完了`「続いて、」/ `用件聴取へ`「それでは、」/ `代表案内 終話へ`「かしこまりました。」)→ 次段 hearing/subflow ブロックの導入文言にマージ。
- `用件を判断 / _1 / _2 / _3` の 4 箱は役割が違う: `_1` は代表案内分岐込み(用件聴取直後)、`_2` はキャンセル時のキャンセル理由ルート分岐、`_3` は変更時の連絡事項ルート分岐、`用件を判断` 自体は予約日 vs 予約候補日フリーの初期分岐。**役割を取り違えると分岐先が壊れる**ので元資料の `次の箱` 列を必ず確認。
- `smsflag1（再診）` / `smsflag2（変更）` は **Dr.JOY 側へ送る区分フラグ**(SMS 送信種別)を context にセットするだけの空箱。Gen3 では script ブロックで context をセット。
- 用件聴取の `肯定エンティティ` / `肯定（エンティティ）` / `否定エンティティ` / `否定（エンティティ）` / `肯定否定` 等の **微妙な命名違いが多数 同居**しているのは Gen1 の慣習。Gen3 では SKILL_B(Yes_No 判定)で 1 本化。
- `予約日聴取.yaml` の `生年月日の判定` 箱は **大正/昭和/平成/令和の元号判定**だが、**予約日では実質使わない**。Gen3 では date OpenAI(YYYY-MM-DD 出力)+ DTMF 並走で代替。
- `診療科数確認.yaml` の `用件を判断` 箱には **`再診` ルートが無い**(変更 / キャンセルのみ)。**再診は診療科数確認をスキップして直接単一診療科 hearing**(`診療科聴取へ`)。
- `診療科聴取（医師名）.yaml` 内の `①テストの為のルート変更` / `①聞き取りできず TESTルート変更` 箱の **丸数字「①」は環境依存文字で禁止**。Gen3 では ASCII または安全な日本語に書き換える。同様に `（医師名）` のような全角括弧入りファイル名 / モジュール名は scaffold で URL 安全な形に変換が必要。
- 患者氏名は PatientName サブフロー 1 枠に充当。**「お名前を、フルネームでおっしゃってください」**(「受診される方の」のような区別なし)。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字(①②③…⑩)等の環境依存文字は禁止。
- **ContextMatchRouter.reference_module は subflow を参照する場合 `jump_{step名}`**。
- **CMR conditions は必ず最後に `match: "other"` + next 明示**。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。`<? $... ?>` 構文は `<% var %>` に変換。
