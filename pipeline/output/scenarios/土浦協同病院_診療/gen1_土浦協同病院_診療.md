### 移管方針
- 主流れ: opening → API_DRJOY → 時間判断 → 8時半前_1（時刻分岐: 8時半前/17時以降/時間内）→ 用件聞き取り（4分岐）→ 用件別ルート → 個人情報聴取 → 紹介状分岐 → termination
- 用件は **4 分類**: `診察（=新規予約）` `予約変更` `予約キャンセル` `予約確認`。発話＋ DTMF 1/2/3/4 両対応。各分岐直後に echo_back 4 本 → Pattern A
- **新規予約ルートは複数前段ゲート**: `予約対応.yaml`（紹介状/健診結果/それ以外 3 分岐）→ 紹介状あり時のみ紹介元医療機関名聴取、健診結果あり時のみ「予防医療センターで受けたか？」確認 → 否定なら他院健診 termination（近隣医療機関ご受診）。それ以外なら通常診察予約ルート（再診確認 → 必要なら医師指示確認 → 選定療養費確認 → 症状確認 → 複数診療科確認 → 診療科聞き取り → 診察券番号 → 患者氏名 → 生年月日 → 連絡先）
- 予約変更ルート: `変更理由` 自由文 → `希望日聞き取り` → 診察券番号 → 患者氏名 → 生年月日 → 連絡先
- 予約キャンセルルート: `予約日の聞き取り` → `用件の確認_2`（変更/キャンセル再判定）→ `キャンセル理由` → `予約希望確認`（次の予約希望 Yes/No、Yes なら診察券番号〜連絡先聴取）
- 予約確認ルート: `確認内容の聞き取り` 自由文 → 診察券番号 → 患者氏名 → 生年月日 → 連絡先
- 連絡先 subflow 戻り値で `紹介状分岐`（時間内＆紹介状有り → 8時半前_2 で時刻再判定 → 営業時間内なら地域連携室転送、営業前/終了なら折り返しアナウンス。それ以外なら 完了フラグ → 用件別完了アナウンス）

### 施設固有の特殊ルール
- **受付時間**: 祝日除く月-金 8:30-17:30。8:30 前 / 17:30 以降は専用時間外 TTS（折り返しは翌診療日 8:30 以降）。年末年始休みは `時間判断.yaml` 内 `11/24分岐_1` で 12/27〜1/4 を捕捉、年明けは 1/5 8:00 から再開。Gen3 では acceptance_times + Script に集約
- **代表電話番号 3 種使い分け**:
  - 病院代表 029-830-3711: 折り返し連絡発信元、用件聴取不可・電話番号 NG・リハビリ・代表案内系の termination
  - 予防医療センター 050-1725-6511: 健診結果ありで予防医療センター以外の他院健診なら近隣医療機関案内、当院予防医療センターなら専用案内
  - 地域連携室: 紹介状あり時のみ Brekeke Blind Transfer（`地域連携室へ転送`「紹介状をお持ちの方は、このまま地域連携室に転送いたします。少々お待ちください。」→ API_DRJOY_1 経由 → `地域連携室転送` サブフロー）
- **症状確認は新規予約ルート前段ゲート**: 予約対応 → それ以外 → 再診確認 → 医師指示確認（再診で No の場合）→ 選定療養費確認 → 症状確認（自由文）→ 次へ_2 → 診察券番号
- **選定療養費確認**: 「当院では診療費とは別に、11000 円の選定療養費を徴収しています。ご了承の上、受診を希望されますか？」 Yes/No hearing。No なら `終話へ` で「選定療養費をご了承いただけない場合は、診療予約を行っておりませんので、近隣医療機関をご受診ください」 termination
- **再診確認 → 医師指示確認 の段差**: 「予約されるのは現在通院中の診療科ですか？」（再診確認）No → 「今回は土浦協同病院の医師の指示でのご予約でしょうか？」（医師指示確認）→ Yes/No で診療科聞き取りへ進むかどうか判定
- **複数診療科ルート**: 「今回希望する診療科は複数ありますか？」 Yes/No → Yes なら診療科聞き取り（メイン）→ `患者サポート外来` 確認 → 複数診療科確認_1 → 診療科フリー（自由文で 2 つ目以降を聴取）→ 用件の確認_1（リハビリだけ代表案内、診察 → 次へ_2、予約変更 → リハビリだけ代表案内）でループ。Gen3 では再帰呼び出しの代わりに**複数診療科ループ最大 3 回までで打ち切り** + 診療科あとで確認 termination
- **診療科聞き取りは分岐あり/なしの 2 段**: `00_00_分岐あり診療科` で「呼吸器」「消化器」「脳神経」「眼科」「整形外科」「産婦人科」が選ばれた場合のみ second turn で「外科 / 内科」「形成 / 整形」「産科 / 婦人科」を再確認、それ以外は `01_00_分岐なし診療科` で 50 科以上の単発判定
- **AI 非対応診療科**: リハビリテーション科 → 代表案内 termination、患者サポート外来 → 代表案内、産婦人科の二次健診（予約対応・健診結果ルート経由）→ `産婦人科終話へ`（直接総合受付へお越しください、月-金 8:30-11:30、健診結果・マイナンバーカードか資格確認証・医療受給者証・診察券持参）
- **紹介状あり時のみ地域連携室 Brekeke Blind Transfer**: 連絡先聴取後の `紹介状分岐` で「時間内＆紹介状有り」→ `8時半前_2` 再判定 → 営業時間内 → `地域連携室へ転送` → API_DRJOY_1 → `地域連携室転送` サブフロー。営業前/終了なら `営業前アナウンス` / `営業終了アナウンス`（折り返しは翌診療日 8:30 以降）→ termination
- **携帯番号で締め文言切替**: 携帯あり → 「2 日以内にショートメッセージ、もしくは病院代表番号、029-830-3711 からお電話にてご連絡」、携帯なし → 「2 日以内に病院代表番号、029-830-3711 からお電話にてご連絡」
- **DTMF 1〜4 並列**: 用件聞き取りで 1=診察予約 / 2=変更 / 3=キャンセル / 4=確認、予約対応で 1=紹介状 / 2=健診結果 / 3=それ以外
- **「もしもし」「もう一度」検知**: 用件聞き取り入口で `もしもし` `もう一度` の専用フォローアップ箱

### director が見落としやすいポイント
- `API_DRJOY_CONTEXT_SETTINGS` / `API_DRJOY` / `API_DRJOY_1` 系は Gen3 では opening 冒頭の標準処理に吸収 → 明示ブロック化不要（ただし `API_DRJOY_1` は地域連携室転送の前段で使われる固有処理 → 紹介状ルートの Brekeke Blind Transfer 直前のフラグセットとして残す）
- 「次へ」「次へ_1〜_3」「連絡先電話番号聞き取りへ」（"では次に、" "それでは、" "続いて、" "最後に、ご連絡先のお電話番号をお伺いします。"）は短い繋ぎ TTS → 次段ブロック先頭 TTS にマージ
- セリフ none 振り分けのみの箱（用件の確認 / _1 / _2、用件を判断 / _1、完了フラグ、完了フラグ（代表案内）、時間外フラグ、携帯電話番号チェック、紹介状分岐、複数診療科flagセット、複数診療科確認_1、患者サポート外来、リハビリだけ代表案内 / _1、肯定否定 / _1〜_5、8時半前_1 / _2、時間判別へ、時間判断、転送チェックポイント、電話切断へ / _1、確認内容の聞き取りへ、聴取失敗→フリー聴取へ）は独立ブロック不要
- `肯定否定.yaml` は 5 種類（_1〜_5）の同型 Yes/No サブフローを使い回し → Gen3 では用途別に hearing ブロック直接埋め込みで OK、わざわざ subflow 化不要（再診確認 / 医師指示確認 / 選定療養費確認 / 健診場所確認 / 予約希望確認 / 複数診療科確認 の 6 用途）
- `代表案内` `代表案内_1` の文言（"祝日を除く、月曜から金曜の、8 時 30 分から 17 時に、代表電話へおかけなおしください。…電話番号は、029‐830‐3711 です。繰り返します。…"）は **3 回繰り返し読み上げ** → 文言短縮しないこと。電話番号 029‐830‐3711 のハイフンは **`‐`（U+2010 ハイフン）でなく `-`（U+002D）に統一**してから case 出力
- 予約日の聞き取り subflow は和暦判定（大正/昭和/平成/令和）を全枚挙しているが、Gen3 では generate_by_OpenAI の output_format=datetime + addCurrentDate: Yes で吸収 → director判断 で単純な日付 hearing に
- 連絡先聴取の `市外局番5桁/4桁/3桁/2桁` 区切り処理は Gen3 phonetype subflow で吸収 → ノート不要
- `END_戻る：内科発話` `END_戻る：歯科口腔外科` `END_戻る：救急外来` 等の戻り値 context は SUBFLOW_RETURN_CONTEXTS 設計で main 側 CMR 分岐
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字や全角スラッシュ・括弧は環境依存 → 禁止
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**（scaffold prefix）を使う
- **CMR conditions は最後に `match: "other"` + next 明示**
- 患者氏名はサブフロー 1 枠 PatientName に充当（受診者本人前提、入電者と分離設計は無し）
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
- 「<? $予約日の読み ?>」「<? $連絡先電話番号（ハイフン区切あり） ?>」等の Gen1 動的タグは Gen3 では `<% var %>` 構文へ
