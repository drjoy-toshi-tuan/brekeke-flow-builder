## 4. generator への注意事項

### フロー生成時の注意

1. **3フロー分のJSONを生成すること**: 中之島CL$健診（メイン）、中之島CL$氏名聴取、中之島CL$電話番号聴取
2. **電話番号聴取サブフロー内に6つの saveCompletionFlag2db を配置すること**: smsFlag 1〜6 に対応する6モジュール。分岐は `@General$Script` で clinicalDepartment × full-body_checkup を判定する方式を推奨
3. **END_人間ドック / END_その他 の TTS は各1モジュールで共有可能**: 施設による文言差がないため、6つの saveCompletionFlag2db から2つの END TTS に集約できる
4. **全 TTS/STT/Retry Counter に save2db サブモジュールを接続すること**
5. **Script モジュールは `script_` プレフィックス必須**: script_携帯判別、script_携帯かその他、script_携帯ルート、script_その他ルート、script_結果返却_氏名
6. **冒頭チェーンの順序に注意**: wait → saveContextModel2DB → acceptance_times → incoming-classifier（本設計での順序）

### properties 生成時の注意

1. **3フロー分のプロパティを生成すること**
2. **BLOCKER の TTS 文言（時間外・非通知）は `TODO_要確認` のまま出力すること**
3. **電話番号確認の TTS には `{additionalPhoneNumber}` の変数展開が必要**
4. **携帯確認の TTS には `{telephoneNumber}` の変数展開が必要**
