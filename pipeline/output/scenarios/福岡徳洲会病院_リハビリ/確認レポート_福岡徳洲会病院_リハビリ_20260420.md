### 抽出した要件サマリー

- **フロー構成**: 冒頭アナウンス → 患者氏名聴取 → 用件聴取 → status 判定 → 終話（成功 / 失敗）の超シンプル 3 ステップ構成。
- **聴取項目**: patientName（氏名・カタカナ変換あり）、reason（用件・自由発話）の 2 項目のみ。function schema には他 14 フィールドの required 定義があるが聴取せず空文字保存でよい。
- **復唱・再確認**: 完全禁止（§II, §1-8）。Re-confirmation・DOB Re-confirmation モジュールを一切使わない。
- **再質問**: 各項目 1 回のみ。固定文言（完全一致）で再質問し、超過時は聞き取れたテキストのまま次項目へ skip（§1-4-1）。
- **氏名カタカナ変換**: §1-7 に従い OpenAI でカタカナ正規化、判別不可時は聞き取れた表記で保存。
- **数字表記ルール**: §1-5「よん/し→4」「ご/ごお/ごー→5」を OpenAI プロンプトで強制。
- **status 判定**: patientName または reason が非空 → `1`、両方空 → Gen2 `0` を Gen3 禁止のため `2` に置換、代表案内 → `2`。
- **FAQ 対応**: 禁止（§1-2）。RAG サブフロー一切不使用。
- **時間外・非通知・転送**: customer_doc に文言記載なし。時間外・非通知はテンプレート既定文言で仮実装、転送は非実装。
- **着信電話番号**: 先頭 0 を省略禁止（§1-3）。
