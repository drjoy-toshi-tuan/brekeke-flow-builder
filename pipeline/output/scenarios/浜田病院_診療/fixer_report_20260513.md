### ⚠️ 2-6. 日付の年推定ロジックの月日判定境界

#### **W-001: 月日のみ入力時の年推定ロジックが「月日の大小判定」に依存**

**対象モジュール**: 全 datetime プロンプト（受診希望日_産科 等）

**現在のロジック**（例：OpenAI_受診希望日_産科）:
```
発話が「月日のみ」または「MMDD 4桁」の場合、システム日付を基準に以下のように年を決定します。
・発話された月日が、システム日付の月日と同じかそれ以降であれば、システム日付と同じ年を採用します。
・発話された月日が、システム日付の月日より過去であれば、来年（システム日付の翌年）を採用します。
```

**例**:
- システム日付: 2026-05-13
- 入力: 「5月20日」 → 2026-05-20（同年）✅
- 入力: 「4月15日」 → 2027-04-15（来年）✅
- 入力: 「5月12日」 → 2026-05-12（同年 or 来年？）❓

**曖昧性**: 「同じかそれ以降」の「それ以降」がシステム日付（時刻も含む）を指すのか、月日のみを指すのか。実装時に「2026-05-13 09:00:00」のような時刻が付く場合、「05-13」発話は同日と判定されるのか、それとも「以降」に該当するのか。

**影響度**: 中程度（STEP2 で具体的な日付パターン（A）や（D）が先に抽出される場合が多いため、月日のみ判定は限定的）

**修正方針**:
- ロジックの表現を「発話月日 >= システム日付の月日」「発話月日 < システム日付の月日」という数値比較に統一
- 或いは、「5月13日」のように入力と同日の場合の扱いを明示的に文言

### ⚠️ **W-003: キーワード定義の曖昧な単語が存在**

**対象モジュール**: OpenAI_用件3, OpenAI_電話SMS選択

#### **OpenAI_用件3 の「さん」キーワード問題**

STEP2 - 数字モード判定 - 「産後ケア」:
```
### 産後ケア
3
3番
3ばん
3です
3でお願いします
さん
さんばん
さんです
```

**問題**:
- 「さん」は敬語接尾辞として他の単語に容易に含まれる
- 例：「無痛分娩についてお聞きしたいんですが」→ 「さん」が含まれている？いいえ。
- しかし「3だんですが」と発話した場合、STT が「さんだんですが」と誤認識した場合「さん」が含まれる
- 実装レベルでの部分マッチ処理次第では誤判定リスク

**実際の問題例**:
- ユーザー: 「3でお願いします」 → STT: 「さんでお願いします」 → 正しく「産後ケア」と判定 ✅
- ユーザー: 「3番」 → STT: 「さんばん」 → 正しく「産後ケア」と判定 ✅
- ユーザー: 「無痛分娩について聞きたいんですが」 → STT: 「無痛分娩についてお聞きしたいんですが」 → 「さん」は含まれていない ✅
- **しかし** ユーザー: 「〇〇さんの件で」 → 誤判定リスク ❌（患者の名前提示を含む場合）

**修正方針**:
- STEP2 の「さん」単独キーワードを削除し、「さんばん」「さんです」など必ず複合キーワードにする
- 或いは、「さん」を STEP2 から除外し、STEP3 のキーワードマッチに統合

### ⚠️ **W-004: 「ケア」キーワードの過度な一般性**

**対象モジュール**: OpenAI_用件3

```

ケア

- しかし、複合キーワード（「無痛」+「ケア」など）が含まれない限り、「ケア」のみで分娩予約にマッチしない

**リスク**: 中程度（STEP2 で数字モード判定が先に行われるため、言語分析は限定的）

**修正方針**:
- 「ケア」の前に「産後」や「出産」などの修飾語が含まれることを条件にする方法がないか検討
- 或いは、「ケアについて」「ケアのことで」など、より文脈を含むキーワードに変更

### C-002: キーワード「さん」「に」の単独含有による誤判定リスク
- **修正担当**: prompter <prompter>
- **対象モジュール**: OpenAI_用件3（「さん」）, OpenAI_電話SMS選択（「に」）
- **重大度**: **Critical**
- **問題**:
  - **「さん」は敬語接尾辞**: 「患者さん」「Mr.さん」など他の発話に容易に含まれる。例えば「3番についてお聞きしたいんですが…」という発話でも「さん」が含まれる可能性
  - **「に」は助詞**: 「電話に出ろ」「2にしてください」など、複数の文脈で出現。特に「電話にしてください」→「に」が含まれて SMS 判定される誤り
  - STEP2 での部分マッチ処理が厳密でない場合、単語境界を考慮しない単純な部分マッチで誤判定
- **修正方針**:
  - OpenAI_用件3 の STEP2「さん」単独キーワードを削除、「さんばん」「さんです」等の複合キーワードのみに統一
  - OpenAI_電話SMS選択 の STEP2「に」単独キーワードを削除、「にばん」「にです」等の複合キーワードのみに統一

### W-001: 月日のみ入力時の年推定ロジック「以降」定義が曖昧
- **修正担当**: prompter <prompter>
- **対象モジュール**: 全 datetime プロンプト（受診希望日_産科 他 4個）
- **重大度**: **Warning**
- **問題**:
  ```

  - 「同じかそれ以降」の「それ以降」がシステム日付の時刻を含むのか不明確
  - 例：システム日付が「2026-05-13 09:00:00」の場合、ユーザーが「5月13日」と発話 → 同日と判定されるのか、それとも「以降」に該当するのか
  - 実装時に境界条件で齟齬が生じる可能性
- **修正方針**:
  - ロジック表記を「発話月日 >= システム日付の月日」という数値比較に統一
  - 「同じ日付の場合は同じ年、日付が過去なら来年」と明確に記述
