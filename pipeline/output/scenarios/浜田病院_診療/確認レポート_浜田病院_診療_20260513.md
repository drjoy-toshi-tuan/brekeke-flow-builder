### 判定した作業種別

- **作業種別**: 新規フロー作成（Pattern 1）
- **施設名**: 医療法人愛賛会浜田病院（産科・婦人科クリニック）
- **シナリオ名**: 診療
- **対象フロー**: 浜田病院$診療（メインフロー）+ 4 サブフロー（診察券番号 / 氏名 / 生年月日 / 電話番号）

### 抽出した要件サマリー

- **緊急性判断（最優先分岐）**: 陣痛・破水・出血・胎動が少ない・つわりがひどい・症状が急に重くなった → 即時 Blind Transfer to 06-6708-7200
- **転送失敗時の救済**: 06-6708-7200 への掛け直し案内を発話して終話（END_転送失敗）
- **用件分岐 3 段階**:
  - 用件① 3 択: 診察予約・変更 / 分娩・産後ケア / その他問い合わせ
  - 用件② 3 択（診察系のみ）: 新規予約 / 予約変更 / キャンセル
  - 用件③ 3 択（分娩・産後ケア系のみ）: 分娩予約 / 無痛分娩 / 産後ケア
- **新規予約**: 産科/婦人科分岐、産科は 16 週確認、婦人科は発熱有無確認あり
- **予約変更**: 現在の予約日 → 希望予約日 → 体調不良か
- **キャンセル**: 現在の予約日 → キャンセル理由
- **分娩予約**: 予約不要・来院案内・予約金 5 万円・持参物案内（説明①）→ そのまま終話
- **無痛分娩**: 初産婦不可・経産婦は紹介状必須（説明②）→ 紹介状有無 hearing → なしは終話（END_紹介状なし案内）、ありは個人情報チェーンへ
- **産後ケア**: 産後 2 か月までの方対象・HP 予約誘導のみ（説明③）→ そのまま終話
- **その他問い合わせ**: 内容自由テキスト聴取 → 電話 or SMS 折返し選択 → 個人情報チェーン
- **個人情報サブフロー連鎖**（合流後共通、raw.md L34-L40 順序）: ID 番号 → 患者名 → 生年月日 → 連絡先電話番号
- **復唱有無**: ID 番号(あり) / 患者名(なし) / 生年月日(あり) / 電話番号(あり、固定電話確認含む)
- **最後の問い合わせ**: 「その他にお聞きしたいことはございますか。」hearing。あり/なし 両方とも終話分岐へ
- **終話分岐**: 2 段の context_match_router で classification を参照し、用件別の END_* に振り分け
- **SMS 追加項目**（AI 電話では聴取せず SMS で回収）: 出産予定日 / 出産経験 / 帝王切開 or 普通分娩 / 住所 / 身長 / 体重 / 無痛分娩希望 / 女性医師希望

## 6. 次のアクション

1. 上記 **BLOCKER** 項目（特に office_id / status 番号 / sms_flag テンプレ番号）を顧客・Dr.JOY 担当者に確認
2. 解消した値を設計書 YAML の対応箇所に反映（`TODO_要確認` の置換）
3. **NON-BLOCKER** 項目で重要なもの（特に 16 週確認の配置、「最後の問い合わせ」yes 時挙動）を元 PDF フロー図で再確認
4. 上記が完了したら orchestrator パイプライン（または `@generator` 直接呼出）で実行を進行
5. **Dr.JOY 側設定事項**（SMS 文面、営業時間スケジュール、認証設定、画面表示）はフロー稼働前に別途設定
6. **本番投入前に必ず通話テスト**で以下を確認:
   - 緊急性判断 → Blind Transfer 動作（実機転送先での着信確認）
   - 16 週確認の配置が顧客意図と一致しているか
   - 「最後の問い合わせ」yes 時の挙動が運用に支障ないか
   - 各終話 TTS 文言が施設運用と合致しているか
   - SMS 追加項目の送信タイミング（受付完了直後 / 翌営業日 等）
