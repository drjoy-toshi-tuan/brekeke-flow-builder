### 🔴 C-002: 出力値ミスマッチ（担当者確認系プロンプトと YAML KEYWORD_PATTERNS）【要修正】

**発見箇所:**

OpenAI_担当者確認_入院（および全 4 つの担当者確認プロンプト）

**問題の詳細:**

プロンプト（line 458-473）に記載の「わかる」キーワード：
```
わかる
わかります
分かる
分かります
わかっています
わかってる
知っている
知ってます
存じています
決まっています          ← YAML に無い
お決まりです             ← YAML に無い
はい
ええ                     ← YAML に無い
そうです                 ← YAML に無い
```

設計書YAML（line 209）のKEYWORD_PATTERNS：
```regex
分かる|わかる|わかり|分かり|はい|存じ|知って
```

**マッピング検証:**

| プロンプトキーワード | YAML正規表現で検出可否 | 備考 |
|---|---|---|
| わかる | ✅ わかる | 完全一致 |
| わかります | ✅ わかる | 部分一致 |
| 分かる | ✅ 分かる | 完全一致 |
| 分かります | ✅ 分かる | 部分一致 |
| わかっています | ✅ わかり | 部分一致 |
| わかってる | ✅ わかり | 部分一致 |
| 知っている | ✅ 知って | 部分一致 |
| 知ってます | ✅ 知って | 部分一致 |
| 存じています | ✅ 存じ | 部分一致 |
| **決まっています** | ❌ NO MATCH | 検出不可 |
| **お決まりです** | ❌ NO MATCH | 検出不可 |
| はい | ✅ はい | 完全一致 |
| **ええ** | ❌ NO MATCH | 検出不可 |
| **そうです** | ❌ NO MATCH | 検出不可 |

**影響:**

YAML の KEYWORD_PATTERNS を使用する実装では、以下の入力が「わかる」として認識されません：
- 「決まっています」→ マッチせず
- 「お決まりです」→ マッチせず
- 「ええ」→ マッチせず
- 「そうです」→ マッチせず

これらは自然な応答なので、本来マッチすべきです。

**同様の問題が他の担当者確認プロンプト（外来、連携、行政）にも存在します。**

**修正方針:**
- YAML KEYWORD_PATTERNS を拡張して、プロンプトに記載のキーワードをすべて捕捉する
- または、プロンプトから YAML に無いキーワード（決まっています、お決まりです、ええ、そうです）を削除する

**修正担当**: prompter  <prompter>

**問題:**

OpenAI_相談区分確認（line 349）：
```
新規
初めて
はじめて
初回          ← このキーワード
→ 新規
```

設計書YAML（line 180）のKEYWORD_PATTERNS：
```regex

```

**差分:**
- プロンプトに含まれる: `初回`

**分析:**

「初回」（しょかい）= 初めての回、第1回  
「初診」（しょしん）= 初めての診察、初めての外来訪問  

意味は似ているが、用語として異なります。実装では「初診」が優先されるので、「初回」という回答は NO_RESULT に落ちます。

**修正方針:**
- 顧客の想定するユーザー応答を確認してどちらを採用するか決定
- 両方を許容する場合は YAML を `新規|初めて|はじめて|初回|初診` に拡張

### 🟡 W-002: 複合・曖昧条件ロジックとYAML不一致【要設計確認】

**問題:**

OpenAI_担当者確認_入院（および全 4 つの担当者確認プロンプト）の STEP3 に、以下の複雑な条件判定ロジックがあります（line 433-438）：

```
【最優先】複合・曖昧・条件付きの判定 → わからない
- 「わかる」系の語と「わからない」系の語の両方を含む
- 「わかる」系の語に、曖昧・不確実を表す語（多分/たぶん/かも/かもしれません/一応/多少）が伴う
- 「わかる」系の語の後に逆接（が/けど/けれど/ただし/でも/しかし）が続き、否定が示される
- 「いない」「いません」「ありません」等を含む
```

しかし設計書YAML（line 209）の KEYWORD_PATTERNS は単純な正規表現のみ：

```regex
{"regex":"分かる|わかる|わかり|分かり|はい|存じ|知って","result":"わかる"}
{"regex":"分からな|わからな|不明|いいえ|知らな","result":"わからない"}
```

これは「決まっていない」は捕捉しますが、「わかるが/けど」という逆接ロジックや「多分わかる」という不確実マーカー併有時の処理は実装されていません。

**影響:**

プロンプトで定義した複雑な条件判定が script（n_choice）では実装されず、単純な部分一致のみになる可能性。

**修正方針:**
- YAML KEYWORD_PATTERNS で複雑な条件を表現する方法を検討
- または、こうした複雑な判定が必要な場合は、OpenAI を使用する設計に戻す必要がある

**修正担当**: prompter  <prompter>
