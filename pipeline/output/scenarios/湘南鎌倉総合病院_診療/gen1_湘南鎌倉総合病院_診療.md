### 移管方針
- Gen1「箱」構造 → Gen3 9 ブロック型に変換。冒頭は opening + 冒頭_アナウンス（2 ブロック 1 セット）。挨拶 TTS は「お電話ありがとうございます。湘南鎌倉総合病院の、予約専用、AI 電話です」。
- 用件分岐は **Pattern A**（用件聴取 OpenAI 直後に 5 分岐: 予約確認 / 定期受診 / 予約変更 / 予約キャンセル / 新規予約）。**用件 echo_back 必須**（main.yaml 用件聞き取り.yaml 内に確定確認は無いが、Gen3 標準で echo_back 配置）。
- 用件別の合流仕様（main.yaml `用件を判断` 箱から読む）:
  - **新規予約** → `新規予約の流れ_1`「新規のご予約は AI 電話では受付できません」→ 予約専用ダイヤル案内で終話（**新規は本フロー受付対象外**）
  - **予約確認** → `予約確認の流れ_1`「ご予約の確認は AI 電話では受付できません」→ 予約専用ダイヤル案内で終話（**確認も本フロー受付対象外**）
  - **定期受診 / 予約変更 / 予約キャンセル** → `予約変更・キャンセル・定期受診へ` → 診療科：1_29修正（心療内科非対応）→ 診療科判断_1（診療科 × 用件マトリクス）→ 通常ルート
- 通常ルート: 診療科 → 理由聞き取り → 詳細聞き取り（予約確認なら診察券番号 / 変更・キャンセルなら次へ_1 → 診察券番号）→ 患者氏名 → 診察券有無 → （無し）生年月日 → 連絡先電話 → 携帯電話番号チェック → 通話完了。
- 診療科分岐は **Pattern B（後段 ContextMatchRouter）併用必須**。診療科 × 用件のクロス分岐があり、診療科だけでは終話先が決まらない。
- サブフロー（患者氏名 / 生年月日 / 連絡先電話 / 診察券）は Jump to Flow 経由。電話番号区切り.yaml は連絡先電話番号 subflow に内蔵。
- 入電日時間判断.yaml（年末年始 12/30〜1/5 + 平日 8:00〜19:00 + 土曜 8:00〜13:00）→ date_of_call_classifier ブロック。**addCurrentDate: Yes 自動付与対象**。
- 紹介状の有無と紹介元医療機関確認.yaml → 紹介状ありの場合のみ紹介元医療機関を hearing で追加聴取（**ただし新規予約は受付不可で先に切るため、本フローでは未使用の可能性あり** → director 判断）。

### 施設固有の特殊ルール
- **新規予約・予約確認は AI 受付対象外**: 用件聞き取りで 5 分岐するが、新規/確認は即「受付できません」アナウンス → 予約専用ダイヤル `0467-84-8922`（祝日除く 月〜金 13:00〜16:30）案内 → 終話。AI で完走するのは **定期受診 / 予約変更 / 予約キャンセル** の 3 用件のみ（main.yaml の `用件セット（新規予約）` `用件セット（予約確認）` 参照）。
- **受付時間**: 祝日除く、月〜金 8:00〜19:00、土曜 8:00〜13:00。日曜祝日は終日時間外。
- **年末年始休止**: 12/30〜1/5 まで自動音声休止（年末年始アナウンス: 「12 月 30 日から 1 月 5 日まで自動音声による電話対応を休止」+「急患は直接 8 番窓口へ」）。1/5 は平日扱いで時間内のみ受付（入電日時間判断.yaml `（平日）時間の分岐_1` 参照）。
- **代表電話**: `0467-46-1717`（予約以外用件・聞き取り失敗時の案内、3 回繰り返し）。
- **予約専用ダイヤル**: `0467-84-8922`（新規予約・予約確認の受付不可案内、3 回繰り返し）。
- **直通電話案内が複数**:
  - **人工膝関節センター**: `0467-46-7172`（センター予約は AI 不可、直通案内）
  - **セカンドオピニオン**: `0467-45-8923`（セカンドオピニオン予約は AI 不可、直通案内）
  - **予防医学センター / 禁煙外来**: `0467-44-1454`（両者で同一専用ダイヤル）
- **診療科 × 用件マトリクス**（main.yaml 診療科判断_1 / 診療科判断_2 / 用件の確認_1〜_7 を参照）:
  - **AI 受付不可（直通案内）**: 人工膝関節センター / セカンドオピニオン / 予防医学センター / 禁煙外来 → どの用件でも上記直通案内 → 専門ダイヤル案内 → 病院 HP 案内 → 5 回まで繰り返しで切断。
  - **代表電話案内（放射線科 TTS）**: 放射線｜検査 / 放射線｜その他 / 人工透析 / 臨床心理士 → 「ご用件は AI 電話では受付できません」+ 代表電話案内（音声ガイダンス「その他のお問い合わせ 4 番」）。
  - **救急外来**: 現在の時間判断（8:30 まで / 17:00 まで / それ以外）→ 時間内/時間外で別 TTS の代表電話転送案内（`転送コメント（時間内）` / `転送コメント（時間外）`）。
  - **特定診療科の予約キャンセルのみ別文言**: 産科 / 放射線腫瘍科 / 内視鏡 → `予約キャンセルの流れ（特定診療科）`「3 日以内に折り返し電話」。それ以外の診療科の予約キャンセル → `予約キャンセルの流れ（その他診療科）`「予約キャンセルを受付いたしました」（折り返し無し）。
  - **通常ルート**（用件の確認_5 で振り分け）: 定期受診 / 新規予約 → `次へ_1`（診察券番号へ）／ 予約変更 / 予約キャンセル → `次へ`（予約日へ）。**用件で先に診察券か予約日かが分岐する**点に注意。
- **47 診療科対応**: 心療内科は対応外（main.yaml サブ yaml 名「診療科：1_29修正（心療内科非対応）」が示す通り）→ 診療科聴取で心療内科が出た場合の振る舞いはサブ yaml 内で完結（director はそのまま subflow 経由で呼ぶだけ）。
- **理由聞き取り（変更/キャンセル理由）**: 予約変更/キャンセルでは `理由聞き取り.yaml` で「薬が足りなくなった」「予定が合わない」等の自由文聴取あり → hearing ブロック（output_format=text）で吸収。聴取後 `転送へ（変更理由から）` `転送へ（キャンセル理由から）` 経由で通常ルートに合流（実際には代表電話案内へ落ちる失敗パスのみ）。
- **3 件の出口 termination パターン**:
  - 通常完走（定期受診/変更/キャンセル）: `通話完了`「3 日以内に折り返し」+ 終話「お電話ありがとうございました」+ END_切断
  - 受付不可: 各用件別の専門ダイヤル案内 → 5 回繰り返しで切断
  - 聞き取り失敗: `代表電話案内（聞き取りできず）`「お電話が遠いようで、回答が確認できませんでした」+ 代表電話案内 + 「その他のお問い合わせ 4 番」
- **DTMF**: yaml に DTMF 受付の記述は無い（数値以上判定は時間判定のカウンタのみ）→ DTMF 並走は不要（director 判断、SKILL_C 自由テキストで吸収）。

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `API_DRJOY_1` → `時間判別へ` → `入電日時間判断` → `チェックポイント` → `用件聞き取り` の 7 箱は Gen3 では opening + date_of_call_classifier + 用件 hearing の 3 ブロックに集約。
- 「セリフ: none」かつ振り分けのみの「箱」（`用件の確認 / _1〜_7`、`用件を判断`、`診療科判断 / _1 / _2`、`診察券有無`、`診察券無し`、`診察券有り`、`携帯電話番号チェック`、`完了フラグ / _1`、`チェックポイント / _1`、`電話切断へ`、`次へ_3 / _4`、`用件セット（5 種）`、`用件セット（未取得）`、`転送失敗`、`混み合っている旨で切断`、`API_DRJOY / _1 / _2` 等多数）は Gen3 で独立ブロック不要 → 前段の next 書き換え or context_match_router の一部として吸収。
- **`案内回数カウント / _1 / _2` の 5 回上限**: 専門ダイヤル / 予約専用 / 代表電話の 3 経路それぞれに「5 回繰り返したら切断」のカウンタあり → Gen3 では subflow の retry_count か script で簡素化（director 判断、Gen3 デフォルトの retry_count=2 とは別の上位ループ）。
- **`現在の時間の取得` → `現在の時間判断`** は救急外来分岐用の **その場の時間判定**（8:30 まで / 17:00 まで）であり、入電日時間判断.yaml とは別物。Gen3 では救急外来分岐内に小さな date_of_call_classifier を追加するか、addCurrentDate: Yes で取った値を context_match_router で再判定するか **director 判断**。
- **「予約以外」エンティティ**で予約以外用件（くすり/会計/書類/症状 等）を拾うと `代表電話案内（予約以外の用件）`「こちらは予約専用 AI 電話となっております」→ 代表電話案内に直行。用件聴取の最終フォールバックとして必須。
- **連絡先電話番号聞き取り後の `携帯電話番号チェック`**: 携帯番号取得済か判定 → 完了フラグ。電話番号 subflow 内で携帯/固定の context 保存があれば不要、無ければ subflow 戻り値の context で context_match_router 1 個追加。
- **「セカンドオピニオン専用ダイヤル / 予防医学センター専用ダイヤル / 人工膝関節センター専用ダイヤル」の TTS は数字のリテラル読み上げ**（「0、4、6、7、の、4、6、の、7、1、7、2」）→ 設計書転記は元 TTS のまま。`<% var %>` 動的置換は不要。
- **代表電話案内の音声ガイダンス分岐文言**「その他のお問い合わせの 4 番を押してください」は **代表電話側の Brekeke IVR が別途処理する前提**（本フロー外）。Gen3 では TTS 文言として転記するだけでよい。
- **`紹介状の有無と紹介元医療機関確認.yaml` は新規予約ルート専用**だが、新規予約は AI 受付不可で先に切られるため **本フローでは実質未使用** → director 判断（subflow ブロック化を省略するか、念のため残すか）。
- **47 診療科 entity** は entity_classification.md で R1=582 件展開済 → STT 辞書は scaffold/properties が自動取り込み。director は転記不要。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。
