### 抽出した要件サマリー

Customer Doc から抽出した主要要件:

- **聴取順序（施設固有）**: 3営業日以内確認 → 受診者名 → 生年月日 → 連絡先電話番号 → 用件①（clinicalDepartment：人間ドック/健康診断/その他の検診/精密検査/問い合わせ）→ 人数 → 用件②（classification：予約/変更/キャンセル/問い合わせ）。**他施設と異なり用件①が用件②より先**
- **用件①で問い合わせ選択時**: 人数聴取スキップ → 問い合わせ内容聴取 → 追加の質問聴取 → 終話判定（classification 空欄で通過）
- **精密検査の終話分岐最優先ルール**: clinicalDepartment=精密検査 の場合、用件②を**無視**して連絡先電話番号種別（携帯/固定）のみで終話文を決定。キャンセル文言は使用禁止
- **終話判定は3変数組み合わせ（Pattern B）**: clinicalDepartment × classification × phonetype の組合せで5種終話（END_精密_携帯 / END_精密_固定 / END_予約変更問合せ_携帯 / END_予約変更問合せ_固定 / END_キャンセル）→ Script モジュールで実装
- **3営業日以内確認**: 冒頭「はじめに、3営業日以内の予約に関するお問い合わせでしょうか？」→ 肯定で END_予約代表案内（0537-28-8133 を3回繰り返し、status=2, smsFlag=0）
- **非通知即時切断**: 「発信者番号を通知して…186 を追加いただくことで」施設固有 TTS、opening 直後に実施
- **3名以上折返**: numberOfReservations>=3 で即 END_折返案内（status=2, smsFlag=0）
- **当日/翌日拒否**: 受診希望時期／現在の予約日で「当日」「翌日」発話 → END_当日代表案内
- **復唱ルール**: 受診者名・問い合わせ内容・追加の質問・受診希望コース・希望オプション・受診希望時期・現在の予約日・変更内容は復唱禁止。**生年月日のみ復唱許可**（DOB Re-confirmation）
- **連絡先電話番号4分岐**: 非通知（opening で弾き済み）/ 050 / 携帯(090/080/070) / 固定・その他 でアナウンス文言が異なる。携帯のみ「今おかけいただいている ○○ でよろしいですか？」確認文使用
- **連絡先電話番号リトライ上限**: 3回（他項目の2回と異なる）。anonymous 着信の超過時のみ専用アナウンスで終話、それ以外は折返案内
- **希望オプション例外**: retry=1、リトライ超過で空欄セットして次へ（終話しない）
- **additionalPhoneNumber 初期化禁止**: 聴取完了まで着信番号を絶対にセットしない
- **単語正規化ルール**: 「検針/返信/天候診断」→健診、「人間ロック/ドック/ロック」→人間ドック、「薬/お薬」→予約、「取り直したい/別の」→変更、「やめる/取り消し」→キャンセル
- **受診希望コースリスト**: 16コース。各コースに 0〜5 個のレコメンド対象キーワード（クーポン/市の検診/協会けんぽ等で複数候補提示）
- **オプションリスト**: 19項目（脳ドック〜骨密度検査）
- **受診希望コース分岐**: 用件①=人間ドックはコース聴取スキップ、健康診断/その他の検診/精密検査は用件別のコース聴取TTS使用
- **dtmf/SSML タグ除去**: Gen2 の `<dtmf digit="8" />` と `<speak type="telephone">` は TTS プロンプト禁止、IVR プロパティ側で設定
- **代表電話の使い分け**: 予約代表 0537-28-8133（3回繰り返し）、AI非対応質問案内 0537-21-5555
- **smsFlag 決定**: status=1 かつ キャンセル→2、status=1 かつ 予約/変更/問合せ→1、それ以外→0
- **desiredConsultation はフリーテキスト**、reservationDate は YYYY-MM-DD 形式。再ヒアリングで生年月日系定型文を使わない
- **RAG/FAQ 利用なし**（移管ノート明記）
- **acceptance_times 不使用**（Gen2 yaml に時間外分岐なし）
