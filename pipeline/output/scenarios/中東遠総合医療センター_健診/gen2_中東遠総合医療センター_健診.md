### 移管方針
- Gen2 の「箱」構造を scenario_flow ブロックへ素直に1:1で変換（opening → announcement → hearing × N → context_match_router → termination）。
- 入口で **非通知即時切断** の例外分岐あり → opening 直後に incoming-classifier で非通知を判定し END_非通知即時切断 へ。これを最優先で配置。
- **Date of Call Classifier 不要**（営業時間外案内は yaml にない。acceptance_times 判定は運用側）。ただし「現在の予約日」「受診希望時期」に「当日/翌日」発話があった場合は END_当日代表案内 へ分岐（→ Pattern A: 各 hearing 直後の OpenAI 分岐で処理）。
- **分岐パターン**: 用件②（予約/変更/キャンセル/問い合わせ）分岐は Pattern A（classification 判定直後に分岐）。**終話判定は Pattern B**（用件①×用件②×連絡先電話番号種別の3条件組合せのため、末尾の ContextMatchRouter に集約して END_精密_携帯 / END_精密_固定 / END_予約変更問合せ_携帯 / END_予約変更問合せ_固定 / END_キャンセル の5分岐）。
- **サブフロー化**: 受診者名 / 生年月日 / 連絡先電話番号 は Gen3 標準サブフロー（Jump to Flow）。連絡先電話番号は incoming_number 種別（非通知/050/携帯/固定）で4分岐 → 4つの hearing ブランチをサブフロー内で分ける。
- **受診希望コース聴取の分岐**: 用件①（人間ドック/健康診断/その他の検診/精密検査）に応じてコース聴取文言が分岐（人間ドックは聴取スキップ）。context_match_router で 4 ブランチ + スキップ分岐。
- FAQ/RAG 利用なし。

### 施設固有の特殊ルール
- **精密検査の終話分岐最優先**: clinicalDepartment="精密検査" の場合、用件②（予約/変更/キャンセル/問合せ）を**無視**して連絡先電話番号種別のみで終話文を決定する。キャンセルであっても「ご予約のキャンセルを承りました」文言は使用禁止。→ 終話 ContextMatchRouter のプロンプト条件順で精密検査を最優先に置くこと。
- **smsFlag 決定ルール**: status="1" かつ classification="キャンセル" → smsFlag="2"、status="1" かつ予約/変更/問合せ → smsFlag="1"、それ以外 → smsFlag="0"。
- **checkpoint 値**: 3営業日以内確認入口="0"、完了="1"、代表案内="2"、折返案内="7"。
- **additionalPhoneNumber 初期化ルール**: 連絡先電話番号聴取**前は絶対に着信番号をセットしない**（ブランク）。途中切断時もブランクのまま。→ 着信時の初期化処理で incoming_number を additionalPhoneNumber にコピーしないこと。
- **連絡先電話番号の4分岐**: 非通知/050/固定/不明 → 「ショートメール受信できる携帯番号を」文言。携帯（090/080/070）のみ「今おかけいただいている {番号} でよろしいですか？」確認文。確認文は携帯以外で**絶対に使用禁止**。
- **連絡先電話番号リトライ超過時**: anonymous の場合のみ「こちらからお電話失礼」END_連絡先エラー終話、それ以外は折返案内。リトライ上限3回（他項目は2回）。
- **3名以上の予約**: numberOfReservations>=3 で即 END_折返案内（checkpoint=7, smsFlag=0）、以降のヒアリングに進まない。
- **3営業日以内確認**: 冒頭で「3営業日以内の予約に関するお問い合わせでしょうか？」と聴取、「はい」で END_予約代表案内（0537-28-8133 3回繰り返し）。
- **当日/翌日予約の拒否**: 受診希望時期 / 現在の予約日で「当日」「翌日」発話 → END_当日代表案内（0537-28-8133）。過去日付は未来化して受付、10月以降に1〜9月発話は翌年扱い。
- **受診希望時期と現在の予約日の区別**: desiredConsultation はフリーテキスト、reservationDate は日付形式（YYYY-MM-DD 00:00:00）。受診希望時期の再質問では生年月日系の定型文（「生まれた年…」等）を**絶対に使わない**。
- **復唱禁止範囲**: 受診者名/問い合わせ内容/追加の質問は復唱禁止。**生年月日のみ「○○ですね」の復唱を許可**（yaml の「生年月日復唱_内部」箱）。
- **受診希望コースのレコメンド**: 「クーポン」「市の検診」「協会けんぽ」等のキーワードで複数候補をレコメンド。Gen3 では該当コース聴取モジュールの OpenAI プロンプトに埋め込む。
- **代表電話番号の使い分け**: 予約代表案内 = 0537-28-8133（3回繰り返し）、AI非対応質問案内 = 0537-21-5555。混同しないこと。
- **用件②=問い合わせ**: question に発話内容。予約/変更/キャンセルフロー途中の追加質問は nextQuestion（別項目）。
- **単語正規化**: 「検針/返信」→「健診」、「人間ロック/ドック/ロック」→「人間ドック」、「薬/お薬」→「予約」、「取り直したい/別の」→「変更」、「やめる/取り消し」→「キャンセル」。「検診」の前に「乳がん/子宮がん/子宮頸がん」がある場合は除外。
- **dtmf digit="8"**: 生年月日聴取時のセリフに `<dtmf digit="8" />` タグが入っているが、TTS プロンプトに SSML は禁止。**dtmf は IVR プロパティ側で設定**、TTS 文言からは除去。
- **telephone タグ**: 連絡先確認時の `<speak type="telephone">{}</speak>` も同様に SSML 禁止ルール対象。IVR プロパティで digit 読み上げ設定。

### director が見落としやすいポイント
- 改訂履歴: 2026/03/26 版（冒頭コメント）。それ以前版との差分記述は customer_doc に無し。
- **聴取順序**: 3営業日以内確認 → 受診者名 → 生年月日 → 連絡先電話番号 → **用件①（clinicalDepartment）→ 人数 → 用件②（classification）**。用件①が用件②より先に来る点に注意（他施設と逆）。
- 問い合わせフローは用件①で「問い合わせ」選択時に**人数聴取をスキップ**して直接 question へ（用件②スキップ）。yaml の「用件1聴取」箱の分岐参照。
- 用件②=問い合わせの場合も後続は追加の質問 → 終話判定に合流（キャンセル以外と同じ終話分岐）。
- option 聴取はリトライ1回で空欄許容・終話せず次へ進む（1-5-1 オプション例外）。他項目と挙動が違う。
- status の決定: checkpoint 値優先（0/1/2/7）→ それ以外は必須項目の充足で 1/0 判定。
- endpoint 列挙: 冒頭切断/時間外/電話転送/電話案内/途中切断/通話完了。「時間外」値は yaml に対応する終話箱が無いため現状不使用。
- function の required に `medicalCardNumber` / `healthInsuranceAssociation` が含まれるが**聴取フローでは聞かない**（空欄で必須充足扱い）。display_type 用の項目定義のみ。
