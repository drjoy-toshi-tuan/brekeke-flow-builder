## 検出事項

### W-001: 当日確認の NO_RESULT 処理が不明確  
- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_当日確認
- **問題**: 
  - STEP4 で「NO_RESULT が返る条件：無音、フィラーのみ、意味不明な発話」と記載
  - しかし、設計書では「NO_RESULT は default で別日扱い（安全側）」と明記
  - プロンプト内で NO_RESULT 時の処理フロー（別日への fallback）が明記されていない
- **修正方針**: STEP4 に「NO_RESULT の場合も別日として扱い、通院歴確認へ進む」と明記すること

### W-002: 通院歴確認の NO_RESULT デフォルト処理が未記載  
- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_通院歴確認
- **問題**:
  - STEP4 で「NO_RESULT：両方含む場合のみ」と記載
  - 設計書では「NO_RESULT は default で新規扱い（紹介状確認へ進む安全側）」と明記
  - プロンプトの STEP4 では新規 default への記載がない
- **修正方針**: 
  - STEP4 の「その他」セクションに「NO_RESULT → 新規として紹介状確認へ」と明記
  - または STEP3 直後に「STEP4 以外のいずれにも該当しない場合 → NO_RESULT、新規扱い」と追記

### W-003: 紹介状確認の分岐処理が不完全  
- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_紹介状確認
- **問題**:
  - 設計書では「あり → 新規_診療科、なし → 新規_診療科、NO_RESULT → 新規_診療科（Gen1 仕様通り紹介状有無は context 記録のみ、業務継続）」と明記
  - プロンプトでは STEP4「あり/なしの両方含む場合 → NO_RESULT」のみ記載
  - 正常系「あり」「なし」それぞれが新規_診療科へ遷移すること、NO_RESULT も同じパスへ進むことが明記されていない
- **修正方針**:
  - STEP4 に以下を追記：「以下のいずれかに該当する場合 → NO_RESULT（ただし新規_診療科へ続行）：あり/なしの両方含む、どちらのキーワードも含まない、無音・フィラーのみ・意味不明」

### W-004: 予約日聴取の月日のみ補完ルール記述が設計書と異なる  
- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_予約日聴取
- **問題**:
  - 設計書：「年は本日以降の最も近い同日に補正」（簡潔な表現）
  - プロンプト：「月日のみ（M月D日 / M/D）→ 基準日の年を補完。基準日と同月日なら基準日の年、基準日より過去なら翌年、未来なら基準日の年」（複合記述）
  - 入出力例で検証：例「基準日 2026-04-20、入力『3月5日』→ 出力『2027-03-05』」は正確
  - **実装上は正確だが、記述が冗長で曖昧性がある**
- **修正方針**:
  - 記述を簡潔に：「月日のみの場合は基準日の年を補完。基準日より過去か同日なら翌年、未来なら基準日の年に補完」
  - または「月日のみが基準日より過去か同日なら翌年に補完。未来なら基準日の年に補完」と統一

### W-004b: 次予約希望日確認のキーワード拡張が設計書を超過  
- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_次予約希望日確認
- **問題**:
  - 設計書キーワード（あり）：「はい / あります / 希望あり」の3種類
  - プロンプトキーワード（あり）：上記 3 種+「ある」「有り」「有ります」「希望日あり」「お願いします」「予約したい」「取りたい」「決まっています」「決まってます」（計13種）

  - プロンプトキーワード（なし）：上記 4 種+「ない」「無い」「なし」「無し」「希望日なし」「未定」「決まっていません」「決まってない」「わからない」「分からない」（計14種）
