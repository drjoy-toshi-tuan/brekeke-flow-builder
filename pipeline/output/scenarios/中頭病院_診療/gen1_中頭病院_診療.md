### 移管方針
- 主流れ: opening → API_DRJOY → 時間判別 → 予約種別確認（当日手術 or 健診紹介状の有無で代表案内へ離脱判定 Yes/No）→ 用件聞き取り（4 分岐: 予約変更 / 予約キャンセル / 予約確認 / 診察予約）→ 診察券番号確認 → 診療科聞き取り → 用件別合流 → 個人情報聴取（氏名 / 生年月日 / 連絡先）→ 完了アナウンス → termination
- 用件は **4 分岐** + 用件聴取不能時の `用件NG` → 連絡先のみ取得して終話する救済ルート
- 新規予約（=診察予約）のみ「乳腺外科・産婦人科・形成外科」限定 → 診療科聞き取り後段で `新規用診療科.yaml` を呼び出し対象外科目は受付外として代表案内へ離脱
- 予約変更 / キャンセル: 診療科 → 予約日聞き取り → 検査確認（胃カメラ/大腸カメラ/CT/MRI Yes/No）→ 連絡先電話番号 → 携帯チェック → 完了アナウンス
- 予約確認: 診療科 → WEB確認案内（SMS で URL 送付）→ 終話 / または連絡先 → 完了アナウンス（携帯あり=SMS、なし=折り返し電話）の 2 系統
- 診察予約 (新規): 紹介状有無 subflow → 紹介状なしは ちばなクリニック ナビ案内へ離脱、紹介状あり → 医療機関名聞き取り → 診療科確認 → 患者氏名 → 連絡先 → 携帯チェック → 完了アナウンス
- 完了アナウンスは用件×携帯フラグで 4 種使い分け（変更 / キャンセル / 新規 / 確認、それぞれ「3 診療日以内に折り返し電話 or SMS」）

### 施設固有の特殊ルール
- **予約種別確認ゲート**: opening 直後に「ご用件は当日の手術・入院予約 / 健康診断・人間ドック / 乳児健診紹介状をお持ちの方に該当しますか？」を Yes/No → Yes は **0570-09-1300（中頭病院ナビ） / 0570-09-1301（ちばなクリニックナビ）** へ離脱、No のみ正規ルート進行
- **新規予約は 3 科目限定**: `新規用診療科.yaml` で「乳腺外科・産婦人科・形成外科」のみエンティティ定義。それ以外を発話されたら「当院の新規予約はこの 3 科のみ対応」案内 → 終話。Gen3 では新規ルート専用診療科 hearing を別 enum で分離
- **診療科は分岐あり/分岐なしの 2 段構成**: `診療科聞き取り.yaml` で「呼吸器/消化器/脳神経/婦人科」が分岐あり（外科/内科の聞き返し）、その他 27 科は分岐なし。Gen3 では診療科 OpenAI で一次正規化 → CMR で分岐あり 4 値のみ二段目 hearing
- **検査確認 Yes/No**: 「胃カメラ、大腸カメラ、CT、MRI の予約検査はありますか？」を変更 / キャンセルの後段で hearing。Yes → 連絡先聴取、No → 連絡先聴取（分岐先は同じだが検査有無 context を保存）
- **代表電話の使い分け**:
  - **098-939-1300**: 中頭病院代表電話（電話番号 NG / 診療科 NG 等の最終フォールバック、3 回繰り返し読み上げ）
  - **0570-09-1300**: 中頭病院ナビダイヤル（当日手術・入院予約向け）
  - **0570-09-1301**: ちばなクリニックナビダイヤル（健診・人間ドック・乳児健診紹介状向け）
- **WEB確認案内（予約確認用）**: 携帯ありなら SMS で WEB 確認 URL 送付、なしなら「現在の番号では SMS 送信不可、HP 確認 or 携帯から再発信を」案内 → 終話。3 種類の終話文言を使い分け
- **時間外/年末年始アナウンス**: 「月-金 8:30-17:30、土 8:30-11:30、日祝・年末年始休み、紹介予約は月-金 8:30-17:30 / 土 8:30-11:30」を保持。`紹介予約のみ` 箱（時間外でも紹介予約のみ受付）の特例は director 判断（acceptance_times の時間外用 termination で代表 0570-09-1301 案内に統一可）

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` と `API_DRJOY` は Gen3 opening 標準処理で吸収 → 明示ブロック化不要
- 「次へ」「次へ_1」「次へ_2」「続いて」「それでは」「接続詞」「タイトル」（セリフ「では次に、」「続いて、」「ありがとうございます。」等）は短い繋ぎ TTS → 次段ブロック先頭 TTS にマージ（director デフォルト）
- 「セリフ: none」かつ振り分けのみの箱（`用件を判断` / `_1〜_5`、`完了フラグ`、`携帯電話番号チェック` / `_1〜_3`、`時間外接続へ`、`返答確認`、`カウント`、`再質問カウント`、`電話番号NG`、`診療科NG`、`検査有り`、`用件を診察予約に`）は独立ブロック不要 → 前段 next 書き換えか CMR の一部として吸収
- 「肯定否定」サブフローは予約種別確認 / 紹介状有無確認の **共通 Yes/No 確認** → Gen3 では各ポイントに Yes/No hearing を都度配置（共通 subflow 化はしない、scaffold が固定プロンプト埋め込み）
- `紹介状無しは受け付けない` / `紹介状無しは受け付けない_1` は新規予約で紹介状なしの離脱 termination → ちばなクリニック ナビ 0570-09-1301 案内固定
- `紹介予約のみ` 箱（時間外で紹介予約のみ受付）は **時間外特例ルート** → director 判断、acceptance_times 統一の方が安全
- `テスト用` 系の名残箱は無し（**`時間外テスト用外し` 等は ちばな側のみ**）
- `第1受診希望日` / `第2受診希望日` は同じ `第1受診希望日.yaml` を参照する流用 → Gen3 では希望日 1 / 2 の 2 段 hearing として連続配置（datetime + addCurrentDate: Yes scaffold 自動付与）
- 患者氏名サブフローは **PatientName 1 枠**に充当（受診者と入電者の分離は Gen1 にも無し）
- 「<? $連絡先電話番号（ハイフン区切あり） ?>」「<? $入電番号（ハイフン区切りあり） ?>」等の Gen1 動的タグは Gen3 では `<% var %>` 構文へ
- entity_classification.md によると分岐あり診療科（呼吸器/消化器/脳神経/婦人科）は STT 19 件のみ、分岐なし診療科は 396 件と大量 → STT 辞書で吸収、OpenAI 同義語は最小限
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字等の環境依存文字は禁止
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`** を使う
- **CMR conditions は必ず最後に `match: "other"` + next 明示**
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
