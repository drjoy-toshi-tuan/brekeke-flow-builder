### C-001: 搬送or来院確認 — JSON分岐条件とプロンプト出力値の不一致
- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_搬送or来院確認
- **重大度**: **Critical**
- **問題の詳細**:
  - プロンプトは出力値として「救急搬送」「直接来院」「NO_RESULT」を定義
  - しかし、JSONの分岐条件（scenario_flow L132-136）では：
    ```yaml
    conditions:
      - match: "救急搬送"
      - match: "other"                      # = 直接来院
    ```
  - つまり、条件に「match: 直接来院」が明示されていないため、プロンプトが出力「直接来院」を生成しても、JSON では「other」分岐で評価される
  - 結果として、出力「直接来院」と「NO_RESULT」が JSON の条件評価上で区別されない
  - **効率性・保守性の問題**: プロンプトが「直接来院」を出力する設計が、JSON で有意義に評価されていない

- **修正方針**: 以下のいずれかを選択して統一せよ（JSON修正が必要な場合は fixer の後流対応が発生）：
  1. **推奨案A**: JSON conditions を修正して `match: "直接来院"` を明示的に追加する
  2. **推奨案B**: プロンプトの出力値から「直接来院」を削除し、「NO_RESULT」のみで統一して JSON の「other」分岐へ自動遷移させる

- **補足**: scenario_flow の conditions に「match: other」という設計は、「救急搬送以外の全て」を意味する scaffold の約定。プロンプト側が「直接来院」を明示的に出力することが設計意図か、または「NO_RESULT」で十分かを確認した上で修正すること。
