### 抽出した要件サマリー

材料から読み取った技術要件:

- **24時間受付**: 救急搬送のため acceptance_times は使用しない（時間外終話パスは互換性のためデフォルト保持）
- **冒頭アナウンス**: 「医誠会無料救急搬送です。AI電話が対応いたします。」
- **主分岐（hearing enum 2 値）**: 救急搬送（DTMF 1 / 救急/搬送/緊急）/ 直接来院（DTMF 2 / 直接/自分で/行きます）。Pattern A での hearing 直後分岐
- **救急搬送ルート（移管ノート指定の包括的シーケンス）**:
  施設名 → 住所 → 状態 → バイタル → 患者氏名 → 個人情報（生年月日年齢性別） → 連絡先 → 緊急手術確認 → Blind Transfer
- **直接来院ルート**:
  選定療養費案内（紹介状無し時 9,900 円） → 患者氏名 → 個人情報（生年月日年齢性別） → 症状聴取 → 連絡先 → 終話（はしか流行案内付き）
- **転送先**: 転送1=080-9923-2523 → 転送2=080-7031-1207 の 2 系統順次フォールバック（Blind Transfer）
- **終話文言（救急搬送）**: 「到着時間は担当者より折り返しご連絡いたします...」+ はしか流行案内 + 「それでは、失礼いたします。」
- **終話文言（直接来院）**: 「受診のお申し込みを受け付けました。」+ はしか流行案内 + 「それでは、お気を付けてお越しください。」
- **聴取失敗フォールバック**: 「3回失敗⇒終話へ（番号OK）」（入電番号への折り返し案内） / 「連絡先聴取失敗」（施設名と住所で調べて折り返し案内）
- **緊急時オーバーライド**: ベット逼迫時の公設救急優先案内（運用フラグでの切替）
- **個人情報聴取は自由テキスト 1 ステップ**: Gen1 と同じく「生年月日・年齢・性別」を 1 発話で聴取（emergency context での dialog 短縮）
