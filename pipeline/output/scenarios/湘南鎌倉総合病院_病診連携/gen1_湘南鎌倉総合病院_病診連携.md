### 移管方針
- Gen1「箱」構造 → Gen3 9 ブロック型へ変換。主な対応:
  - `挨拶`（callback コンテキスト説明）→ opening + 冒頭_アナウンス（2ブロック1セット）
  - `問い合わせ内容①聞き取り`（サブyaml）→ hearing ブロック（用件、output_format=text の自由聴取）。**用件分類による分岐は無し**（病診連携は単一フロー）
  - `施設名・部署・担当者名`（サブyaml）→ **1 ブロックの hearing**（医療機関名 / 所属部署 / 担当者氏名を 1 発話で受ける合成聴取）。output_format=text、3 項目を OpenAI 側で同時抽出
  - `連絡先電話番号聞き取り` + `電話番号区切り`（サブyaml）→ Gen3 標準 PhoneNumber サブフロー 1 枠（携帯/固定/IP の桁数判定・ハイフン整形は subflow が内蔵）
  - `END_電話番号聞き取り不完了` → 連絡先聞き取り不可 announcement（「後ほど担当者にて確認」）→ 終話に合流
- **本フローに用件分岐は無い**（用件 hearing は記録のみで分岐に使わない）→ Pattern A/B いずれも不要、scenario_flow は直線的

### 施設固有の特殊ルール
- **callback ボット**: opening TTS が「お呼び出ししましたが、お繋ぎできませんでした。AI電話がご用件をお伺いし、担当者から折り返し電話連絡いたします。」となっており、**院内代表が病診連携の架電に出られなかった際に AI が一次受けして用件・連絡先を記録するシナリオ**。通常の inbound 受付ではない点に director は留意（acceptance_times 不要、call_transfer 不要、IVR Disconnect は incoming-classifier 直結でなく opening 後）
- **call_transfer なし**: フロー内に転送箱が存在しない。call_transfer ブロックは作らない。最後は終話のみ
- **date_of_call_classifier なし**: 時間帯 / 曜日 / 年末年始の分岐箱が yaml 中に一切無い。受付時間判定ブロックは生成しない
- **ヒアリング項目は 3 つだけ**: ① 用件（自由文）/ ② 医療機関名・部署・担当者名（合成自由文）/ ③ 連絡先電話番号（subflow）。氏名 / 生年月日 / 診察券番号 / 診療科 / 予約日 等の通常聴取は **無し**（PatientName/DOB/PatientID/診療科サブフロー不要）
- **47 診療科の entity 値は使われていない**: main.yaml に診療科リストはあるが、実フロー内に診療科聴取箱が無い。entity_classification.md の R1 48 件は **STT 辞書には積まないか、用件 hearing の補助として use_template 指定にとどめる**（director 判断、デフォルトは積まない）
- **連絡先電話番号聞き取り不可** 箱（main.yaml l.162）→ 専用 announcement「大変申し訳ございません、お電話番号を聞き取ることができませんでした。後ほど、担当者にて確認させていただきます。」を出して **終話に合流**（切断ではない）。電話番号 subflow の retry_count=2 を超えた失敗 next 先として配置

### director が見落としやすいポイント
- **施設名・部署・担当者名は 1 ブロック合成聴取**（3 ブロックに分けない）。元 yaml の聴取文「医療機関名、所属部署、お名前をおっしゃってください。」をそのまま使う想定。output_format は text、保存先 context は 3 つに分けて OpenAI が抽出する設計（or 1 つの自由文 context にまとめて担当者に運用で読ませる）→ director は **どちらを採るか移管ノートで明示してくれた方が generator が迷わない**（推奨: 1 自由文 context で担当者運用に委ねる）
- **電話番号復唱**: 連絡先電話番号聞き取り.yaml l.101–121 に「ご連絡先の電話番号は、〜、でよろしいでしょうか?」の復唱あり → PhoneNumber サブフロー内蔵の echo_back で吸収（director は echo_back 重複配置禁止）
- **入電番号取得確認** → CMR ロジック（入電番号未取得時の分岐）は subflow 内処理。Gen3 では PhoneNumber サブフロー仕様に従い、director は外側で入電番号 context を渡すだけ
- **携帯電話番号の追加聴取**（連絡先 yaml l.397–468「携帯電話をお持ちの場合は番号を、お持ちでない場合は『持っていない』と」）→ 緊急時連絡先として固定電話の後に **携帯番号を追加で聞く 2 段聴取**。これは PhoneNumber サブフローの標準仕様外なので、**サブフロー後段に hearing ブロックを追加**するか、subflow を 2 連で呼ぶ必要あり。director は明示してほしい
