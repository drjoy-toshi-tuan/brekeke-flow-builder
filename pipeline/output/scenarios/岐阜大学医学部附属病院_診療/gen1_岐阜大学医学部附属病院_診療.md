### 移管方針
- Gen1「箱」→ Gen3 9 ブロック型へ変換。opening + 冒頭_アナウンス(2 ブロック 1 セット必須)。挨拶 TTS は「お電話ありがとうございます。岐阜大学医学部附属病院の、予約専用、AI電話です」+ 「番号通知あり」箱の「受付には診察券番号が必要ですので、お手元に無い方は…代表電話へおかけ直しください…なお、当院からの折り返しのご連絡をもって確定とさせていただきます」を冒頭_アナウンスに合流。
- **用件分岐は Pattern A**(用件聞き取り OpenAI 直後に 4 分岐: 再診予約 / 予約変更 / 予約キャンセル / 予約確認 + DTMF 1〜4 並走)。echo_back(再診予約復唱 / 予約変更復唱 / キャンセル復唱 / 予約確認復唱)を各分岐直後に配置。
- **冒頭の WEB or AI電話 分岐**: 用件聞き取り**前**に「WEB ページ入力 vs AI 電話受付」の Y/N 質問あり(眼科・脳神経内科以外)。WEB 希望 → `WEB入力番号聞き取り`(連絡先電話番号サブフローで完結 → END_WEB確認の流れ)。AI 電話希望 → 通常の用件聞き取りへ。Gen3 では opening 直後に hearing ブロック(Y/N)+ context_match_router で WEB レーン/AI レーンを切り分ける(director 判断、ここはサブフロー化せず main 内で素直に展開推奨)。
- **時間外チェック(時間外チェック.yaml)**: 平日(月〜金)9:00〜15:00、年末年始 12/29〜1/4 を個別判定 + 祝日チェック(サブシナリオ呼び出し)→ date_of_call_classifier ブロックに集約。**addCurrentDate: Yes 自動付与対象**。
- **入電番号判別**: incoming-classifier 側で `058-230-6123 / 6124 / 6000 / 非通知` を判別 → 6123/6124/6000 の場合は「6月2日から、岐阜大学医学部附属病院、予約センターの電話がAI対応に変わりました」**転送時アナウンス**を追加して通常フローへ。非通知 → 「186 を付けておかけ直しください」案内 → 切断。Gen3 では incoming-classifier の Pattern2 判定で吸収。
- **稼働開始日 6/2**: `稼働前分岐` `6/2分岐` `稼働前アナウンス`(6/2 午前 9 時開始)箱あり。**移管時点(5/1 起草、稼働は 6/2)で既に稼働済みになっている見込み。設計書では 6/2 9:00 アナウンスは再現不要**(director 判断、設計書からは削除)。要マネージャー確認。
- サブフロー(患者氏名 / 生年月日 / 電話番号 / 診察券番号)は Jump to Flow 経由でサンプル JSON 完コピ。
- 自由テキスト hearing は **理由、確認内容の聞き取り.yaml の「予約確認内容/再診理由/キャンセル理由確認/変更理由確認」 と 聴取失敗時聞き取り(フリー聴取).yaml** の 2 種類。output_format=text、SKILL_C 自由テキスト、5 秒無言で打ち切り。

### 施設固有の特殊ルール
- **受付時間**: 平日(月〜金)9:00〜15:00、祝日除く、年末年始 12/29〜1/4 休診。時間外アナウンスは祝日明示「祝日を除く、月曜から金曜の、9時から15時」、年末年始アナウンスは別文言「祝日、年末年始を除く、…」あり。**Gen3 では時間外と年末年始で TTS 分割**(director デフォルト適用、acceptance_times に集約)。
- **代表電話案内(当日受診/初診予約/聞き取り不可)**: 058-230-6000。`当日→代表案内` 箱の TTS で **3 回繰り返し読み上げ**「お電話番号は、058-230-6000、です」 → 切断。失敗系 termination_pattern にも代表番号を含める。
- **紹介受診重点医療機関**: `紹介状有無聞き取り.yaml` あり、紹介状なしの場合「初診時選定療養費 7,700 円」案内 TTS。**ただし main.yaml の現行フローからこのサブ yaml は呼び出されておらず**(終了セクションに `END_紹介状有無聞き取り` の参照名のみ存在し、生きているフロー上の呼び出し箱はない)→ **Gen3 では再現しない**(director 判断、移管時点で運用停止と解釈、要マネージャー確認)。新規予約は受付せず代表案内なので紹介状判定が不要。
- **新規予約は AI 受付不可**: `番号通知あり` TTS で「初診予約、並びに、今日受診希望の方は、代表電話へおかけ直しください」と冒頭で明示。AI で受け付けるのは 4 用件(再診/予約変更/キャンセル/確認)のみ。
- **診療科ハイラルキ二段(5 グループ)**: ① 親エンティティ「呼吸器 / 消化器 / 脳神経 / 放射線 / 腎臓」発話 → ② `分岐(xxx)の聞き取りへ` 箱で内/外科 or 細分化を聞き直し → ③ 確認復唱(`<? $診療科の読み ?>`)。**放射線のみ 3 分岐**(放射線科 / 放射線診断・IVR / 放射線治療)。**腎臓 = 腎臓内科 / 腎移植外科**。Gen3 では **親 hearing → context_match_router → 子 hearing 5 本(放射線含む)→ echo_back** で再構成(director 判断、信州大と同方針、子 hearing は subflow 化せず hearing ブロックを並べる推奨)。
- **対象外診療科 = 予約センター転送**: `診療科分岐` 箱で「眼科 / 脳神経内科」発話 → 058-230-6125 へ Blind Transfer。**転送先は 1 系統のみ**(信州大と同様)。`予約センターへ転送` TTS は「<? $診療科の読み ?> については予約センターの担当者へ転送いたします」、転送失敗時は `転送失敗→切断へ` で「ただいま電話が混み合っております…再度おかけ直しください」案内(director デフォルト 8 該当だが TTS 文言は施設固有のため明示)。
- **予約変更の追加質問**: ① 予約日聴取 → ② 予約希望時期聴取 → ③ 残薬確認(残薬確認 + 肯定否定確認(残薬有無).yaml は **3 値分岐**: 1=はい / 2=いいえ / 3=処方されていない、3 値の echo_back 不要、`かしこまりました。` で次へ)→ ④ 変更理由(自由テキスト)。
- **予約キャンセルの追加質問**: ① 次回予約希望の有無(`次回予約希望の有無` + `肯定否定確認(次回予約希望).yaml`)→ 希望ありなら予約希望時期 + 残薬確認 + 確認内容聴取に合流、希望なしなら確認内容(キャンセル理由)聴取へ。
- **予約確認**: 予約確認内容を自由テキスト聴取(「次回の予約日時を教えて欲しい」等)。
- **再診予約**: 診療科 → 患者氏名 → 生年月日 → 用件確認 → 連絡先電話番号 → 終話。理由聴取は `理由、確認内容の聞き取り` で「再診理由」発話。
- **予約希望時期で「今日」を発話 → 代表案内へ**: `予約希望時期の聞き取り.yaml` で「今日/今から/当日/本日」を検出 → `END_今日→代表案内へ`。当日予約は AI 不可なので代表案内へ合流。
- **3 回失敗時**: 各サブで `count再質問` 2 回到達 → `END_聞き取り失敗`「申し訳ございません。AIが正しく聞き取ることができませんでした。次の質問に進みます」→ **次の聴取項目に進む寛容モード**(信州大は代表案内に転送だったがここはスキップ継続)。Gen3 では retry_count + skip フラグで吸収。
- **予約日聴取は DTMF 4 桁並走可**: 月+日の 4 桁、過去日付なら +1 年で確定する自動補正ロジックあり(director デフォルト適用、Gen3 標準 subflow が吸収)。
- **DTMF 受付(用件聞き取り)**: 1=再診予約 / 2=予約変更 / 3=キャンセル / 4=確認(冒頭ガイダンスで明示、4 分岐で並走)。
- **復唱(echo_back)対象**: 用件 4 種 + 診療科のみ。氏名/生年月日/電話番号/診察券はサブフロー内で完結。
- **救済語/同音異義救済**: 診療科 yaml に大量にあり(「Suica → 麻酔科疼痛治療科」「BB/GB/DB → 耳鼻咽喉科頭頸部外科」「カレー類 → 糖尿病代謝内科」「ハート/ペース → 循環器内科」等)。**entity_classification.md / entity_openai_synonyms.md / entity_stt_patch.json に全件あり、director は転記不要**。

- **「予約変更」と「予約キャンセル」が複数経路で出現**: ① main の `用件の確認 / _1 / _2 / _3`、② `理由、確認内容の聞き取り.yaml` の `用件の確認`、③ `用件聞き取り.yaml` の `セットした用件の分岐 / _1`。**Gen3 では用件 context を一度確定したら以降は context_match_router で参照する**ことで重複聴取を避ける(信州大・北里と同方針)。
- **`セットした用件の分岐_1`(当日予約 NG 後の再診/新規振り分け)**: 「現在、通院中の診療科のご予約ですか？」 Y/N で再診/新規セットを切り分けている。**Gen3 では新規予約は受付不可なので、ここで「いいえ(新規)」 → 代表案内 termination に直結**(director 判断、`END_新規セット` をそのまま代表案内 termination へ繋ぐ)。
- **診療科確認復唱(`診療科を確認`)で「分岐なし/分岐あり/対象外/内科」発話が再度出た場合**は `失礼しました` → `count再質問_3` → `再聴取`(聞き直し)。**「肯定否定」(はい/いいえ等の混在発話)は `読み替え` 箱経由**で診療科名を Gen1 内部表記に正規化(血液感染症内科 → 血液・感染症内科、耳鼻咽喉科頭頸部外科 → 耳鼻咽喉科・頭頸部外科、放射線診断IVR → 放射線診断・IVR、リハビリテーション部リハビリテーション科 → リハビリテーション部・リハビリテーション科)。**Gen3 では診療科 context は OpenAI 出力の正規化済み診療科名 1 個に統一**し、中黒入りの正式表記は OpenAI 正規化辞書(entity_openai_synonyms.md)で吸収(director 判断)。
- **「内科」単独発話 → わからないセット → `読み替え`**: 「内科です」だけだと診療科不明 → そのまま読み替え経由で診療科確定不可とする扱い(信州大の「呼吸器感染症アレルギー内科」括り表記とは違う)。Gen3 では OpenAI 側で内科単独 = 「内科」と返してもらい、context_match_router で「内科のみ → 失敗 → 再聴取」分岐に乗せる(director 判断)。
- **`電話番号区切り.yaml` は読み上げ用補助サブシナリオ**(電話番号を桁数別に「090-1234-5678」等にハイフン区切り)→ Gen3 では TTS 動的値構文 `<% var %>` の readable 形式で吸収(director 判断、別ブロック化不要、信州大・北里と同方針)。
- **連絡先電話番号聞き取り** vs **WEB入力番号聞き取り** の 2 系統: 前者は AI 電話レーンの最後で連絡先聴取、後者は WEB 入力レーンで申込番号として連絡先を聴取。Gen3 では PhoneNumber サブフロー 1 本に集約し、入口を 2 系統用意するか、context フラグで分けるかは director 判断。Pattern A(用件直後分岐)に乗せず**主軸は AI 電話レーン**で組み、WEB レーンは opening 直後の専用 termination/サブフローで完結させる方が事故が少ない(director 判断)。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**(丸数字 ①②③⑩等は使用禁止)。元資料に出口記号は無いが、`非通知(完了フラグ8)` `時間外へ(完了フラグ7)` `予約キャンセルの流れ(その他診療科)` のような括弧記法はそのまま使わず、`END_非通知NG` `END_時間外` `END_予約キャンセル` 等の安全名に置換。
- **ContextMatchRouter.reference_module は subflow を参照する場合 `jump_{step名}`**(scaffold が付ける prefix)を使う。subflow ブロックの step 名そのもの(例: `電話番号聴取`)ではダメ。
- **CMR conditions は必ず最後に `match: "other"` + next 明示**。`TODO_other_target` プレースホルダーを残さない。
- **転送先は 1 系統**(予約センター 058-230-6125、Blind Transfer、眼科/脳神経内科のみ)。新規予約や予約変更/キャンセルは AI で完結し、Dr.JOY 側で折り返し連絡する設計(再診予約は「3 診療日以内」、変更/キャンセル/確認は「医師確認の場合 1 週間以上」と TTS で明示)。
- **終話 TTS の集約**: `終話`「お電話ありがとうございました。」/ `当日→代表案内`(058-230-6000、3 回繰り返し)/ `非通知`(186 案内)/ `時間外アナウンス` / `年末年始アナウンス` / `本人or家族確認聞き取り失敗→終話`(**ただし現行フローからの呼び出しなし**、再現不要)/ `稼働前アナウンス`(移管後不要)→ Gen3 では携帯/固定の TTS 分割(director デフォルト 9)+ termination_patterns 集約。
- **TTS は `{tts_g:...}` 小文字のみ、SSML 禁止**。Gen1 変数記法 `<? $診療科の読み ?>` `<? $予約日の読み ?>` `<? $連絡先電話番号(ハイフン区切あり) ?>` `<? $入電番号(ハイフン区切りあり) ?>` は Gen3 では `<% var %>` に変換。
- 患者氏名は PatientName サブフロー 1 枠に充当(patient 1 名のみ、入電者 = 受診者前提の設計)。
