## 修正①: お問い合わせ用件で「結果の確認」画面に遷移する問題

**対象フロー**: 福岡県新水巻病院$健診
**通話ID**: 184000013186, 184000013595

### 原因

`用件分岐`（ContextMatchRouter）の組み合わせ設定において、
「お問い合わせ」と「結果確認」の遷移先が**入れ替わっている**。

現状（誤）:
- 組み合わせ4: module1Value4=`お問い合わせ`, module2Value4=`お問い合わせ` → nextModuleName=`相談_結果確認`
- 組み合わせ5: module1Value5=`結果確認`, module2Value5=`結果確認` → nextModuleName=`相談_問合せ`

正しい設定:
- 組み合わせ4: `お問い合わせ` → `相談_問合せ`（問い合わせTTSに遷移すべき）
- 組み合わせ5: `結果確認` → `相談_結果確認`（結果確認TTS→`入力_相談_結果確認`に遷移すべき）

### 修正内容

`用件分岐`モジュール（modules["用件分岐"]）の `next` 配列を以下のように修正する:

```json
[
  {"condition": "^1$", "label": "組み合わせ1", "nextModuleName": "予約_受診歴"},
  {"condition": "^2$", "label": "組み合わせ2", "nextModuleName": "変更_予約日"},
  {"condition": "^3$", "label": "組み合わせ3", "nextModuleName": "変更_予約日"},
  {"condition": "^4$", "label": "組み合わせ4", "nextModuleName": "相談_問合せ"},
  {"condition": "^5$", "label": "組み合わせ5", "nextModuleName": "相談_結果確認"},
  ...（6〜11番は空のまま）
]
```

> **注意**: ConditionMatchRouterの条件値（`^1$`〜`^5$`）はBrekekeが内部生成するもので変更不要。変更するのは `nextModuleName` のみ。
> `params`（module1Name/module2Name/moduleNValueN）は変更不要。

## 修正②: お問い合わせ分岐にて文字起こしが反映されない問題

**対象フロー**: 福岡県新水巻病院$新水RAG（サブフロー）
**通話ID**: 184000013186, 184000013595

### 原因分析

`FAQ検索`（Jump to Flow）から`新水RAGフロー`に遷移後、`入力_相談_お問い合わせ`（Speech to Text）でSTTは動作しているが、openAI_相談_問合せにデータが渡らない、またはループ制御のロジックに問題がある可能性がある。

新水RAGフローの `入力_相談_お問い合わせ` の `probability`（現在未調査）が低すぎる場合、音声認識結果が破棄されて `NO_RESULT` として扱われている可能性がある。

### 修正内容

`入力_相談_お問い合わせ`（Speech to Text）モジュールの以下パラメータを確認・修正する:

1. `probability` を `0.55` → `0.4` に下げる（認識閾値を下げて認識率を上げる）
2. `profile_words` に汎用問い合わせ向けキーワードが含まれているか確認する

また、修正①（遷移先の入れ替え）が原因で `お問い合わせ` 選択時に `相談_結果確認` → `入力_相談_結果確認` に遷移していた可能性が高い。修正①を適用することで、本来の `相談_問合せ` → `FAQ検索` → 新水RAGフロー → `入力_相談_お問い合わせ` の経路が正しく通るはず。

**修正①適用後に再テストして本問題が解消するか確認すること。解消しない場合は上記probability調整を追加実施する。**

## 修正③: 生年月日の聴取精度が低い問題

**対象フロー**: 福岡県新水巻病院$個人情報新水（サブフロー）
**通話ID**: 184000013518, 184000013665

### 原因分析

1. **`openAI_患者_生年月日` の success 遷移先が空欄**: `next` 配列の `success`（condition=`^.*$`）の `nextModuleName` が空文字になっている。正規化後の遷移先が未設定のため、フローが止まっている可能性がある。

2. **`probability` が `0.55`**: STT認識閾値が高めに設定されており、明確な発話でも一部が棄却される可能性がある。

3. **`stop_play_when_speech` が `"No"`**: TTS発話中に患者が発話しても検知しない設定になっている。

4. **profile_wordsの不足**: 現在の辞書には「令和」が含まれていない。また「平成」も未登録の可能性がある。

### 修正内容

**修正A: `openAI_患者_生年月日` の success 遷移先を設定する**

`modules["openAI_患者_生年月日"]["next"]` の success条件（`^.*$`）の `nextModuleName` を、サブフロー内の次の正常遷移先（script_結果返却 またはメインフロー返却モジュール）に設定する。

個人情報サブフローの構造を確認し、生年月日正規化後に返却スクリプトに遷移するよう修正すること。

**修正B: `入力_患者_生年月日` の `probability` を調整する**

```json
"probability": "0.45"
```
（0.55 → 0.45 に引き下げ。認識閾値を下げることで確実に聞こえている発話を受け取る）

**修正C: `stop_play_when_speech` を `"Yes"` に変更する**

```json
"stop_play_when_speech": "Yes"
```

**修正D: `profile_words` に令和・平成の読み仮名を追加する**

以下を `profile_words` の先頭に追加する:
```
令和 れいわ
平成 へいせい
昭和 しょうわ
```

> 現在の辞書に「昭和 きょうわ」「昭和 きょうは」の誤読補正は存在するが、「昭和 しょうわ」（正しい読み）が未登録の可能性があるため、正読も追加する。
> 「令和」「平成」も同様に追加する。
