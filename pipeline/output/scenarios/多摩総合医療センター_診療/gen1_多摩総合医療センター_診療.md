### 移管方針
- Gen1 main.yaml の箱群 → Gen3 9ブロック型に再マッピング。冒頭(opening + 冒頭_アナウンス) → 時間判断(date_of_call_classifier: 平日8:30-17:00/年末年始12/28-1/4) → 診察券準備TTS → 用件聴取 → 用件別ルーティング → 個人情報聴取 → 受付完了アナウンス → termination の8〜9ブロック構成。
- **用件は Pattern A（OpenAI直後分岐）**。5分岐: 診察(初診)/再診/キャンセル/変更/確認/その他。Gen1の「用件の分岐」「用件の分岐_1」「用件の分岐_2」「用件の分岐_3」が用件×工程軸で重複している → Gen3 は用件分類1回だけ行い、以降は context で制御するようフラット化。
- **受診歴分岐と診療科分岐は Pattern B**（後段 ContextMatchRouter）。受診歴有/無、診療科名（ゲノム/腫瘍内科/血液/緩和ケア/女性専用/救急/救命・集中/リハビリ/看護/リウマチ/精神神経/産婦人科/小児 等）で受付可否・誘導先が変わる。
- date_of_call_classifier は時間判断.yaml 準拠（受付時間外・年末年始で分岐、時間外TTSは予約センター042-323-9200 / 代表042-323-5111 / WEBフォーム案内を含む）。
- サブフロー: PatientName(患者氏名1枠) / DateOfBirth(生年月日) / PhoneNumber(電話番号) / PatientID(診察券) を使用。紹介元医療機関名・紹介元電話番号・前回症状・変更キャンセル理由・予約日は hearing ブロック（自由テキスト/日付）で直接処理。

### 施設固有の特殊ルール
- **初診は紹介状必須（紹介状聴取.yamlを介す）**。初診で紹介状あり → 紹介元医療機関名 → 紹介元電話番号 → 以降の個人情報聴取 へ。再診は紹介状不要で前回症状確認へ。
- **診療科ごとの受付可否マトリクス**（director は診療科聞き取り.yaml と診療科の分岐_0/1/2 を必ず照合）:
  - 初診受付不可（医療連携室へ誘導）: ゲノム診療科/腫瘍内科/血液内科/緩和ケア内科
  - 初診受付不可（AI電話では受付不可）: リハビリテーション科/看護外来
  - 受付停止中: 女性専用外来（初診・再診とも不可）
  - 別機関誘導: 小児科 → 小児総合医療センター・地域/紹介元へ
  - 救急/救命・集中治療科: 代表電話へ誘導
  - 変更/キャンセル特殊: 精神神経科・産婦人科(妊婦検診) → AI電話では不可、用件聞き取って後ほど折返し
  - 再診で救命・腫瘍・緩和ケア: 代表電話へ誘導
  - リウマチ外科/リウマチ膠原病内科: 紹介元医療機関聴取に回す特殊ルート
- **時間帯分岐（現在の時間判断）**: 8:30前/15:30前 → 本日折り返し or SMS、それ以降 → 翌営業日以降折返し（3営業日以内案内）。これは受付時間内の中での「今日中 or 翌営業日以降」判定。date_of_call_classifier ではなく hearing 後段の script 的分岐として実装。**director 判断**: Pattern B で実装推奨。
- 受付完了TTSは用件ごとに「診察予約/予約キャンセル/予約変更/予約日時確認」の申し込みを受付けました、を出し分け → 共通の「3営業日以内に折返しor SMS」アナウンス → 終話。
- 電話番号の有無チェック（携帯電話番号チェック_0〜3）は Dr.JOY API から取得済みかの判定。**director 判断**: Gen3 では context 値で制御、なしの場合のみ電話番号聴取サブフローを通す。

### director が見落としやすいポイント
- Gen1 yaml で「用件の分岐」系が4種類あるが、**用件は1回だけ分類**し Gen3 では context_match_router で使い回す。重複ブロックを作らない。
- 紹介元医療機関・紹介元電話番号は**初診の紹介状ありルートのみ**。再診・変更・キャンセル・確認では聞かない。
- PatientName は 1 枠のみ（患者氏名聴取.yaml）。入電者氏名・受診者氏名の区別はなく患者本人氏名のみ。
- 電話番号は「電話番号聴取」「電話番号聴取復唱なし」「電話番号区切り」「電話番号区切り_1」の4本があるが、Gen3 は PhoneNumber サブフロー1本に統合。
- 診察券は「診察券番号の聴取」「診察券復唱なし」の2本 → Gen3 は PatientID サブフロー1本。
- 「その他」用件 → その他アナウンス（予約センター/代表電話に誘導して終話）。ヒアリングせず即誘導。
- 受付時間外TTSは**WEBフォーム誘導の一文**を含む（他院と異なる）。omit しないこと。
- 冒頭TTS「こちらは多摩総合医療センターの、予約専用、AI電話です。」+ 「紹介状や、診察券をお持ちの方はお手元にご用意ください。」は opening + 冒頭_アナウンス の2ブロック1セットで配置。
- 終話分岐の END_切断 は「切断待ち」経由で入力待ちあり。Gen3 termination は標準構成でOK。
- TTS は `{tts_g:...}` 小文字、SSML 禁止。電話番号読み上げ「042-323-9200」等はポーズ処理を Gen3 TTS に委ねる。
