## 変換上の判断事項

1. **着信番号判別ノード（050-1707-0320/050-1707-4509）**: Gen1では予約センター直通番号として特別扱いしていたが、Gen3では `incoming-classifier` の標準分岐に統合。別途 IVRプロパティで office_id 設定が必要。

2. **入電時間判断 ConversationNode**: Gen1の `ConversationNode` 「入電時間判断」（受付時間内/受付時間外/年末年始 3分岐）を `acceptance_times` モジュールに変換。年末年始対応は acceptance_times の設定（IVRプロパティ）で対応。

3. **診療科フリー ConversationNode**: Gen1の用件確認ノードの複数選択肢（集学的がんセンター等16項目）を OpenAI 分岐方式に変換。`用件_特殊分岐` と `OpenAI_再診診療科`/`OpenAI_新規診療科` で対応。

4. **婦人科の特殊処理**: Gen1では「3日以内に予約が必要」かの確認後に転送。Gen3では `婦人科_パス` → `転送前_婦人科` → `転送_婦人科予約センター` の直接転送パスとして実装。

5. **聴取不可時の折り返しエラーフラグ**: Gen1の複雑な連絡先エラーフラグ処理（連絡先が取れない場合の折り返し氏名・電話番号聴取）を `通話失敗` → 転送パスに単純化。

6. **雑談ノード（commubo.StartDigressNode）**: Gen3では不使用。無視した。

7. **API_DRJOY ノード**: Gen1では saveCompletionFlag2db に相当。`完了フラグ_*` モジュールとして変換。

8. **API_DRJOY_CONTEXT_SETTINGS ノード**: コンテキストモデル設定に相当。`コンテキスト設定` (saveContextModel2DB) として変換。

9. **着信番号050番号の扱い**: Gen1では 050-1707-0320 と 050-1707-4509 への着信を予約センター直通として別処理していたが、Gen3の `incoming-classifier` は番号ではなく種別（固定/携帯/非通知等）で分岐するため統合。

## 人間が確認・設定すべき事項

1. **個人情報サブフローの作成**: `docs/reference/bivr/samples/個人情報サブフロー.bivr` をベースに、「北里」プレフィックスの4サブフロー（氏名聴取・生年月日聴取・電話番号聴取・診察券番号聴取）を作成すること。

2. **acceptance_times の設定**: 受付時間は平日（土日祝除く）8:30〜17:00。年末年始（12/27〜1/4）の特別対応も設定すること。

3. **転送先電話番号**: 3件の転送先番号を設定すること（折り返し予約センター・眼科予約センター・婦人科予約センター）。

4. **incoming-classifier の office_id**: BrekekeのIVRプロパティで設定すること。

5. **IVRプロパティの発話テキスト確認**: `output/properties_北里大学病院_診療.md` の内容を確認し、病院スタッフと発話内容を調整すること。

6. **診療科リストの確認**: Gen1で参照されていた診療科（集学的がんセンター・心理室等）が OpenAI プロンプトに適切に含まれているか確認すること。

7. **saveCompletionFlag2db のパラメータ**: 各用件のstatusとsmsFlagを業務要件と照合すること。

8. **050番号の入電経路**: 予約センター直通番号（050-1707-0320/050-1707-4509）の取り扱いをシステム担当者と確認すること。
