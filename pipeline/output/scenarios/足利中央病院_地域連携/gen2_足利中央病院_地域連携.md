## Gen2→Gen3 移管ノート

### フロー概要

- n8n なし。標準的な Gen2 地域連携フロー。
- 通話対象: **医療機関の地域連携室スタッフからの問い合わせ**を想定（一般患者向けではない）。紹介元医療機関からの連絡を受けて、相談・紹介/日程調整/その他問い合わせを受け付ける。
- 受付制限:
  - **当日予約不可**: 希望日時が「今日」の場合は代表電話（0284-72-8401）案内で終話（endpoint="電話案内", status="2"）。
  - **緊急案件**: 通話開始直後の緊急確認で「はい」の場合、即座に 0284-72-8401 へ転送（endpoint="転送", status="3", history="緊急案件"）。
  - **移管方針**: 当日判定は `希望日時` 聴取後にブロック型 `date_of_call_classifier` を配置する想定。緊急判定は `generate_by_OpenAI`（B型 Yes/No）で実装。
- 用件分類: **相談・紹介 / 日程調整 / その他** の3分類（enum値は "相談・紹介"、"日程調整"、"その他"、""）。
  - 「相談」と「紹介」は同一サブフローに合流（Gen2 yaml で「相談_患者氏名聴取」へ両方合流する構造）。
- FAQ/RAG: Gen2ではFAQ記載項目のみRAG検索、記載なしの質問は「申し訳ございません。ご質問いただきました内容はAI電話では対応できませんので、終話後に改めてご連絡をいたします。」と案内し、直前のヒアリング項目を再発話。**Gen3ではRAGサブフローを配置するかを director が判断**（Gen2資料にFAQ本体が含まれていないため、既定では RAG サブフロー省略＋FAQ不適合アナウンスのみとする）。

#### 診療科の読み仮名・同義語マッピング（STT 揺れ吸収用）
- **内科**: ないか / いっぱん / そうごう
- **循環器内科**: じゅんない / じゅんかんき / しんぞう
- **人工透析内科**: じんこうとうせき
- **消化器内科**: しょうかき / い / おなか
- **整形外科**: しんけい ※注意: 「しんけい」は神経内科とも重複、generate_by_OpenAI の正規化プロンプトで両科を候補提示させる
- **皮膚科**: ひふ
- **神経内科**: しんけい

#### リストにない診療科の扱い
「当院に○○科はございませんが、○○科でしょうか？」とレコメンドし、近しい診療科に寄せる（例: 呼吸器→呼吸器内科のように回答するが、当院にはそもそも呼吸器科がないため、該当がない場合は正規化サブフロー内で対応。移管方針: generate_by_OpenAI（D型 正規化）に「リスト外の場合は最も近い科を1件だけ提案」の固定ルールを含める）。

#### 用件単語の同義語（classification 正規化用）
- 「診察」と判定する単語: 印刷 / 定期 / 新薬 / 再診 / 最新 / 禁忌 / 診療
- 「予約」と判定する単語: 薬 / お薬
※ただし本シナリオの分類は「相談・紹介 / 日程調整 / その他」の3択であり、上記単語は **日程調整または相談・紹介** に寄せる（director 判断が必要）。移管方針: generate_by_OpenAI（A型 分類）のプロンプトに上記同義語表を含め、3択に収束させる。

#### 読み仮名の固定ルール
- "直通" は「チョクツウ」と読む（「ナオミチ」は禁止）。TTS 発話テキストでは必要箇所をカタカナ固定。

### function スキーマ（主要フィールド）

function 名: `dialogue_completed`（聴取内容を保存）

**required 全21フィールド**:
- `classification` (enum): "相談・紹介" | "日程調整" | "その他" | ""  → display_type: CLASSIFICATION
- `patientName` (TEXT): 患者名（カタカナ）
- `patientDateOfBirth` (DATE_OF_BIRTH): 生年月日 (例) 1998-06-09 00:00
- `medicalCardNumber` (NUMBER): 診察券番号（本シナリオでは聴取しないが function 定義上 required）
- `hospitalName` (TEXT): 医療機関名（入電者所属の施設名）
- `contactName` (TEXT): 入電者氏名
- `additionalPhoneNumber` (PHONE_NUMBER): 連絡先電話番号 (例) 0312345678 / 08012345678
- `history` (TEXT): 緊急性確認（"緊急案件" | "通常"）
- `reservationDate` (DATE): 予約日時 (例) 2016-01-02 00:00（本シナリオでは聴取せず、`DesiredreservationDate` とは別管理）
- `DesiredreservationDate` (TEXT): 予約希望日（日程調整サブフローのみ）
- `reason` (TEXT): 予約変更または予約キャンセルの理由（本シナリオ未使用だが required）
- `institution` (TEXT): 検査項目（本シナリオ未使用だが required）
- `introduction` (TEXT): 紹介状（本シナリオ未使用だが required）
- `disease` (TEXT): 病名（相談・紹介サブフローのみ）
- `details` (TEXT): 内容確認（相談・紹介/日程調整/その他すべてで使用）
- `replyDeadline` (TEXT): 返信期限（日程調整サブフローのみ）
- `question` (TEXT): 最後の問い合わせ
- `checkout` (TEXT): 途中切断ステータス。途中切断時の質問項目名を代入。候補: 冒頭切断 / 用件確認 / 緊急性確認 / 紹介目的 / 検査項目 / 内容確認 / 診療科 / 患者名 / 生年月日 / 連絡先電話番号
- `endpoint` (enum): 冒頭切断 | 時間外 | 電話転送 | 電話案内 | 途中切断 | 通話完了
- `status` (enum): "0" | "1" | "2" | "3"（0=途中切断, 1=通話完了, 2=電話案内, 3=転送）
- `smsFlag` (enum): "0" | "1" | "2"

**status/endpoint 正規化ルール**（出力直前）:
- 通話完了（終話アナウンス後切断）: endpoint="通話完了", status="1"
- 代表電話案内 / 受付不可 / 宛名不明: endpoint="電話案内", status="2"
- 転送: endpoint="転送", status="3"
- 未取得スロットあり/途中切断: endpoint="途中切断", status="0"

### 特記事項

1. **1回聴取禁止 / 再ヒアリング禁止**（最重要）:
   - 一度聴取した内容（診療科 / 患者氏名 / 用件など全情報）は絶対に聞き直さない。
   - 複数情報を一度に発話された場合も、既取得情報は再確認せず不足項目のみ質問。
   - **Gen3 移管方針**: generate_by_OpenAI 各モジュールの prompt に「既取得 context の再質問禁止」「不足項目のみ質問」ルールを含める。`SKILL_quality_criteria.md` の4本柱に準拠。

2. **復唱確認の可否**:
   - **復唱する**: 用件聴取後 / 患者生年月日聴取後 / 電話番号聴取後（`<speak type="telephone" breakc="300ms">{}</speak>ですね。` フォーマット）/ 希望日時聴取後（日程調整サブフローのみ必須）
   - **復唱しない**: 患者氏名 / 施設名 / 入電者氏名 / 病名 / 内容 / 診療科 等
   - Gen3ブロック設計では復唱必要項目のみ echo_back:true を指定。

3. **緊急案件の即時転送**:
   - 冒頭アナウンス直後に緊急確認 → "はい" で `<forward department="担当者" phone_number="0284728401"/>` を発話し転送。
   - history="緊急案件" を格納、status="3", endpoint="転送"。

4. **施設名の特別正規化**:
   - "明日は赤十字病院" と STT された場合 → hospitalName="足利赤十字病院" に固定正規化。
   - generate_by_OpenAI（D型正規化）の prompt に上記マッピングを明記。

5. **電話番号聴取サブフロー**:
   - 入電番号取得済み（非通知でない） → 「今おかけの{incoming_phone_number}でよろしいですか？」で確認、"はい" で入電番号流用。
   - 非通知 or "いいえ" → 直接聴取。
   - バリデーション: 10桁または11桁。11桁なら先頭3桁が 090/080/070/060/050 のいずれか。9桁/10桁で先頭0欠落時は先頭に0補完。不正時は再聴取。
   - 最終的に `<speak type="telephone" breakc="300ms">{additionalPhoneNumber}</speak>ですね。` で復唱、"いいえ" で再聴取。
   - **移管方針**: 共通サブフロー（電話番号）をそのまま `Jump to Flow` で呼ぶ。memory: feedback_subflow_connection_rule に従いインライン展開禁止。

6. **患者氏名のカタカナ判定**:
   - カタカナでなければ "うまく聞き取りができませんでした、ゆっくり話すとAIが聞き取れません。普通のスピードではっきりと発話してください。" と発話して再聴取。
   - 苗字のみの場合 "下のお名前もお伺いしてもよろしいですか？" と追加聴取。
   - "カタカナで" と質問することは禁止（入電者に表記を意識させない）。

7. **生年月日**:
   - 西暦で聴取（例: "1980年7月1日のように"）。和暦発話時は西暦変換。
   - 復唱後 "いいえ" で再聴取。

8. **希望日時（日程調整のみ）**:
   - 必須復唱。STT で「2月」「4月」が誤認識されやすいため、今日の日付基準で補正（例: 今日4/15に "2月20日" → "4月20日でよろしいでしょうか" と復唱）。
   - 予約日（reservationDate）と予約希望日（DesiredreservationDate）は別物。予約日聴取がない場合に希望日時を予約日へコピー禁止。
   - 10月以降に1〜9月を言われた場合は翌年に解釈。
   - 「明日」「明後日」「来週の月曜」等の相対表現は YYYY-MM-DD に変換。
   - **当日判定**: `date_of_call_classifier` ブロックで今日と比較、今日なら代表電話案内。

9. **返信期限の確認（日程調整のみ）**:
   - "返信期日のご希望がありましたらお話しください。" → replyDeadline に格納。
   - 「順番に対応しておりますので、しばらくお待ちください」の定型応答を generate_by_OpenAI プロンプトに含める。

10. **最後の問い合わせ（共通）**:
    - 全サブフローで終話前に "そのほかに何かお問い合わせは有りませんか？" を発話、question に格納。

11. **FAQ/RAG の扱い**:
    - FAQ にある質問 → RAG回答後、元のヒアリング項目を再発話して戻る。
    - FAQ にない質問 → "申し訳ございません。ご質問いただきました内容はAI電話では対応できませんので、終話後に改めてご連絡をいたします。" と発話後、**直前ヒアリング項目の質問文を一言一句変えず再発話**。
    - 例外: 「その他の用件」の「内容」ヒアリング中にFAQ不適合返答をした後は、質問を繰り返さず **即終話アナウンス**。
    - **Gen3 移管方針**: 元資料にFAQ本体がないため RAG サブフローは既定で配置しない。配置する場合は各hearingブロック後にRAGサブフローを挿入し、RAG no_result 時は FAQ不適合アナウンスをメインフロー側のTTSで発話（memory: feedback_rag_subflow_tts に従いメインフロー冒頭TTSは置かない）。

12. **終話アナウンス（日程調整/相談・紹介）**:
    - 「ご依頼を承りました。3診療日以内に担当者から折り返しご連絡いたします。それでは失礼いたします。」
    - 発話後 status="1", endpoint="通話完了", smsFlag="1" を格納して切断。

13. **口調・言葉づかい**:
    - 全て丁寧語。"かしこまりました" は可。"ありがとうございます。" "お気軽に" は禁止。
    - 質問時は "おっしゃってください" 禁止、"お話ください" に統一。
    - 回答受領後に "ありがとうございます。" を言わず即次へ進む。

14. **`<dtmf />` タグ**:
    - 削除禁止。連絡先電話番号・用件確認で使用（`<dtmf/>`、`<dtmf digit="1"/>`）。
    - Gen3 ではプロパティ側 TTS テキストにそのまま埋め込む。

15. **SSML の扱い（重要）**:
    - Gen2資料には `<speak type="telephone">` `<forward>` `<dtmf/>` 等のタグが登場するが、**Gen3 では SSML 禁止**（memory: feedback_tts_ai_no_ssml）。
    - 電話番号読み上げは `{tts_g:...}` 内で通常テキストとして組み立て、読み仮名制御は発話テンプレート側で対応。
    - `<forward>` タグは Gen3 の `call_transfer` ブロックに置き換え、phone_number をプロパティで管理。

16. **再質問時の固定テンプレート**:
    - 「申し訳ありません。うまく聞き取れませんでした。ゆっくり話すとAIがうまく聞き取れません。普通のスピードではっきりとお話しください。再度伺います。（元の固定質問文を厳密再提示）」
    - ユーザー発話の挿入は禁止。Gen3 では Retry モジュールのTTS発話として固定。

17. **サブフロー化推奨項目**（memory: feedback_subflow_connection_rule）:
    - 患者氏名 / 生年月日 / 電話番号 は共通サブフローを `Jump to Flow` で呼ぶ。メインにインライン展開しない。
    - `classification` 分岐は `context_match_router` でサブフロー呼び出し（3分岐、catch-all は "その他"）。

18. **catch-all 仕様**（memory: feedback_brekeke_catchall_pattern）:
    - incoming-classifier の「その他」は `^*$`。
    - context_match_router の用件分岐は3件すべてを明示条件として並べ、catch-all は "その他" を既定とする（Brekeke 仕様準拠）。
