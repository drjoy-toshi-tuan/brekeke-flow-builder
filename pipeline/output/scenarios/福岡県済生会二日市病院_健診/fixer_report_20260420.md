#### C-003: 用件確認の「検診」条件付きマッピングの実装曖昧

**対象**: OpenAI_用件確認 (Line 2-118)

**問題**:

- 予約: 「検針」「返信」「天候診断」「返金控寸断」「検診（乳がん/子宮がん/子宮頸がん除く）」

**プロンプト側の実装（Line 62-104）**:

```markdown

### 変更
変更

ドック

### 予約（条件付き：検診）
「検診」という語を含み、かつ直前に「乳がん」「子宮がん」「子宮頸がん」の語が
**含まれない**場合に限り → 予約
```

**曖昧性**:

1. **「直前に」の定義が不明確**
   - 例: 「子宮がん検診」と発話された場合
     - 「検診」を含む ✓
     - 「乳がん」「子宮がん」「子宮頸がん」を含む ✓
     - 「子宮がん」が「検診」の**直前**に存在する → 条件に該当しない → 「予約」に分類されない
     - **正しい判定**（期待値: NOT 予約）
   
   - 例: 「乳がん検診」と発話された場合
     - 「検診」を含む ✓
     - 「乳がん」を含む ✓
     - 「乳がん」が「検診」の直前に存在する → 条件に該当しない → 「予約」に分類されない
     - **正しい判定**（期待値: NOT 予約）
   
   - 例: 「検診」のみ発話された場合
     - 「検診」を含む ✓
     - 「乳がん」「子宮がん」「子宮頸がん」を含まない ✓
     - 条件に該当する → 「予約」に分類される
     - **質問**: 「検診」単独（乳がんなど対象を指定しない）は「その他」ですか、それとも「予約」ですか？

2. **設計書との整合性の問題**
   - 設計書では「検診（乳がん/子宮がん/子宮頸がん除く）」を「予約」に含めると記載
   - つまり「一般的な検診」は予約対象、「乳がん検診」「子宮がん検診」「子宮頸がん検診」は非対象

   - プロンプトの実装では「直前に」という表現で、複数の除外ワードが並列的に処理される可能性があり、ロジックの厳密性が不足

**修正方針**:
1. 「直前に」の定義を明確にする（例: 「スペースで区切られた直前トークン」「任意の距離内」など）
2. または、正規表現を使用して「(乳がん|子宮がん|子宮頸がん)検診」パターンを明示的に除外

### W-003: DTMF モード（数字選択）が未実装

**重大度**: ⚠️ **Warning**

**対象モジュール**: すべての enum 型モジュール（用件確認、受診歴確認、前回と同じ内容確認、受診希望内容_初回/再診、変更内容、予約希望日）

`dtmf_max_length: null` により **DTMF は使用しない** 設計。

**判定**:

**修正方針**:

- DTMF 対応が必要な場合: prompter が各プロンプトに DTMF マッピングを追加
