### 抽出した要件サマリー

主要要件:
- 用件は **5 分岐**: 健診予約 / 日時変更 / キャンセル / 企業問合せ / 問い合わせ
- 健診予約は SMS 送信ルート（受診票確認 → 電話番号サブフロー → 携帯あれば SMS、なければ WEB 案内終話）。氏名・生年月日聴取はなし
- 日時変更 / キャンセル / 問い合わせは個人聴取 4 点セット（氏名・生年月日・電話番号 + 内容/日時）→ 完了フラグ → 終話
- 企業問合せは 内容 → 企業名担当者名 → 企業連絡先電話番号 → 完了フラグ → 終話
- 終話は **入電時刻が営業時間内/外で TTS が切り替わる 2 段判定**（聴取中に営業時間境界をまたいだ場合の文言調整）
  - 営業時間内: 「内容を確認の上、担当者よりご連絡いたします」
  - 営業時間外: 「当施設の営業日に担当者より、ご連絡いたします」
  - キャンセルのみ時間によらず固定文言「ご予約のキャンセルを承りました」
- 営業時間: **月曜〜土曜 8:00 - 16:00**（祝日除く。`入電曜日時間判断.yaml` 由来）
- GW / お盆等の連休中アナウンス: Gen1 ではハードコード（5/1〜5/7、お盆復帰 8/16）。Gen3 では Dr.JOY 側 acceptance_times の休業日設定で代替（NON-BLOCKER 参照）
- 復唱項目: 予約日 / 受診希望日 / 希望時間 / 生年月日 / 電話番号（Gen1 で `<? $... ?>` 動的タグあり）
- DTMF 入力対応: 用件確認、受診票確認、登録番号、生年月日、電話番号系（Gen1 の DTMF分岐ノードに合わせて Gen3 でも DTMF_AmiVoice）
