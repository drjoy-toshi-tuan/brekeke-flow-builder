### 移管方針
- 既存ノート `gen1_新潟県けんこう財団_健診.md`（3施設用）の **設計をベースに、長岡固有差分のみ**当ノートに記述。共通部分（用件 5 分岐、SMS 送信判定、折り返し確認、当日予約 10:30 締切、終話パターン、コンテキストフィールド、DTMF 化方針、AmiVoice 例外）は 3施設用ノート参照
- 冒頭文言を「長岡健康管理センターの、予約専用、AI電話です。」固定に置換。施設選択 DTMF（東新潟/西新潟/長岡）は **削除** → 3 施設用と異なり、用件聴取直後すぐ受診票有無確認へ進む
- 用件 5 分岐: 1=健診予約 / 2=日時変更 / 3=キャンセル / 4=企業問合せ / 5=問い合わせ。OpenAI 用件正規化辞書は 3施設用と共通エンティティを再利用
- 主流れ: opening → API_DRJOY → 入電曜日時間判断（acceptance_times）→ GW/お盆 2 系統の特別アナウンス分岐 → 用件聴取（通常 or 連休中）→ 用件分岐 → 各ルート → 氏名 / 生年月日 / 連絡先 → 完了フラグ → 入電曜日時間判断 → 営業時間内/外 別の完了アナウンス → 終話
- 完了フラグ後に**もう一度 acceptance_times を通す**理由は、聴取中に営業時間が切り替わった場合の終話文言切替（営業時間内 → 「内容を確認の上、担当者よりご連絡」 / 営業時間外 → 「当施設の営業日に担当者より、ご連絡」）。Gen3 でも 2 段判定を維持

### 施設固有の特殊ルール
- **施設名 = 「長岡健康管理センター」固定**: 3施設用が「東新潟/西新潟/長岡」を施設選択 DTMF で切り替えるのに対し、長岡用は施設選択ステップ自体が削除されている。施設別休業日も「長岡」固定で扱える（director デフォルトの receiving_facility 単一値）
- **GW 期間ハードコードあり (`GWアナウンス` `5/1時間分岐` `5/2 ~ 5/6` `5/7時間分岐` `通常営業へ`)**: 「ただいま休業期間中ですので、予約・変更・キャンセルのみ受付」+ 「その他のお問い合わせは、5月7日以降にご連絡ください」TTS。Gen3 では `script_営業日判定` の holidays 配列に GW 7 日分を入れて、休日アナウンス TTS を読み上げる方式に統合 → 文言は GW 限定なので施設テンプレート文言「営業日に担当者より、ご連絡いたします」型で統一（director デフォルト）
- **お盆期間ハードコードあり (`挨拶（連休中）` の「8月16日土曜日以降にご連絡ください」)**: お盆休業 → 復帰日 8/16 をハードコード。これも holidays 配列に集約 → 復帰日アナウンスは director デフォルトに含めない（休日アナウンスのみ流す）
- **連休中専用 完了 TTS が 4 種類**: `連休中：健診予約` `連休中：予約変更` `連休中：健診予約_1` `連休中：予約変更_1` の 4 箱が同一文言「予約の確定は、当施設の営業日に担当者より、ご連絡いたします」→ Gen3 では 1 本に圧縮可能
- **時間分岐ハードコード閾値**: 営業時間内/外の判定が `8時まで` / `16時まで` の 2 段（時間で分岐 箱）。3施設用と異なり 8:00 開始 / 16:00 終了の 1 系統のみ → acceptance_times に 8:00〜16:00（曜日依存は入電曜日時間判断 yaml を参照、director 判断）
- **代表電話番号は 3施設用と異なる**（長岡施設の代表番号が別）→ customer_doc から長岡用代表番号を確定（main.yaml には電話番号 TTS が無く、SMS / 折り返しのみで完結）。**フリーダイヤル / 代表番号案内アナウンスは無し**（main.yaml に番号読み上げ箱が存在しない）
- **「(保留)折返し方法確認.yaml」は dead code**: main.yaml `（保留）折返し方法確認` 箱が存在するが、`SMS送信へ` 直結で実質 SMS のみ → Gen3 では削除（director 判断）
- **用件聞き取り（連休中）.yaml は通常版用件聞き取り.yaml と TTS 文言違い**: 連休中は「予約・変更・キャンセル・問い合わせ・企業問合せの 5 つから〜」のような連休中限定の文言になっている可能性 → director が yaml の中身を確認して 2 系統 prompt 用意（or 1 本に統合し前段 announcement で告知）

### director が見落としやすいポイント
- 3施設用にあった `施設選択（東新潟/西新潟/長岡）` subflow は **長岡用には存在しない** → 施設別 holidays / 施設別代表番号の分岐は不要。受診票確認 → 登録番号 → 受診希望日 へストレートに進む
- `分岐：連休中` `分岐：連休中_1` `分岐：連休中_2` `分岐：連休中_3` の 4 箱は **時間内/外 × 健診/変更** の 2x2 マトリクスで連休中フラグを立てる中継 → Gen3 では context に `holiday_flag=true` を持って単一 termination で `連休中：完了` 文言を出す方が単純
- `用件の分岐_1` `用件の分岐_2` の 2 段分岐: _1 は受診希望日聴取後（健診予約は氏名へ、日時変更は希望時間へ）、_2 は予約日聴取後（日時変更→受診希望日、企業問合せ→企業担当者名、デフォルト→氏名）→ 既存 3施設用ノートと同一構造（CMR で吸収）
- `折り返し確認へ(*NG)` 11 箱は **すべて `折り返し確認` 直結**で実質 1 本に集約可能（director デフォルト）
- 完了フラグは **聴取完了後の acceptance_times 再判定** + 営業時間内/外で完了 TTS を切り替える起点 → Gen3 では date_of_call_classifier 系の前段 OpenAI（addCurrentDate: Yes scaffold 自動付与）を 1 段挟むのが妥当
- `企業：連絡先電話番号聞き取り.yaml` は企業問合せルートのみ通る → 個人 4 ルートでは通常の連絡先聴取 subflow（連絡先電話番号聞き取り.yaml）を使用、混在しない
- Gen1 動的タグ `<? $予約日の読み ?>` 等は echo_back に集約済み → `<% var %>` 構文への置換チェックのみ
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字禁止
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**
- **CMR conditions は必ず最後に `match: "other"` + next 明示**
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
