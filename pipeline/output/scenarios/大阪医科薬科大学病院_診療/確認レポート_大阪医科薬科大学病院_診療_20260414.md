### 抽出した要件サマリー

- **用件分岐**: 3択（変更 / キャンセル / 確認）。DTMF 1=変更, 2=キャンセル, 3=確認
- **変更ルート聴取項目**: 診療科 → (歯科口腔外科アナウンス) → 現在の予約日 → 次回予約希望時期 → 検査有無 → 変更理由 → 診察券番号 → 患者名 → 生年月日 → 連絡先電話番号 → 終話
- **キャンセルルート聴取項目**: 診療科 → (歯科口腔外科アナウンス) → 現在の予約日 → キャンセル理由 → 診察券番号 → 患者名 → 生年月日 → 連絡先電話番号 → 終話
- **確認ルート聴取項目**: 診療科 → 確認内容 → 診察券番号 → 患者名 → 生年月日 → 連絡先電話番号 → 終話
- **歯科口腔外科特別アナウンス**: 変更/キャンセル時に「手術に関する予約変更はAI電話では受付できません」を発話
- **診療科**: 31科 + 登録なし。音声認識誤変換補正ルール10件あり
- **復唱**: 生年月日、診察券番号、連絡先電話番号のみ許可。患者名の復唱は禁止
- **診察券番号**: 3桁以上必須、'C'→'4'置換ルール、「わからない」スキップ可
- **smsFlag**: 変更/確認=1, キャンセル=2（用件別に異なる）
- **非通知**: Gen2ではstatus=7。Gen3ではstatus=2にマッピング
- **電話番号聴取ロジック**: incoming-classifierで携帯/固定判定 → 携帯は着信番号確認 → 固定/不明は携帯有無確認から聴取
