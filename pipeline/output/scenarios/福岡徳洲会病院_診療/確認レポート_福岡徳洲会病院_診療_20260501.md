3. パイプライン実行は以下の順:

   - `@prompter` で OpenAI モジュールの prompt を埋め込み

   - `@reviewer` でプロンプトレッドチーム校閲
   - `validator.py` + `tester.py` で構造/プロンプト/到達性検証
   - `@fixer` で残存 Critical をブロック単位修正
   - `build_bivr.py` で .bivr 生成
4. Dr.JOY 側設定事項（セクション 3）はフロー稼働前に別途設定してください。
5. .bivr インポート後、テスト発信で以下の経路を最低限確認:
   - 用件 = 変更 → 通常診療科 → 全項目聴取 → END_受付完了_変更
   - 用件 = キャンセル → 通常診療科 → 全項目聴取 → END_受付完了_キャンセル
   - 用件 = 確認 → 通常診療科 → 確認_内容聴取 → 全項目聴取 → END_受付完了_確認
   - 用件 = 新規（DTMF 4） → END_新規受付不可
   - 診療科 = 呼吸器 → 呼吸器分岐確認 → 内科 → 通常進行
   - 診療科 = 呼吸器 → 呼吸器分岐確認 → 否定 → 用件聴取に戻る
   - 診療科 = 健康診断 → END_健康診断代表案内
   - 診療科 = 歯科口腔外科 / リハビリテーション科 → END_転送案内_歯科口腔外科リハビリ
   - 診療科 = ペインクリニック → END_麻酔科代表電話案内

   - 非通知 → END_非通知
