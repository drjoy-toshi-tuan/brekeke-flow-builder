### 移管方針
- Gen1「箱」構造 → Gen3 9 ブロック型に変換。八尾と同じく**新規予約フローは持たない**（用件 yaml は 3 分岐: 変更 / キャンセル / 確認）。ただし用件 yaml の DTMF 4 = 新規 を専用 termination に流す程度の最小実装あり。
- 冒頭は opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「お電話ありがとうございます。福岡徳洲会病院の、予約専用、AI 電話です」。
- 用件分岐は **Pattern A**（用件聴取 OpenAI 直後に 3 分岐: 予約変更 / 予約キャンセル / 予約確認）。echo_back 文言は元資料に明示なし → **director 判断**（用件は echo_back 必須ルールに従い、3 種固定文言で付与）。DTMF 4 = 新規予約は専用 termination で「新規予約は受付できません、代表電話へ」案内。
- 診療科分岐は **Pattern B（後段 ContextMatchRouter）併用**。診療科聴取後の経路: 分岐なし診療科（30 科以上の通常進行）/ 00 分岐あり診療科（呼吸器・脳神経で内/外を再聴取）/ 01 分岐あり診療科（産婦人科で産科/婦人科を再聴取）/ 健康診断（→ 健康診断代表案内終話）/ 歯科口腔外科 or リハビリ科（→ 転送案内終話）/ ペインクリニック（→ 麻酔科代表電話案内終話）/ 分からない（→ 通常進行）。
- サブフロー（患者氏名 / 生年月日 / 電話番号 / 診察券）は Jump to Flow 経由。**用件 = 確認の場合は診察券聴取をスキップ**。変更/キャンセルは診察券→氏名→生年月日→連絡先のフルセット。
- 時間判断.yaml（祝日チェック → 12/26〜1/4 の年末年始判定 → 平日 8〜20 時の時間判定）→ date_of_call_classifier ブロック。**addCurrentDate: Yes 自動付与対象**。

### 施設固有の特殊ルール
- **受付時間**: 祝日除く、月〜金 8:00〜20:00（徳洲会内で最も長い、湘南藤沢 9-16/9-14・八尾 8-16/8-11・札幌 9-17 と異なる）。
- **代表電話**: `092-573-6622`（病院代表、対象外診療科・聞き取れず終話・健康診断代表案内・新規予約案内で使用）。
- **代表電話の「受付時間」案内が代表電話と通常時間外で異なる**: `代表電話案内` 箱（聞き取れず終話）の文言は「土日祝を除く、月〜金 13:00〜16:00」、通常時間外案内は「月〜金 8:00〜20:00」、麻酔科代表電話案内は「月〜金 9:00〜12:00」、健康診断代表案内は「月〜金 10:00〜16:00 / 土 10:00〜12:00」。**4 種類の異なる受付時間案内 TTS が混在**。director は混同しないよう注意。
- **ペインクリニック直通（麻酔科 = 緩和ペインクリニック等）**: `092-514-6137`（平日午前専用、`麻酔科代表電話案内` 終話 TTS で 2 回繰り返し読み上げ）。
- **歯科口腔外科 / リハビリ科**: 専用 `END_転送案内（歯科口腔外科/リハビリ科）` 終話。元資料では END 名のみで TTS 本文未定義 → **director 判断**（八尾の歯科時間帯ルーティングのような複雑な仕組みは無いと推定）。
- **健康診断**: `END_転送案内（健康診断）` または直接 `健康診断代表案内` 箱経由で「健康診断は AI 電話では受付不可、代表電話へ。受付時間は月〜金 10:00〜16:00 / 土 10:00〜12:00」固定 TTS で終話。
- **分岐あり診療科は 3 系統**:
  - **呼吸器** → `呼吸器内科` / `呼吸器外科`（同義語に「ないか」「外科」「形が」「経過」「結果」等の音響的曖昧語が大量）
  - **脳神経** → `脳神経内科` / `脳神経外科`
  - **産婦人科** → `産科` / `婦人科`
  - 各分岐の TTS: 「呼吸器内科と呼吸器外科の、どちらですか?」等。否定で戻すと用件聴取からやり直し。
- **リハビリ確認内容サブフロー**: 用件 = 変更/キャンセル/確認、診療科 = リハビリ科 を選んだ場合に「内容は訓練と、診察のどちらでしょうか」で訓練/診察 Yes/No 確認。**ただし main.yaml では `リハビリ確認内容` の next が `分からない分岐` で終わっており、訓練/診察の判定値は後段で参照されない**（情報収集のみ）。
- **追加問い合わせ_追加診療科**: 変更/キャンセル/確認の途中で「他に診療科があればおっしゃってください。ない場合は、ありません、とおっしゃってください」と追加聴取。**変更時は「他に予約を変更する診療科」/ 確認時は「ご確認内容についてお伺いします」と TTS 本文が `用件分岐` で変わる**。
- **理由聴取**: 変更時「変更の理由をおっしゃってください」/ キャンセル時「キャンセルの理由をおっしゃってください」と用件で TTS が変わる。確認用件では理由聴取はなし。
- **変更希望日**: 変更時「予約希望日をおっしゃってください」/ 確認時「ご確認内容についてお伺いします」と用件で TTS が変わる。**変更時は予約日（現在）→ 理由 → 変更希望日 の 3 段聴取**。
- **予約日 DTMF 4 桁機能**: 予約日.yaml にシャープ/アスタ + 4 桁数字受付（過去日なら +1 年で確定）。
- **3 つの「予約 ◯◯ の流れ」終話 TTS が同一文言**: 予約変更 / キャンセル / 確認 すべてで「3 診療日以内の、担当者からの折り返し電話、もしくは、SMS にて受付完了となります。折り返しは、当院代表番号、092-573-6622 よりおかけいたします」と完全一致。**携帯/固定で TTS 分岐させる札幌・湘南藤沢方式は採用しない**。
- **DTMF 受付**: 用件聴取（1=変更 / 2=キャンセル / 3=確認 / 4=新規不可案内）と Yes/No 確認。予約日聴取（シャープ/アスタ + 4 桁）。
- **「もしもし」「もう一度」フォールバック**: 用件 yaml で「もしもし」発話 → 1 回目「はい、聞こえております」/ 2 回目以上「はい、もしもしと聞こえております」と回数別 TTS。
- **復唱（echo_back）対象**: 用件（3 種）/ 診療科 / 予約日 / 分岐確認 Yes/No。
- **入電番号取得分岐は無し**: 八尾の `終話電話番号確認` や湘南藤沢の `WEBか電話か` のような入電番号取得有無で分岐する仕組みは持たない。

### director が見落としやすいポイント
- 冒頭の `API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `API_DRJOY` → `時間判断` → `接続詞_7` → `用件` の 6 箱は Gen3 では opening 冒頭に吸収（時間判断は date_of_call_classifier として独立）。**`接続詞_7` の TTS 「新規予約に関するお問い合わせは、このお電話では受付できません。それでは、」は冒頭_アナウンス内に組み込む**。
- セリフ none 振り分けのみの「箱」（用件の確認、携帯電話番号チェック / _1 / _2、用件を判断 / _1〜_4、完了フラグ、診療科分岐、分からない分岐、各種NG 7 種、API_DRJOY_CONTEXT_SETTINGS、年末年始 等）は独立ブロック不要 → 前段の next 書き換え or context_match_router の一部として吸収。
- 用件 yaml の **エンティティ「用件」値定義に「新規」が含まれる**（同義語: ドック / 健康診断 / 健診 / 検診 / 人間ドック / 特定健診 / 予約 / 進学 / 就職 / 雇い入れ 等 28 個）。OpenAI 側で「新規」発話を検出した場合は **健康診断代表案内 or 新規予約不可案内**に流す。
- 用件 yaml の `セットした用件の分岐` で **「健診予約」判定の next が空欄**になっており、フロー的に未定義のループが存在 → 元資料のバグ相当。**Gen3 では「新規予約 / 健診予約」共通の END_新規受付不可 termination に集約して解消**。
- 診療科聴取 yaml の `分岐確認へ` 箱に `<? $分岐あり診療科の読み ?>` `<? $診療科の読み ?>` 等の Gen1 構文あり → **Gen3 では `<% var %>` に変換**。
- 診療科聴取 yaml の `00 分岐あり診療科読み取得` `分岐（呼吸器）読み取得` 等の **「読み取得」サブシナリオ呼び出し**は元資料で内部仕様が見えない → Gen3 では確認 echo_back TTS の中で分岐先診療科名を直接埋め込む（Script モジュール不要）。
- `祝日チェック` は元資料で「サブシナリオ呼び出し」扱い → カレンダー Script モジュールが必要（docs/brekeke/モジュール選定ガイド_v2.md §3.8 参照）。
- リハビリ確認内容 yaml は「訓練 / 診察」Yes/No と書きつつ実体は **フリー入力で SKILL_E 系 hearing**。**判定値を後段で使わない情報収集のみ**なのでサブフロー化までは不要。
- ContextMatchRouter.reference_module で **subflow を参照する場合は scaffold が付ける `jump_{step 名}` prefix を使う**。湘南藤沢徳洲会・八尾徳洲会・札幌徳洲会で同種ミスを記録済み。
- **CMR conditions は必ず最後に `match: "other"` + next 明示**（catch-all）。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。括弧付き END 名（例: `END_転送案内（歯科口腔外科/リハビリ科）`）は **Gen3 で半角括弧と日本語スラッシュを除去して ASCII 化**。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。`<? $... ?>` 構文は `<% var %>` に変換。
