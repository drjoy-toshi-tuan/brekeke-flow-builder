### 移管方針
- Gen1 の「箱」構造 → Gen3 scenario_flow 9 ブロック型に変換。主な対応:
  - 入電番号判断 → incoming-classifier。Gen1 では「IVRからの転送」と「上記以外」の 2 値分岐（非通知/海外/携帯/固定の 4 値分岐ではない）。Gen3 では incoming-classifier の `^IVR_TRANSFER$` / それ以外で振り分け、opening TTS だけ差し替える設計が妥当（director 判断）
  - opening + 冒頭_アナウンス（2 ブロック 1 セット）: IVR 転送経由は「こちらは健康診断の予約関連専用、AI 電話です。」のみ、上記以外は「倉敷中央病院付属予防医療プラザの、予約関連専用、AI 電話です。この通話は、通話の内容確認およびサービス向上のため、録音させていただいております。」を流す
  - 用件聞取り（復唱無し）.yaml → hearing ブロック（用件分類 generate_by_OpenAI）。**5 分岐**: 新規 / 変更 / キャンセル / 問い合わせ / 折返し連絡。各分岐直後に echo_back（用件の復唱）を 5 本配置 → Pattern A
  - 氏名 / 生年月日 / 電話番号（連絡先聴取 + 電話番号区切り 2 本セット）→ subflow ブロックで Gen3 標準サンプル JSON 完コピ
  - 予約日聞き取り / 希望時期 / 変更項目：受診日orその他 / 内容聞き取り / 追加質問 → 各 hearing ブロック（OpenAI 正規化、output_format は datetime / text / enum を director が判断）
  - 転送可否判断.yaml（受付時間/曜日/祝日 ＋ 担当者転送）→ acceptance_times（曜日・時刻判定）+ Script モジュール（祝日チェック）を冒頭 opening 直後に配置。addCurrentDate: Yes 自動付与の対象は date_of_call_classifier 系の前段
  - 締めあいさつ → termination ブロック内。携帯電話番号チェック結果で「電話 or ショートメール」「電話のみ」の 2 文言を切り替え（携帯チェックは scaffold 既定の電話番号区切り subflow 戻り値 phonetype を使用、SUBFLOW_RETURN_CONTEXTS 経由）
- 用件別の合流先（main.yaml の「次へ_*」「用件の分岐_1〜_3」を読み取り）:
  - 新規: 内容聞き取り → 希望時期 → 入電者氏名 → 生年月日 → 連絡先 → 締めあいさつ → 終話
  - 変更: 変更項目（受診日 or その他）→ 受診日なら希望時期、その他なら内容聞き取り → 予約日聞き取り → 入電者氏名 → 生年月日 → 連絡先 → 終話
  - キャンセル: 予約日聞き取り → 入電者氏名 → 生年月日 → 連絡先 → 終話
  - 問い合わせ: 内容聞き取り → 入電者氏名 → 生年月日 → 連絡先 → 終話
  - 折返し連絡: 折返し連絡該当 TTS → 終話（個人情報聴取スキップ）

### 施設固有の特殊ルール
- **受付時間**: 日曜祝日・年末年始除く、月〜金 8:00〜16:00、土曜 8:00〜12:00。土曜は不定休のため受付時間外/かけ直し案内 TTS で「土曜は不定休のためホームページの『営業日のご案内』をご確認ください」を必ず読み上げる（代表電話案内・転送ガイダンス両方に同文言あり）
- **代表電話番号**: 086-422-6800（予防医療プラザ専用番号）。代表案内では「ガイダンス 2 番を押してください」と必ず付記する（IVR の 2 番が予防医療プラザに繋がる前提）
- **3 回案内後カット**: 「案内回数カウント」箱で 3 以上で電話切断へ。Gen3 では gen_properties.py の HP 案内文言（「ご案内している番号は病院ホームページでもご確認いただけます」）を 2 回目挿入、3 回目で termination ブロックへ
- **用件 5 分岐は復唱必須**: Gen1 の用件聞取り（復唱無し）.yaml の名前は誤解を招くが、サブフロー内に「用件の復唱（新規/キャンセル/変更/問い合わせ/折返し連絡）」が定義されている → Gen3 でも各分岐直後に echo_back 5 本必須。直後肯定/否定/否定単語/再質問の処理は OpenAI 正規化＋ContextMatchRouter で吸収
- **「もしもし」検知**: 用件聞き取り入口で「もしもし」発話を検知 → 1 回目「はい、聞こえております」、2 回目以上「はい、『もしもし』と聞こえております」で再質問へ戻す → Gen3 では用件 OpenAI のプロンプトで救済語扱いする or 前段に独立 hearing を置くかは director 判断
- **入電者 = 受診者前提**: Gen1 では患者氏名サブフロー名が「入電者氏名」のみ → PatientName サブフロー 1 枠に充当。受診者と入電者の分離設計は無し
- **変更項目の 2 分岐**: 「受診日」「受診日以外の変更（その他）」のみ。受診日なら希望時期へ、その他なら内容聞き取りへ。default は内容聞き取り（main.yaml「変更項目の分岐」箱より）
- **携帯電話番号チェックで締め文言切替**: 連絡先 subflow 戻り値で携帯あり → 「担当者からお電話、もしくは、ショートメールにてご連絡」、なし → 「担当者から折り返しご連絡」。Gen3 では context_match_router で phonetype を分岐、各分岐に termination ブロックを 2 本ぶら下げる
- **当日予約・年末年始の特別分岐は無し**: Gen1 main.yaml 内に当日カット箱・年末年始分岐箱は存在しない → director デフォルト（acceptance_times に集約・受付時間外で統一）でよい
- **「追加質問」は問い合わせ・新規・変更時に必須**: Gen1 では「他に、オペレーターにお伝えになりたいことがあれば、お話下さい」を聴取してから締めへ。Gen3 では各用件最終段の hearing ブロックとして配置（合流前）→ ただし問い合わせ/折返し連絡では subflow 後即終話のパスもある（main.yaml「用件の分岐_2」「用件の分岐」の差を director が読み取る）

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` と `API_DRJOY` 系 3 箱（`API_DRJOY` / `API_DRJOY_1` / `API_DRJOY_2`）はすべて Gen3 では opening 冒頭の標準処理に吸収 → 明示ブロック化不要
- 「次へ」「次へ_1〜_5」（セリフ「それでは、」「続いて、」「次に、」「最後に、」）は短い繋ぎ TTS → 次段ブロックの先頭 TTS にマージ（director デフォルト）
- 「セリフ: none」かつ振り分けのみの箱（`用件の分岐` / `用件の分岐_1〜_3` / `変更項目の分岐` / `通話完了` / `通話完了_1` / `完了フラグ` / `入電番号判断` / `入電日時間判断ノードへの接続` 等）は Gen3 では独立ブロック不要 → 前段の next 先書き換えか context_match_router の一部として吸収
- 「CP 〜」プレフィックス箱（`CP 用件聴取不可` / `CP 受付時間外` / `CP 変更項目聴取不可` / `CP内容聴取不可` / `CP 予約日聴取不可` / `CP 氏名聞き取れず` / `CP 生年月日聴取不可` / `CP 連絡先聴取できず転送`）は **Compliance Path 系のリトライ上限失敗** → Gen3 では各 hearing/subflow の `error_jump` 先 termination に集約（氏名・生年月日聴取失敗のみ転送可否判断へ、それ以外は終話へ）
- 「★転送可否と入れ替え」箱は元資料時点で名残（テスト用）→ 削除して直接転送ブロックへ
- 転送先は **Brekeke Blind Transfer ではなく「代表番号 086-422-6800 にかけ直し」+「ガイダンス 2 番」**を音声案内する設計（Gen1 の `転送コメント` / `転送ガイダンス` / `転送ガイダンス` の流れ）。担当者直結の Blind Transfer は無し → call_transfer ブロック不要、termination + 番号読み上げで完結（director デフォルト 6「Blind Transfer デフォルト」は適用しない、本案件は番号案内型）
- 「転送可否判断」サブフロー（曜日 + 時刻 + 祝日）は受付時間内/外の判定のみ → Gen3 ではメインフロー冒頭の acceptance_times + Script（祝日リスト）で代替し、メイン内に subflow ブロックは置かない
- 予約日聞き取り.yaml には DTMF 4 桁（月日）入力 + 過去日付なら + 1 年補正のロジックあり → Gen3 では Brekeke 標準の datetime hearing + addCurrentDate: Yes で過去日補正は OpenAI 正規化のプロンプトで吸収
- 生年月日聴取.yaml の「予約日比較（生年月日扱いの数値範囲チェック大正〜令和）」は予約日では使われない誤参照 → そのまま無視してよい（Gen1 のサブフロー流用バグの残骸）。Gen3 では生年月日聴取サブフロー JSON が独立処理
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字（①②③…⑩）等の環境依存文字は禁止
- **ContextMatchRouter.reference_module は subflow を参照する場合 `jump_{step名}`（scaffold が付ける prefix）を使う**。subflow ブロックの step 名そのものではダメ
- **CMR conditions は必ず最後に `match: "other"` + next 明示**
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
- 「<? $予約日の読み ?>」「<? $連絡先電話番号（ハイフン区切あり） ?>」「<? $入電番号（ハイフン区切りあり） ?>」等の Gen1 動的タグは Gen3 では `<% var %>` 構文へ。subflow 内 echo_back に集約済みのため、メインフローの直接参照は不要
