### 重大問題: 日付フォーマット不一致

#### C-001: 日付出力フォーマット誤り（CRITICAL）

<prompter>

**対象モジュール**:
- `OpenAI_予約変更_現在の予約日`（行1252-1357）
- `OpenAI_キャンセル_現在の予約日`（行1619-1724）

**問題**:

プロンプトは以下のフォーマットを指定（行1277、1707）:
```
出力整形：yyyy/mm/dd 形式で出力する。月日は必ず2桁ゼロパディング。
```

入出力例でも確認（行1342-1356、1709-1723）:
```
- 入力「3月4日」 → 出力「2026/03/04」
```

しかし設計書では以下を要求（step_details「予約変更_現在の予約日」）:
```yaml
notes: "yyyy-MM-dd 00:00:00 形式で格納。復唱（echo_back）あり"
```

**不一致内容**:
1. **区切り文字**: スラッシュ(`/`)  vs ハイフン(`-`)
2. **時刻部分**: 時刻なし vs `00:00:00` を含む

**影響**:
- システム側で日付解析時に形式エラーが発生する可能性
- DB 保存形式との不一致でデータ整合性が失われる可能性

**修正方針**:

プロンプトの STEP6「出力整形」の指定を以下に変更:

```
## 【STEP6：出力整形】
yyyy-MM-dd 00:00:00 形式で出力する。
- 年は4桁ゼロパディング
- 月日は2桁ゼロパディング
- 時刻は常に「00:00:00」を付加
```

入出力例も改訂:
```
入出力例（基準日: 2026/02/24）:
- 入力「3月4日」 → 出力「2026-03-04 00:00:00」
- 入力「2026年3月4日」 → 出力「2026-03-04 00:00:00」
- 入力「令和8年3月4日」 → 出力「2026-03-04 00:00:00」
```

#### W-001: 診療科プロンプムの複数マッチ時の UX

<prompter>

**対象モジュール**:
- `OpenAI_診察予約_診療科`（行825-989）

**問題**:

STEP4「語句一致判定」で複数の診療科ラベルが同時にマッチした場合、以下が指定（例: line 828）:
```
※複数の診療科ラベルが同時に一致した場合は NO_RESULT
```

**具体例**:
- ユーザー入力「内科」 → 高齢者総合診療科、循環器内科、消化器内科、呼吸器内科、腎・高血圧内科、膠原病内科、糖尿病・内分泌内科、脳神経内科、外科・消化器外科（複数マッチ）

- ユーザーは「内科」と言っただけで「わかりません、もう一度」と繰り返される

- 再聴取時の TTS アナウンスが無ければ、ユーザーが混乱する

**改善提案**:

再聴取時の TTS アナウンスで「詳しく教えてください」など明確化することを推奨。プロンプト側では現在の NO_RESULT ロジックで問題ありませんが、運用側で以下を確認:

1. 再聴取時の TTS が「詳細な診療科名をお答えください」等を案内するか
2. リトライ上限達成時に `no_result_default='登録なし'` で吸収するか

#### W-002: echo_back における日付フォーマット

<prompter>

**対象モジュール**:
- `OpenAI_予約変更_現在の予約日`（行1252-1357、echo_back: true）
- `OpenAI_キャンセル_現在の予約日`（行1619-1724、echo_back: true）

**問題**:

設計書で両モジュールに `echo_back: true` が指定されており、復唱確認があります（step_details: 予約変更_現在の予約日 line 1305 等）。

しかし以下が不明瞭：
- **プロンプト出力**: `2026-03-04 00:00:00`（日時）
- **echo_back での復唱**: 日付だけ「2026年3月4日」と読み上げるか、時刻も「00時00分」を含めるか

**現在の設計**:
プロンプト側は `yyyy-MM-dd 00:00:00` を指定しますが、echo_back は IVR プロパティ側で制御されるため、プロンプト側では「時刻は読み上げない」という明記がありません。

**改善提案**:

プロンプトに以下を追記することで明確化:

```
## 【echo_back 対応】
出力した日付（yyyy-MM-dd 00:00:00）は、IVR の復唱処理で「YYYY年MM月DD日」形式で読み上げられます。
時刻部分（00:00:00）は復唱では読み上げられません。
```

**対象**: 全 enum/classify モジュール（STEP4「それ以外」）
