### 移管方針
- 冒頭は opening + 冒頭_アナウンス の2ブロック1セット。冒頭文は「こちら牛久愛和総合病院の、予約専用ＡＩ電話です。今日、明日の予約につきましては代表番号へおかけなおしください。」を原文どおり TTS。入力待ち無しで次ブロックへ。checkpoint='0' を初期値として代入。
- **冒頭直後にワクチン優先判定 → 複数人予約判定 → 用件確認の3段ガード**（yaml 構造に忠実）。Pattern A（OpenAI 直後分岐）でそれぞれ代表案内終話に流す。director は intent 確認より先にこの2ガードを置くこと。
- 用件確認は classification を `予約 / 確認 / 変更 / キャンセル / 検診 / 問い合わせ` の6値で取得（function enum は `新規/再診/変更/キャンセル/確認/問い合わせ`、yaml は intent ベース）。「予約」のみ次に新規/再診（patientType）確認、それ以外は患者基本情報セクションへ直行。
- 氏名／DOB／電話番号／診察券番号は **サブフロー経由（Jump to Flow）**。連絡先電話番号は incoming 採用パターン（「今おかけいただいている電話番号でよろしいですか？」）を電話番号サブフロー内で処理。
- 診療科判定は Pattern A：リハビリ／小児科／健診センター／耳鼻咽喉科 → 各代表案内終話、子宮がん検診／乳がん検診 → 検診サブフロー、それ以外 → 診療科2確認（同日他科）→ フロー分岐判定。
- フロー分岐判定（intent 別）：予約 → 紹介状確認（新規のみ）→ 担当先生 → 予約希望日 → その他問い合わせ。変更／キャンセル → 担当先生 → 現在予約日 → 予約希望日（変更のみ）→ 理由 → その他問い合わせ。
- 終話パターンは termination_patterns に **①通話完了_固定 / ②通話完了_携帯（SMS） / ③代表案内_ワクチン / ④代表案内_複数人 / ⑤代表案内_リハビリ / ⑥代表案内_小児科 / ⑦代表案内_健診 / ⑧代表案内_耳鼻科 / ⑨代表案内_当日翌日 / ⑩自費検診不可** の10系統で組む（director 判断で系統統合可）。
- RAG / FAQ は元資料に記載なし → 不要。「その他問い合わせ」は question フィールドへ自由文格納し、なくなるまで繰り返し聴取（自己ループ）。
- Date of Call Classifier：当日／翌日判定（予約希望日聴取直後）に必須。「今日」「明日」「明後日」を着信日時から自動換算するルールあり。

### 施設固有の特殊ルール
- **ワクチン優先判定（最優先ガード）**：「インフルエンザ／コロナ予防接種・ワクチン」系は診療科判定や予約フローへ進まず、冒頭直後で代表案内終話（checkpoint=2 / smsFlag=0）。「帯状疱疹／シングリックス」は **皮膚科に確定**して通常フローへ（混同禁止）。
- **複数人予約禁止**：1通話につき患者1名のみ。「二人」検出で代表案内終話。**全フローに最優先で適用**。
- **診療科キーワード揺れの特殊救済**：「できないか」→ 血液内科、「鹿／会社／車」→ 歯科口腔外科、「産科」→ 眼科、「しきゅうがん／いきゅうがんけんしん／ちきゅうがんけんしん／きゅうがんけんしん」→ 子宮がん検診、「小児神経／小児心理／小児アレルギー／小児腎臓／小児腎」→ 小児科（小児循環器科に置換禁止）、「小児循環器」→ 小児循環器科。prompt で吸収。
- **分類語の揺れ**：「印刷／定期／新薬／再診／最新／禁忌」→ 診察、「薬／お薬／集約／要約」→ 予約、「天気」→ 新規 として扱う。用件確認 prompt に必須。
- **数字 4/5 の誤認識救済**：診察券番号で「C」「Sea」→ 4、「号」→ 5 へ置換。発話に数字が含まれない場合は再ヒアリング。
- **総合診療科・整形外科の特例**：変更／キャンセルの分岐をせずに **そのまま先に進む**（「最後の案内」へ直行）。新規予約フロー上では総合診療科は耳鼻咽喉科と同様「予約制でない」アナウンスで終話する点に注意（5-3-1 と 5-3-2 で挙動異なる、director 判断必要）。
- **診療科 = リハビリ／小児科／健診センター／耳鼻咽喉科 で代表番号 3回繰り返し**：「電話番号は『029-873-3111』です。繰り返します。〜」を 3 回 TTS する文言を保持（テンプレ短縮禁止）。健診センターのみ 029-873-4334。
- **連絡先電話番号 = 入電番号確認パス**：「今おかけいただいている電話番号でよろしいですか？」→「はい」で diam 確定、その他で別番号聴取。Yes/No 系プロンプトで `肺→はい / いいです→はい` の救済必須。
- **smsFlag ルール**：endpoint='通話完了' の場合のみ判定。携帯（^0(90|80|70|60)）かつ最後の案内が「ショートメールでご連絡」文言なら 1、それ以外（冒頭切断／途中切断／時間外／電話転送／電話案内）は **必ず 0**（1を入れてはいけない）。
- **status マッピング**：全 not empty で 1、代表番号案内（029）で 2、checkpoint=2 で 2、checkpoint=7 で 7、それ以外 0。enum は `0/1/2/7` のみ。
- **通話終了アナウンス**：「なお、予約日につきましてはご希望に添えない場合がございます。３営業日以内にご連絡いたします。お電話ありがとうございました。それでは失礼いたします。」を ①通話完了_固定 ②通話完了_携帯 双方の末尾に必ず付ける。
- **検診（子宮がん／乳がん）の症状確認分岐**：症状なし → 受診券有無 → 市町村 → 同時予約有無 → 折り返し終話。受診券無し → 自費検診不可で終話。症状あり → 専門外来（産婦人科／乳腺科）扱いで選定療養費案内 → 担当先生 → 予約希望日へ。

### director が見落としやすいポイント
- **改訂履歴 2026/04/20「【1-11. 営業時間外の予約不可】ルール削除」**：時間外の代表案内ブロックを置かない。yaml 側にも時間外分岐は無い（incoming-classifier では他 4 分岐のみ）。
- **「予約日と予約希望日は別項目」**（5章）。予約日のヒアリングが無い場合に予約希望日の中身を予約日に入れてはいけない。reservationDate=DATE 形式 / DesiredreservationDate=フリーワード（日付形式に整形しない）の二重管理。
- **「10月以降で1月から9月の日付を言われたときは翌年」**の年度判定ルール（5章）。日付正規化 prompt に必須。「2025年8月26日は火曜日」を前提として曜日計算可（ただし**曜日は発話しない**）。
- **「曜日は発話しない」**（1-7）。入電者が「10月1日水曜日」と発話しても日付のみ記録、矛盾しても再質問・訂正禁止。
- **「氏名は復唱禁止／電話番号と診察券番号のみ復唱許可」**（1-2 / 冒頭ルール）。復唱フォーマットは Brekeke の `<% %>` で別途プロパティ実装。SSML（`<speak type="telephone">`）はリテラル無効なので、director は SSML を設計書に書かないこと（feedback_tts_dynamic_value_syntax）。
- **「もしもし対策」記載なし**（元資料に明記なし）→ director 判断で Retry 標準パターンを採用。
- **生年月日リトライ仕様の明記なし** → director 判断で DOB サブフロー標準（Max Retry 設定）を踏襲。
- **5-3-1（新規予約）と 5-3-2（再診予約）で診療科分岐の挙動が異なる**：5-3-1 は総合診療科で終話（予約制でない案内）、耳鼻咽喉科も終話。5-3-2 でも同じ。一方、5-1（予約確認）と 5-2（変更／キャンセル）では総合診療科は終話せず最後の案内へ進む。**用件×診療科の二軸分岐**を CMR で組む必要あり。
- **「ありがとうございます」「お気軽に」禁止**、「おっしゃってください」→「お話ください」統一（1-0）。全 TTS 文言の語彙チェック。
- **「ご質問いただいた内容は AI 電話ではご対応できませんので…」**（1-4）の汎用 FAQ 案内文言。RAG 不在のため scenario_flow には組まないが、その他問い合わせブロックの prompt に注記必要。
- checkout（途中切断ステータス）は enum 無し自由文。director は step_details 側で代入ルール（'冒頭切断' / '用件確認' / '予約希望日' / '予約日' / '診療科２' / '診察券番号' / '理由' / '診療科' / '患者名' / '生年月日' / '連絡先電話番号'）を明記する必要あり。
