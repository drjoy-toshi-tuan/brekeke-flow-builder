### 抽出した要件サマリー

- **冒頭ガード**: ワクチン優先判定（インフル／コロナ系のみ。帯状疱疹は皮膚科確定）→ 複数人予約禁止 → 用件確認の3段ガード。OpenAI 直後分岐（Pattern A）で用件確認 hearing 1 つに統合
- **用件分類**: 予約／確認／変更／キャンセル／検診／問い合わせ の 6 値（function enum は `新規/再診/変更/キャンセル/確認/問い合わせ`、yaml は intent ベース）。「予約」のみ次に新規／再診（patientType）確認、それ以外は患者基本情報セクションへ直行
- **個人情報サブフロー**: 氏名 → 生年月日 → 電話番号（incoming 採用パターン）→ 診察券番号。氏名と生年月日は復唱なし、電話番号と診察券番号のみ復唱あり
- **診療科判定**: 35 科 + 「登録なし」（no_result_default）。リハビリ／小児科／健診センター／耳鼻咽喉科 → 各代表案内終話、子宮がん検診／乳がん検診 → 検診サブフロー、それ以外 → 診療科 2 確認 → フロー分岐判定
- **フロー分岐**（用件別）: 予約 → 紹介状確認（新規のみ）→ 担当先生 → 予約希望日 → 当日翌日判定 → その他問い合わせ。変更 → 担当先生 → 現在予約日 → 予約希望日 → 当日翌日判定 → 変更理由 → その他問い合わせ。キャンセル → 担当先生 → 現在予約日 → キャンセル理由 → その他問い合わせ。確認・問い合わせ → その他問い合わせ
- **検診フロー**: 子宮がん／乳がん検診 → 症状確認 → 症状なし → 受診券確認 → あり → 市町村 → 同時予約確認 → その他問い合わせ。受診券なし → END_自費検診不可。症状あり → 検診_症状あり案内 → 担当先生（予約フロー新規ルートに合流）
- **当日翌日判定**: Script モジュール（custom）。予約希望日のフリーワードに「今日」「明日」「明後日」が含まれているか、または着信日時から見て当日／翌日に該当するかをカレンダー計算で判定。当日／翌日なら END_代表案内_当日翌日 へ
- **その他問い合わせ自己ループ**: OpenAI で「なし／any」判定。生発話を question フィールドに連結保存。「any」なら『ご要望を承りました。順番に対応しておりますので、しばらくお待ちください』とアナウンスして自己ループ
- **終話パターン**: 異常系 3（END_非通知／END_時間外／END_聴取失敗）+ 代表案内系 7（ワクチン／複数人／リハビリ／小児科／健診／耳鼻科／当日翌日）+ 自費検診不可 + 通話完了系 2（固定／携帯）の合計 13 パターン。電話番号 3 回繰り返しのテンプレ短縮禁止
- **status マッピング**: 第3世代規約に準拠し 1（通話完了）／2（代表案内）／3（聴取失敗）／6（時間外）のみ使用。元資料の status='0' / '7' は不採用
- **smsFlag**: END_通話終了_携帯 経由のみ smsFlag=1。それ以外は全て smsFlag=0（元資料 1-13）
- **営業時間チェック**: 改訂履歴 2026/04/20 で削除されたため `use_acceptance_times: false`、時間外判定なし
