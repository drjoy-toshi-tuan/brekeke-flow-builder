- **シナリオ名**: 病診連携

### 抽出した要件サマリー

- **着信処理**: 平日 8:30〜17:15 のみ受付。土日祝・年末年始・時間外は END_時間外。非通知は END_非通知。
- **冒頭挨拶**: 「お電話ありがとうございます。信大病院・連携室AI電話です。」
- **共通聴取**: 施設名 → 担当者名 → 用件確認（4分類, DTMF 1〜4 並走 + 音声）。
- **用件 4 ルート**:
  - **FAX到着確認**: 用件詳細（診療報告 / 問い合わせ回答 / これから受診の方の追加情報 / その他）。「その他」のみ自由テキスト → 受付完了アナウンス。
  - **情報提供依頼**: 当日予約日確認 (Yes/No) → 患者氏名 → 生年月日 → 診療科 → 用件詳細（治療の経過 / MRIの実施可否 / その他）→ その他のみ自由テキスト → 返答期限 → 受付完了。
  - **入院・転院依頼**: 当日予約日確認 → 患者氏名 → 生年月日 → 診療科 → 希望時期 → 医師間共有 (Yes/No) → 医師名 → その他問合せ → 受付完了「入院・転院依頼を受付いたしました。診療科よりお返事いたします。」
  - **その他問合せ**: 用件詳細（紹介状の送信依頼 / 問い合わせの進捗確認 / その他の問合せ）→ 「その他の問合せ」のみ自由テキスト経由、それ以外は患者氏名 → 生年月日 → 診療科 → 受付完了。
- **PatientName サブフロー**: 1 本（`{group}$氏名聴取_{date}`）を 3 ルートから共有して呼び出し。
- **47 診療科**: main.yaml の診療科 entity から `clinicalDepartment.range_values` と `output_labels` に転記。`no_result_default: "登録なし"` を設定。
- **代表案内**: リトライ上限到達 → 「0263-37-3370」を 3 連呼するアナウンス → status=3。
- **復唱（echo_back: true）対象**: 施設名聴取 / 用件確認 / 用件詳細_FAX到着確認 / 用件詳細_情報提供依頼 / 用件詳細_問合せ の 5 ステップのみ。
- **同音異義救済**: OpenAI プロンプト層で吸収（店員/入金/返品/天神依頼→入院・転院依頼、Suica/受信→これから受診の方の追加情報、位置/イチ→1番系、ファックス→FAX到着確認 等）。
