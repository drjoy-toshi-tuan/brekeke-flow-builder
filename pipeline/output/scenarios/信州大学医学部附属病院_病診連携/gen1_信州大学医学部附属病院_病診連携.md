### 移管方針
- Gen1「箱」→ Gen3 9 ブロック型へ変換。opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「お電話ありがとうございます。信大病院・連携室AI電話です。」
- **入電情報あり API_DRJOY_CONTEXT_SETTINGS → API_DRJOY → 施設名_1 → 入電情報から取得した情報をコンテキストセット → 施設名確認**: 入電番号から施設名既知の場合は施設名サブフローをスキップ → `施設名確認`「<? $復唱 ?>、様、ご連絡ありがとうございます。」TTS で復唱のみ。未取得なら通常の施設名 hearing → 担当者名 hearing。Gen3 では opening 直後に施設名既知/未知の CMR を 1 段挟む（director 判断、浦添と同方針）。
- **用件分岐は Pattern A**（用件 hearing OpenAI 直後に 4 分岐: FAX到着確認 / 情報提供依頼 / 入院・転院依頼 / その他問合せ + DTMF 1〜4 並走）。echo_back（「FAX到着確認ですね。」「情報提供依頼ですね。」「入院・転院依頼ですね。」「お問合せですね。」）を各分岐直後に配置。
- **用件別の追加分岐**:
  - **FAX到着確認** → `用件②（FAX到着確認）` で 4 分岐 hearing（診療報告 / 問い合わせ回答 / これから受診の方の追加情報 / その他、DTMF 1〜4 並走）。「その他」のみ `内容確認.yaml` で自由テキスト聴取 → 完了フラグ。
  - **情報提供依頼** → `当日予約日確認` → `患者氏名` → `生年月日フリー` → `診療科` → `用件②（情報提供依頼）` で 3 分岐 hearing（治療の経過 / MRIの実施可否 / その他、DTMF 1〜3 並走）。「その他」のみ `内容確認.yaml` で自由テキスト → `返答期限（情報提供依頼）` で期限自由テキスト → 完了フラグ。
  - **入院・転院依頼** → `当日予約日確認` → `患者氏名` → `生年月日フリー` → `診療科` → `希望時期（入院・転院依頼）`（自由テキスト 5 秒打ち切り）→ `医師間共有（入院・転院依頼）`（Yes/No、DTMF 1/2 並走）→ `医師名（入院・転院依頼）`（自由テキスト）→ `その他問合せ` 自由テキスト → 完了フラグ → `入院・転院依頼`「入院・転院依頼を受付いたしました。診療科よりお返事いたします。」 termination。
  - **その他問合せ** → `次へ_3` → `用件②（問い合わせ）` → `その他問合せ分岐`（その他の問合せ → `その他問合せ_1` 自由テキスト / それ以外 → `患者氏名` フロー合流）。
- **当日予約日確認**: 「資料はいつまでに必要か」相当の Yes/No 確認 hearing（浦添と同等）。
- **入電日時判別** → `祝日チェック` サブシナリオ + 平日 8:30〜17:15 + 土曜は実質受付外（END_受付時間外） → date_of_call_classifier ブロックに集約。**addCurrentDate: Yes 自動付与対象**。
- **テスト系箱**（テスト番号判別 / テスト：DrJOY渡部 / テスト：DrJOY清水 / テスト：DrJOY / テスト施設名 / 稼働前）は **移管対象外**。

### 施設固有の特殊ルール
- **受付時間**: 平日（月〜金）8:30〜17:15、土日祝・年末年始休（時間外（稼働日：月ー金へ修正）箱の TTS で明記）。
- **代表案内**: 0263-37-3370 を 3 連呼（`代表案内` 箱「0263-37-3370、0263-37-3370、0263-37-3370」）。聞き取り不可・各種 NG（用件NG / 生年月日NG / 氏名NG / 企業NG / 問い合わせ①NG / 問い合わせ②NG / 当日NG）の合流先 → 失敗系 termination_pattern に集約。
- **転送**: `転送` サブシナリオ → `転送完了` → `END_切断`。`転送不可` 箱「現在、混み合っており転送ができません。…職員より折返しご連絡させていただきます。」あり。**ただし元 yaml の主要動線では転送は使われていない**（API_DRJOY_1 経由の `転送` のみ）→ Gen3 では call_transfer ブロック必要かどうか director 判断。
- **47 診療科 entity**: main.yaml に診療科リスト 47 件。診療科 hearing は情報提供依頼 / 入院転院依頼の中盤で使われる → STT 辞書 `hearing_shinryoka_basic` use_template + additional_words 47 件。
- **完了フラグ TTS**: `完了フラグ`「かしこまりました。」のみ → Gen3 では浦添方針に合わせ「かしこまりました。<受付日時>。」を採用するか元のまま「かしこまりました。」のみ採用するか **director 判断**。
- **3 回失敗時**: 各サブの `count再質問` 3 回到達 → `END_3回聞き取れず` → main の各 NG 箱 → `代表案内` → `終話`。
- **救済語/同音異義救済**: 用件層で「店員 / 入金 / 返品 / 天神依頼 → 入院転院依頼」「Suica / 受信 → これから受診の方の追加情報」「位置 / イチ → 1 番系」等多数。
- **DTMF 受付（用件聞き取り）**: 用件層 1=FAX到着確認 / 2=情報提供依頼 / 3=入院転院依頼 / 4=その他問合せ。冒頭 TTS で明示。
- **復唱（echo_back）対象**: 施設名 / 用件 4 種 / 用件②（FAX到着確認 4 種・情報提供依頼 3 種）のみ。

### director が見落としやすいポイント
- 冒頭の API_DRJOY_CONTEXT_SETTINGS → API_DRJOY → 挨拶 → 入電日時判別 → 施設名_1 → 入電情報から取得した情報をコンテキストセット → 施設名確認 の 7 箱は Gen3 では opening + 冒頭_アナウンス + date_of_call_classifier + 施設名既知/未知 CMR + 施設名サブフロー（既知時はスキップ）に吸収。
- セリフ none 振り分けのみの箱（API_DRJOY_1、用件のセット、セットした用件の分岐、用件セット分岐 / _1〜_5、次へ / _1〜_4、その他問合せ分岐、用件NG〜当日NG 各 NG、転送完了）は独立ブロック不要。
- **`用件セット分岐 / _1〜_5` の 6 段ネスト**: 用件 4 種を異なる前段から再分岐するために 6 個ある。**Gen3 では用件 context を一度確定したら以降は context_match_router で参照する**。
- **`次へ` `次へ_1〜_4` の 5 種**: 単に「続いて、」「それでは、」を発話して次箱へ繋ぐつなぎ TTS。
- 施設名_1 / 施設名_2 / 施設名確認 / 施設名確認_1 の 4 段ネスト: 入電番号から施設名取得 / 取得失敗 / 復唱の経路違い。
- **`当日予約日確認` の用途**: 元 yaml l.247-252 で情報提供依頼/入院転院依頼の前に呼ばれるが、サブシナリオ実体は別 yaml。中身は「お急ぎですか / いつまでに必要ですか」相当を 1 hearing に統合 director 判断。
- **用件②（問い合わせ）→ その他問合せ分岐 → その他問合せ / 患者氏名** の二分岐: `その他の問合せ` 一致なら自由テキスト聴取、それ以外（紹介状の送信依頼 / 問い合わせの進捗確認 等）なら患者氏名 hearing へ。
- **入院・転院依頼の医師間共有 hearing**: Yes/No + DTMF 1/2 のみ。Gen3 では generate_by_OpenAI（Yes/No 判定 SKILL_B）+ DTMF 並走、output_format=enum [Yes, No]。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**（丸数字 ①②③ 等は使用禁止）。元資料の `用件②（FAX到着確認）` `用件②（情報提供依頼）` `用件②（問い合わせ）` `希望時期（入院・転院依頼）` `医師間共有（入院・転院依頼）` `医師名（入院・転院依頼）` `返答期限（情報提供依頼）` 等の **②・全角括弧（）はモジュール名で使えない** → `yoken2_fax` `yoken2_johoteikyo` `yoken2_toiawase` `kibojiki_nyuteninrai` `ishikan_kyoyu_nyuteninrai` `ishimei_nyuteninrai` `hentokigen_johoteikyo` 等の ASCII 安全名に置換。
- **ContextMatchRouter.reference_module は subflow を参照する場合 `jump_{step名}`**（scaffold が付ける prefix）を使う。
- **CMR conditions は必ず最後に `match: "other"` + next 明示**。
- **TTS は `{tts_g:...}` 小文字のみ、SSML 禁止**。Gen1 変数記法 `<? $復唱 ?>` は Gen3 では `<% var %>` に変換。
- 患者氏名は PatientName サブフロー 1 枠に充当（patient 1 名のみ、医療機関スタッフからの架電だが受診者氏名 = 患者氏名）。
