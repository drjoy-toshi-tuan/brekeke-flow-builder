### 移管方針
- Gen1「箱」構造 → Gen3 scenario_flow 9 ブロック型へ変換。主な対応:
  - `API_DRJOY_CONTEXT_SETTINGS` / `API_DRJOY` / `挨拶` → opening + 冒頭_アナウンス（2 ブロック 1 セット）。冒頭文言「お電話ありがとうございます。愛仁会総合健康センターの、予約専用 AI 電話です。」
  - `入電日時判断` → acceptance_times（曜日・時刻判定）+ Script（祝日/年末年始判定）→ 時間内/時間外で振り分け、時間外は `時間外切断へ` → `携帯チェック_1`（携帯あり=WEB 予約案内、なし=`時間外案内` 終話）
  - `用件聴取.yaml` → hearing（**DTMF 2 値分岐**: 1=新規予約 / 2=予約以外（変更・キャンセル・問い合わせ））+ echo_back（用件の復唱：健康診断 / 変更）→ Pattern A（max_dtmf_length: 1）。**音声入力は受付けず DTMF のみ**で確定
  - 新規ルート（DTMF=1）: 用件の分岐 → 受診希望日聴取 → 完了フラグ → 終話挨拶（「3 営業日以内に担当者から折り返し電話、もしくはショートメールにてご連絡」）
  - 予約以外ルート（DTMF=2）: 用件の分岐_1 → 携帯チェック → **携帯あり: WEB 予約希望確認**（DTMF 1=WEB / 2=TEL）→ WEB 選択時は `WEB予約フォーム案内`（SMS 送付）、TEL 選択時は受診者名（担当者名）聴取へ → タイトル → 生年月日聞取り → それでは → 連絡先聴取 → 用件の分岐 → **連絡先後の二次分岐**: 「予約以外（その他）」は携帯チェック → WEB 予約案内、それ以外は受診者名聴取 → 現在の予約日聴取 → 完了フラグ → 終話挨拶
  - **携帯なし**: WEB 予約案内をスキップして直接受診者名（担当者名）聴取 → 通常フロー
  - 氏名 / 生年月日 / 連絡先 / 電話番号区切り → subflow ブロックで Gen3 標準サンプル JSON 完コピ。氏名は受診者名（担当者名）の 1 枠 → PatientName 1 枠に充当
  - 受診希望日聴取 / 現在の予約日聴取 → datetime hearing（OpenAI 正規化、addCurrentDate: Yes scaffold 自動付与）

### 施設固有の特殊ルール
- **用件入口は DTMF 2 値のみ**: 倉敷の 5 分岐（新規/変更/キャンセル/問い合わせ/折返し連絡）、蘇生会の 5 分岐（新規/変更/キャンセル/問い合わせ/企業）と異なり、**愛仁会は DTMF 1 / 2 の 2 値のみ**。音声入力は受付けない（`用件の確認` 箱で DTMF エンティティのみ判定）。Gen3 でも DTMF 専用 hearing を 1 本配置、OpenAI 正規化は不要（director 判断: scaffold の DTMF input ブロックで吸収）
- **新規ルートは受診希望日のみ聴取**: 健診種類（人間ドック / 協会けんぽ / その他）の聴取は無し → 担当者折り返しに任せる設計。Gen3 でも種類 hearing は不要
- **予約以外ルート (DTMF=2) は中で再分岐**: 連絡先聴取後の `用件の分岐` で「予約以外（その他: アクセス / 質問 / 書類 / 症状 等）」エンティティが含まれていたら携帯チェック → WEB 予約案内へ離脱、それ以外（変更/キャンセル/予約確認）は通常通り受診者名 + 現在の予約日聴取へ。**OpenAI 正規化が必要な唯一の用件分類**
- **WEB 予約希望確認は携帯ありのみ**: `携帯チェック` で携帯あり判定された場合のみ「WEB 予約フォームから申し込みも可能。WEB は 1、電話は 2」を聞く。携帯なしの場合は無条件で電話聴取に進む
- **WEB 予約フォーム案内**: DTMF=1 で WEB 選択時、「お電話終了後にショートメールで URL をお送りいたします。URL をタップし予約申し込みを完了させてください」→ 終話（SMS 送付フラグ立て）
- **DTMF 入力 LoopCount 5 回**: WEB 予約希望確認の DTMF 入力で 5 回失敗 → `入力確認できないので TEL へ` → 通常電話聴取に切り替え
- **代表電話番号**: 072-692-9291（健康管理センター直通）。代表電話案内 TTS では「祝日を除く、月-金 8:45-17:00、土 8:45-12:00 に直通へかけ直し」と案内 → 各 CP 系（希望日 / 氏名 / 生年月日 / キャンセル日 / 用件 / 連絡先）のリトライ上限失敗時に集約
- **時間外案内**: 「土日祝・年末年始除く、月-金 9:00-15:00、土 9:00-12:00」（こちらが本番受付時間と推定、director 判断で acceptance_times をこちらの値で設定）。代表電話案内の「8:45-17:00」とは差分あり、時間外案内の値を優先
- **転送無し / Blind Transfer 無し**: 担当者直結転送は無し → 全ルート最終段で番号案内型 termination（CP 系）or 完了フラグ → 終話挨拶（折り返し約束）
- **「予約以外」は予約に関する話ではなく、アクセス/書類/症状/会計等の問い合わせ全般**: エンティティ「予約以外」に「アクセス、お見舞い、くすり、会計、行き方、伺い、質問、処方箋、書類、症状、証明書、診断書、病棟、副作用、文書、聞きたい」等が定義。これらは WEB 予約案内（SMS）に誘導してフォーム転送する独自運用

### director が見落としやすいポイント
- 冒頭の `API_DRJOY_CONTEXT_SETTINGS` と `API_DRJOY` は Gen3 opening 標準処理で吸収 → 明示ブロック化不要
- 「タイトル」（セリフ「それでは次に、」）「それでは」（セリフ「それでは、」）等の繋ぎ TTS は次段ブロック先頭 TTS にマージ（director デフォルト）
- 「セリフ: none」かつ振り分けのみの箱（`用件の分岐` / `_1`、`完了フラグ`、`携帯チェック` / `_1`、`時間外切断へ`、`入電時間判断へ 開発中はシナリオへ繋ぐ`、`連絡先聞き取れずCP`、`希望日聞き取れず ＣＰ`、`氏名聞き取れず ＣＰ`、`生年月日聞き取れず ＣＰ`、`キャンセル日聞き取れず CP`、`用件聞き取れず CP`、`LoopCount`）は独立ブロック不要 → 前段 next 書き換えか CMR の一部として吸収
- `入電時間判断へ 開発中はシナリオへ繋ぐ` 箱は **開発時の暫定迂回箱の名残** → Gen3 では削除して直接 acceptance_times へ
- `タイトル_1`（セリフ「ご予約以外の方、お電話で予約の申し込みをご希望の方は、」→ `時間外案内`）は **時間外で携帯ありの WEB 案内 → DTMF 失敗時の最終フォールバック**で時間外案内に合流する迂回箱 → director 判断、acceptance_times の時間外用 termination に集約
- 「CP 〜」プレフィックス箱は Compliance Path 系のリトライ上限失敗 → Gen3 では各 hearing/subflow の `error_jump` 先 termination（代表電話 072-692-9291 案内）に集約
- 用件聴取サブフローの「end_用件取得できず」（DTMF 3 回失敗）は CP 用件聞き取れずの `代表電話へ` に合流
- 連絡先聴取後の `用件の分岐` は **OpenAI で「予約以外（その他）」を判定する唯一の OpenAI 分類点** → エンティティ「予約以外」の同義語列挙が肝（アクセス / 質問 / 書類 等 22 件）
- 「<? $連絡先電話番号（ハイフン区切あり） ?>」「<? $入電番号（ハイフン区切りあり） ?>」等の Gen1 動的タグは Gen3 では `<% var %>` 構文へ
- 受診者名サブフローは「受診者または担当者」と聴取するが Gen3 では PatientName 1 枠に充当（受診者と担当者の分離設計は無し）
- 生年月日サブフローは予約以外ルートでも聴取 → カームと同じ運用（director 注意）
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字等の環境依存文字は禁止
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`** を使う
- **CMR conditions は必ず最後に `match: "other"` + next 明示**
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
