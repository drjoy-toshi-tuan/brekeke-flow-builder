### 移管方針
- Gen1「箱」構造 → Gen3 9 ブロック型に変換。`挨拶` → opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「お電話ありがとうございます。湘南記念病院の、予約専用、AI 電話です。当院は、2026 年 1 月 1 日より順次、外来診療を縮小し、2026 年 3 月 31 日をもってすべての外来診療を終了させていただきます。」
- `時間判断` サブシナリオ（現在の曜日と時間取得 → 祝日チェック → 1/4 / 12/30 / 月～金 9-15 時 / 土曜 / 年末年始の階段分岐）→ date_of_call_classifier ブロック（addCurrentDate: Yes 自動付与対象）。
- 用件分岐は **Pattern A**（用件 OpenAI 直後に 4 分岐: 診察 / 変更 / キャンセル / 確認）。各分岐に echo_back（予約変更復唱 / 予約キャンセル復唱 / 確認復唱）を直配置。診察分岐のみ「予約判別」（診察のご予約ですね？）→ 新患判断（当院でのご受診は初めてですか？）→ **新患=初診なら（初診）代表へ ルートで終話**、否定なら再診として処理という追加分岐がある。
- 用件確定後、**用件種別 × 新患 / 再診 の組合せで遷移先が異なる**（main.yaml `用件を判断` `用件を判断_1` 箱）。Pattern B（後段 ContextMatchRouter）で実装妥当。
  - 確認 → 予約確定タイミングの説明 → 確認内容の聞き取り subflow → 診察券 → 氏名 → 生年月日 → 電話 → 終話
  - 変更 / キャンセル → 予約確定タイミングの説明_1 → 診療科 → 予約日 → 患者氏名 → 生年月日 → 電話 → 終話
  - 再診（新患=否定） → 診療科 → 予約希望日 → 二科目診療科聴取 → 診察券 → 氏名 → 生年月日 → 電話 → 終話
  - 新規（新患=肯定） → （初診）代表へ → 代表番号案内 → 切断（**AI で受け付けない**）
- サブフロー（患者氏名 / 生年月日 / 電話番号 / 診察券）は Jump to Flow 経由。`電話番号区切り.yaml` は `連絡先電話番号聞き取り.yaml` 配下から呼ばれる読み上げ整形用 → Gen3 サブフローテンプレ内蔵で完結、director は触らない。
- `予約日の聞き取り.yaml` は「日にち先行発話 / 月先行発話 / 数字のみ / DTMF 4 桁」+ 0 桁詰め + 日付比較（比較日より前 → +1 年で確定 / 以降 → そのまま確定）。Gen3 では date OpenAI（YYYY-MM-DD 出力）+ DTMF 並走で代替。
- `予約希望日.yaml` はフリーワード受付（`受診を希望されている日付や時期を、ご自由にお話ください。`）→ hearing ブロック（text 自由テキスト / SKILL_C）。echo_back なし、無言判定で完了。
- `確認内容の聞き取り.yaml` も同様にフリーワード hearing（「次回の予約日時を教えて欲しい」など）。
- `二科目診療科聴取.yaml` は再診ルート専用の追加診療科 / 伝達事項聞き取り（フリーワード）。「ない場合は『ありません』」の誘導文言あり。

### 施設固有の特殊ルール
- **2026-03-31 外来診療終了**: 挨拶 TTS と `（初診）代表へ` 箱に終了アナウンスが組み込まれている。**初診（新患）ルートは AI で受付不可**、代表電話へ強制誘導。`形成外科` `婦人科` `脳神経外科` `皮膚科` の 4 診療科は `END_外来終了`（既に 2025-12 で終了）→ 専用 TTS（「`<? $clinical_department ?>` は、2025 年 12 月をもって外来診療を終了いたしました」`代表へ_2`）。
- **受付時間**: 月～金 9:00～15:00、土日祝・年末年始は時間外。年末年始は 12/30〜1/4 が休業（1/5 月曜 9 時から再開）→ 専用 TTS（`年末年始` 箱）。
- **代表電話**: 0467-32-3456。`代表誘導：0467-32-3456` 箱で読み上げ、`回数count` で 2 回繰り返してから終話（合計 2 回）。Gen3 では call_transfer / termination 内 TTS で 2 回繰り返し含める。
- **Dr.JOY 番号判別**: `API_DRJOY` 後の `Dr.JOY 番号判別` 箱で `05017074509` だった場合のみ営業時間判定をスキップして `診察券用意案内`（営業時間内固定）に直結する分岐あり。Gen3 では incoming-classifier or context_match_router で `caller_to == "05017074509"` 判定（director 判断、scaffold が標準対応していなければ augment ブロック）。
- **当日予約は AI 不可**: `診察券用意案内` 直後に `当日予約排除（5/22 更新）` サブフロー（「明日以降の受診予約についてでよろしいですか？」）→ 否定なら `END_当日希望` → `当日予約→代表へ` 箱（「この電話では、本日の予約を承ることができません」）→ 代表電話案内で切断。
- **診療科 13 科は AI 受付不可（代表誘導）**: 放射線科 / 検査科 / リハビリテーション科 / 栄養科 / 内視鏡室 / 発熱外来 / 腫瘍内科 / 健康診断 / 予防接種 / 検査 / 栄養指導 / アートメイク / アロママッサージ / 乳腺外科 → `END_代表案内（13診療科）` → `代表へ` 箱（「`<? $clinical_department ?>` については、月曜日から金曜日の 9 時～16 時の間に、代表電話へ」）。
- **産婦人科 → 婦人科への自動付け替え**: `産婦人科/検査→乳腺排除` 箱で「産婦人科」エンティティ命中時 → 「当院には産婦人科はございません。婦人科でよろしいでしょうか？」と確認 → 肯定なら婦人科として扱う（ただし婦人科自体は `END_外来終了` 対象なので結局終了案内に流れる）。Gen3 では context_match_router 1 段で吸収可能。
- **3 回失敗時**: 用件 / 診療科 / 予約日 / 確認内容で `count再質問` `count上記以外` 系（5 以上で次層、2 以上で終話）→ `3回聞き取れず` → `END_聞き取れず` → 代表誘導で切断。Gen3 では termination_patterns に集約。
- **DTMF 受付**: 用件聞き取り（1=診察 / 2=変更 / 3=キャンセル / 4=確認）と予約日聴取（シャープ / アスタ + 月 2 桁 + 日 2 桁）。Gen3 でどう扱うかは director 判断（OpenAI プロンプトで吸収 / SKILL_C 自由テキスト + DTMF 並走）。
- **「もしもし」リカバリ**: 用件聞き取りの最初で `countもしもし`（1 回目「はい、聞こえております」、2 回目「『もしもし』と聞こえております」、3 回目で `END_聞き取れず`）。OpenAI プロンプトに「もしもし」誤検知排除を入れる必要あり。
- **復唱（echo_back）対象**: 用件 4 種（診察 / 変更 / キャンセル / 確認）+ 診療科 + 予約日。氏名・生年月日・電話・診察券はサブフロー内で完結、メインに echo_back を出さない。
- **連絡猶予 TTS（共通）**: `3 診療日以内の担当者からの折り返し電話、もしくは SMS にて確定となります。` を全 4 用件の終話前に配置。**携帯有無での出し分けはなし**（西宮渡辺と異なる）。

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `API_DRJOY` → `時間外ノードへ` → `Dr.JOY 番号判別` の 5 箱は Gen3 では opening 冒頭処理に吸収。`API_DRJOY_CONTEXT_SETTINGS` / `API_DRJOY` は明示ブロック化不要。ただし **Dr.JOY 番号判別による営業時間スキップ分岐**は明示ブロック必要（incoming-classifier or 専用 context_match_router）。
- 「セリフ: none」かつ振り分けのみの「箱」が大量（`次へ` / `次へ_1〜_5` / `用件の確認` / `用件を判断 / _1` / `完了フラグ` / `セットした用件の分岐` / `予約判別` / `新患判断` / `初診確認へ` / `分岐なし診療科セット` / `代表案内案件` / `切断待ち` / `時間外ノードへ` / `時間外_1` / `用件のセット` / `予約日セット（DTMF）` / `そのまま確定` / `プラス1年後で確定` / `0桁詰め` / `日の置換` / `月の置換` / `日付チェック_1〜_3` / `日チェック` / `月チェック` / `産婦人科/検査→乳腺排除` / `分岐なし診療科読み取得` / `診療科読み取得` / `診療科確認へ` / `用件再設定へ` / `否定` / `用件のセット` / `用件確認へ` / `予約変更復唱→用件を確認 入口` 系の中継箱）。Gen3 で独立ブロック不要、前段の next 書き換え or context_match_router の一部として吸収。
- `次へ_1〜_5`（セリフ「では次に、」「次に、」「続いて、」「それでは、」「最後に、」）等の **短い繋ぎ TTS は次段 hearing/subflow ブロックの導入文言にマージ**するか、announcement ブロックで個別配置するかは director 判断。
- **再診 / 変更 / キャンセル / 確認 各復唱（echo_back）の遷移先**は全部 `用件を確認` 箱に集約 → そこから Yes/No 判定で `END_戻る`（次工程）/ `否定`（再質問）/ `count再質問_3 → 否定`（再質問→3 回失敗で終話）。Gen3 では SKILL_B（Yes_No 判定）+ retry_count=2 統合で 1 ブロックに収まる。
- 用件聞き取り中の `肯定エンティティ` / `肯定（エンティティ）` / `否定エンティティ` / `否定（エンティティ）` / `肯定否定` 等の **微妙な命名違いが多数同居**しているのは Gen1 の慣習。Gen3 では SKILL_B（Yes_No 判定）で 1 本化。
- `予約日の聞き取り.yaml` の `生年月日の判定`（大正/昭和/平成/令和の元号判定）は **予約日では誤った遷移名が残っているだけ**で、実際は単に `日付あり` のチェック → `聞き取り完了` に流すだけ。元号判定ロジックそのものは生年月日聞き取り側に存在。director 混同しやすい。
- `予約希望日.yaml` と `確認内容の聞き取り.yaml` は**フリーワード hearing**（SKILL_C 相当）。echo_back なし、無言判定（5 秒で再質問、2 回失敗で完了 or 終話）。Gen3 では hearing(text) で簡素化。
- `二科目診療科聴取.yaml` は**再診（用件=診察＋新患=否定）ルート専用**。変更 / キャンセル / 確認ルートでは呼ばれない。director は「すべての診療科聞き取りに二科目を付ける」誤解をしないこと。
- **新患判断（用件＝診察＋新患=肯定）→ （初診）代表へ → 代表案内 → 切断** のルートは AI 受付対象外。**新規予約フローを Gen3 に組まない**（director は新規予約パスを誤って実装しないこと）。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字（①②③⑩等）等の環境依存文字は禁止。`END_代表案内（13診療科）` の括弧と数字混じりはそのまま使えるが、scaffold が衝突した場合は `END_代表案内_13診療科` 等に置換。
- **ContextMatchRouter.reference_module は subflow を参照する場合 `jump_{step名}`（scaffold が付ける prefix）を使う**。subflow ブロックの step 名そのものではダメ。
- **CMR conditions は必ず最後に `match: "other"` + next 明示**。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。`<? $clinical_department ?>` `<? $予約日の読み ?>` `<? $診療科の読み ?>` 等の Gen1 変数記法は Gen3 では `<% var %>` に変換。
- 患者氏名は PatientName サブフロー 1 枠に充当（patient 1 名のみ、入電者 = 受診者前提）。
- entity_classification.md / entity_stt_patch.json / entity_openai_synonyms.md にエンティティは自動分類済み。director は転記不要、scaffold/properties が自動取り込み。
