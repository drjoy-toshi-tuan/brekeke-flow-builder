### 移管方針
- 冒頭は opening + 冒頭_アナウンス の2ブロック1セット。冒頭アナウンスは**「本日より2診療日以内の内容はお受けできない」旨を含む**ので、原文どおりそのまま TTS に展開する（削らない）。入力待ち無しで個人情報同意確認へ接続。
- 個人情報同意確認は Yes/No 分岐（Pattern A: OpenAI直後分岐）。否定→代表番号「029-887-1161」案内で終話（checkpoint=7 / status=7 / endpoint=電話案内）、肯定→診察券番号サブフローへ。
- 氏名／DOB／電話番号／診察券番号は **サブフロー経由（Jump to Flow）**。診察券番号サブフロー、PatientName サブフロー、DOB サブフロー、電話番号サブフロー（memory: feedback_subflow_connection_rule / feedback_patientname_subflow_allocation）。
- 連絡先電話番号は yaml の「連絡先電話番号確認→YES: incoming 採用 / NO: 聴取→復唱→NG なら再聴取1回のみ」の流れ。**mobile（070/080/090/060）のみ連絡方法聴取ブロックを挟み、固定/非通知（050・10桁・anonymous・unknown）はスキップして用件確認へ**。Pattern B（後段 ContextMatchRouter）で分岐合流。
- 用件確認（予約 / 変更 / キャンセル / その他問い合わせ = 確認）で classification を設定し4分岐。「予約」選択時のみ次に受診歴確認（新規/再診）を実行、新規系は紹介状→選定療養費→診療科判定、再診は診療科直行。
- 予約受付不可分岐あり（紹介状「無」かつ診療科が循環器内科・脳神経内科・呼吸器内科・メンタルヘルス・代謝内分泌内科・感染症科・呼吸器外科の7科）→ 受付不可アナウンスで終話（checkpoint=8 / status=8）。
- 小児科のみワクチン確認→インフル確認の追加サブルート。それ以外の診療科は共通の予約希望日聴取に合流。
- 終話パターンは termination_patterns に ①診療予約完了（新規/再診/予約）②予約変更完了 ③予約キャンセル完了 ④確認完了 ⑤同意拒否（代表案内）⑥新規受付不可 ⑦汎用代表案内 の **7系統**。
- RAG は元資料に記載なし → 不要。FAQ 的内容は「その他問い合わせ」= 確認ルートで details に自由文格納。

### 施設固有の特殊ルール
- **肯定語の誤変換救済**：「肺」「いいです」「廃墟化します」→ 肯定扱い。Yes/No 系 prompt に必須注記（prompter 担当）。
- **数字 4/5 の誤認識救済**：「for」「C」→ 4、「号」「go」→ 5。診察券番号・電話番号ブロックの prompt 側で対処。
- **分類語の揺れ**：「印刷/定期/新薬/再診/最新/禁忌」→「診察」扱い、「薬/お薬/要約」→「予約」扱い、「電気/でんき」→「新規」扱い。用件確認ブロックの prompt に明記。
- **新規受付不可7科**：循環器内科／脳神経内科／呼吸器内科／メンタルヘルス／代謝内分泌内科／感染症科／呼吸器外科。**紹介状「有」なら受付、「無」なら受付不可**。再診ルートではこのガードは発動しない。
- **紹介状「有」のみ医師指定確認**（doctor='有'/'無' のみ、医師名は取得しない）。紹介状「無」では医師指定確認をスキップ。
- **選定療養費アナウンス（7,700円）は全文発話完了まで次へ進まない**ガードあり（script ブロックで入力待ち無し・完了後に診療科聴取）。
- **status/checkpoint/endpoint のマッピング**：checkpoint=7→status=7/endpoint=電話案内（代表案内・同意拒否）、checkpoint=8→status=8/endpoint=通話完了（受付不可）、予約日または希望日確認まで到達で status=1、代表案内のみ status=2、それ以外 status=0。
- **smsFlag**：携帯番号と判定された時点で smsFlag=1 を確定（「SMS」でも「電話」でも）。終話アナウンス（①②③④）出力時点で smsFlag=1。それ以外（途中切断・代表案内・受付不可）は smsFlag=0。
- **診療科リスト25科**：function enum に準拠。キーワード揺れ（「め/めか」→眼科、「はいしゃ」→歯科口腔外科 など）は prompt 側で吸収。
- **「もしもし」対策**：「もしもし」のみの発話は無視し、「お声は届いております。」で同じ質問を再提示する（Retry系のハンドリング）。

### director が見落としやすいポイント
- 『**予約日と予約希望日は別項目**。予約日のヒアリングがなかった場合に予約希望日の中身を予約日に入れてはいけない』。変更ルートは reservationDate（日付）と DesiredreservationDate（フリーワード）の**両方を別項目として聴取**。キャンセルは予約日のみ。
- 『10月以降で1月から9月の日付を言われたときは、翌年にする』の年度判定ルール。日付正規化 prompt に必須。
- 『一度に複数情報を発話した場合、すでに伝えられた内容は絶対に復唱・確認せず、不足項目のみ質問』が共通ルール。generate_by_OpenAI 全プロンプトに反映。
- **checkpoint / status / smsFlag / endpoint は内部フラグ**。AI は絶対に発話しない（TTS 文言混入防止）。
- 生年月日は **2回認識失敗で空欄セット・次へ進む**（エラーアナウンスなし）。DOB サブフロー側で Max Retry=1、超過時は次ブロックへフォールスルー。
- 携帯番号で入電している場合、**contact と smsFlag が未確定のまま用件確認に入らないこと**（進行ガード最重要）。連絡先種別判定ブロックで分岐させてから用件確認へ。
- 「診療科がわからない」と言われたら**空欄のまま進める**、症状・理由の追加質問禁止（診療科特定のためのレコメンドは「〇〇でしょうか？」形式で行うが、不明時は追加質問しない）。director 判断で Retry 済みの場合は clinicalDepartment='登録なし' または空欄にフォールバック。
- 患者氏名は**苗字のみ判定時のみ**下の名前再質問を許可（PatientName サブフロー側で処理）。カタカナ化チェックも同サブフローで実施。
- checkout（途中切断ステータス）は質問中断箇所を記録する enum 無しの自由文。director は step_details 側で checkout 値の割り当てルールを指示する必要あり。
