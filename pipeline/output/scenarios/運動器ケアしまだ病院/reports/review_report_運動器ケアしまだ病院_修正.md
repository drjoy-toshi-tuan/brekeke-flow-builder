## 修正内容

### ① SMS未送信の調査と修正

**調査結果:**
`失敗flag`（`saveCompletionFlag2db`）の `smsFlag` パラメータが空文字になっていた。
他のフラグモジュール（`変更終話flag`, `キャンセル終話flag` 等）は `smsFlag: "1"` または `"0"` が設定されていた。失敗時はSMSを送らない仕様として `"0"` に統一。

| 対象モジュール | 修正前 | 修正後 | 対象フロー |
|---|---|---|---|
| `失敗flag` | `smsFlag: ""` | `smsFlag: "0"` | 通常・日曜（両方） |

**その他のSMSフラグ状況（変更なし・確認済み）:**
- `変更終話flag`, `キャンセル終話flag`, `確認終話flag`, `予約flag`: `smsFlag: "1"` (SMS送信あり) ✓
- `転送成功flag`, `転送成功flag2`: `smsFlag: "0"` (転送ステータス) ✓
- `連携室時間外flag`, `予約_個人の問い合わせ`: `smsFlag: "0"` ✓

### ② 転送が「途中切断」になる事象の修正

**調査結果:**
転送モジュールへの遷移チェーンを確認。

- `リハビリ転送`: TTS(`リハ専用転送`) → wait(`転送_wait`) → `リハビリ転送` → 転送成功時 `転送成功flag` → `切断_リハ転送`
- `転送_代表`: TTS(`転送_代表アナウンス`) → wait(`転送wait3`) → `転送_代表` → 転送成功時 `転送成功flag2` → `切断_転送成功`
- `転送_地域連携室`: TTS(`転送_地域連携連携室アナウンス`) → wait(`転送_wait2`) → `転送_地域連携室` → 転送成功時 `転送成功flag2` → `切断_転送成功`

`saveCompletionFlag2db` は転送成功コールバック後に呼ばれる構造になっており、正常。
**JSON修正なし。** ただし、転送失敗時（`Failed`）に転送モジュール自身にループしている点（無限リトライ）は既存設計のため変更なし。

### ③ 電話番号再聴取時の切断修正

**問題:** `リトライ_患者_連絡先電話番号` の `Retry` 分岐の `nextModuleName` が空文字になっており、リトライ時に遷移先が存在しなかった。

| 対象モジュール | condition | label | 修正前 nextModuleName | 修正後 nextModuleName | 対象フロー |
|---|---|---|---|---|---|
| `リトライ_患者_連絡先電話番号` | `true` | `Retry` | `""` | `"患者_連絡先電話番号"` | 通常のみ |

**確認:** 日曜フローの `リトライ_患者_連絡先電話番号` は `nextModuleName: "患者_連絡先電話番号"` が正しく設定されていた（変更不要）。

**補足:** `入力_患者_連絡先電話番号` は `drjoy^AmiVoice$Speech to Text` 型（DTMFなし）。`患者_連絡先電話番号` TTS の `stop_by_dtmf` は `"No"` であることを確認。電話番号入力は後続の `入力_患者_連絡先電話番号_非通知`（`DTMF AmiVoice STT Input`）で DTMF対応している。

### ④ 診察券復唱の修正

**調査結果:**
診察券番号の聴取〜復唱フローを追跡：

```
患者_診察券番号_接続詞(TTS) → 患者_診察券番号(TTS) → 入力_患者_診察券番号(DTMF STT)
  → 診察券番号_わからないの分岐(OpenAI)
    - 数字 → 患者_診察券番号_復唱(Re-confirmation, nodeName="入力_患者_診察券番号")
              → 入力_患者_診察券番号_復唱(AmiVoice STT)
              → 入力_患者_診察券番号_AI(OpenAI)
              → 患者_氏名_接続詞(TTS)
    - わからない → 患者_氏名_接続詞(TTS)
```

`患者_診察券番号_復唱` の `nodeName` は `"入力_患者_診察券番号"` に正しく設定されており、DTMF入力値を復唱する構造になっている。
**JSON修正なし。**
