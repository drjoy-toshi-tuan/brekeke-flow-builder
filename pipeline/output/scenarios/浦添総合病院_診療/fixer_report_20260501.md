#### C-001: OpenAI_用件確認 - 敬称「さん」が判定条件を汚染

**対象モジュール**: OpenAI_用件確認（STEP2：数字モード判定）  
**問題**: 

```markdown
## 【STEP2：数字モード判定】

### 予約確認
- 3
- 3番
- 3ばん
- さん        ← 敬称「さん」が「3番」（予約確認）の一部として定義
- さんばん
```

**ユースケース（失敗例）**:
- ユーザー発話: 「河村さんに予約変更をお願いしたいです」
- STT認識後: 「河村さんに予約変更をお願いしたいです」
- 正規化後: 「河村さんに予約変更をお願いしたいです」
- **STEP2判定**: 「さん」を含む → **「予約確認」として誤分類**
- **期待値**: 「予約変更」
- **実結果**: 「予約確認」 ✗

**原因**: 「さん」は医療シーンで敬称として非常に頻出するが、同時に「3」の数字読みとしての使用もある。部分マッチ方針により、助詞と混同して敬称が判定条件として機能してしまう。

**修正方針**: 
- 「さん」「さんばん」を STEP2 から削除し、STEP3 の語句一致判定に移動することを検討する、OR
- STEP2 で「さん」にマッチする際に、前後の文脈（敬称パターン：「〇〇さん」「名前さん」）を排除する正規化ロジックを追加する。
