## 検出事項

### W-001: STEP4 の和暦年号範囲チェックが不完全

- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_患者生年月日
- **重大度**: Warning
- **問題**: 
  - L95-97 で和暦の年号範囲が「昭和1〜64年」「平成1〜31年」「令和1〜」と括弧内に記載されているが、範囲外の年号（例：「昭和65年4月1日」）に対する明示的な処理がない
  - STEP4 の判定ロジックは「{元号}{年数}年{月}月{日}日の構造に完全一致」と「西暦変換後がシステム実行年以下」のみで、年号の年数の有効性をチェックしない
  - 例：「昭和65年4月1日」は構造的に完全一致し、西暦 1925+65=1990 年に変換される。昭和は1989年（昭和64年）で終わるが、プロンプトではこれを検出できない。結果として「1990-04-01」が出力される（実現象としての誤り）

- **修正方針**: 
  - STEP4 L100-105 の「抽出ルール」に以下を追加：
  ```
  年号の年数が有効範囲内であることを確認：
  - 昭和: 1 ≤ {年数} ≤ 64
  - 平成: 1 ≤ {年数} ≤ 31
  - 令和: {年数} ≥ 1
  上記範囲外の場合は NO_RESULT
  ```

### W-002: STEP1 正規化と STEP3/4 判定の対象入力が曖昧

- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_患者生年月日
- **重大度**: Warning
- **問題**: 
  - STEP2-6 では「正規化後の入力が」という表現があり、一見すると STEP1 の正規化後の入力を対象にしていることが明らかだが、STEP3・STEP4 の「完全一致」判定時の対象入力が暗黙的である
  - 例：入力「1990 年 4 月 1 日」（スペースを含む）は STEP1 で「1990年4月1日」に正規化される（L60）。その後 STEP3 で「YYYY年M月D日」との完全一致を判定する際、正規化後の「1990年4月1日」を対象にするのか、元の「1990 年 4 月 1 日」を対象にするのかが明確でない
  - 実装者がこの曖昧性により、異なる入力に対して異なる判定を行う可能性がある

- **修正方針**: 
  - STEP2 の冒頭に以下を追加：
  ```
  ## 【STEP2-6 の評価対象】
  以下のすべての判定（STEP2 ～ STEP6）は、STEP1 で正規化された入力に対して行う。
  元の入力ではなく、正規化後の入力と比較すること。
  ```

### W-003: 記号削除対象リストにハイフン「-」が含まれていない理由が不明記

- **修正担当**: prompter  <prompter>
- **対象モジュール**: OpenAI_患者生年月日
- **重大度**: Warning
- **問題**: 
  - L59 の記号削除対象が「、。,.・:;！!？?「」『』""''／/」と列挙されているが、ハイフン「-」が含まれていない
  - ユーザーが「1990-04-01」のような形式で入力した場合、STEP1 でハイフンは削除されず「1990-04-01」のままになる
  - その後 STEP2 で「半角数字のみで構成され、かつ桁数が 8」という条件に不合格になり NO_RESULT となるが、この設計意図が不明確
  - 実装者が「ハイフンを削除対象に追加すべき」と誤解する可能性がある

- **修正方針**: 
  - L59 の記号削除対象の説明に以下を補足：
  ```
  記号を削除（、。,.・:;！!？?「」『』""''／/）
  
  注: ハイフン「-」は削除対象に含めない。理由：
  - DTMF 入力では「-」は出現しない（DTMF は 0-9、*、# のみ）
  - 音声入力では「1990年4月1日」のように発話されるため、「1990-04-01」形式は想定外
  万が一ユーザーが「1990-04-01」と言及した場合（稀）、STEP2 の「半角数字のみ」条件で自動的に NO_RESULT になり安全
  ```

## まとめ

### 強み
1. **攻撃耐性**: プロンプトインジェクション・役割反転への防御が明示的かつ厳格
2. **出力制約**: 「1 語のみ」「YYYY-MM-DD のみ」という制約が強く明記されており、予測不可能な出力リスクが低い
3. **アルゴリズムの詳細性**: STEP1-6 で機械的ルールが十分に詳細に記載されており、実装者が判定ロジックを誤解する余地が少ない

### 改善推奨
1. **W-001**: 和暦年号の範囲チェック（昭和1〜64年、平成1〜31年）を STEP4 のロジックに明示的に統合する必要がある。実在しない年号に対する処理を明確化
2. **W-002**: STEP1 正規化後の入力を対象にした判定であることを、全 STEP の冒頭で明記し、実装者の混乱を排除
3. **W-003**: 記号削除対象にハイフンが含まれない理由を補足説明。DTMF 無関連性と想定外性を明記
