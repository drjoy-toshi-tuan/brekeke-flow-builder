### 判定した作業種別

- **作業種別**: Gen1→Gen3移管
- **施設名**: 徳島大学病院
- **シナリオ名**: 診療
- **対象フロー**: `徳島大学病院$診療`（メイン）+ サブフロー5本（診察券番号聴取/氏名聴取/生年月日聴取/電話番号聴取/RAG検索）
- **判定根拠**: 入力資料が `gen1_徳島大学病院_診療.md`（Gen1移管ノート）+ commubo Gen1 yaml 14本。pipeline_state.json の `pattern: 4` も Gen1→Gen3 移管パターンを示唆

### 抽出した要件サマリー

- **冒頭処理**: wait → コンテキスト設定 → 着信分類（非通知判定）→ 営業時間チェック（平日7:00-20:00, 年末年始休）→ 冒頭_アナウンス
- **体調確認**: 冒頭直後にY/N質問（"体調が悪くて本日診察を受けたい、もしくは予約を変更したいですか？"）。Y→END_体調相談（外来直接案内）/ N→用件確認
- **用件確認**: 3択（変更/キャンセル/確認）+ DTMF 1/2/3 対応 + echo_back復唱。新規予約はAI受付対象外
- **変更ルート**: WEB予約案内 → Y/N分岐
  - Y → 電話番号聴取SF（携帯確定）→ END_変更_WEB（SMS送信文言含む）
  - N → AI共通レーン合流
- **キャンセルルート**: 診療科聞き取り → 予約日 → 理由（キャンセル）→ 個人情報 → 折り返し案内 → END_キャンセル_AI
- **確認ルート**: 確認内容聴取（text）→ 個人情報 → 折り返し案内 → END_確認_AI（診療科・予約日聴取スキップはGen1原本準拠）
- **診療科聞き取り**: 7群階層（呼吸器/消化器/脳神経/口腔/歯科/小児/放射線）+ 単一leaf 35件。親hearing→ContextMatchRouter→子hearing（6群）or END_代表案内（放射線）or 予約日（leaf直接）
- **代表案内**: 放射線科系（3種）はAI受付不可。代表電話 088-633-9071 を3回繰り返してEND
- **個人情報**: 4サブフロー（診察券→氏名→生年月日→電話番号）を順次Jump
- **RAG検索**: Gen1→Gen3移管デフォルト（パターン2）で全正常終話前に配置
- **終話**: 6パターン（END_変更_WEB / 変更_AI / キャンセル_AI / 確認_AI / 体調相談 / 代表案内）+ 異常系3パターン（非通知/時間外/聴取失敗）
