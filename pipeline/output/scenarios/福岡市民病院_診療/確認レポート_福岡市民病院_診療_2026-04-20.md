### 抽出した要件サマリー

**フロー目的**: 外来予約に関する問合せ（変更/キャンセル/予約確認）の受付を自動化。新規予約（初診）は紹介状＋直接来院運用のため代表電話案内で終話。

**用件分類（classification）**:
- `変更` → 予約変更ルート
- `キャンセル` → 予約キャンセルルート
- `予約確認` → 予約確認ルート
- `新規` → 代表電話案内後に終話（Gen2 本体では classification に保存しないが、Gen3 ではルーティング兼 Dr.JOY 側識別のため enum 値として追加）

**Gen2 特有の順序**: 一般的な Gen3 フロー（用件→診療科→個人情報）と異なり、**共通の個人情報を先に聴取**してから用件を確認する。順序を元資料どおり保持:

```
冒頭 → 氏名 → 生年月日 → 診察券番号 → 診療科 → 診療科2 → 用件確認 → 用件別聴取 → 電話番号 → 用件別終話
```

**診療科（15科 + 登録なし）**:
内科/外科/整形外科/脳神経外科/心臓血管外科/呼吸器外科/消化器外科/乳腺外科/小児外科/泌尿器科/婦人科/眼科/耳鼻咽喉科/皮膚科/麻酔科（ペインクリニック）

**診療科2**: 複数診療科通院時の副診療科。「ない/ないです」→「なし」で格納、それ以外は診療科と同じ enum リスト。

## 7. 特記事項

- **Gen2 特有の聴取順序（個人情報先）**: 一般的な Gen3 パターン（用件→診療科→個人情報）と異なり、共通の個人情報（氏名→生年月日→診察券番号→診療科→診療科2）を用件確認より先に聴取する。Gen2 原文の挙動互換を優先して順序を維持した。通話体験を Gen3 標準に寄せたい場合は scenario_flow の並び替えで対応可能
- **生年月日の復唱禁止**: Gen2 プロンプトの明示ルールに従い `samples/json/生年月日聴取_復唱なし.json` を採用。Gen3 でデフォルト採用する DOB Re-confirmation モジュールは使わない。copy_subflows 時に通常版ではなく復唱なし版を指定すること
- **「新規」ルートの扱い**: Gen2 function では新規予約時 classification を保存しない設計だったが、Gen3 では Dr.JOY 側の可視化のため enum 値「新規」（order=4）を追加して代表案内終話前に save。Dr.JOY 側のレコード処理で区別可能にしている。classification を保存しない純粋 Gen2 互換が必要なら、`END_代表案内_予約新規` の前に classification 保存ステップを省略する構造に変更可能
- **smsFlag の用件別 3 本立て**: Gen2 function では smsFlag を 1/2/3/4 で切替えていた（1=汎用完了、2=変更、3=キャンセル、4=予約確認）。Gen3 では用件別に termination_pattern を分離し、各 pattern で smsFlag を静的設定した。動的 smsFlag 判定（ContextMatchRouter 経由）より追跡容易で、Dr.JOY 側 SMS テンプレも smsFlag 別に用意できる
- **診察券番号の「わからない」/「不明」フォールバック**: Gen2 ルールで聴取不能時に特殊文字列を保存する仕様。Gen3 の診察券番号聴取サブフロー OpenAI プロンプトに追記が必要（標準テンプレになければ prompter で追加）
- **Customer Docs 読み込み方法**: 元資料は約 1500 行で 1 回読込では 25k トークン制限超過。3 分割（0-500 / 500-1000 / 1000-末尾）で全量読込した上で設計書を起草。移管ノートだけで起草せず、必ず元資料から診療科リスト・TTS 文言・function enum を取得
