### 抽出した要件サマリー

- **共通ヒアリング**: 受診者名 → 生年月日 → 連絡先電話番号 → ID番号（診察券） → 用件 の固定順。全て個別サブフロー経由（Jump to Flow）
- **用件4分岐（DTMF）**: 1=予約 / 2=変更 / 3=キャンセル / 4=問い合わせ。分類不可時は問い合わせにフォールバック。リトライ2回（例外的に多め）
- **予約ルートは健診の種類3分岐（DTMF）**: 1=川崎市 / 2=健保組合 / 3=その他
  - 川崎市 → 受診内容1 → 人数受診時期 → 最後の問い合わせ
  - 健保組合 → 健保名 → 受診希望コース → オプション → 人数受診時期 → 最後の問い合わせ
  - その他 → 希望検査 → 最後の問い合わせ
- **変更ルート**: 予約日 → 変更項目 → 最後の問い合わせ
- **キャンセルルート**: 予約日 → 受診内容2 → 最後の問い合わせ
- **問い合わせルート**: 確認内容 → 最後の問い合わせ
- **終話分岐**: 最後の問い合わせ後、classification → (予約の場合) menu → phonetype の順に context_match_router で3段ネスト分岐
- **終話数**: 正常系 8 + キャンセル 1 + 代表案内 1 + 聴取失敗 1 + 非通知 1 + 時間外 1 = 13 種類
- **smsFlag**: 通常=1 / キャンセル=2 / 異常=-1 の3値（Gen2 仕様を Gen3 に保持）
- **status**: 正常=1 / 代表案内=2 / 聴取失敗=3 / 時間外=6 / 非通知=2（Gen2 の status=0 は Gen3 禁止のため 3 or 2 に再マップ）
- **SSML 例外**: 電話番号復唱の `<speak type="telephone">`、生年月日・用件・健診種類の `<dtmf/>` タグは原文保持（director 判断で Gen3 SSML 禁止ルールの例外扱い）
- **RAG**: 元資料に該当機能なし → `rag_subflow.pattern: none`
