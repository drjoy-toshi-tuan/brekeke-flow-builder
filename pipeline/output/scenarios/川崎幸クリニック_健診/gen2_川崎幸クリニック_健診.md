### 移管方針
- 冒頭は opening + 冒頭_アナウンス の2ブロック1セット。冒頭文言は「お電話ありがとうございます。川崎幸クリニックの、健診予約専用ダイヤルです。」を固定で使用（要約禁止）。直後は共通ヒアリングへ無入力遷移。
- 共通ヒアリングは **受診者名 → 生年月日 → 連絡先電話番号 → ID番号 → 用件①** の固定順。**受診者名/生年月日/電話番号/診察券（ID番号）は全てサブフロー経由（Jump to Flow）**。DTMF 聴取・桁数チェック・復唱確認はサブフロー側に閉じ込め、メインにはインライン展開しない。
- 用件① は **4択 DTMF（1=予約 / 2=変更 / 3=キャンセル / 4=その他お問い合わせ）**で classification を格納し Pattern A（OpenAI直後分岐）で4分岐。分類不可時は「その他お問い合わせ」に流す。
- 予約ルートは更にネスト分岐：**健診の種類 3択 DTMF（1=川崎市 / 2=健保組合 / 3=その他）** → 各ルートで必要項目を個別聴取。この第2段分岐も Pattern A で問題ない（director 判断、ルート数が多ければ Pattern B 合流も検討可）。
- 終話系統は **6種類**（予約-川崎市／予約-健保組合／予約-その他／変更／キャンセル／その他お問い合わせ）× 固定電話/携帯電話で文言差し替えあり。電話番号種別判定は incoming_phone_number 型の分岐ではなく、**聴取済み additionalPhoneNumber の先頭3桁判定**（090/080/070/060/050→携帯、それ以外→固定）で script ブロック or termination_patterns に2パターン定義する（director 判断）。
- TTS は全て `{tts_g:...}`（小文字）で発話。`<speak type="telephone">` `<dtmf />` `<dtmf digit="1" />` `<dtmf digit="8" />` タグは**電話番号復唱・生年月日再質問・用件ブロック・健診の種類ブロック**等のプロパティに原文どおり保持（SSML 禁止ルールの例外、director 判断）。
- RAG は元資料に記載なし → 不要。フロー外質問は reason3（※本文記述）あるいは question に自由文格納。
- 健診ドメイン特有：**担当者名・会社名は PatientName サブフロー対象外**。patientName は**受診者本人**のみ。健保・仲介会社名（companyname）は健保組合ルート内の hearing ブロックで受ける。

### 施設固有の特殊ルール
- **status/smsFlag は「固定発話が再生されたタイミング」でのみ更新**。終話処理が走っても固定発話未再生なら status=0 を保持。AI は内部フラグを絶対に発話しない。
- **smsFlag 値は 3 値**：正常終話（健保組合/川崎市/予約その他/変更/その他お問い合わせ）= 1、**キャンセル終話のみ = 2**、それ以外 = 0。通常の 0/1 バイナリではない点に注意。
- **status=2 は代表案内（044-541-5210 への誘導）発話時のみ**。電話番号再質問3回超過での代表案内終話も該当。endpoint="電話案内"（※元資料は"代表案内"表記もあるが enum は「電話案内」）。
- **電話番号桁数チェックは最大3回リトライ**（他項目は1回まで）。3回目超過で代表案内（044-541-5210）へ誘導し終話。
- **氏名は苗字のみ判定時の再案内1回のみ許可**。再聴取後も不一致なら「聞き取れた表記で保存して次へ」。氏名サブフロー側で対応。
- **数字 4/5 の誤認識救済**："for"/"C"→4、"号"/"go"→5。電話番号・診察券・生年月日ブロックのプロンプト（prompter 担当）に注記。
- **予約日と予約希望日は別項目**。reservationDate は yyyy-MM-dd 00:00:00、Preferred_date はフリーワード（日付明確時のみ yyyy-MM-dd）。変更・キャンセル系で両方聴取する場面では混用禁止。
- **10月〜12月着信時に1〜9月の日付発話は翌年扱い**。予約日正規化プロンプトに必須記載。
- **Yes/No の肯定/否定キーワード（復唱確認時のみ適用）**：肯定=「そうです/はい/あります/する/そうそう/あっています/合っています/いい」、否定=「違います/ちゃうちゃう/いえいえ/いや/いいえ/じゃない」。
- **氏名複数名発話時**：1人目のみ patientName にフルカタカナ、2人目以降は reason に所定形式（カタカナ氏名＋yyyy-MM-dd、1人1行）格納。

### director が見落としやすいポイント
- 『**終話文が additionalPhoneNumber の固定/携帯で2パターン**』（川崎市ルートは特に長文で、特定健診受診券の自己負担金額確認文言が**固定では「無料/無料と＊のどちらか」**、**携帯では「無料の横にアスタリスク印字の有無」**と内容が異なる）。原文を要約せず保持。
- 『**健診の種類 3択 DTMF は "1=川崎市 / 2=健保組合 / 3=その他"** で、その他コース内部の判定（人間ドック/Aコース/ABコース/協会けんぽ等）は希望検査 hearing の自由文に格納』。enum 化しない。
- 『**変更ルートは "予約日 → 変更項目（メニュー or 日付） → 最後の問い合わせ → 終話"** のシンプル構成』。「日程変更」と「変更内容」の区別は reason に自然文で格納（構造分岐しない）。
- 『**キャンセルルートの終話文は smsFlag=2、文言は「必要がある場合のみ折り返し」一択**』（固定/携帯の分岐なし）。他ルートと異なる構造。
- 『**ID番号（診察券）は「分からない」許容**』。「診察券の登録番号をお話しください。番号が分からない場合は、「分からない」とお話しください。」→ 未入力で次へ進める。診察券サブフロー側にスキップ分岐必須。
- 『**最後の問い合わせ（FAQ 受け）は全ルート必須**』。「そのほかに何かお問い合わせは有りませんか？」→ 無ければ終話、あれば question/reason に格納して終話。
- 『**不明瞭時の終話は必ず status=0**』。無音・聞き取り不能・切断による強制終了では終話アナウンスが再生されても status は 0 のまま。
- 『**再質問1回ルールの例外は電話番号のみ（最大3回）**』。氏名・コース・オプション等は**たとえ不明確でも1回以上再質問せず次へ**。
- 改訂履歴（20260323）：sms制御が 2値→3値（キャンセル=2 新設）に変更された形跡あり。smsFlag の 0/1/2 enum 定義を漏らさない。
