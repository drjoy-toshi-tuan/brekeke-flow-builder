## 設計上の注意点（migrator/generator 向け）

### NOTE-1: 児童思春期外来×初診の判定タイミング

診療科=児童思春期外来かつ classification=初診 の組み合わせを OpenAI モジュール（`OpenAI_児童思春期_初診判定`）で判定する。この判定は `saveContext2DB` で保存済みの classification を参照する形で実装すること。

classification の値は用件確認→受診歴確認のステップで確定している前提のため、診療科聴取後に判定を行う。

---

### NOTE-2: 当日予約は用件聴取段階で分岐

「今日」「本日」等のキーワードは冒頭アナウンス時点では分岐せず、用件聴取の OpenAI 分岐（`OpenAI_用件_区分`）で検出して `END_代表案内` へ遷移する設計とする。

冒頭アナウンスにも代表電話番号の案内を含めているため、入電者は事前に代表番号を認識できる。

---

### NOTE-3: 入電者氏名サブフローの扱い

入電者氏名（代理の場合）は、標準の個人情報サブフローとは別に扱う。保存先は `callerName` であり、受診者氏名（`patientName`）とは異なる contextName を使用する。

個人情報サブフロー.bivr の「氏名サブフロー」を流用する場合、保存先 contextName の置換に注意すること。

---

### NOTE-4: 診察券番号サブフローの用件別適用

診察券番号は以下の用件で聴取する：
- 再診予約: 聴取あり
- 変更: 聴取あり
- キャンセル: 聴取あり
- 確認: 聴取あり
- **初診予約: 聴取なし**（初診なので診察券が存在しない）

初診パスでは `診察券番号サブフロー` への Jump をスキップする設計とする。

---

### NOTE-5: 予約希望日・確認内容はフリーワードで保存

- `desiredAppointmentDate`（予約希望日）: 日付形式に変換せず、発話内容をそのままテキストで保存
- `confirmationContent`（確認内容）: 入電者の発話をそのままテキストで保存
- `appointmentDate`（既存予約日）: YYYY-MM-DD 形式に変換して保存（変更・キャンセル時）

OpenAI の正規化ルールでこれらの違いを明示的に指示すること。

## 第2世代からの主な変更点（移管ポイント）

| 第2世代 | 第3世代 |
|---|---|
| 全フロー1つのOpenAI会話フロー | 複数モジュール＋サブフロー分割型 |
| checkpointはOpenAIが直接制御 | `saveCompletionFlag2db` で設定 |
| smsFlagはOpenAIが直接制御 | `saveCompletionFlag2db` の params で設定 |
| 当日判定は任意のタイミング | 用件聴取の OpenAI 分岐で一元判定 |
| 復唱禁止はプロンプトで制御 | Re-confirmation モジュールで制御（診察券・生年月日・電話番号のみ） |
| リトライは会話内で自然に行う | `Speech Retry Counter` で最大2回まで制御 |
| 非通知・時間外の制御はなし | `incoming-classifier` + `acceptance_times` で分岐 |
| 個人情報はすべて1フローで処理 | 氏名・生年月日・電話番号・診察券番号を個別サブフローに分割 |
