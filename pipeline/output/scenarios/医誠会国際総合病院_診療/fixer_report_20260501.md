### C-001: 複合意図の処理が明示されていない  <prompter>

**重大度**: Critical  
**対象モジュール**: OpenAI_診療科  
**問題**: 

複数の診療科キーワードが同時にマッチした場合の処理が、プロンプトに明示されていない。

現在のプロンプト（STEP3, line 122）では：
```
※**複数のラベルがマッチした場合は上から優先**（特定名称が一般名称より先に評価されるよう順序を厳守）
```

ただし、これは「複合意図を NO_RESULT とすべき」なのか「最初のマッチを返す」なのかが設計として曖昧。

**例**:
- ユーザー入力: 「内科と心臓血管の両方」
- 現在の挙動: 「内科」のみを返す（評価順序により）
- 期待値: NO_RESULT（複合意図は分類不可）か、それとも最初のマッチか？

設計書では NO_RESULT は明示的な戻り値（saveContext2DB を経由してリトライ）であるため、複合意図を NO_RESULT として扱うべき場合、プロンプトに「複合キーワード検出時 → NO_RESULT」を明示すること。

**修正方針**:

STEP3 の判定ロジックに、以下の条件を追加：
```
◆ 複合意図検出時の処理
複数のキーワードが異なるカテゴリで同時にマッチした場合 → NO_RESULT
（例：「内科」AND「心臓血管」が同時に含まれる → NO_RESULT）

ただし、以下は複合意図ではなく「キーワード包含」として処理：
- 「心臓血管」入力で「心臓」キーワードがマッチ → 「心臓血管」を返す（上から優先）
- 「脊椎脊髄外科」入力で「脊椎」「脊髄」がマッチ → 「脊椎脊髄外科」を返す
```

### W-001: キーワード「心臓」が複数カテゴリに含まれる場合の判定が設計通りか不確実  <prompter>

**重大度**: Warning  
**対象モジュール**: OpenAI_診療科  
**問題**:

キーワード「心臓」が以下 2 つのカテゴリに含まれており、STEP3 の評価順序に依存する：

| カテゴリ | 順序 | キーワード |
|---|---|---|
| 心臓血管 | 先 | 心臓血管、心臓血管外科、心血管、**心臓** |
| 循環器 | 後 | 循環器内科、循環器、**心臓**、じゅんかんき |

ユーザー入力「心臓です」の場合、現在は「心臓血管」に分類される（評価順序により）。

**設計上、これが意図通りか確認が必要**:
- 医学的に、単独の「心臓」という言及は「循環器」に分類すべきではないか？
- あるいは、「心臓血管」と「循環器」は診療内容が異なり、「心臓」をキーワードとして持つべきではないか？

特に、プロンプト line 119 の「部分マッチ可」の定義が曖昧なため、「心臓血管という単語内での『心臓』」と「単独の『心臓』」の区別がつきにくい。

**修正方針**:

1. **医学的観点から確認**: 「心臓」単独の言及を「心臓血管」に分類することが妥当か、あるいは「循環器」に分類すべきか
2. **部分マッチの定義を明確化**: プロンプト内で「部分マッチ」の定義を以下のように追加：
   ```
   ◆ 部分マッチの定義
   「部分マッチ可」とは、キーワードが入力テキスト内に「区切られた単語単位」で含まれることを指す。
   例1: キーワード「心臓」、入力「心臓の治療」→ マッチ
   例2: キーワード「心臓」、入力「心臓血管外科」→ マッチ（ただし「心臓血管」が先に評価される）
   ```
3. **評価順序の見直し**: 「特定名称が一般名称より先」の原則を厳密に適用するため、「循環器」の「心臓」キーワードを削除または別キーワード（例：「心臓の調子」など意図を限定するキーワード）に変更

### W-002: STEP2 の「どこ」「どれ」が曖昧で誤分類の可能性あり  <prompter>

**重大度**: Warning  
**対象モジュール**: OpenAI_診療科  
**問題**:

STEP2（line 110-113）で「どこ」「どれ」が「わかりません」系のキーワードとして扱われている：

```
- どこ
- どれ
```

しかし、これらは「分かりません」という意図表現ではなく、日本語の「質問詞」である可能性がある。

**誤分類シナリオ**:
- ユーザー: 「腕の痛みはどこの診療科ですか？」
- キーワード「どこ」が含まれる → STEP2 で「登録なし」に分類
- **期待値**: ユーザーは「診療科を聞いている」のに、「わからない」と判定される ❌

より正確には、「どこ」「どれ」が含まれる場合、それが：
- 「わかりません」の言い換えなのか（例：「どれがいいか分からない」）
- 単なる質問詞なのか（例：「どこの科ですか」）

の区別がつきにくい。

**修正方針**:

STEP2 から「どこ」「どれ」を削除し、より限定的な「わかりません」系キーワードのみに絞る：

```
【修正後の STEP2 キーワード】
- わからない
- 分からない
- わかりません
- 分かりません
- 解らない
- わからん
- 不明  ← 削除候補（医療用語との衝突あり — W-003 参照）
（削除: どこ、どれ）
```

### W-003: STEP2 の「不明」が医療用語としても使用される可能性  <prompter>

**重大度**: Warning  
**対象モジュール**: OpenAI_診療科  
**問題**:

STEP2（line 110）で「不明」が含まれているが、医療会話では「診断が不明」「症状は不明」など、医学的な用語として使用される可能性がある。

**誤分類シナリオ**:
- ユーザー: 「症状がよく分からない、診断は不明なんです」
- キーワード「不明」が含まれる → STEP2 で「登録なし」に分類 ⚠️
- **期待値**: 「わかりません」という意図を反映すべき場合と、医学的に「診断が不明である」という状況説明の 2 パターンがあり、区別が困難

実装上、「不明」を単独のキーワードとして扱うと、医療用語としての用法で誤分類される確率が高い。

**修正方針**:

STEP2 から「不明」を削除し、より限定的な「わかりません」系キーワードのみに絞る：

```
【修正後の STEP2 キーワード（推奨）】
- わからない
- 分からない
- わかりません
- 分かりません
- 解らない
- わからん
（削除: 不明、どこ、どれ）
```
