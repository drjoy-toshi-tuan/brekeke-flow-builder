### 抽出した要件サマリー

#### 全体構造
- 医事課入院係への callback ボット（代表電話混雑時に AI が一次受け）
- フロー骨格: 冒頭 → 初診確認 → [診察券番号 subflow（再診のみ）] → 氏名 → 生年月日 → 電話番号 → 携帯番号 → 用件① → 追加用件 → [用件②（あり時のみ）] → 通話完了 → 終話
- acceptance_times / call_transfer / date_of_call_classifier は不使用

#### 聴取項目（5 件 + サブフロー 4 件）
- **hearing**: 初診確認(enum) / 携帯番号聴取(text) / 用件聴取_1(text) / 追加用件確認(enum) / 用件聴取_2(text)
- **subflow**: 診察券番号聴取 / 氏名聴取 / 生年月日聴取 / 電話番号聴取

#### 主要分岐
- **初診確認 enum**: 「初診」→ 診察券スキップして電話番号聴取へ / 「other(=再診)」→ 診察券番号聴取へ
- **追加用件確認 enum**: 「あり」→ 用件聴取_2 へ / 「other(=なし)」→ 通話完了アナウンスへ

#### TTS（customer_doc から完全転記済み）
- 冒頭_アナウンス: callback 説明 + 3 診療日以内折り返し
- 通話完了_アナウンス: 受付完了 + 折り返し連絡説明 + 代表電話混雑案内
- END_聴取失敗: 代表電話案内 + DTMF 操作（1 番→4 番）案内
- 携帯番号聴取: 緊急連絡用案内 + 「持っていない」発話可能
