### 移管方針
- Gen1「箱」→ Gen3 9 ブロック型へ変換。opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「お電話ありがとうございます。自治医科大学附属さいたま医療センターの、外来予約専用、AI電話です」。
- **用件分岐は Pattern A（5 分岐）**: 用件聞き取り OpenAI 直後に「再診 / 初診 / 変更 / キャンセル / 確認」+ DTMF 1=診察予約 / 2=変更 / 3=キャンセル / 4=確認 並走。**ただし用件聞き取り yaml では「診察予約」を一旦確定させた後、`初再診確認` で再診/初診を二段聴取**する設計。Gen3 では用件 hearing 1 本で 5 値を OpenAI 出力させ、二段聴取は context_match_router で「diagnosis 系（診察予約）→ 初再診 hearing 追加 → echo_back / それ以外 → そのまま echo_back」に整理（director 判断）。
- **時間判断（時間判断.yaml）**: 平日 9:00〜16:00、年末年始 12/26〜1/4 を個別判定 + 祝日チェック → date_of_call_classifier ブロック。**addCurrentDate: Yes 自動付与対象**。年末年始アナウンスは「予約受付再開は 1 月 5 日 9 時から、体調不良による受診は 048-647-2111 まで」と専用番号案内あり。
- **稼働開始日 5/19**: `稼働日前アナウンス` 箱「この番号は、5/19午前9時より、受付開始となります」TTS あり。Gen3 本番投入が 5/19 以降にずれる場合は不要（director 判断、要マネージャー確認）。
- サブフロー（患者氏名 / 生年月日 / 電話番号 / 診察券）は Jump to Flow 経由。
- 自由テキスト hearing は **理由、確認内容の聞き取り** と **医師名聴取（医師名or診療科 / 医師名のみ）** と **予約日都合聞き取り（都合の合わない日時）** の 3 種類。

### 施設固有の特殊ルール
- **受付時間**: 平日 9:00〜16:00、土日祝日除く、年末年始 12/26〜1/4 休診。
- **代表電話案内が 2 系統**: `初診：2回失敗時アナウンス`（048-788-2788、初診紹介受診者向け）と `初診以外：2回失敗時アナウンス`（048-647-4898、再診/変更/キャンセル/確認向け）。初再診不明時は `初再診不明：失敗時アナウンス` で **両番号を並べて 3 回繰り返し**「初診紹介受診の方は、048-788-2788、それ以外の方は、048-647-4898」。Gen3 では用件 context を参照して termination_patterns で 3 種に振り分け（director 判断）。
- **紹介状有無分岐は新規（初診）のみ**: 用件 = 新規 → `紹介状有無聞き取り` → 「紹介状あり」なら通常フロー、「紹介状なし」なら `紹介状無アナウンス`「紹介状を手元にご準備の上、再度お電話ください」→ 終話。**初診時選定療養費の金額案内なし**（信州大・岐阜大では金額提示があったが本施設は無し）。
- **新規予約も AI 受付対象**: 信州大・徳島大・岐阜大とは異なり新規（初診）の受付がフローに残っている。
- **診療科聞き取りはハイラルキ二段（4 グループ）**: ① 親エンティティ「呼吸器 / 消化器 / 脳神経 / 産婦人科」発話 → ② 内/外科 or 産科/婦人科を聞き直し → ③ 確認復唱。**北里大・信州大・岐阜大より分岐が少なく、CT/MRI レーンも無い**。
- **対象外診療科**: 「救急科 / 病理診断科」のみ → `END_対象外診療科`。
- **耳鼻咽喉・頭頸部外科 / 一般・消化器外科 への中黒変換**: OpenAI 正規化辞書（entity_openai_synonyms.md）で吸収。
- **診察予約レーンの追加質問**: 紹介状有無（初診のみ）→ 診療科 → 予約種別確認（**「診察 / 診察以外」の 2 値**、診察以外は CT/MRI/PET-CT/採血/内視鏡 等 30 種以上の細目）→ 予約日 → 診察 → 医師名 or 診療科 / 診察以外 → 予約時間 → 予約日都合 → 氏名 → 生年月日 → 連絡先。
- **予約変更 / キャンセルの追加質問**: 予約種別 → 予約日 → 理由 → 診察券 → 氏名 → 生年月日 → 連絡先。
- **予約確認**: 予約確認内容 → 診察券 → 氏名 → 生年月日 → 連絡先。
- **時間内アナウンス TTS**: `時間内へ` 箱「体調不良等の当日受診のお電話はお受けできませんのでご了承ください」が時間内分岐の冒頭で必ず再生。
- **3 回失敗時**: 各サブで `count再質問` 2 回到達 → `END_聞き取り失敗` → `初再診分岐` で 3 種代表案内に分岐 → 終話。**用件聞き取り段階での失敗のみ `初再診不明：失敗時アナウンス`（両番号案内）に分岐**。
- **予約日 DTMF 4 桁**: 月+日 + 過去日 +1 年補正、Gen3 標準 subflow が吸収。
- **DTMF 受付（用件聞き取り）**: 1=診察予約 / 2=変更 / 3=キャンセル / 4=確認。
- **救済語**: 「Suica → 麻酔科」「BB/GB/DB → 耳鼻咽喉頭頸部外科」「ハート → 循環器内科」等多数。entity_classification.md 参照。
- **復唱（echo_back）対象**: 用件 4 種 + 初診/再診（診察予約レーンのみ）+ 診療科 + 予約種別。

### director が見落としやすいポイント
- 冒頭の `API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `API_DRJOY` → `時間判断` → `時間内へ` → `用件聞き取り` の 6 箱は Gen3 で opening + date_of_call_classifier + 時間内 announcement + 用件 hearing に整理。
- セリフ none 振り分けのみの「箱」（用件分岐 / _1 / _2、用件確認 / _1 / _2、完了フラグ、紹介状無し→終話、かけ直し案内（13 種）、初再診分岐、診察orその他分岐、各種診療科セット、読み取得系、診療科分岐、耳鼻咽喉・一般消化器の中黒変換、用件のセット / セットした用件の分岐、count上記以外 / count再質問、予約日DTMF系、予約種別セット系 等多数）は独立ブロック不要 → 前段の next 書き換え or context_match_router の一部として吸収。
- **用件は二段確定**: ① 用件聞き取り yaml の `セットした用件の分岐` で「診察予約 / 変更 / キャンセル / 確認」4 値 → ② 診察予約だけ `初再診確認` で「再診 / 初診」追加聴取 → 結果として用件 5 値。Gen3 では用件 hearing 1 本で 5 値を OpenAI 出力させ、二段聴取構造は context_match_router の派生で再現。
- **「予約変更」「予約キャンセル」が 3 経路で出現**: main の用件分岐 / 用件聞き取り.yaml セットした用件の分岐 / 理由・確認内容の聞き取り.yaml 用件の確認。Gen3 では用件 context を一度確定したら以降は context_match_router で参照する。
- **`予約種別確認.yaml` は 30 種以上の診察以外項目（CT/MRI/PET-CT/採血/内視鏡/透析/化学療法 等）**: Gen3 では正規化辞書で「診察以外」フラグ + サブカテゴリ保持、その後の予約時間聴取に進めるだけ。**PET-CT のみ専用復唱パスがある**点に注意。
- **`医師名聴取.yaml` は新規/再診で TTS が異なる**: 初診なら「医師名の指定がある場合は」、再診なら「予約票に記載されている先生のお名前、または診療科をお話ください」。Gen3 では用件 context = 初診/再診 で hearing prompt を切替（director 判断）。
- **`予約日都合聞き取り.yaml` は新規予約レーンの最後に**「都合の合わない日時」を自由テキスト聴取（「ありません」発話で skip 可能）。**新規予約のみ呼び出し**、再診/変更/キャンセル/確認では呼び出さない。
- **`電話番号区切り.yaml` は読み上げ用補助サブシナリオ** → Gen3 では `<% var %>` の readable 形式で吸収（director 判断、別ブロック化不要、信州大・岐阜大・徳島大・北里と同方針）。
- **失敗時のかけ直し案内が 13 系統**（用件NG / 紹介状NG / 医師名NG / 理由・確認NG / 予約日NG / 電話番号NG / 生年月日NG / 氏名NG / 診察券番号NG / 診療科NG / 予約種別NG / 予約時間NG / 希望日NG）→ Gen3 では failure_termination 1 本に集約し、用件 context 参照で代表案内文言を切替。**用件NG のみ初再診不明扱いで両番号並べる termination に直結**。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**（丸数字 ①②③⑩等は使用禁止）。括弧記法は `END_用件NG` `END_紹介状なし` `END_対象外診療科` 等の安全名に置換。
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**（scaffold が付ける prefix）。subflow ブロックの step 名そのものではダメ。
- **CMR conditions は必ず最後に `match: "other"` + next 明示**。`TODO_other_target` プレースホルダーを残さない。
- **転送先は無し**（Blind Transfer モジュールは登場しない、徳島大と同様）。失敗系・対象外系は全て代表電話案内 TTS のみ。
- **TTS は `{tts_g:...}` 小文字のみ、SSML 禁止**。Gen1 変数記法 `<? $予約日の読み ?>` `<? $診療科の読み ?>` `<? $reservation_type ?>` は Gen3 では `<% var %>` に変換。
- 患者氏名は PatientName サブフロー 1 枠（受診者 = 入電者前提）。
