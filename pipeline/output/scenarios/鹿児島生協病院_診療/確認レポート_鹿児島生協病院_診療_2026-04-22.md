- **作業種別**: Gen2→Gen3移管

- **シナリオ名**: 診療（眼科専用）

### 抽出した要件サマリー

- **用件 4 択**: 再診 / 変更 / キャンセル / 確認（classification enum）
- **初診弾き**: 受診歴確認で「無」の場合、END_初診受付不可 で終話（受付時間案内を含む長文）
- **個人情報聴取**: 患者氏名 → 電話番号 → 生年月日 の順でサブフロー経由
- **電話番号**: incoming-classifier 3 分岐（携帯は着信番号復唱、固定/IP/非通知は手動聴取）
- **用件別分岐**:
  - 再診・変更 → 予約希望日（フリーワード）
  - キャンセル → 次回予約有無 → 予約希望日（有のみ）→ 最終問合せ
  - 確認 → 問い合わせ内容
- **終話パターン**: 4 系統（再診受付完了/変更受付完了/確認受付完了/キャンセル受付完了）+ 初診受付不可 + 異常系 3 つ
- **smsFlag**: status=1 かつ 再診/変更/確認 = 1、status=1 かつキャンセル = 2、それ以外 0
- **RAG**: なし（元資料に記載なし）
- **診察券番号**: function に required として残るが yaml フローでは聴取しない（director 判断：空文字のまま dialogue_completed に渡す）
- **診療科 (clinicalDepartment)**: 眼科固定値（受診科識別不要）
- **特殊ルール多数**: Yes/No 救済語（肺/いいです/が今/PHVます/P）、数字 4/5 誤認識救済（C/for→4、ごー→5）、用件キーワード救済（印刷/大変/薬品/不審 等）、生年月日復唱形式指定、「もしもし」単独発話処理、予約希望日フリーワード格納
