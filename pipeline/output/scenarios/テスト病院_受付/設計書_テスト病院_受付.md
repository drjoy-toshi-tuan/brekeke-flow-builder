# 設計書 — テスト病院 受付フロー

## 基本情報

| 項目 | 内容 |
|---|---|
| 施設名 | テスト病院 |
| フロー名 | `テスト病院$受付` |
| 作成日 | 2026-03-23 |
| 対象チャネル | 電話（着信） |
| 概要 | 電話着信から冒頭ガイダンス、用件確認（予約／変更／その他）の3分岐を経て終話するシンプルな受付フロー |

---

## フロー全体図

```
[着信]
    ↓
[TTS: 冒頭ガイダンス]
    ↓
[STT: 入力_用件確認]
    ├─ 予約      → [TTS: 終話_予約]
    ├─ 変更      → [TTS: 終話_変更]
    ├─ その他    → [TTS: 終話_その他]
    ├─ timeout   → [リトライ_用件確認]
    ├─ error     → [リトライ_用件確認]
    └─ no_result → [リトライ_用件確認]
                      ├─ true（リトライ可） → [STT: 入力_用件確認]
                      └─ false（上限到達） → [TTS: 終話_失敗]
```

---

## モジュール詳細

### 1. 冒頭ガイダンス
- **type**: `drjoy^Text To Speech$Text to speech`
- **next**: 無条件 → `入力_用件確認`
- **params**:
  - `prompt`: `{tts_g: お電話ありがとうございます。こちらは、テスト病院の自動応答サービスです。ご用件をお話しください。診察のご予約、予約の変更、その他のお問い合わせ、のいずれかでお答えください。}`
  - `stop_by_dtmf`: `false`

---

### 2. 入力_用件確認
- **type**: `drjoy^External Integration$DTMF AmiVoice STT Input`
- **next**:
  - `^予約$` / `よやく` → `終話_予約`
  - `^変更$` / `へんこう` → `終話_変更`
  - `^(その他|問い合わせ)$` → `終話_その他`
  - `^TIMEOUT$` → `リトライ_用件確認`
  - `^ERROR$` → `リトライ_用件確認`
  - `^NO_RESULT$` → `リトライ_用件確認`
- **params**:
  - `prompt`: `{tts_g: 予約、変更、その他、のいずれかをお話しください。}`
  - `timeout`: `30`
  - `retry`: `2`
  - `profile_words`: 下記参照

---

### 3. リトライ_用件確認
- **type**: `drjoy^Text To Speech$Speech Retry Counter`
- **next**:
  - `true`  → `入力_用件確認`
  - `false` → `終話_失敗`
- **params**:
  - `retry_count`: `2`
  - `prompt_true`: `{tts_g: 申し訳ございません。うまく聞き取りができませんでした。もう一度、ご用件をお話しください。}`
  - `prompt_false`: `（終話_失敗モジュールへ遷移するため空白）`

---

### 4. 終話_予約
- **type**: `drjoy^Text To Speech$Text to speech`
- **next**: 通話終了
- **params**:
  - `prompt`: `{tts_g: ご予約の窓口へおつなぎします。少々お待ちください。}`
  - `stop_by_dtmf`: `false`

---

### 5. 終話_変更
- **type**: `drjoy^Text To Speech$Text to speech`
- **next**: 通話終了
- **params**:
  - `prompt`: `{tts_g: ご予約変更の窓口へおつなぎします。少々お待ちください。}`
  - `stop_by_dtmf`: `false`

---

### 6. 終話_その他
- **type**: `drjoy^Text To Speech$Text to speech`
- **next**: 通話終了
- **params**:
  - `prompt`: `{tts_g: その他のお問い合わせ窓口へおつなぎします。少々お待ちください。}`
  - `stop_by_dtmf`: `false`

---

### 7. 終話_失敗
- **type**: `drjoy^Text To Speech$Text to speech`
- **next**: 通話終了
- **params**:
  - `prompt`: `{tts_g: 大変恐れ入りますが、うまくご用件を確認できませんでした。お手数ですが、再度おかけ直しください。}`
  - `stop_by_dtmf`: `false`


## 遷移条件まとめ

| condition | 意味 | 遷移先 |
|---|---|---|
| `^予約$` | 「予約」と発話 | 終話_予約 |
| `^変更$` | 「変更」と発話 | 終話_変更 |
| `^(その他\|問い合わせ)$` | 「その他」または「問い合わせ」と発話 | 終話_その他 |
| `^TIMEOUT$` | タイムアウト | リトライ_用件確認 |
| `^ERROR$` | 認識エラー | リトライ_用件確認 |
| `^NO_RESULT$` | 認識結果なし | リトライ_用件確認 |
| `true` | リトライ可 | 入力_用件確認 |
| `false` | リトライ上限到達 | 終話_失敗 |
