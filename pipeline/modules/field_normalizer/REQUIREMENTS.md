# field_normalizer — 聴取値の単独フィールド正規化

## 目的
商談デモ「フリー発話受付」の終端 `drjoy_finalize`（`docs/brekeke/script_templates/drjoy_finalize.js`）が、
**聴取経路で save2db された生 STT** を Dr.JOY へ送る前にクリーン化するための正規化ロジック。
用件抽出（自由発話前提）とは入力仮定が異なり「フィールド単独入力」前提。

## 入出力
- 入力: `kind`（フィールド種別）＋ `raw`（生 STT 文字列）
- 出力: 正規化後文字列。マッチしなければ `""`（呼び出し側 finalize が `clean or raw` で raw を維持＝冪等）

| kind | 正規化 | 例 |
|---|---|---|
| `name` | 前置（私は/名前は…）除去・末尾マーカー（と申します/です…）で切断・敬称（さん/様…）除去 | 「山田太郎と申します」→「山田太郎」/「山田太郎です」→「山田太郎」/「山田太郎」→「山田太郎」(冪等) |
| `phone` | 全角→半角・区切り除去・`0\d{9,10}` 抽出 | 「電話番号は090-1234-5678です」→「09012345678」 |
| `card` | 全角→半角・区切り除去・`\d{3,10}` 抽出 | 「診察券番号は12345です」→「12345」 |
| `birthday` | 和暦/西暦の `…年…月…日` を **`YYYY-MM-DD 00:00:00`** へ変換（DATE_OF_BIRTH ウィジェットが要求する形式・和暦は ERA_BASE で西暦化）| 「昭和60年4月1日です」→「1985-04-01 00:00:00」/「令和2年1月1日」→「2020-01-01 00:00:00」|
| `department` | 公式診療科辞書の最長一致 | 「内科でお願いします」→「内科」/「耳鼻でお願いします」→「耳鼻咽喉科」 |
| `date` | 末尾丁寧表現（です/の予約です/でお願いします…）除去・自然文維持 | 「6月20日10時の予約です」→「6月20日10時」 |
| `raw` | 無変換（用件種別/概要/自由発話） | — |

辞書（DEPARTMENTS）・手順は `inquiry_extractor` と同一系統。
