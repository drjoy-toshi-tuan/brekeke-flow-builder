# 健診CC コース分類器（checkup_course_classifier）— 仕様

> **#274 universe 化（engine v3・2026-07-08）**: CATEGORIES を実ログ/DR universe の**コース種別**へ差し替え、
> **施設非依存**にした。出力ラベルは universe 6 種別（雇用時健診 / 市区町村健診 / 協会けんぽ・生活習慣病予防健診 /
> 一般・定期健診 / レディース・専門ドック / 人間ドック）＋ その他の健診（generic）。DTMF は **data-driven（DTMF_MAP・@spec）**、
> `facility_offered`（wiring・配列/null）指定時は施設非提供ラベルを `ラベル|OFF_MENU` で返す。FOLDINGS は
> `健康診断→健診` の畳みを外し（実ログ最大の一般健診を種別化）、`検診→健診`・`健康診査→健診` は維持。
> 施設は `FACILITY_OFFERED` 配線＋施設別コース別名・DTMF_MAP 順の差分で instantiate（各々再 P6）。
> 下記の厚木ルーティング表・旧 4 択 DTMF 記述は **v2 当時の一例**（v3 のラベル集合は上記 universe 種別）。

ヘルスケアクリニック厚木 統合CC「コース選択」（n6）の generate_by_OpenAI 分類を置き換える決定論 Script。
「その他コース確認」（-6）も**同一 script の再利用**（SOURCE_MODULE 差し替えのみ）。

## 入出力

- 入力: `$runner.getModuleResult(SOURCE_MODULE)`（文字列 or `{text:...}`）。
- 出力（`$runner.setResult`）:

| 出力 | コース選択(n6)のルーティング | その他コース確認(-6)のルーティング |
|---|---|---|
| `人間ドック` / `協会けんぽ` / `定期健診` | 氏名聴取 | （default → 氏名聴取） |
| `雇用時健診` | END_雇用時健診_HP案内 | END_雇用時健診_HP案内 |
| `その他の健診` | その他コース確認(-6) | （default → 氏名聴取） |
| `NO_RESULT` | リトライ（再質問） | リトライ |

- 設計書 YAML の enum ラベルと完全一致。-6 は `^雇用時健診$` のみ専用配線、それ以外は default で氏名へ
  （course の生テキスト保存は STT モジュール側 save2db で行う＝ラベル化で詳細が落ちない）。

## 判定手順（決定論・上から評価）

1. **正規化**: checkup_intent_classifier と同一（全角→半角・小文字化・記号空白除去＋
   健康診断/健康診査/検診→健診 畳み込み）。
2. **空** → `NO_RESULT`。
3. **数字のみ（DTMF 併用）**: `1`→人間ドック / `2`→協会けんぽ / `3`→定期健診 / `4`→その他の健診 /
   その他の数字→`NO_RESULT`。
4. **わからない検知** → `その他の健診`（※他部品と異なり NO_RESULT ではない。
   -6 が「わからない場合は『わからない』とお話しください」と明示誘導しており、
   わからない＝その他扱いで氏名へ進め折返しでスタッフが確定する設計。図の -6 仕様と整合）。
5. **カテゴリ走査**（部分一致・優先順位順）:
   1. **雇用時健診**（checkup_intent_classifier と同一語彙）
   2. **協会けんぽ**（協会けんぽ/けんぽ/協会健保）
   3. **定期健診**（定期健診/定期の健診 ※畳み込み後）
   4. **その他の健診・明示語**（特定健診/市の健診/市健診/生活習慣病/イーウェル/東振協/レディース/
      法定健診/職場の健診/会社の健診/その他）
   5. **人間ドック**（人間ドック/ドック）
6. **総称の受け皿走査**（部分一致・最後段）: 「健診」を含む（畳み込み後の健康診断/検診含む）→ `その他の健診`。
   特定コース語に全部外れた「健診」言及はその他扱いで -6/氏名へ進め、折返しでスタッフが確定する。
7. 不一致 → `NO_RESULT`。

### 優先順位の根拠

- **雇用時健診が最優先**: HP 誘導が確実に発動するように（旧要件定義「『雇用時健診』に該当した時だけ雇用時健診へ」）。
- **協会けんぽ > 定期健診**: 「協会けんぽの定期健診」は保険者特定が強い情報のため協会けんぽ。
- **定期健診 > 会社の健診**: 「会社の定期健診」→定期健診。
- **明示その他語 > 人間ドック**: 「レディースドック」→その他の健診（レディース系コースは 4 択外）。
- **特定健診・市の健診は n6 ではその他の健診へ**: 受付不可案内（END_受付不可案内）は用件確認(n4)の責務。
  予約フローまで入ってからの発話は -6→氏名→折返しでスタッフが受付可否を案内する安全側に倒す
  （n6 に受付不可分岐を増やす場合は設計書改訂＋再受入）。
- 語彙典拠: 旧要件定義のコース正規化リスト（人間ドック/生活習慣病予防健診/定期健診/法定健診/雇用時健診/
  イーウェル/東振協/特定健康診査/会社の健診/職場の健診/協会けんぽ/レディース健診/健康診断 等 18 語）。
