# checkup_intent_classifier — 健診CC 用件 決定論分類器

ヘルスケアクリニック厚木 統合CC（6 施設共通）の「用件確認」(n4) と「遅刻種別確認」(-4) の
generate_by_OpenAI を置き換える決定論 Script。仕様: `REQUIREMENTS.md`。

## 使い方

1. `script.js` を @General$Script モジュールとして配置（n4 用 / -4 用の 2 インスタンス）。
2. 先頭の設定行 `var SOURCE_MODULE = "__SOURCE_MODULE__";` を入力 STT モジュール名に差し替える
   （**この 1 行のみインスタンスパラメータ**）。
3. 出力 9 値: 予約 / 変更 / キャンセル / 雇用時健診 / 市の健診 / 特定健診 / 遅刻 / その他 / NO_RESULT。
   設計書 enum ラベルと一致。-4 では ^遅刻$ と ^(変更|キャンセル)$ のみ配線し他はリトライへ。

## 判定方式

正規化（全角→半角・小文字化・記号空白除去＋健康診断/健康診査/検診→健診 畳み込み）→
空 / DTMF(1-4) / わからない検知 → **カテゴリ優先順位走査**:
その他(問合せ系) > 変更 > キャンセル > 雇用時健診 > 市の健診 > 特定健診 > 遅刻 > 予約。
動作語がコース語に優先する根拠は旧要件定義 728 行（変更/キャンセル/問い合わせ時の雇用時健診は折返し対応）。
