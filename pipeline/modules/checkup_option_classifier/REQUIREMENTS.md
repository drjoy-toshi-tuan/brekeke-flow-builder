# checkup_option_classifier — 受診オプション検査 抽出・正規化

## 概要

健診センターの「受診オプション検査」選択場面で、患者の自由発話（STT結果）から
オプション検査名を抽出・正規化する決定論スクリプト。
`generate_by_OpenAI`（オプション選択プロンプト）の置換を目的とする。

## 入出力

| 項目 | 内容 |
|---|---|
| 入力 | STT 結果テキスト（自由発話） |
| 出力 | 正規化された検査名（複数は「、」区切り） / "不明" / "無い" / "復唱途中切断" |

## 判定優先順位

1. **マッチあり**（否定除外後）→ 正規化名を「、」結合して返す
2. **不明系**（わからない／まだ決めていない等）かつマッチ無し → `"不明"`
3. **無い系**（追加なし／不要／いらない等）かつマッチ無し → `"無い"`
4. どれにも該当しない → `"復唱途中切断"`

## アルゴリズム

1. **NFKC 正規化** → 全角英数を半角化 → 英字を小文字化
2. **記号・空白ストリップ**（句読点・括弧・ハイフン・スペース等を除去）
3. **音声誤認補正**（ロック/ドッグ/トック → ドック）
4. **FUMEI_PATTERN** テスト → `has_fumei`
5. **NAI_PATTERN** テスト → `has_nai`
6. **否定検出**（`keyword + 否定マーカー` 直結パターンでアイテムを negated セットに）
7. **アイテム一致走査**（定義順・先勝ち・キーワード部分一致）
8. **SUBSUMES 適用**（複合コース一致時に構成単体を除外）
9. **優先順位適用** → 出力

## 否定語処理（制限事項）

- `keyword + 否定マーカー` の直結パターンのみ検出（助詞を挟んだ遠隔否定は未対応）
- 例: "脳ドックはいらない" → 否定OK / "脳ドック、それはいらない" → 否定未検出
- 自己訂正（"〇〇はやめて××で"）は "はやめ" が否定マーカーに含まれるため自動処理される

## SUBSUMES（複合コース包含関係）

複数アイテムが同時マッチした場合、複合コースを優先し構成単体を除外する:

| 複合コース | 除外される単体 |
|---|---|
| 脳ドック＋PET-CTコース | 脳ドック / PET-CTがんコース |
| 胃部内視鏡＋PET-CTコース | PET-CTがんコース |
| ＜お通い＞２日ドック／なでしこ精査コース | 脳ドック |
| ＜お通い＞２日なでしこPET-CTがんコース | PET-CTがんコース |
| ＜お通い＞３日なでしこＰＥＴ－ＣＴがんプレミアムコース | PET-CTがんコース / 脳ドック |
