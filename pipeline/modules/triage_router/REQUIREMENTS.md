# triage_router — 受診相談 決定論トリアージ（要件定義）

- 対象: 商談デモ 受診相談パスC の `受診相談_LLM問診`（generate_by_OpenAI）置換。
- 準拠: JTAS ＋ 総務省消防庁「電話相談」プロトコル。判定仕様の本体は
  `output/scenarios/商談デモ_フリー発話受付/受診相談_トリアージ判定仕様_20260703.md`（問診票＝決定木）。
- 実データ根拠: `reference_consult_triage_real_response_patterns`（患者86%自由発話・純yes/no回答者0）。

## 0. このモジュールが証明すること

「自由発話の症状相談を、LLM を使わず**決定論**で 3 ゴール（救急/看護師/通常）に振り分けられる」こと。
オラクルは A→B→C→D カスケードを純関数で実装し、全分岐を test で網羅して**出力が入力から一意に決まる**ことを機械証明する。

## 1. 出力（3 ゴール・全て推奨トーン）

| 出力 | 意味 | VFB 終端 |
|---|---|---|
| `GOAL1_救急` | 救急車要請を推奨 | END_救急案内（TTS） |
| `GOAL2_看護師` | 看護師接続/早期受診を推奨 | END_看護師案内（TTS） |
| `GOAL3_通常` | 通常受診/自宅療養（LINE推奨）を推奨 | END_通常案内（TTS） |

判定は必ずいずれか1つに落ちる（D=全否定で GOAL3）。`reason` に発火ブロック/フラグを付す（監査用）。

## 2. 入力（答えの取り方＝2 系統）

浜口さん確定（2026-07-03）: **Yes/No 誘導は「本当に救急か」の一番最初（A ブロック）だけ**。それ以外は自由発話フォールバック抽出。

- **A ブロック（一次スクリーニング・閉じた質問）**: `abcd: dict[str, str]`。各値 `"yes" | "no" | "unclear"`。
  - キー: `A1_意識`（呼びかけ反応なし）/ `A2_気道`（話せない・声出ない）/ `A3_呼吸`（息苦しい）/ `A4_循環`（顔色/唇が紫・冷汗）。
  - `"yes"` = 該当（危険側）。`"unclear"`（NO_RESULT）= **安全側で危険扱い**（over-triage・A は生命兆候ゆえ）。
- **A-0 CPA**: 全患者発話テキストに対する固定キーワード検知（心肺停止語）。閉じ質問ですらなく最優先。
- **B/C（自由発話抽出）**: `complaint: str`（主訴発話）＋ `free_texts: list[str]`（以降の自由発話）。
  - 主訴分類・Red Flag スキャン・修飾因子スキャンは、この自由発話テキスト（complaint＋free_texts の連結）に対する**決定論キーワード走査**。

## 3. カスケード（Top-Down Exclusion・短絡）

```
1) A-0 CPA語 検知(全テキスト)                 → GOAL1_救急
2) A-1..A4 いずれか yes/unclear               → GOAL1_救急
3) 主訴分類 → カテゴリ別 Red Flag スキャン
      いずれか発火                             → GOAL1_救急
4) 修飾因子スキャン いずれか発火               → GOAL2_看護師
5) 全否定                                      → GOAL3_通常
```

- **短絡**: 上から評価し最初に成立したゴールで確定。**機械ランクダウン禁止**（上位確定後に下位を見ない）。
- **複数主訴**: A系カテゴリ（頭痛めまい/胸痛）を優先分類。B スキャンは分類カテゴリ＋「全カテゴリ共通の致死語」を併走（放散痛/冷汗/突発激痛/大量出血等は主訴カテゴリに関係なく拾う）。
- **その他カテゴリ**: カテゴリ別 Red Flag は無し（共通致死語のみ）→ 実質 C/D で判定。

## 4. 主訴カテゴリ（分類・最長一致/包含）

`頭痛めまい` / `胸痛` / `腹痛` / `発熱` / `外傷出血` / `その他`。分類は自由発話へのキーワード包含。
複数該当時は A系（頭痛めまい→胸痛）優先、次に外傷出血、発熱、腹痛、最後にその他。

## 5. Red Flag / 修飾因子 = 自由発話キーワード走査（over-triage bias）

- 各 Red Flag は「はい/いいえ質問」ではなく、**自由発話に危険サイン語が含まれるか**で発火（recall 重視・表現ゆれを広く拾う）。
- 例（胸痛）: `放散痛`=`(首|あご|顎|肩|背中|腕).*(痛|広が|放散|しびれ)` / `冷汗`=`冷や?汗|脂汗` / `突発`=`突然|急に|いきなり` / `安静時痛`=`安静|じっとして|寝てて`t...。
- 修飾因子（全カテゴリ共通・C）: `歩行不能`=`歩けな|立てな|動けな` / `抗凝固薬`=`血.*さらさら|ワーファリン|抗凝固|血液.*薬` / `妊娠`=`妊娠|妊婦` / `高齢`=`(6[5-9]|[7-9][0-9]|100).?歳|高齢` / `小児`=`(0|1|2|3|4|5).?歳|乳児|赤ちゃん` / `止血困難`=`血.*止まらな|ずっと出血` 等。
- **キーワード表は data-driven**（`oracle.py` の `RED_FLAGS` / `MODIFIERS` 辞書）。カテゴリ追加・語彙拡張は表を足すだけ（エンジンコード不変）。full セットは判定仕様 §4/§5 準拠。
- **曖昧・未検出の扱い**: Red Flag は「危険サインの陽性検知」なので、語が無ければ非発火（＝そのフラグでは上げない）。over-triage bias は「語を広く拾う」方向で効かせる。A の unclear だけは危険扱い（生命兆候）。

## 6. 決定論・エッジ

- 同一入力 → 同一出力（純関数・乱数/時刻/IO なし）。
- 空入力（complaint 空・free_texts 空・abcd 全 no）→ GOAL3_通常（何も危険サインが無い）。
- NFKC 正規化してから走査（全半角/カナゆれ吸収・既存部品と同方式）。
- reason は必ず発火根拠（`A-0` / `A1_意識` / `B:胸痛/放散痛` / `C:歩行不能` / `D:全否定`）を返す（監査）。

## 7. VFB 実装への写像（Phase 3・参考）

オラクルの純関数は VFB では次に分解される想定（新規エンジン0・[[判定仕様]] §8）:
- A-0/A1-4 = hearing→（CPAはCMRキーワード / ABCDは yes_no_classifier）→CMR。
- 主訴分類 = hearing→n_choice（症状カテゴリ spec）→CMR。
- B/C 抽出 = hearing（開いて聞く）→ script（本 oracle の Red Flag スキャン相当・inquiry_extractor/field_presence 型）→CMR。
- 終端 = END_救急案内 / END_看護師案内 / END_通常案内。
