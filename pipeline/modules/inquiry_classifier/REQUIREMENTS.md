# inquiry_classifier — 要件定義（用件分類・自由発話・決定論）

## 目的
総合相談室の「ご用件」自由発話を **OpenAI を使わず**決定論的に分類する。
亀田 総合相談室の `OpenAI_用件確認`（5択判定エンジン）を置き換える。

> 成果物は **Script モジュールに入れる JS 本体（`script.js`）＋ oracle 認定**まで。
> 実モジュール化（Brekeke 構築・組込）は人手。

## 入出力
- **入力**: 直前の STT 自由発話モジュールの結果。
- **出力**: `相談` / `予約` / `大代表` / `定型案内` / `その他` / `NO_RESULT`。
- **副作用**: `CONTEXT_NAME` に保存（saveContext2DB + setObject）。

## 顧客大原則（2026-06-10 浜口確定）
相談室業務に関係する電話のみ受け、**予約関連は弾く**。
- 優先順位 **相談 ＞ 予約 ＞ 大代表 ＞ 定型案内**（RULES 配列順・先勝ち）。
- 「相談」を含む発話は必ず相談を優先（**相談予約を誤って弾かない**）。
- 相談を含まず予約系を含む発話は必ず予約として弾く。

## 判定順（script.js / oracle.py 共通）
1. 空 / 正規化後空 → `NO_RESULT`
2. 用件なし応答（特にありません/大丈夫です/結構です 等）→ `NO_RESULT`
3. フィラーのみ → `NO_RESULT`
4. 数字のみ → `NO_RESULT`
5. RULES 優先順位マッチ（相談→予約→大代表→定型案内・先勝ち）→ ラベル
6. いずれも非該当だが発話あり → `その他`

## 語彙（亀田 OpenAI 用件プロンプト STEP2 を決定論 regex に転記）
| ラベル | 主なキーワード |
|---|---|
| 相談 | 相談/ご相談/総合相談室/相談室/退院/退院調整/転院/ケアマネ/在宅/介護/医療相談/ソーシャルワーカー/MSW/PSW |
| 予約 | 予約/ご予約/受診予約/外来予約/予約変更/予約確認/受診/診察/外来/キャンセル/取り消し |
| 大代表 | 代表/代表番号/大代表/別の部署/他の部署/別の窓口/かけ間違/間違え/担当部署 |
| 定型案内 | 場所/駐車場/行き方/道順/アクセス/受付時間/診療時間/外来時間/面会時間/何時/営業時間/電話番号/番号 |

## `その他` と `NO_RESULT` の決定論ポリシー（要確認・調整可）
- **NO_RESULT**（再質問へ）: 無音 / 用件なし応答 / フィラーのみ / 数字のみ。
- **その他**（伝言・折返しへ）: 上記以外で、キーワード非該当の実発話。
- OpenAI 版は「無関連発話（雑談・固有名詞のみ）→ NO_RESULT」だったが、決定論版は**安全側（その他＝伝言折返し）に倒す**。雑談を機械的に NO_RESULT 判定するのは決定論では誤判定リスクが高いため。再質問ループより伝言受けの方が顧客影響が小さい。**この方針は調整可**。

## 既知の限界（実コールデータで語彙チューニング前提）
- **キーワード重複**: 「外来時間」「診察時間」は `外来`/`診察`（予約語彙）を含むため、優先順位上 **予約**に倒れうる（本来は 定型案内）。受付時間/診療時間/面会時間 は予約語彙を含まないため安全。実データで `予約` rule に時間系 exclude を足す等の調整を想定。
- キーワードは固定リスト。表記揺れ・新語は実データで追補（n_choice/department_classifier と同様の運用）。
