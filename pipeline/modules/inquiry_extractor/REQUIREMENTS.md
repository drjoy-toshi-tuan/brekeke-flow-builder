# 用件抽出 決定論パーサ（inquiry_extractor）— 仕様 v2（全決定論・OpenAI不使用）

商談デモ「フリー発話受付」シナリオ（`output/scenarios/商談デモ_フリー発話受付/`）の `用件抽出`（`type: script`）の正本。
**自由発話の STT テキストを直接パース**し、用件種別＋各スロット値＋取得有無フラグ＋復唱用の用件概要文へ
決定論的に分解する。**OpenAI は一切使わない**（自由発話の抽出そのものを script で行う）。
OpenAI が残るのは「こぼれたもの＝問い合わせ」の FAQ 対話のみ（本部品のスコープ外）。

> 設計転換（2026-06-16, ユーザー指示）: v1 の案A（OpenAI が区切り文字列を作り→script がパース）を**廃止**。
> v2 は STT 生テキストを script が直接解析する。

## 設計原則
- **高精度優先・ベストエフォート抽出**: 確信が持てる時だけ抽出する。抜けても良い（下流の個人情報順次回収で取得）。
  誤抽出（recall より precision）を避ける。STT はまとめ発話で文字起こしが安定する前提。
- **氏名は「言い方」マーカー依存**（辞書ではない）: 「私は〇〇です」「〇〇と申します」等の自己紹介語法でのみ抽出。
  第三者名（医師名「〇〇先生」等）を患者氏名として誤取得しない。
- **用件種別で予約系/問い合わせ系に二分**: 新規/変更/キャンセル のいずれにも分類できなければ `問い合わせ`（こぼれ）。

## 入出力

### 入力
- `$runner.getModuleResult("用件抽出_入力")`（受入テスト時）/ production はテンプレートが `{{INPUT_MODULE}}` の
  STT 結果を読む（用件フリー聴取の STT。文字列 or `{text:...}`）。**OpenAI を経由しない生 STT テキスト**。

### 出力1: `$runner.setResult(...)`（受入テストで照合する canonical 文字列）
`‖`(U+2016) 区切りで以下を連結（オラクルと JS が必ず同一文字列を返す）:
```
<用件種別>‖<診療科>@<flag>‖<予約希望日>@<flag>‖<予約日時>@<flag>‖<氏名>@<flag>‖<連絡先>@<flag>‖<生年月日>@<flag>‖<診察券番号>@<flag>‖SUMMARY:<用件概要文>
```
- `flag` = `取得済`（値が非空）/ `未取得`（値が空）。
- `用件種別` ∈ {新規, 変更, キャンセル, 問い合わせ}。
- `用件概要文` = script が組み立てた復唱文（§4）。

### 出力2: context への撒き出し（production 配線用・`$ivr.setObject`）
| context key | 値 |
|---|---|
| `用件種別` | 新規 / 変更 / キャンセル / 問い合わせ |
| `用件概要` | 復唱文 |
| `診療科` / `診療科_取得` | 値 / 取得済・未取得 |
| `予約希望日` / `予約希望日_取得` | 〃 |
| `予約日時` / `予約日時_取得` | 〃 |
| `氏名` / `氏名_取得` | 〃 |
| `連絡先` / `連絡先_取得` | 〃 |
| `生年月日` / `生年月日_取得` | 〃 |
| `診察券番号` / `診察券番号_取得` | 〃 |

## 判定手順（決定論・上から評価）

### 0. 正規化（共通）
- `{text:..}`→文字列化。全角数字→半角。前後空白除去。内部の解析は原文も併用（マーカー検出用）。

### 1. 用件種別（評価順＝precision優先）
1. `取消`/`取り消し`/`キャンセル`/`やめ`/`中止` を含む → **キャンセル**
2. （上記以外で）`変更`/`変えたい`/`ずらし`/`ずらす`/`振替`/`振り替え`/`日にちを変`/`時間を変`/`予定を変` → **変更**
3. （上記以外で）`新規`/`初めて`/`はじめて`/`予約`/`受診したい`/`診てもらい`/`みてもらい`/`診ていただき`/`かかりたい` → **新規**
4. いずれにも該当しない → **問い合わせ**（こぼれ→FAQ-OpenAI へ）

### 2. スロット抽出（ベストエフォート・高精度優先）
- **診療科**: 公式診療科辞書（[[department_classifier]] と同一の DEPARTMENTS）を最長一致で照合 → canonical。無ければ空。
- **連絡先（電話番号）**: 数字正規化後、`0` 始まりの **10〜11 桁連続数字** を抽出。無ければ空。
- **生年月日**: 和暦元号（昭和/平成/令和/大正）or 西暦4桁年 を含む `…年…月…日` 表現 → 抽出。年/元号が無い日付は生年月日にしない。
- **予約日／予約日時**: 年/元号を含まない日付・相対日付・曜日・時刻表現
  （`来週`/`今週`/`再来週`/`明日`/`あした`/`明後日`/`あさって`/`今日`/`きょう`/`今度`/`◯月◯日`/`◯曜(日)`/`◯時`）を抽出。
  用件種別=新規 → **予約希望日**、変更/キャンセル → **予約日時** に格納（同一抽出値）。
- **氏名**: 自己紹介語法マーカーでのみ抽出（precision優先・第三者名回避）:
  - 接頭: `私は◯◯`/`わたしは◯◯`/`私、◯◯`/`わたし、◯◯`/`名前は◯◯`/`なまえは◯◯` → `◯◯` を `です`/`と申`/句読点/末尾まで。
  - 接尾: `◯◯と申します`/`◯◯ともうします`/`◯◯と言います`/`◯◯といいます` → `◯◯` を直前の境界（句読点/空白/助詞 は・が・、/先頭）まで遡って取得。
  - 取得値から敬称（さん/様/先生）と前置語を除去。長さ上限 10 文字。`先生` を含む第三者名は対象外。
- **診察券番号**: 原文に `診察券` が含まれる場合のみ、近傍の数字列（3〜10桁、電話番号として抽出済みの桁列は除外）を抽出。
  ラベルが無ければ空（precision優先）。

### 3. 取得フラグ
- 各スロット: 正規化後が非空 → `取得済`、空 → `未取得`。

### 4. 用件概要文（復唱用・script 組み立て）
用件種別と主要スロット（空はスキップ）から自然文を組み立てる:
- 新規: `「新規のご予約、{診療科}、{予約希望日}、{氏名}様、でお間違いないですか？」`（空項目は読点ごと省略）
- 変更: `「ご予約の変更、{予約日時}、{氏名}様、でお間違いないですか？」`
- キャンセル: `「ご予約のキャンセル、{予約日時}、{氏名}様、でお間違いないですか？」`
- 問い合わせ: `「お問い合わせ、でお間違いないですか？」`（YES なら FAQ 対話へ）
> 文言の最終形は IVR プロパティ（gen_properties）側で調整可。本部品は context `用件概要` に組み立て文を入れる。

### 5. 撒き出し・canonical 生成
- §出力2 の context キーを `$ivr.setObject`、§出力1 の canonical 文字列を `$runner.setResult`。

## シナリオ配線（商談デモ フリー発話受付 v2）
```
用件フリー聴取(hearing/text・OpenAIなし) → 用件抽出(本部品・script)
  → 用件概要復唱(announcement) → 確認(YES/NO)
        NO → 用件フリー聴取 へ戻る（再聴取ループ）
        YES → 用件種別分岐(CMR, <% 用件種別 %>)
              新規   → チェック_診療科 → チェック_予約希望日 → 個人情報回収
              変更   → チェック_予約日時 → 個人情報回収
              キャンセル → チェック_予約日時 → 個人情報回収
              other(問い合わせ=こぼれ) → FAQ-OpenAI対話（問合せ対応）
  → 個人情報回収（未取得のみ順次）: 氏名→連絡先→生年月日→診察券番号 → 最終確認 → 終話
```

## パリティ（Python ↔ Brekeke JS）
- `oracle.py` と template/script を同一辞書・同一手順・同順で実装。Nashorn(ES5.1) 想定。
- `String.normalize` 不使用（全角数字→半角・限定的な記号処理のみ）。使用記号は全て BMP。

## 注意・既知の制約
- ベストエフォート抽出。抜けは下流で回収する設計のため、抽出 0 件でもフロー上は正常（再聴取/順次回収で吸収）。
- 診察券番号と連絡先の数字列衝突は、連絡先=10〜11桁・診察券=ラベル必須 で分離（precision優先）。
- 氏名抽出は STT 精度に依存。誤抽出回避を優先し recall は割り切る。
- 1 文字でも改変したら再受入（CLAUDE.md 部品ポリシー）。
