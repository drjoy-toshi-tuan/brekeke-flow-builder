# factory-v2 既定化（flip）判定基準

**方針（2026-06-23 確定）**: 工場の既定生成経路を v1（director が設計書を著述）から
v2（drawio 入口 → emitter で設計書 YAML 合成 → 決定論部品で判定 script を生成）へ切り替える
判断を、感覚ではなく **満たすべきゲートの集合**で行う。本書はそのゲートを定義する。

関連: PR #209（factory-v2 リストラ本体）/ `deterministic-replacement-roadmap.md`（部品置換の進め方）/
`part-certification-spec.md`（engine/spec 二段認定）

## flip とは何か（定義）

- **flip = 新規シナリオ生成・既存再生成の "既定" を v2 経路にすること**。orchestrator / `start_ui.py`
  の既定パスを director(v1) から drawio-emitter(v2) に切り替える。
- flip は「既定の切替」であって「v1 の削除」ではない。**v1 経路は施設単位で opt-out（fallback）として温存**する
  （ゲート6）。flip 後も v1 で作った既存施設は再生成しない限り影響を受けない。

## なぜ基準を先に固定するか

- v2 は決定論率がまだ **57.2%（det 123/215・v2 形式 21 施設実測, PR #209 時点）** で、#2 出口（判定 script の
  自動生成）は yes_no と診療科 B案までしか組まれていない。**未完のまま既定化すると、生成のたびに人手の
  穴埋めが発生して "自動生成" の体を成さない**。
- 一方で「100% 決定論」を待つと永久に flip できない（`block(none)` には施設都合で残る判定点がある）。
  → **"既定化して安全な最低ライン" を明文化**し、そこに到達したら flip する。

## flip ゲート（全て満たしたら既定化可）

| # | ゲート | 合格条件 | 現状（2026-06-23） |
|---|---|---|---|
| 1 | **#2 出口3経路の完備** | yes_no / 診療科 / DOB / 日付 の判定 script が build_script で emit 可能 | yes_no ✅ ・診療科 B案統合済(d5821c6) ✅ ／ **残=DOB(property 駆動ビルダー)・日付(reservation_date_classifier テンプレ化)** |
| 2 | **決定論率の閾値到達** | v2 形式の代表施設群で det 率 ≥ 目標値、かつ `block(none)` 上位施設の未対応点を解消 | 57.2%。`block(none)` 上位 = 松本協立 9 / 沖縄県立南部 7 / 諏訪 6 / Medcity21 6 … が未処理 |
| 3 | **回帰ゼロ（非破壊）** | 既存認定施設（亀田 / 商談 / 厚木）が v2 経路で byte-identical 再生成 | **A3 で実証済 ＝ 充足**（生成挙動不変を `resolve_hearing_backend` surfacing 化で担保） |
| 4 | **入口被覆** | 代表的な drawio パターンが emitter で **qa_validator clean**（CRITICAL 0）に通過 | emitter が必須12セクションを工場デフォルト合成（入口ゲート clean / commit 10e8d04） |
| 5 | **実機 P7 グリーン 1 施設** | 少なくとも 1 施設を v2 フル経路（drawio→生成→P6→P7 実機）でグリーン | 未実施（施設別に別途） |
| 6 | **ロールバック温存** | flip は既定切替のみ。施設単位で v1 経路に opt-out できる手段が用意されている | 設計要（orchestrator / start_ui の経路選択フラグ） |

> **ボトルネックは 1（残 DOB/日付）と 5（フル経路実機 1 本）**。3 は充足、4 はほぼ充足。
> 2 は目標値の合意が必要（暫定 ≥ 85%、ただし施設都合で残る `block(none)` は除外して評価する）。

### ゲート2 の det 率について（評価の注意）

- `block(none)` には「その施設に固有で決定論化しても薄い判定点」が含まれる。これを分母に入れたまま
  100% を目指すのは不経済。**評価は `det / (det + 決定論化すべき残点)` で行い、施設都合の `block(none)` は
  別枠で報告**する（隠れた取りこぼしにしないため log で明示）。
- 閾値の最終値は flip 直前に実測を見て確定する。

## flip の実行（ゲート充足後）

1. orchestrator / `start_ui.py` の既定経路を drawio-emitter(v2) に切替（v1 は opt-out フラグで残す）。
2. README / CLAUDE.md のパイプライン全体像を v2 既定で記述し直す。
3. 既存施設は再生成時のみ v2 に乗る（一斉移行はしない）。
