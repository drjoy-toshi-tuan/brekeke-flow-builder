### 移管方針
- Gen1「箱」→ Gen3 9 ブロック型へ変換。opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は移転告知文を含む長文 1 本: 「お電話ありがとうございます。ユアクリニック秋葉原のAI電話です。当院は3月31日をもちまして移転となり、診療は終了しております。5月からは内科はお茶の水駅前生活習慣病クリニック、小児科はユアクリニックお茶の水での診療となります。詳しくはユアクリニック秋葉原ホームページの移転情報をご覧ください。尚、お問い合わせの在る方は、このあとの3つの質問にお答えください」。
- **用件分岐なし**（全件＝問い合わせ預かりのみ）。OpenAI 用件分類 / 診療科聴取 / 予約日聴取 / 診察券聴取 / 生年月日聴取は無し。
- 主流れ: opening + 冒頭_アナウンス（移転告知一括）→ 入電時間判断（時間外なら時間外アナを冒頭に被せて 3 質問へ進行 / 時間内かつ「院内転送」相当時間帯は API_DRJOY → 院内代表転送 → END_切断）→ Q1: 問い合わせ内容聴取（自由テキスト）→ Q2: 患者名聴取（PatientName 1 枠）→ Q3: 連絡先聴取 → 終話「ありがとうございます。質問は以上です。内容確認後、院内スタッフから土・日、祝日を除く二日以内にショートメッセージもしくはお電話にてご連絡いたします」。
- **時間判断は date_of_call_classifier ブロック**（入電時間判断.yaml + 年末年始設定.yaml の 2 つ。addCurrentDate: Yes 自動付与）。**時間内であっても院内代表転送（人スタッフへブラインド転送）に直結する時間帯あり**（Brekeke `Blind Transfer` モジュール必須）。
- 3 質問すべて「3 回聞き取り失敗救済 時間判別」サブシナリオへフォールバック → 院内転送：聞取り失敗（「申し訳ございません、うまく認識ができませんでした。代表電話に転送いたしますのでそのままお待ちください」）→ Blind Transfer。

### 施設固有の特殊ルール
- **受付時間**: **3/31 移転完了済み・4 月以降は診療終了**。AI 電話側は移転告知 + 問い合わせ預かりのみ稼働。具体時間帯は入電時間判断.yaml 参照（平日 8:30〜19:00 を AI/代表転送で細かく分岐: 8:30 前=時間外 / 8:30〜11:00=代表転送 / 11:00〜15:00=AI / 15:00〜17:00=代表転送 / 17:00〜19:00=AI、土曜は 8:30 前=時間外 / 8:30〜11:00=AI / 11:00〜16:00=代表転送、木曜（8 月中）・祝日は 8:30 前=時間外 / 8:30〜11:00=AI / 11:00〜13:00=代表転送）。**この時間判定は移転前の旧運用残骸**で、移管時に簡略化検討要（director ノート対応 + マネージャー確認推奨）。
- **代表電話番号**: 元 yaml に明示なし（`院内代表転送` サブシナリオが Brekeke ブラインド転送モジュールを呼ぶ前提）。Gen3 では Call Transfer ブロックの `transfer_destination` を properties で別途設定（director は記述せず、マネージャー入力枠を残す）。
- **時間外案内 TTS**: 「本日の診療は終了いたしました。救急の方は東京都医療機関案内サービス、ひまわり03-5272-0303におかけください。当院へお問い合わせの方は、この後にご用件をお伺いいたします。内容確認後、院内スタッフから翌営業日終了までにショートメッセージもしくはお電話にてご連絡いたします」（救急＝ひまわり 03-5272-0303 を案内）。
- **混み合い時 TTS**: Blind Transfer 失敗時「只今お電話が込み合っております。恐れ入りますが、お時間を空けて再度おかけなおしください」→ 終話。
- **3 つの質問のつなぎ TTS**: Q1 = 「まずは1つ目の質問です」/ Q2 = 「かしこまりました。続いて、二つ目の質問です」/ Q3 = 「それでは、最後の質問です。ご連絡先のお電話番号をお伺いします」（**Q2 は入力待ち=あり** で間を空ける、Q3 は連続）。
- **問い合わせ内容聴取**: フリーテキスト 1 本（output_format=text）。「発話あり/なし」「ご用件あり/なし」「5 秒無音」の 3 種で聴取内容チェック → 3 回失敗で END_戻る：聞取りできず（「うまく聞き取りができなかったため、後ほど担当者にて確認をいたします」）。
- **DTMF なし**（全 hearing 発話のみ）。

### director が見落としやすいポイント
- 冒頭 4 箱（API_DRJOY_CONTEXT_SETTINGS / API_DRJOY / 挨拶 / Q1: 問い合わせ内容聴取）は Gen3 で opening + 冒頭_アナウンス（移転告知の長文 TTS 一本化）+ date_of_call_classifier に整理。**API_DRJOY が挨拶の前にある**（他施設は挨拶 → API_DRJOY が多い）が、Gen3 ではどちらでも動くため scaffold デフォルト順を採用してよい。
- セリフ none 振り分けのみの「箱」（完了フラグ / 完了フラグ_1 / 通話完了 / API_DRJOY 系 / 入電時間判断 / 年末年始設定）は独立ブロック不要 → 前段の next 書き換えに吸収。
- 「Q2: 二つ目の質問へ」「Q3: 連絡先へ」のつなぎ TTS は **次段サブフローの冒頭 hearing TTS にマージ**。独立 announcement ブロック禁止。
- **用件分類なし**: 移転終了告知 + 問い合わせ預かり 1 本。湘南藤沢/八尾/福岡徳洲会/井上健診型の用件 3〜5 分岐構造をコピペで持ち込まないこと。
- **PatientName 1 枠**: 患者本人 = 入電者前提（subject="patient"、受診者氏名/入電者氏名の二段聴取は無し）。
- **時間判断は移転前残骸**: 元 yaml の 平日 8:30〜19:00 細分化はもはや実運用と乖離。「終日・全件 AI 受付（移転告知 + 問い合わせ預かり）」に簡略化するか、現状維持するかは **マネージャー確認必須**（director は元 yaml そのまま記述し、簡略化提案を移管ノート備考に残す）。
- **Blind Transfer ブロック必須**: 院内代表転送（時間内）/ 院内転送：聞取り失敗（3 回失敗時）/ END_院内代表転送 はすべて `call_transfer` ブロック。Gen3 標準モジュール。`transfer_destination` プロパティは properties 側でマネージャーが入力。
- **「3 回聞き取り失敗救済 時間判別」サブシナリオの位置付け**: 失敗時に時間判定をやり直して院内転送可能か再判断する gating 機構。Gen3 では date_of_call_classifier の context を使い回せばサブフロー化不要（director 判断で簡略化）。
- 連絡先聴取.yaml の電話番号区切り `市外局番5桁/4桁/3桁/2桁` は Gen3 phonetype subflow で吸収 → ノート不要。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**（全角コロン `：` 全角チルダ `〜` 半角小文字 `Q1` 等の混在禁止、`END_AI ※時間外アナ有` の `※` も除去）。
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**。CMR conditions 末尾は必ず `match: "other"` + next 明示。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。Gen1 動的タグ `<? $... ?>` → Gen3 `<% var %>`（本フローは静的 TTS のみ）。
