## Gen2→Gen3 移管ノート

### フロー概要

- n8n なし。標準的な Gen2 診療フロー（予約変更・キャンセル・確認の3用件のみ受付）。
- **新規予約は受け付けない**: 「予約をとりたい」等の発話を検出したら代表案内アナウンスを出力して終話（status=2, smsFlag=0）。当院は初診予約を行わず、紹介状持参＋直接来院が必要な運用。
- **受付制限**: 2診療日以降のご予約に関する問合せのみ。CT・MRI・内視鏡・エコー等の予約変更は不可（冒頭アナウンスで案内するが、Gen2 ではチェックなし）。
  - **移管方針**: director 判断。時間チェックが必要なら Date of Call Classifier を追加、不要なら冒頭アナウンスの案内のみで運用。
- 用件分類: **変更 / キャンセル / 予約確認**（3用件のみ、新規・再診なし）。
- 聴取順序が特徴的: **共通の個人情報（氏名→生年月日→診察券番号→診療科→診療科2）を用件聴取より先に聴取**してから用件分岐。斗南病院等の「用件→共通項目」型とは順序が逆。
- 類義語登録: 1件、診療科類義語 1件（元資料の更新履歴より）。

### 冒頭アナウンス

「お電話ありがとうございます。 福岡市民病院の、予約専用AI電話です。こちらのお電話では、2診療日以降のご予約に関するお問合せのみ受け付けております。なお、CT・MRI・内視鏡・エコー等のご予約の変更はできません。お手元に診察券をご準備の上、お話しください。」

※Gen2 は冒頭アナウンス直後にユーザー回答を求めず、即座に氏名聴取へ進む仕様。Gen3 の opening ブロックは `input_wait: false` で構成し、announcement→氏名 hearing に直結させる。

### フロー構造

```
冒頭アナウンス（status=0 / smsFlag=1 セット）
  → 氏名聴取（復唱禁止・リトライ1）
  → 生年月日聴取（復唱禁止・リトライ1）
  → 診察券番号聴取(リトライ1)
      → 診察券番号復唱（はい／いいえ）
          └─ いいえ → 診察券番号_否定再聴取 → 診察券番号復唱（ループ1回まで）
  → 診療科聴取(復唱禁止・リトライ1)
  → 診療科2聴取（復唱禁止・「ない」→「なし」格納・リトライ1）
  → 用件聴取（classification: 変更／キャンセル／予約確認）
      │   ※「予約をとりたい」等を含む → END_代表案内_予約新規（status=2 / smsFlag=0）
      → 用件復唱（はい／いいえ）
      │   └─ いいえ → 用件_否定再聴取 → 用件復唱（1回まで）
      └─ 用件分岐
          ├── 変更    → 予約日聴取 → 変更希望日聴取 → 電話番号判定
          ├── キャンセル → 予約日聴取 → 電話番号判定
          └── 予約確認 → 内容確認（inquiry）→ 電話番号判定
  → 電話番号判定（incoming_phone_numberで分岐）
      ├── 携帯（050/060/070/080/090始まり） → 携帯番号確認（はい／いいえ）
      │       ├─ はい → 終話判定
      │       └─ いいえ → 電話番号_否定・非通知用聴取 → 電話番号復唱
      └── 非通知／固定電話／空白 → 電話番号直接聴取 → 電話番号復唱
              └─ 電話番号復唱（はい／いいえ）
                  └─ いいえ → 電話番号_否定再聴取 → 電話番号復唱（1回まで）
  → 終話判定（classification で分岐）
      ├── 変更    → END_終話_変更    （status=1 / smsFlag=2）
      ├── キャンセル → END_終話_キャンセル（status=1 / smsFlag=3）
      └── 予約確認 → END_終話_予約確認（status=1 / smsFlag=4）
```

### 特記事項

#### 1. 1回聴取禁止 / 再ヒアリング禁止（Gen2 の最重要ルール）

- 一度聴取した内容（氏名・問い合わせ・担当医・紹介元医療機関等、**全ての情報**）は、いかなる理由があっても再度聞き直さない。不明確でもそのまま記録し次の質問に進む。
- Gen3 の generate_by_OpenAI モジュール（Eテンプレ自由テキスト）のプロンプトに**必ず埋め込む**。

#### 2. 復唱ルール（項目別で厳格に分岐）

**復唱必須（3項目）**: 用件 / 診察券番号 / 連絡先電話番号
- 復唱後肯定 → 確定して次へ。
- 復唱後否定 → 各項目専用の「復唱後否定用の再質問フォーマット」で1回だけ再聴取。失敗時は「復唱後否定の再聴取失敗」アナウンスを出力し次の聴取ステップへ。

**復唱禁止（4項目）**: 氏名 / 生年月日 / 診療科 / 診療科2
- 「○○ですね」「○○と承りました」「○○で承知しました」等の**再提示アナウンスは出力禁止**。
- 回答認識直後に次の聴取質問を即開始。Gen3 ブロック生成で echo_back セクションを**生成しない**ように scenario_flow で明示（scaffold が自動的に echo_back を挿入しないよう、YAML で `echo_back: false` を明記）。

#### 3. 再聴取（リトライ）ルール

- 全項目で**最大1回**のリトライ。2回目失敗で「聴取失敗」と判断。
- 失敗時アナウンス（固定文、一言一句厳守）:
  「申し訳ございません。ご回答内容が確認できませんでした。担当より折り返し連絡のうえ、再確認いたします。次の内容をお伺いします。」
- 再質問アナウンス A（通常無効回答用）: 「ご回答内容が確認できませんでした。もう一度お話しください。質問を繰り返します。＜直前質問＞」
- 再質問アナウンス B（復唱後否定→再聴取→無効回答用）: 用件／診察券番号／連絡先電話番号ごとに固定文あり。
- 再質問アナウンス C（ユーザーから「もう一度」要求）: 「質問を繰り返します。＜直前質問＞」

#### 4. 数字・電話番号・診察券番号の正規化

- 診察券番号: 3桁以上の数字必須、「C」→「4」、「ハイフン」「はいふん」「の」「iPhone」「AiPhone」等を検出したら「-」に変換。数字と「-」以外は再聴取。
- 電話番号: 10桁or11桁以外は再聴取。11桁は 090/080/070/050 のいずれかで始まる必要あり。9～10桁で0欠落なら先頭に「0」補完。
- 氏名: カタカナ完全統一（STTの漢字誤変換は同音カタカナへ強制補正）。
- 「4」「５」の誤変換: 「よん」「し」→4、「ご」「ごお」→5。「for」「C」「号」「go」は数字変換しない。

#### 5. 和暦→西暦換算（生年月日）

- 大正=1911+年数 / 昭和=1925+年数 / 平成=1988+年数 / 令和=2018+年数。
- YYYY年MM月DD日形式で patientDateOfBirth に格納。

#### 6. incoming_phone_number 分岐（電話番号聴取の出し分け）

- 携帯（090/080/070/060/050開始）→ 「電話番号（携帯電話番号）」アナウンス出力し、肯定なら Incoming_phone_number を additionalPhoneNumber に格納。否定なら「電話番号（否定の場合・非通知の場合）」で再聴取。
- 非通知 / anonymous / 空白 / 上記以外の0始まり（固定電話）→ 直接「電話番号（否定の場合・非通知の場合）」で聴取。
- **ユーザーが肯定するまで additionalPhoneNumber は空（status=0保持）**。

#### 7. 「予約をとりたい」系発話の検出→代表案内

- 用件聴取時に「よやく」「よやくしたい」「よやくをとる」「よやくをとりたい」「やく」「やくです」「やくいれたい」「やっしたい」「やっとりたい」「やくおねがい」等を検出 → END_代表案内_予約新規 出力→切断。status=2, smsFlag=0。

#### 8. 日付扱い

- 予約日は「言われた日程＋当日の年度」を足して YYYY-MM-DD 00:00:00 で登録。
- 10月以降の入電で1月～9月の日付を言われた場合は翌年に繰り越し。
- **予約日と予約希望日は別項目**。予約日が未聴取のまま予約希望日の値を予約日に流用してはいけない。

#### 9. SSML → {tts_g:...} 小文字形式への変換

Gen2 資料中の SSML タグは Gen3 では一切使用しない。
- `<speak type="digits" breakc="300ms">{}</speak>` → `{tts_g:<診察券番号読み上げ>}` に置換（桁区切り読みは properties 側で吸収）。
- `<speak type="telephone" breakc="300ms">{}</speak>` → `{tts_g:<電話番号読み上げ>}`。
- `<dtmf/>` → DTMF入力はIVR側のモジュール設定で実現。TTSには埋めない。
- `<break time="Xms"/>` → 使用せず、文中の句読点／改行で代替。

#### 11. 口調・禁止語

- すべて丁寧語。「かしこまりました」可。
- 「ありがとうございます。」「お気軽に」は禁止。
- 「おっしゃってください」ではなく「お話ください」に統一。
- プロンプト（FAQ）外の質問が来た場合:
  - 直前が「お問い合わせ」系 → 1回目「うまく聞き取りができませんでした。もう一度お話しください。」／2回目以降「申し訳ございません。こちらのお電話で回答ができませんので別途ご連絡させていただきます。」
  - それ以外 → 「ご回答いただいた内容を確認できませんでした。質問を繰り返します。」＋直前質問を復唱。

#### 12. ターン制御ルール

- 質問出力直後に「聞こえませんでした」等のエラー案内を自発的に出さない。
- ユーザー発話終了は1.5秒の無音継続で判定（発話途中での打ち切り禁止）。
- 無関係語句を拾っても勝手に別トピックへ遷移しない。
