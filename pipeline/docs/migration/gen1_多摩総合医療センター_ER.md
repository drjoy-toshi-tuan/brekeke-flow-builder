### 移管方針
- **入電日時間判断ノード／時間内と3月18日以降の判断**: Gen1 では 3/18 移行アナウンス切替の暫定分岐が残っているが、移管時点では役目を終えているため **acceptance_times に集約（Dr.JOY 側責任）**。`イーアール専用AI電話変更のお知らせ` 箱は Gen3 では作らない。
- **冒頭 → 確認質問 → 3条件 Yes/No → スタッフ転送 or 受診案内**の 1 本フロー。3条件確認(タイトル) → Yes は転送、No は受診案内パスへ進み、最終的に「来院案内」で終話。
- 転送はスタッフ転送（Blind）1 系統のみ。`転送①` と `転送②` の 2 サブシナリオがあるが、Gen1 上の使い分けは便宜的なもので、Gen3 では call_transfer 1 ブロックに統合。
- `回答の分岐_0/1/2/3` は Yes/No + 「もう一度」復唱の繰り返しを場面ごとにコピーしただけ。**hearing 1 ブロック + echo_back（復唱あり）+ retry_count=2 で統合**。

### 施設固有の特殊ルール
- **3 条件確認質問（タイトル箱）**: 「一ヶ月以内に当院を退院した／一ヶ月以内に当院で手術を受けた／外来化学療法室に通院中」のいずれかに該当するかを Yes/No で聴取。**該当（Yes）→ スタッフ転送**、**非該当（No）→ 受診案内 → 受診希望 Yes/No → 来院案内 or 他院誘導**。Yes/No の判定は OpenAI 出力（肯定/否定）で行い、「もう一回」発話のときだけ直前 TTS を再生（Pattern A）。
- **救急受診の安全アナウンス**: 冒頭で「大けが・意識不明・大量出血・激しい頭痛・強い胸の痛み・呼吸困難・手足の麻痺」の場合は通話を切って 119 を案内するという長い注意喚起 TTS が必須。announcement ブロックで保持。
- **緊急性低の受診案内**: 「待ち時間 3 時間以上／医師指定不可／眼科・歯科・泌尿器科は対応困難／妊娠関連はかかりつけへ」の説明 TTS も必須。短縮せず原文維持。
- **No/受診せず終話パス**: 「シャープ7-1-1-9（#7119）」の案内が含まれる。読み上げ表記は `<%='♯7119'%>` ではなく **「シャープ ななひゃくいちじゅうきゅう」** などスムーズに読める形に整える（Gen1 は「シャープ7、1、1、9」表記）。
- **Yes 後来院案内の固定文言**: 「保険証かマイナンバーカード、および、お薬手帳をお持ちの上、気をつけて来院ください」を必ず流してから終話。
- **代表番号フォールバック**: 聞き取り失敗 2 回 → 「代表 042-323-5111 にお掛け直しください」で終話（hearing.retry_count=2 のデフォルト失敗 TTS をこの文言に上書き）。

### director が見落としやすいポイント
- **ER シナリオに名前/DOB/電話/診察券のヒアリングは無い**。PatientName/DOB/Phone/PatientID 系サブフローは **使わない**。冒頭の 3 条件 Yes/No → 転送/受診案内のみ。
- **「もう一度」エンティティ**は全 4 箇所の Yes/No hearing で「直前 TTS を再生」として echo_back 復唱で吸収可能。専用の repeat 分岐ブロックは作らない。
- **タイトル_5（さらにご質問がありますか？）→ Yes → タイトル_5 自身に戻る**ループが Gen1 にある。Gen3 では retry_count を超えたら終話に流すのが安全（無限ループ禁止）。回答の分岐_1 の Yes 先「タイトル_5」は **「以下、受診案内…」を再度流す（タイトル_1 へ戻す）** が本来意図と推察される（director 確認推奨）が、安全側に倒すなら **質問回数 2 回上限で終話_1（お電話ありがとうございました）**。
- **STT 辞書**: hearing_yesno_common と echo_back_yesno のテンプレ流用 + カナバリアント 7 件のみ。施設固有の固有名詞は無い。
- **OpenAI 正規化**: 「異常部／丈夫／大丈夫」など STT 誤認識を肯定に正規化、「当てはまらない」を否定に正規化する synonym パッチが必要。entity_openai_synonyms.md を直接参照。
- **エンドメッセージは 2 系統**: (a) Yes 転送系の前置き「スタッフにおつなぎいたします。診察券をご用意の上…」(b) No 終話系「お電話ありがとうございました。それでは失礼いたします」。混在しないよう termination 分岐を分ける。
