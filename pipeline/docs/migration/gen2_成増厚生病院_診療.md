### 移管方針
- incoming-classifier は phone_type 4分岐（非通知／050=IP／070/080/090=携帯／それ以外の0始まり=固定）。catch-all 無し。
- 冒頭アナウンス後、用件確認（dtmf=1〜4＋「今日」→代表案内）で **Pattern A（OpenAI直後分岐）** を採用。分岐先は「診察予約／変更／キャンセル／確認／代表案内（当日予約）」の5系統。
- 診察予約（用件=1）の直後に受診歴分岐（初診/再診）→ 診療科聴取（dtmf=1〜6）で **二段分岐**。ここも Pattern A が自然。診療科分岐で「アルコール＝dtmf=1」は即アルコール病棟案内終話（checkpoint=7）、「児童思春期＝dtmf=4」は受診歴との組み合わせ判定（初診→HP案内終話、再診→通常聴取）なので、ここは **Pattern B（ContextMatchRouter で classification×診療科の組合せ判定）** が適切。
- 氏名／生年月日／電話番号／診察券番号はサブフロー化必須（インライン展開禁止）。RAG は本シナリオでは不要（FAQ聴取が無く、FAQ外は定型返しのみ）。
- Date of Call Classifier は不要（営業時間外分岐なし・「今日」判定はOpenAI側の文字列マッチで処理）。
- 用件確認の3回連続判定不能→進行不可終話の再試行カウンタは script ブロック＋ContextMatchRouter で director 判断。
- 電話番号聴取は incoming_phone_number が取得できている場合は「この番号でよいか？」の Yes/No 分岐あり → 電話番号サブフロー側で取り込む（director 判断）。

### 施設固有の特殊ルール
- **「肺」→「はい」救済**、**「いいです」→肯定** の Yes/No 救済キーワードあり（プロンプト【5.】）。SKILL_B 系 Yes_No プロンプトに追記必須。
- **当日予約は必ず代表案内（03-3939-1191）終話**。トリガーワード: 「今日の予約」「本日の予約」「きょう」「ほんじつ」「今日」「本日」（プロンプト【5.】）。代表案内の直後に checkpoint='2', smsFlag='0'。
- **アルコール相談（診療科=dtmf=1）は即アルコール病棟案内終話**（050-1725-6865）。checkpoint='7', smsFlag='0'。紹介状以降の聴取には絶対に進まない。
- **児童思春期外来＋初診 → HP案内終話**。checkpoint='2', smsFlag='0'。再診の場合は通常聴取を続行。
- 診療科選択キー固定表: dtmf=1→アルコール／2→精神科／3→内科／4→児童思春期／5→老年期こころの外来／6→整形外科。音声「さん」「み」などの数字キーワード検出時もこの表に従う。
- 診療科リストに無い科（例: 消化器科）は「当院に○○はございませんが、○○でしょうか？」とレコメンド返答。
- 用件判定不能が **3回連続** で進行不可終話（最大2回まで再質問）。checkpoint='0'、endpoint='電話案内' 相当。
- 診察券番号は **5桁以内**（他施設と桁数違い）。「わからない」発話時は無音で次へ、補足発話禁止。
- checkpoint enum: **0／1／2／7**（通常・代表案内・アルコール病棟の3終話で値が変わる）。status enum も 0/1/2/7。smsFlag: キャンセル=2、他受付完了=1、案内系=0。
- 「診察」誤認救済: 「印刷／定期／新薬／再診／最新／禁忌」→診察、「薬／お薬」→予約、「電気／でんき」→新規、「ぴか／ちか／しか」→歯科（→代表案内）。

### director が見落としやすいポイント
- **冒頭アナウンスで歯科・発熱外来・当日予約は代表案内誘導**: 「本日のご予約と歯科受診、発熱外来に関するお電話は代表電話におかけ直しください。」（customer_doc L13）。冒頭TTSに必須。
- **「用件」と「診療科」のみ復唱必須、他は復唱禁止**（L116, 124）。氏名サブフローで復唱を入れないこと。電話番号・診察券番号は専用フォーマット `<speak type="telephone">`／`<speak type="digits">` → Gen3 では SSML 禁止なのでプロパティ側で同等表現に置換（director は TTS 形式を `{tts_g:...}` で記述）。
- **生年月日は初回＋聞き直し2回の計3回上限**（L493-497, 999-1004）。3回目以降は無言で次へ。DOB サブフロー側の Retry 回数制限で対応。
- **予約日と予約希望日は別フィールド**（L370-376）。予約希望日は文字列フリーワード、予約日は YYYY-MM-DD。10月以降に1〜9月が言われたら翌年扱い。
- **入電者氏名（代理時のみ取得）** は「本人確認=いいえ」の分岐で agentcheck='代理' とともに入電者氏名を聴取（L417, 920）。本人の場合はスキップ。
- **改訂履歴 2026/01/20**: 「入電者側の終話トリガー制御」追加（L94-96）。情報聴取中に入電者発話を理由とした終話遷移禁止 → prompter の SKILL_C 系に明示的なハードルール追記必須。
- **改訂履歴 2025/12/19**: 診療科分岐追加・SMSflag制御function追加（L96）。旧版（L601〜）には無い処理なので必ず最新版を採用。
