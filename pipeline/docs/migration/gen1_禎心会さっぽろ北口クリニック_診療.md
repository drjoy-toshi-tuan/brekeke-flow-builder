### 移管方針
- Gen1「箱」構造 + サブシナリオ 14 本（用件聞き取り / 患者氏名聞き取り / 生年月日聞き取り / 連絡先電話番号聞き取り / 電話番号区切り / 確認内容の聞き取り / 予約日（フリー聴取）/ 予約希望日 / 紹介状有無聞き取り / 症状ヒアリング / 追加要望 / 診療科 / 診療科フリー聴取 / 時間確認）→ Gen3 scenario_flow 9 ブロック型へ変換
- 主な対応:
  - `API_DRJOY_CONTEXT_SETTINGS / API_DRJOY / 挨拶` → opening + 冒頭_アナウンス。冒頭文言「お電話ありがとうございます。禎心会さっぽろ北口クリニックの、予約専用電話です。」
  - `時間確認` サブシナリオ → opening 直後の acceptance_times（曜日 + 時刻）+ Script（祝日 / 年末年始判定）。**営業時間が曜日で 2 種類**（月火水金 8:25〜17:00 / 木土 8:25〜12:30）→ acceptance_times の曜日別 windows 定義必須。日祝・年末年始休
  - `用件聞き取り.yaml` → hearing（generate_by_OpenAI、4 分岐: 予約変更 / 新規予約 / 予約キャンセル / 予約確認）+ DTMF 1〜4 並列。直後 echo_back 4 本 → Pattern A
  - `用件を判断` `_1` `_2` `_3` の 4 連鎖 → Gen3 では context_match_router 1〜2 枚に圧縮（`完了フラグ` で context 確定 → 1 段目で「予約確認」だけ 確認内容の聞き取り、それ以外は氏名聴取_1 → 生年月日_1 → 用件を判断_3 で「予約確認 / 予約変更 / キャンセル」を再分岐 → 各 announcement 終話 / 「新規予約」だけ予約案内 → 診療科 → 用件を判断_1 で「予約変更 / キャンセル / 新規予約」分岐 → 予約日 or 症状ヒアリング → 連絡先 → 締め）
  - `診療科 / 診療科フリー聴取` → Gen3 では診療科 enum hearing + フォールバック自由文 hearing の 2 段構成（diagnosis_dept パターン）。**「健診・ドック」または「ワクチン接種」が選ばれた場合のみ症状ヒアリングをスキップ**（main.yaml `健診ドックワクチン接種で症状聞かない。` 分岐より）
  - `予約日（フリー聴取）/ 予約希望日` → datetime hearing。新規予約は予約希望日（コース確定後）、変更/キャンセル/確認は予約日（フリー聴取）。両方とも addCurrentDate: Yes
  - `紹介状有無聞き取り` → 新規予約ルートのみ Yes/No hearing（ただし「健診・ドック / ワクチン接種」は通らない）
  - `症状ヒアリング / 追加要望` → 新規予約ルートのみ自由文 hearing 2 段（output_format: text）
  - `患者氏名聞き取り / 生年月日聞き取り / 連絡先電話番号聞き取り / 電話番号区切り` → subflow ブロック Gen3 標準サンプル JSON 完コピ
  - `確認内容の聞き取り` → 予約確認ルートのみ自由文 hearing
  - `携帯電話番号チェック` → 連絡先 subflow 戻り値 phonetype を context_match_router で分岐。携帯あり → END_新規予約の流れ_1（電話 or SMS 案内）、無し → END_新規予約の流れ（電話のみ）
- 用件別の合流先（main.yaml の連鎖 `用件を判断` `_1` `_2` `_3` を整理）:
  - **新規予約**: 予約案内 announcement → 診療科 → 用件を判断_1（新規 → 健診ドックワクチン分岐）→ 健診ドックワクチン除外 → 紹介状有無 → 症状ヒアリング → 追加要望 → 患者氏名 → 生年月日 → 連絡先 → 携帯電話番号チェック → END_新規予約の流れ(_1)
  - **予約変更**: 患者氏名 → 生年月日 → 用件を判断_3 → 新規予約・予約変更・キャンセル・再診予約へ → 診療科 → 用件を判断_1（変更 → 次へ → 予約日 → 用件を判断_2 → 予約希望日 → 連絡先）→ 携帯チェック → 初診再診変更の流れ announcement → 終話
  - **予約キャンセル**: 患者氏名 → 生年月日 → 用件を判断_3 → 診療科 → 用件を判断_1（キャンセル → 予約日（フリー聴取）→ 用件を判断_2（キャンセル → 連絡先））→ 予約キャンセルの流れ announcement → 終話
  - **予約確認**: 患者氏名_1 → 生年月日_1 → 用件を判断_3（確認 → 確認内容の聞き取り）→ 連絡先 → 予約確認の流れ announcement → 終話

### 施設固有の特殊ルール
- **代表電話番号**: 011-709-1131（聴取失敗 / 時間外時の案内、`代表案内` 箱、2 回読み上げ）。札幌禎心会病院（本院 011-712-1131）とは異なる
- **本院誘導 announcement 2 種**: `禎心会病院案内へ`（「当クリニックでの診療は終了となりました。札幌禎心会病院へお問い合わせください。電話番号は050-1722-2947です」2 回読み上げ）と `禎心会病院案内へ_1`（「当クリニックでは診療を行っておりません。札幌禎心会病院へお問い合わせください…」）。**main 上は到達経路が見当たらない** → director判断: 「特定診療科は本院対応」相当の分岐があったテスト残骸の可能性。customer_doc を再確認し、有効ルートが無ければ削除可
- **「健診・ドック / ワクチン接種」は症状ヒアリング・紹介状確認をスキップ**: 新規予約 + 診療科分岐で `健診・ドック` または `ワクチン接種` 一致 → 追加要望へ直行（main.yaml `健診ドックワクチン接種で症状聞かない。`）
- **完了フラグが 2 種**: `完了フラグ` と `完了フラグ：健診・ドック` の 2 つあり、診療科判別後どちらに振り分けるかで Dr.JOY 側のフラグが変わる → save2db の context 値を 2 種使い分け
- **当日予約は対応不可**: `当日は代表へ` 箱（「当日のご予約に関しては、こちらのAI電話ではお受けできません。代表電話までおかけなおしください」）。札幌本院と同パターンだが、main 上どこから呼ばれるか経路が不明確 → director判断: 予約希望日 datetime hearing 直後に「本日かどうか」分岐を 1 段挟む or サブフロー側で吸収
- **3 診療日以内折返し SMS / 電話**: 携帯あり → 電話または SMS、固定 → 電話のみ。文言は「3診療日以内に、担当者から折り返し電話、もしくは、SMSにてご連絡いたします」（_1）/ 「3診療日以内に、担当者から折り返し、電話にてご連絡いたします」
- **氏名サブフロー 2 回参照**: `患者氏名聞き取り`（用件分岐後直接）と `患者氏名聞き取り_1`（予約確認 / 変更 / キャンセル後 用件を判断 後）の 2 系統あり。**Gen3 では subflow ブロックを 1 つだけ定義し、両参照を 1 ブロックに集約**（PatientName 1 枠原則）

### director が見落としやすいポイント
- 冒頭の `API_DRJOY_CONTEXT_SETTINGS` / `API_DRJOY` は Gen3 標準処理に吸収 → 明示ブロック化不要
- 「次へ」/「次へ_1〜_4」（「では次に、」「続いて、」「それでは、」）は短い繋ぎ TTS → 次段ブロック先頭 TTS にマージ
- セリフ none 振り分けのみの箱（`用件の確認` / `用件を判断` / `用件を判断_1〜_3` / `携帯電話番号チェック` / `診療科判別` / `健診ドックワクチン接種で症状聞かない。` / `時間外接続用` / `時間内案内` / `切断へ` / `完了フラグ` / `完了フラグ：健診・ドック`）は独立ブロック不要 → CMR or save2db に吸収
- `タイトル` 箱（「ご希望をお伺いしますので、この後の質問にお答えください。」）は時間内案内 → 用件聞き取り の繋ぎ → opening + 冒頭_アナウンスの 2 ブロック目に取り込む
- `営業前` 箱（「4月1日の9時より受付開始いたしますので…」）は **特定日付ハードコード**（年度依存）→ Gen3 では削除して標準受付時間外文言で統一、稼働日告知は customer_doc 経由で properties 反映
- `予約案内`（「ご予約のご案内をいたします。なお、直近のご予約には対応できない場合がございます。」）は新規予約ルートの導入 announcement → 診療科聴取前の冒頭 TTS
- 聞き取り NG 系（`診療科聞き取りNG` / `用件聞き取りNG` / `氏名聞き取りNG` / `生年月日聞き取りNG`）は **すべて代表案内 → 切断へ** に集約 → Gen3 では各 hearing/subflow の error_jump 先 termination に統一
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字禁止
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**
- **CMR conditions は必ず最後に `match: "other"` + next 明示**
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
- Gen1 動的タグ `<? $var ?>` は `<% var %>` 構文へ
- **diagnosis_subject 聴取は無し**: 受診者 = 入電者前提、PatientName 1 枠で完結（受診者と入電者の分離設計なし）
