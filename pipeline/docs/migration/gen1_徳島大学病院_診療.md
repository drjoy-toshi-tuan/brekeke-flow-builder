### 移管方針
- Gen1「箱」→ Gen3 9 ブロック型へ変換。opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「お電話ありがとうございます。徳島大学病院の、予約専用 AI 電話です」。
- **用件分岐は Pattern A（3 分岐）**: 用件聞き取り OpenAI 直後に「変更 / キャンセル / 確認」+ DTMF 1=変更 / 2=キャンセル / 3=確認 並走。echo_back（予約変更復唱 / 予約キャンセル復唱 / 確認復唱）を各分岐直後に配置。**新規予約・初診予約系のレーンは存在しない**ため director は安易に 4 分岐化しない。
- **冒頭の体調確認**: 用件聞き取り**前**に「体調が悪くて本日診察を受けたい / 予約を変更したい」Y/N 質問あり（体調確認.yaml）。Yes → `終話（体調相談に該当するため）`「予約されている外来診療科へ直接おかけ直しください」専用 termination。No → 通常用件聞き取りへ。Gen3 では opening 直後に hearing（Y/N、SKILL_B）+ context_match_router で体調不良レーン / 通常レーンを切り分ける（director 判断）。
- **時間判断（時間判断.yaml）**: 平日（月〜金）7:00〜20:00、年末年始 12/27〜1/4 を個別判定 + 祝日チェック（サブシナリオ呼び出し）→ date_of_call_classifier ブロックに集約。**addCurrentDate: Yes 自動付与対象**。受付時間が信州大（8:30〜16:30）/ 岐阜大（9:00〜15:00）/ 北里大（8:30〜17:00）より大幅に広い（**早朝 7 時受付**）点に注意。
- **TEST 番号登録ロジック**は運用テスト用入電番号（050-1707-0320 等 + 担当者携帯番号 14 件）を判別して、テスト時は時間判断をスキップして直行する仕組み。**Gen3 では再現不要**（director 判断、本番設計から削除）。
- サブフロー（患者氏名 / 生年月日 / 電話番号 / 診察券）は Jump to Flow 経由でサンプル JSON 完コピ。
- 自由テキスト hearing は **理由聴取.yaml の「変更」「キャンセル」「確認内容聴取」** の 3 種類（用件別に TTS 文言を切替）。output_format=text、SKILL_C 自由テキスト、無言 10 秒で打ち切り（信州大 5 秒より長め）+ 上限 2 回再質問。

### 施設固有の特殊ルール
- **受付時間**: 土日祝日を除く、月〜金 7:00〜20:00。年末年始 12/27〜1/4 休診（時間判断.yaml の `日付分岐` `日付分岐_1` で個別比較、年末年始 TTS は「受付開始は 2026 年 1 月 5 日月曜日の朝 8 時 30 分から」と明示）→ Gen3 では `年末年始` TTS の「8 時 30 分から」が 7 時受付と矛盾していそうな点は施設仕様としてそのまま（要マネージャー確認）。
- **代表電話案内（放射線診断科 / 放射線治療科 = AI 受付不可）**: 088-633-9071。`代表案内判別` で「放射線診断科」「放射線治療科」発話 → `代表へ`「<? $診療科の読み ?>は、AI 電話では承ることができません」TTS → `088-633-9071`「番号は、088-633-9071 です」→ `繰り返しカウント`（3 回到達で切断）→ `繰り返し`「繰り返します。」 → 番号案内を最大 3 回繰り返す。**北里大の代表電話 12 系統とは違い、徳島大は 1 系統のみ**。Gen3 では termination_pattern + retry_count で吸収。
- **新規予約は AI 受付不可**: 用件聞き取り yaml の 3 分岐に「新規」が無く、用件分岐_1（完了フラグ後）も「変更 / キャンセル / 確認」のみ。冒頭 TTS で「予約変更 / 予約キャンセル / 予約確認のいずれか」と明示する想定（director 判断、用件聞き取り.yaml 自体には冒頭ガイダンスがないので main 側で挿入）。
- **紹介状有無分岐は無い**: 大学病院系で信州大・岐阜大・北里大には紹介状サブフローがあるが、徳島大には紹介状サブフローも紹介受診重点医療機関 TTS も存在しない。Gen3 でも紹介状ブロックは作らない（director 判断、対象用件が変更/キャンセル/確認のみで初診ガードが不要なため）。
- **診療科ハイラルキ二段（7 グループ）**: ① 親エンティティ「呼吸器 / 消化器 / 脳神経 / 口腔 / 歯科 / 小児 / 放射線」+ 「分岐なし診療科」（一発確定の科 = 整形外科 / 眼科 / 耳鼻咽喉科 / 皮膚科 等 297 surfaces）→ ② `分岐(xxx)の聞き取りへ` で内/外科 or 細分化を聞き直し（**呼吸器=2 値・消化器=2 値・脳神経=2 値・口腔=3 値・歯科=4 値・小児=4 値・放射線=2 値**）→ ③ 確認復唱（`<? $診療科の読み ?>`）。Gen3 では **親 hearing → context_match_router → 子 hearing 7 本（4 値分岐の歯科・小児はサブフロー化せず hearing ブロックを並べる推奨、信州大・岐阜大と同方針）→ echo_back**（director 判断）。
- **診療科確認時のコンテキスト入れ替え**: `コンテキスト入れ替え` 箱で「呼吸器・膠原病内科」「内分泌・代謝内科」のような中黒入り正式表記をセットし直すロジック → Gen3 では OpenAI 正規化辞書（entity_openai_synonyms.md）で吸収（director 判断、別ブロック化不要）。
- **予約変更のみ WEB 利用案内**: 用件分岐で「変更」 → `WEB利用` サブフロー（「予約変更は WEB でも受付」案内 → 携帯番号取得済みなら携帯宛 SMS / 取得失敗なら携帯番号聴取 → SMS 送信案内 → 終話）。**WEB レーン側では氏名・生年月日・診察券などの個人情報は取得しない**（短縮ルート）。Gen3 では用件 = 変更の context 確定後に announcement + Y/N hearing + PhoneNumber subflow（携帯確定）+ termination で完結（director 判断）。
- **予約キャンセル / 予約確認のフロー**: 用件確定 → 診療科聞き取り → 予約日聴取 → 理由聴取（キャンセル理由 or 確認内容）→ 完了フラグ → 用件分岐_1 で再分岐 → 折り返し案内「3 診療日以内に担当者から折り返し電話、もしくはショートメッセージにてご連絡」→ 終話「お電話ありがとうございました。それでは失礼いたします」。
- **予約変更（AI 電話レーン）のフロー**: 用件確定 → WEB 利用 Y/N 分岐（NG なら）→ 診療科聞き取り → 予約日聴取 → 理由聴取（変更理由 + 次回予約希望日を **同じフリーテキスト 1 本で聴取**）→ 完了フラグ → 入電番号取得確認 → 連絡先電話番号聞き取り（入電未取得時のみ）→ 折り返し案内 → 終話。
- **新規予約（再診ベース）の追加質問**: 既存予約に対する変更/キャンセル/確認のみのため、出産予定日・残薬確認・ペースメーカー等の追加聴取は無し（信州大・岐阜大・北里大とは異なる）。
- **3 回失敗時**: 各サブで `count再質問` 2 回到達 → `END_聞き取りできず` → main の **失敗系 termination が無く**、サブ側の END から呼び出し元の next に戻って次の聴取に進む寛容モード（岐阜大と同方針、信州大の代表案内転送とは違う）→ director は失敗時に強制切断 / 代表案内ではなく次質問へ進める設計を維持。
- **電話番号区切り yaml は 2 本**: メイン（連絡先電話番号聞き取り内で利用）と `電話番号区切り_1`（WEB 利用 / 携帯番号サブフローで利用）。読み上げ用補助で内容は同等 → Gen3 では TTS 動的値構文 `<% var %>` の readable 形式 1 本に集約（director 判断、別ブロック化不要、信州大・岐阜大・北里大と同方針）。
- **救済語/同音異義救済**: 診療科 yaml に大量にあり（「Suica → 麻酔科」「BB/GB/DB → 耳鼻咽喉科」「ハート → 循環器内科」「インプラント → 口腔インプラント」「シンゲ → 心臓血管外科」等）。**entity_classification.md / entity_openai_synonyms.md / entity_stt_patch.json に全件あり、director は転記不要**。
- **DTMF 受付（用件聞き取り）**: 1=変更 / 2=キャンセル / 3=確認（用件聞き取り.yaml の用件エンティティに「1番 / 2 / 2番 / 3 / 3番 / 一番 / 二番 / 三番」あり）。冒頭ガイダンスでも明示する設計を推奨。
- **復唱（echo_back）対象**: 用件 3 種 + 診療科のみ。氏名/生年月日/電話番号/診察券はサブフロー内で完結。

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `API_DRJOY` → `TEST番号登録` → `時間判断` → `はじめに` → `体調確認` → `用件へ` → `用件聞き取り` の 9 箱は Gen3 では opening 冒頭処理 + date_of_call_classifier + 体調確認 hearing + 用件 hearing に整理（**TEST 番号登録は本番設計から削除**）。
- 「セリフ: none」かつ振り分けのみの「箱」（`完了フラグ` `用件分岐_1 / _2` `代表案内判別` `終話_1 / _2` `電話切断へ` `入電番号取得確認` `携帯電話番号チェック / _1` `電話種別` `分岐なし診療科セット` `00分岐あり診療科セット` `わからないをセット` `各セット系（呼吸器/消化器/脳神経/口腔/歯科/小児/放射線）` `読み取得 / _1〜_6` `分岐あり診療科読み取得 / _1` `診療科読み取得` `分岐確認へ` `count上記以外 / _1〜_8` `Count再質問 / _1〜_9` `聞き取り失敗へ / _1〜_6` `コンテキスト入れ替え` `音声確認用フラグ` `フラグ on` `桁数チェック` `区切り` `携帯3-4-4` `IP3-4-4` `固定2/3/4/5-x-4` `電話番号蓄積` `桁数不足など` `用件のセット` `セットした用件の分岐` `用件再設定へ` `2 回聞き取れず` `用件を確認` `分岐あり否定` `わからないをセット` 等）は Gen3 で独立ブロック不要 → 前段の next 書き換え or context_match_router の一部として吸収。
- **「予約変更」「予約キャンセル」「予約確認」が複数経路で出現**: ① main の `用件分岐 / _1 / _2`、② `用件聞き取り.yaml` の `セットした用件の分岐`、③ `理由聴取.yaml` の `用件判別`。**Gen3 では用件 context を一度確定したら以降は context_match_router で参照する**ことで重複聴取を避ける（信州大・岐阜大・北里大と同方針）。
- **`用件分岐_2` は予約日聴取後の再分岐**: 「変更 → 入電番号取得確認」「キャンセル/確認 → 連絡先電話番号聞き取りへ」を context 参照で振り分け。Gen3 では `context_match_router` で機械的に分岐 → 独立ブロック不要。
- **`代表案内判別` は診療科聞き取り直後**: 放射線診断科 / 放射線治療科 を発話した場合のみ代表電話案内へ分岐し、それ以外は予約日聴取へ進む。Gen3 では subflow（診療科聞き取り）の return context を context_match_router で評価し、放射線科 2 種 → termination、それ以外 → 予約日 hearing へ流す（director 判断、SUBFLOW_RETURN_CONTEXTS 解決ルールに乗る）。
- **`体調確認.yaml` の冒頭 `冒頭` 箱「まず初めに、」は短い繋ぎ TTS** → Gen3 では announcement ブロック化するか体調確認 hearing の導入文言に統合するか director 判断。`冒頭_1` 箱の本質問 TTS が長いため announcement で先出ししても自然。
- **`診療科を確認` 箱で「肯定否定」(はい/いいえ等の混在発話)→ `コンテキスト入れ替え` 経由**: 「呼吸器・膠原病内科」「内分泌・代謝内科」を中黒入り正式表記でセットし直す処理。**Gen3 では診療科 context は OpenAI 出力の正規化済み診療科名 1 個に統一**し、中黒入り正式表記は OpenAI 正規化辞書で吸収（director 判断、信州大・岐阜大と同方針）。
- **小児の 4 分岐セリフのタイポ**: `小児科の聞き取りへ` 箱 TTS に「3 番、小児小児歯科」（「小児」が 2 回繰り返し）+ 「4 番、小児摂食外来」あり。Gen3 では文言修正して「3 番、小児歯科 / 4 番、小児摂食外来」に整える（director 判断、ただし要マネージャー確認）。
- **WEB 利用は携帯番号取得時のみ進行**: `携帯電話番号チェック`（入電番号 = 携帯）→ `携帯番号` TTS で「今おかけいただいている電話に WEB 予約ページをお送りできますが、希望されますか？」、`携帯電話番号チェック`（入電番号 ≠ 携帯）→ `携帯無し` TTS で「携帯電話番号宛に WEB 予約ページをお送りできますが」と異なる TTS。**TTS 文言が携帯/固定で変わる点は director デフォルト 9（携帯/固定 TTS 分割）の派生**だが、**今回の主役は「入電番号が携帯か固定か」で文言を切替**するロジックなので明示が必要。
- **`用件聞き取り.yaml` の「セットした用件の分岐_1」は別の retry 系**: 用件再聴取直前のラスト確認パスで、ここからは `用件再設定へ` → `用件のセット` → `セットした用件の分岐` で再分岐に戻る。Gen3 では retry_count で吸収。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**（丸数字 ①②③⑩等は使用禁止）。元資料に出口記号は無いが、`088-633-9071` のような **数字始まりのモジュール名は ID として使えない**ため `END_代表案内_放射線` 等の安全名に置換（信州大・岐阜大と同様）。
- **ContextMatchRouter.reference_module は subflow を参照する場合 `jump_{step名}`**（scaffold が付ける prefix）を使う。subflow ブロックの step 名そのもの（例: `診療科聴取`）ではダメ。
- **CMR conditions は必ず最後に `match: "other"` + next 明示**。`TODO_other_target` プレースホルダーを残さない。
- **転送先は無し**（Blind Transfer モジュールは登場しない）。代表電話案内 088-633-9071 は TTS 案内のみで実際の転送は行わず、利用者に手動で掛け直してもらう設計（信州大・岐阜大は Blind Transfer ありだったので注意）。
- **終話 TTS の集約**: `終話`「お電話ありがとうございました。それでは失礼いたします」/ `折り返し案内`（3 診療日以内、SMS 可）/ `終話（体調相談に該当するため）`（体調不良 → 外来直接案内）/ `時間外`（受付時間外、月〜金 7:00〜20:00 案内）/ `年末年始`（1/5 朝 8:30 から再開案内）→ Gen3 では termination_patterns 集約（director デフォルト 9 の携帯/固定 TTS 分割は WEB 利用レーン以外では限定的、要 director 判断）。
- **TTS は `{tts_g:...}` 小文字のみ、SSML 禁止**。Gen1 変数記法 `<? $診療科の読み ?>` `<? $予約日の読み ?>` `<? $分岐あり診療科の読み ?>` `<? $連絡先電話番号（ハイフン区切あり） ?>` `<? $入電番号（ハイフン区切あり） ?>` は Gen3 では `<% var %>` に変換。
- 患者氏名は PatientName サブフロー 1 枠に充当（patient 1 名のみ、入電者 = 受診者前提の設計）。
