### 移管方針
- Gen1 の「箱」構造 → Gen3 scenario_flow 9 ブロック型に変換。主な対応:
  - `挨拶`（箱） → opening ブロック + 冒頭_アナウンス（2ブロック1セット必須）
  - `用件聞き取り`（サブyaml） → 用件分類の generate_by_OpenAI モジュール（hearing ブロック）。定期予約 / 予約変更 / 予約キャンセル の 3 分岐
  - `入電曜日時間判断(8/1以降)` + `年末年始日付分岐` + `（土）時間の分岐` → date_of_call_classifier ブロックにまとめる（addCurrentDate: Yes 自動付与対象）
  - 氏名 / 生年月日 / 電話番号 / 診察券番号 の各サブ yaml → subflow ブロックでサンプル JSON 完コピ（director は詳細を書かない）
  - 診療科聞き取り → hearing ブロック（text/enum、診療科 no_result パターン適用）
  - 予約日 / 希望日 / キャンセル理由 / 他の要望 → hearing ブロック群
- 用件分岐は **Pattern A**（用件 OpenAI 直後に 3 分岐）で組める。echo_back（定期予約復唱 / 予約変更復唱 / 予約キャンセル復唱）を各分岐直後に配置
- 用件別フロー分岐:
  - 定期予約: 診療科 → 予約日 → 希望日 → 氏名 → 生年月日 → 診察券 → 電話 → 終話
  - 予約変更: 診療科 → 予約日 → 氏名 → 生年月日 → 診察券 → 電話 → 終話
  - 予約キャンセル: 診療科 → 予約日 → キャンセル理由 → 他の要望 → 氏名 → 生年月日 → 診察券 → 電話 → 終話
  - 詳細合流パターンは main.yaml の「次へ_1〜_6」箱から director が読み取る

### 施設固有の特殊ルール
- **当日予約は AI では受付不可** → `当日予約→かけ直し` 箱のアナウンスを流して切断（代表案内パスと統合）
- **受付時間**: 日曜祝日・年末年始除く、月〜金 8:00〜18:00、土曜 8:00〜17:00（受付時間外アナウンスの文言はこれに従う）
- **8/1 以降の特例**: 時間外アナウンスに「月〜金は 24 時間、土は 0〜17 時」の別バージョンあり（`受付時間外アナウンス（8/1以降）`）。director は両方の文言が元資料にある点に注意
- **年末年始の段階分岐**: 12/28 まで / 12/29（18 時で分岐）/ 12/30〜1/2 / 1/3（土・17 時で分岐）/ 1/4 まで特別処理。date_of_call_classifier 内で分岐するか、後段 context_match_router で処理するかは director 判断
- **代表電話番号**: 04-2953-9995（かけ直し案内 TTS に 3 回繰り返しで含める）
- **3 回失敗時**: `3回失敗後のかけ直し依頼` 箱のアナウンスで代表番号案内して切断。termination_patterns に含める
- **復唱（echo_back）対象**: 用件（定期予約 / 予約変更 / キャンセル）のみ。氏名・生年月日・電話番号・診察券番号は各サブフロー内で復唱済みのため重複禁止
- **救済語/キーワード**: 用件聞き取り.yaml で DTMF 入力（1=定期予約 / 2=変更 / 3=キャンセル）に対応。Gen3 でも DTMF 受付を generate_by_OpenAI のプロンプト側で吸収するか、ContextMatchRouter で明示するかは director 判断（SKILL_A 参照）

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` と `API_DRJOY` は Gen3 では opening 冒頭の標準処理に吸収される → 明示ブロック化不要
- 「共通」セクションの `次へ` / `次へ_1〜_6`（セリフ「では次に、」「続いて、」「最後に、」等）は Gen3 では **announcement ブロックの短い繋ぎ TTS** として個別に配置するか、hearing ブロックの導入文言に統合する（director 判断、ただし省略しすぎると自然さが落ちる）
- 「セリフ: none」かつ振り分けのみの箱（`用件の分岐`, `日付分岐へ`, `完了フラグ` 等）は Gen3 では **独立ブロック不要**、前段の next 先を書き換えるか context_match_router の一部として吸収
- `代表案内へ（○○NG）` の 10 個の箱はすべて `完了フラグ（代表案内）` → `３回失敗後のかけ直し依頼` に合流 → **Gen3 では 1 つの termination ブロックに集約**して各 hearing の失敗パスから参照
- `入電曜日時間判断` 内の `祝日チェック` は元資料で「サブシナリオ呼び出し」扱い → Gen3 では **カレンダー Script モジュール**が必要（docs/brekeke/モジュール選定ガイド_v2.md §3.8 参照）。addCurrentDate: Yes 自動付与の対象
- 年末年始の複雑分岐（`年末年始日付分岐` / `（土）時間の分岐` / `（12/29）時間の分岐` / `年末年始日付の分岐`）は **営業日計算 + 時間判定の組合せ** → Pattern B（後段 ContextMatchRouter）で実装が妥当
- サブフロー（氏名 / 生年月日 / 電話 / 診察券）は Gen3 サンプル JSON 完コピで、director が細部を書く必要なし（feedback_subflow_connection_rule.md 準拠、Jump to Flow 経由）
- 患者氏名は PatientName サブフロー 1 枠に充当（patient 1 名のみ、入電者 = 受診者前提の設計になっている）
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
