### 移管方針
- Gen1「箱」構造 → Gen3 9 ブロック型へ変換。挨拶 TTS は「お電話ありがとうございます。真生会富山病院、アイセンターの、AI 電話です」で opening + 冒頭_アナウンスの 2 ブロック 1 セット。
- 用件分岐は **Pattern A**（用件聞き取り OpenAI 直後に 4 分岐: 診察予約 / 変更 / キャンセル / 確認問合せ）。さらに診察予約は subflow 復唱直後に「初めて受診ですか？」Yes/No 判定（initial/follow-up 分岐）が入る **2 段構え** → ここは Pattern A 連結 or 後段 ContextMatchRouter で組むかは director 判断。
- echo_back（用件復唱）: 「診察のご予約ですね？」/「ご予約の変更で、よろしいですか？」/「ご予約のキャンセルでよろしいですか？」/「確認など、その他問い合わせ、でよろしいですか？」の 4 種。診察予約のみ復唱後に初再診確認に進む（他 3 種は echo_back の Yes で確定）。
- 用件別フロー分岐（main.yaml `用件の確認_1` `用件を判断` から director 読取）:
  - 新規予約（初診）: 紹介状有無 → 受診内容 → 患者氏名 → 生年月日 → 連絡先電話 → 折り返し希望時間帯 → 終話アナウンス（新規予約の流れ）
  - 再診: 受診内容 → 診察券 → 患者氏名 → 生年月日 → 連絡先電話 → 折り返し希望時間帯 → 終話アナウンス（再診の流れ）
  - 変更: 予約日 → 受診内容 → 診察券 → 患者氏名 → 生年月日 → 連絡先電話 → 折り返し希望時間帯 → 終話アナウンス（変更の流れ）
  - キャンセル: 予約日 → 理由・確認内容 → 診察券 → 患者氏名 → 生年月日 → 連絡先電話 → 折り返し希望時間帯 → 終話アナウンス(予約キャンセルの流れ）
  - 確認問合せ: 理由・確認内容（問合せ内容）→ 患者氏名 → 生年月日 → 連絡先電話 → 折り返し希望時間帯 → 終話アナウンス（問い合わせの流れ）
- 氏名 / 生年月日 / 電話 / 診察券 は subflow ブロック（Jump to Flow）で定型化。`電話番号区切り.yaml` は連絡先電話番号サブフロー内のヘルパー → Gen3 サブフローで吸収。
- 時間外チェック.yaml（`祝日チェック` サブシナリオ呼び出し + 月水金 8:30-17:00 / 火木土 8:30-12:00 + 日祝・年末年始除外）→ acceptance_times に集約。**addCurrentDate: Yes 自動付与対象**。
- entity_classification.md 参照: STT 辞書は use_template 9 種 + 専門眼科用語 (CO 対応 / 看護師対応 / 医事課対応) は additional_words 直書き、OpenAI 正規化 186 件。

### 施設固有の特殊ルール
- **受付時間**: 日祝・年末年始除く、月・水・金 8:30〜17:00、火・木・土 8:30〜12:00（**土曜は午前のみ**、他施設と異なる変則的な曜日別運用）。受付時間外 / 失敗時アナウンスにも同一文言を含める。
- **診療科は眼科（アイセンター）専用** → 多診療科マトリクスは存在せず、診療科聞き取りブロックは不要。代わりに `受診内容聞き取り.yaml` で **CO対応 / 看護師対応 / 医事課対応 / その他** の 4 区分を分類（眼科特有の専門用語: アトロピン / オルソケラトロジー / レーシック術前検査 / 円錐角膜 / プリズム眼鏡 / 遮光眼鏡 / OPE予約 / ボトックス / 涙の手術 / 入院日変更 等）。
- **受診内容の用件別表示変更**: 新規予約・再診 → 「ご希望の受診内容をお話ください」/ 変更 → 「変更される内容について、診察、検査、手術など、項目をお話ください」/ キャンセル → 「キャンセルされる受診内容をおっしゃってください」。同一サブフローで TTS 切替 → Gen3 では用件 context を見て hearing 導入文言を切替（director 判断）。
- **アイセンター直通電話案内**: 0766-52-6979（2 回失敗時アナウンスで 3 回繰り返し）。月・水・金 8:30〜17:00、火・木・土 8:30〜12:00 の時間帯案内付き。
- **新エンジン番号**: 050-1724-9926（main.yaml `新エンジン移行後アナウンス` 内、現行運用の AI 電話番号）。**移管時点で Gen3 が新エンジン側** のため、移行案内分岐そのものは Gen3 では実装不要（director 判断、現運用と一致するか念のため確認）。
- **DTMF 受付**: 用件聞き取り（1=予約 / 2=変更 / 3=キャンセル / 4=確認・その他）と紹介状 Yes/No（1=はい / 2=いいえ）で受付。Gen3 では generate_by_OpenAI のプロンプト側 or ContextMatchRouter で吸収（SKILL_A 参照、director 判断）。
- **予約日聴取の DTMF 4 桁入力**: シャープ・アスタリスク + 4 桁数字（MMDD）受付、過去日なら +1 年で自動確定する仕様あり。Gen3 でどう扱うかは director 判断（SKILL_C 自由テキスト + DTMF 並走）。
- **紹介状有無は新規予約でのみ聴取** → 再診 / 変更 / キャンセル / 確認 ルートでは紹介状ブロック不要（main.yaml `用件を判断` で `新規予約` のみ `紹介状有無聞き取り` に流れる）。
- **2 回失敗時の振り分け**: `入電番号取得確認` で **入電番号取得済み** → `2回失敗アナウンス（折り返し）`「当院よりお電話いたします」、**非通知** → `2回失敗アナウンス（アイセンター案内）` で直通番号案内。Gen3 では context_match_router 1 個に集約可能。
- **代表案内案件**（健診 / ドック等）→ 真生会富山病院本体側の番号案内。アイセンターでは扱わないため、用件聞き取り内で「代表案内」エンティティが拾われた場合の専用パスが必要かは director 判断（現状 main.yaml に明示パスは無いため、用件再聴取 or 別番号案内で吸収）。
- **復唱（echo_back）対象**: 用件（4 種）+ 初再診確認（診察予約のみ）+ 紹介状有無 + 予約日 のみ。氏名・生年月日・電話・診察券はサブフロー内で完結、メインに echo_back を出さない。
- **折り返し希望時間帯聴取は全用件で必須**（`折り返し希望時間帯聞き取り` → `完了フラグ` → `用件の確認`（最終終話分岐））。終話 TTS は用件別 5 種に分岐（新規予約 / 再診 / 変更 / キャンセル / 問い合わせ）。

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `API_DRJOY` → `新エンジン移行後アナウンス(完了フラグ：7)` → `新エンジン移行前分岐` → `8/1分岐` → `時間外チェック` の冒頭チェーンは、**Gen3 では opening 冒頭処理 + acceptance_times に吸収**。新エンジン関連の分岐ロジック（移行前/8月1日/移行後）は **2026-05 時点で旧番号は廃止済み前提のため Gen3 では実装不要**（director は現運用に新エンジン番号 050-1724-9926 を使っているか念のため確認）。
- `用件の確認_1`（main.yaml 共通セクション）と `用件の確認`（用件聞き取り.yaml 内）は **別の分岐**。前者は `理由・確認内容聴取後の用件再判定`（変更/キャンセル/再診/新規予約 で次のサブフロー先を切替）、後者は echo_back の Yes/No 判定。Gen3 では context（用件）が既にセット済みなので前者は context_match_router で素直に組める。
- `判定: エンティティ含む` で参照されるエンティティ値（CO対応 / 看護師対応 / 医事課対応 / 用件 / 肯定 / 否定 / 分からない 等）は **entity_classification.md / entity_stt_patch.json / entity_openai_synonyms.md にすべて自動分類済み** → director は転記不要、scaffold/properties が自動取り込み。
- `用件の確認_2`（理由・確認内容聞き取り後）は「キャンセルなら `次へ_4`（折り返し電話番号案内 → 用件の確認_1 で再分岐）」「それ以外は `次へ`（診察券聴取）」という地味な分岐 → Gen3 では context_match_router 1 個で吸収。
- 紹介状無しでも **切断されず受診内容聞き取りに進む**（紹介状 OK/NG 共通で `次へ_5` → `受診内容聞き取り`）。多摩総合のような「紹介状なし → 切断」ではない点に注意。アイセンター運営方針として紹介状がなくても新規受付するため。
- 用件聞き取り.yaml の `count再質問_X` / `count上記以外_X` 系（7 系統あり）は Gen3 では **hearing.retry_count=2 に統合 + count再質問_7 のみ「否定回答後の用件再聴取」として hearing.error_block で別扱い**（director 判断、ただし統合しすぎると否定→再質問の文脈が崩れる可能性あり）。
- 予約日.yaml の `日付チェック_1〜_3` `月チェック` `日チェック` `0桁詰め` `数字を日付に` 等の DTMF 4 桁正規化ロジックは Gen3 では **subflow 内 Script モジュール** or **OpenAI に YYYY-MM-DD 出力させて + 過去日なら +1 年補正** で対応（director 判断、SKILL_C 参照）。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。`<? $予約日の読み ?>` のような Gen1 変数記法は Gen3 では `<% var %>` に変換。
- 患者氏名は PatientName サブフロー 1 枠（入電者 = 受診者前提）。本院は患者単独受診シナリオで、家族代理通話のケアは元資料に明示なし → director は subflow 仕様デフォルトで OK。
