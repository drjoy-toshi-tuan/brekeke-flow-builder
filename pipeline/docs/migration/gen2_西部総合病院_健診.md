### 移管方針

- **Pattern 判定**: 用件聴取（予約/変更/キャンセル/問い合わせ）は **Pattern A**（generate_by_OpenAI 直後に ContextMatchRouter で4分岐）。受診歴確認（初めて/2回目以降）、前回と同じ内容確認（はい/いいえ）、電話番号確認（はい/いいえ）はいずれも Yes/No 分岐で Pattern A。
- **Date of Call Classifier**: **不要**。営業時間外分岐（before_open / in_hours / after_close）の発話分岐は customer_doc に定義されているが、プロンプトでの時間帯出し分けは 1-7 の注記程度で、終話アナウンスは用件×電話種別のみで決定するため。
- **サブフロー化**: 氏名聴取（苗字のみ→下の名前再聴取、複数名分岐）、生年月日聴取、連絡先電話番号聴取（Incoming_phone_number 種別判定＋確認/直接聴取/桁数リトライ3回）は必ず Jump to Flow でサブフロー化。メインにインライン展開禁止。
- **電話番号サブフロー分岐**: Incoming_phone_number が携帯（090/080/070/060）の場合のみ「今おかけいただいている〜」確認フロー、それ以外（050/固定/anonymous/unknown/空欄）は直接聴取フローに script モジュールで振り分け。
- **終話ブロック構造**: termination_patterns は **電話種別（携帯 / 固定）× 用件（予約 / 変更 or 問合せ / キャンセル）= 6パターン** を定義。加えて聴取失敗（リトライ2回超過）の END_聴取失敗案内 を別 termination として持つ。
- **リトライ上限**: 用件聴取 = 2回、電話番号桁数チェック = 3回。リトライ超過時は共通の END_聴取失敗案内 に合流。

### 施設固有の特殊ルール

1. **施設名の二重表記（最重要）**: customer_doc 内で施設名が「西部桜クリニック」（さくらクリニック）と書かれている箇所がある。これは**人間が意図的に変更したもので正しい**。TTS 発話文言は customer_doc に従い「西部さくらクリニック」で生成すること。yaml の `basic_info.facility_name` のみ「西部総合病院」（施設管理上の正式名）で統一する。END_聴取失敗案内 の中に「西部総合病院代表電話の電話番号とは異なる番号ですので…西部さくらクリニック直通電話番号が分かる場合は…」という両表記混在の文言がある点に注意（これも原文通りに再現）。
2. **復唱禁止ルール**: 受診者名（patientName）は**復唱しない**。それ以外の項目（生年月日・電話番号・コース等）は「○○ですね。」で復唱する。電話番号のみ `<speak type="telephone" breakc="300ms">` 相当のフォーマットだが、Gen3 では SSML 不可のため `{tts_g:...}` 内で電話番号桁区切り発話に変換（prompter / properties 側で処理）。
3. **複数名入電対応**: 氏名聴取で複数名が発話された場合、最初の1名のみ patientName に格納 → 生年月日聴取時に専用アナウンス「受診される方の生年月日を西暦で、1人目の方から順番に、人数分お話ください。」に分岐する。氏名サブフローから生年月日サブフロー起動時に分岐フラグを渡す設計が必要。
4. **additionalPhoneNumber 格納規則**: (a) 口頭/DTMF で1回でも番号が提示されたら必ず additionalPhoneNumber に格納（空欄禁止）。(b) 携帯着信で「はい、そうです」確認時は Incoming_phone_number をコピー格納。(c) 番号未提示時のみ空欄可。「今おかけいただいている〜」表現は Incoming_phone_number が携帯の場合に限る。
5. **SMS 案内の出し分け**: Incoming_phone_number または additionalPhoneNumber のいずれかが携帯番号（090/080/070/060）の場合のみ SMS 案内文言を使用。固定番号しかない場合は SMS 案内なし版の終話文言へ。
6. **予約日の年跨ぎ補正**: 月日のみ発話 → 着信年を補完。ただし 10月以降の入電で 1〜9月の日付を指定された場合は**翌年**とする。例: 2025年11月1日に「5月1日」→ 2026-05-01。
7. **キーワード正規化ルール（誤認識救済）**: 「検針/返信/天候診断/返金控寸断」→健診、「人間ロック/ドック/ロック/人間独」→人間ドック、「薬/お薬/要約/通訳」→予約、「取り直したい/別の/変えたい」→変更、「やめる/取り消し/取消」→キャンセル。「検診」は健診と判断するが、**前に「乳がん」「子宮がん」「子宮頸がん」が付く場合は除外**（検診のまま）。
8. **終話文言の「かしこまりました。」抑制**: 追加質問で「折り返しの電話してほしい時間」を伝えられて「ご要望を承りました。ご希望に添えない場合もございますのでご了承ください。」を発話した直後のみ、終話アナウンス先頭の「かしこまりました。」を**付けない**分岐を用意する（6終話パターンそれぞれに「通常版」「ご要望承りました直後版」の2系統が customer_doc に定義されている）。
9. **折り返し対応問合せ**: 「折り返し対応の時間を聞かれた」→「順番に対応しておりますので、しばらくお待ちください」固定応答。「折り返し希望時間を伝えられた」→「ご要望を承りました。」固定応答。これらは追加質問ブロック内の分岐として設計。
10. **プロンプト非対応質問のエスカレーション文言**: 「お話しいただきました内容はAI電話では対応ができません、恐れ入りますが、祝日を除く月曜～金曜の8：45～17：00、土曜は8：45～12：00に、西部さくらクリニック直通番号へおかけなおしください。…電話番号は048-854-1114です。」を別途用意。

### director が見落としやすいポイント

- **改訂履歴**: customer_doc 冒頭に「2026/04/07 病院名を西部さくらクリニックに変更」の注記あり。発話文言は「西部さくらクリニック」で統一する（特殊ルール1 参照）。
- **聴取順序の固定**: customer_doc 1-3 で「**用件を先に聞かないでください。指示の順番通りに聞いてください。**」と明記。順序は **氏名 → 生年月日 → 連絡先電話番号 → 用件**。この順序を崩すと違反。
- **受診歴分岐の両方入口**: 「初めて」「はい」→ Medical_history=「初めて」→受診コースへ。「2回目以降」「いいえ」「前回も受診した」「昨年も受診した」「診察券あります」→ Medical_history=「2回目以降」→前回と同じ内容確認へ。ContextMatchRouter の含む条件を両系統で漏れなく定義すること。
- **前回と同じ内容確認の分岐先**: 「はい」→ checking_details=「前回と同じ」→**予約希望日**へ直行（受診コース聴取をスキップ）。「いいえ」→ checking_details=「前回と違う」→受診コース聴取。この**スキップ構造**を見落とさない。
- **本日予約確認は「変更」専用**: customer_doc 明記「キャンセル・予約・問い合わせでは絶対に発話しない」。変更フローのみで発話。肯定時は today_reservation=「当日」＋現在日時を 0000-00-00 00:00:00 形式で代入、否定時は「ー」。
- **キャンセル時の予約日一致チェック**: reservationDate が着信日と同一なら today_reservation=「当日」、それ以外は「－」（全角ハイフン）。customer_doc 明記のロジック。
- **predict_date 不採用、Preferred_date は原文格納**: 「9月上旬の午前中」「8月26日」などそのままテキスト格納。日付正規化しない（予約希望日のみ。reservationDate は yyyy-MM-dd 00:00:00 形式で正規化）。
- **電話番号桁数チェックの期待値**: 090/080/070/060/050 開始 → 11桁、それ以外 → 10桁。リトライは3回まで。超過時は END_聴取失敗案内。
- **endpoint enum**: 冒頭切断 / 時間外 / 電話転送 / 電話案内 / 途中切断 / 通話完了 の6値。status は 0/1/2、smsFlag も 0/1/2（enum は同じだが別フィールド）。
- **function スキーマ特記**: `medicalCardNumber`（ID番号）と `company_name`（企業名/団体名/組合名）は required に含まれるが、customer_doc フロー内では明示的な聴取ステップが薄い。氏名聴取時に企業名が発話されたら company_name に格納し、「受診をされる方のお名前をお話ください」へ再誘導する分岐が必要（director は氏名サブフロー設計時に確認）。
