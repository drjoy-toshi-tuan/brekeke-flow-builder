### シナリオ概要
入電者は **調剤薬局の担当者**（患者本人ではない）。担当者が処方箋に関する疑義をすずな皮ふ科クリニックに照会する一次受付フロー。患者氏名・生年月日は薬局担当者からの代理入力。

### 設計方針
- **flow_type**: 1flow。氏名/生年月日/電話番号/診察券番号の標準4点セットは患者本人由来ではないため、PatientName / Phone / MedicalCard / DateOfBirth の各標準サブフローは原則使用しない。患者名・患者生年月日は hearing として直接 main に置く（小山記念病院_薬剤部 と同型）。
- **薬局名の自動判定**: raw.md L57-63 / L223 「薬局名（電話番号登録済み）→ ○○さんですね。」が必須機能。実装は **`incoming-category-classifier` + `Phone2Name` の組み合わせ**（`incoming-classifier` ではない）。Dr.JOY 電話帳に 4 薬局を事前登録しておく前提で、`incoming-category-classifier` がヒットしたリスト名を返す → `Phone2Name` の `FOUND_KATAKANA_NAME_DEFAULT_TMP: "recipient_name様、お電話ありがとうございます。"` で薬局フリガナを動的差込 → 担当者名へ。NO_RESULT / TIMEOUT / ERROR は **薬局名 hearing にフォールバック**（電話帳未登録・障害時用、必須）。詳細は `docs/brekeke/モジュール選定ガイド_v2.md §1.4` / `モジュール詳細設定ガイド_1.md §4.6 / §6.2` / `docs/reference/incoming_category_examples/`（松本協立病院相談室 bivr が参考実装）を参照。**小山記念病院_薬剤部は 77 薬局あって自動判定を諦めた前例だが、すずな件は 4 薬局で Dr.JOY 電話帳運用が成り立つため自動判定を実装する**。
- **患者生年月日は DTMF 併用必須**（raw.md L228、L34 に「DTMF」明記）。音声単独ではない点に注意。stt_type は DTMF_AmiVoice 系を選択する。
- **用件分岐方式**: 用件モジュール自体は診療フロー側の責務（疑義照会は用件⑤として既存に追加されるだけ）。本シナリオの起点は「用件＝疑義照会」と classification 確定済みの状態。よって本フロー単独に DateOfCallClassifier や Pattern A/B 分岐は不要。ただし冒頭=opening+冒頭_アナウンスの2ブロックセットは必須（既存と共通の文言で良い）。
- **着信時間外/非通知**: 既存 `output/scenarios/すずな皮ふ科_診療/` の incoming-classifier / acceptance_times をそのまま踏襲。本フロー単独で再設計しない。
- **冒頭_アナウンス文言**: 既存診療シナリオ「お電話ありがとうございます。すずな皮ふ科クリニックAI電話です。」をそのまま使う。疑義照会専用の前置きは不要。

### 施設固有の特殊ルール
- **電話番号登録済み判定の TTS**: 「○○さんですね。」（薬局名そのものを差し込む単発確認、復唱モジュールではない）。echo_back=false。
- **疑義照会終話文言**: 「かしこまりました。お掛けいただいた番号に折り返しいたします。お電話ありがとうございました。」（raw.md L231）。小山記念病院_薬剤部 END_Z1 と同一文言で流用可。
- **聴取順**: 薬局名（自動 or 聴取）→ 担当者名 → 患者名 → 患者生年月日(DTMF) → 疑義内容 → 終話。診療科聴取は **無し**（皮膚科クリニックのため診療科分岐不要）。小山記念病院_薬剤部の clinicalDepartment hearing は本施設には配置しない。
- **復唱**: raw.md アナウンス一覧の「復唱有無」は全項目「無し」。echo_back=false で統一。連絡先電話番号復唱は本フローには出現しない（折り返し先は incoming_phone_number 固定）。

### director が見落としやすいポイント
- raw.md の **基本設定1/2 と アナウンス一覧 は表組みが大きく崩れている**（L68-173, L175-231）。聴取項目名と発話文言の対応がずれて読める箇所がある。**必ず元 PDF (P.2/P.4/P.8) を併読**し、raw.md だけで起草しない（feedback_director_must_read_source_doc / feedback_png_misread）。
- **status 値**: raw.md L120-142 に「未処理 1 / 処理中 3 / SMS返信待ち 2 / 処理済み 4 / 途中切断 0」と読めるが表崩れがひどい。**Gen3 では status=0 は使わない（status=2 に置換）**。本フロー終話の status 値は元 PDF で確認、不明なら **BLOCKER 化**。
- **SMS 文言・SMS 送信有無**: raw.md L70-90 に SMS 文言列が存在するが、疑義照会終話で SMS を送るかは記載が見えない。**疑義照会は薬局相手で SMS 送信なし(sms_flag=-1)が自然**だが、推測せず BLOCKER 化推奨。
- **薬局リストは 4 件のみ**（raw.md L59-62）。`profile_words` には全 4 件をヨミ付きで登録。`{施設}` を含む店舗名表記は不要（クオール薬局/ミツヤ薬局/ミネ薬局（ミネドラッグ）/日本調剤）。**列挙系のためノートに全件転記はしない、元資料を director が直接読むこと**。
- **電話番号登録判定の正解モジュール**: `incoming-category-classifier`（`drjoy^Incoming$incoming-category-classifier`）を使う。`incoming-classifier`（回線種別判定）の保存値を ContextMatchRouter で読み直す設計は誤り（前回 director がこの誤選択をして B-3 BLOCKER 化した経緯あり、memory `feedback_incoming_category_classifier` 参照）。Dr.JOY 電話帳の登録は人間作業として確認レポートに上げる。
- **「○○さんですね。」TTS の動的差し込み**: **Phone2Name の `recipient_name` プレースホルダ**を使う 1 本構成（電話帳のフリガナを動的差込）。薬局名リテラルを埋め込んだ TTS を 4 本並べる方式は推奨しない（電話帳変更で TTS 群の更新が必要になるため）。詳細は `docs/brekeke/モジュール詳細設定ガイド_1.md §4.6` を参照。
- **稼働時間**: raw.md L111「24時間365日（3/2（月）のみ9時開始）」とあるが、これは診療フローの設定。**疑義照会フローも同じ稼働時間で良いか元 PDF で要確認**、不明なら BLOCKER 化。
- **非通知**: raw.md L128「非通知受入拒否：設定する」「非通知入電への対応：かけ直し案内」。既存診療シナリオ同様、本フロー単独で非通知切断経路を持つ。
