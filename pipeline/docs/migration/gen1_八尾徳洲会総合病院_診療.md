### 移管方針
- Gen1「箱」構造 → Gen3 9 ブロック型に変換。湘南藤沢徳洲会と違い、**新規予約フロー（初診/再診/紹介状）は持たない**。既存予約に対する「変更 / キャンセル / 確認」と「歯科・口腔外科 / 健診への振り分け」が軸。
- 冒頭は opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「お電話ありがとうございます。八尾徳洲会総合病院の、予約専用、AI 電話です」。
- 用件分岐は **Pattern A**（用件聴取 OpenAI 直後に 4 分岐: 予約変更 / 予約キャンセル / 予約確認 / 歯科・口腔外科）。各分岐に echo_back（「ご予約の変更でよろしいですか?」「ご予約のキャンセルでよろしいですか?」「ご予約の確認でよろしいですか?」「歯科・口腔外科でよろしいですか?」）を直配置。**用件は echo_back 必須**。
- 診療科聞き取りは **Pattern B（後段 ContextMatchRouter）併用**。診療科.yaml の振り分けが「分岐なし診療科（=代表電話 0 7 2-993-8501 案内で終話）/ 検査名（=代表電話案内）/ 歯科口腔外科（=歯科転送）/ 健診センター（=健診転送）/ 病児保育診察（=代表電話案内）/ わからない（=通常進行）」と特殊なので、診療科 hearing → CMR で分岐先を決める二段構成にする。
- サブフロー（患者氏名 / 生年月日 / 電話番号 / 診察券）は Jump to Flow 経由。**診察券は無し有り両方とも次は連絡先電話番号聴取**へ進む（main の `診察券無し` `診察券有り` どちらも次が `連絡先電話番号聞き取り` → 分岐意味なし、独立ブロック化不要）。
- 時間判断.yaml（祝日チェック → 日付分岐 → 平日/土曜の時間判定 → 年末年始）→ date_of_call_classifier ブロック。**addCurrentDate: Yes 自動付与対象**。
- entity_classification.md / entity_stt_patch.json / entity_openai_synonyms.md は scaffold/properties が自動取り込み。director は転記不要。

### 施設固有の特殊ルール
- **受付時間**: 日曜祝日除く、月〜金 8:00〜16:00、土曜 8:00〜11:00（湘南藤沢の「月〜金 9〜16・土 9〜14」と異なる）。
- **代表電話**: `072-993-8501`（5 回繰り返しで案内、全代表案内ルートで使用）。
- **健診センター直通**: `072-993-6505`（健診センター案件で 11 時前にアクセスした場合、「本日予約の方は代表へ、それ以外は平日 11〜16 時に健診直通へ」と案内。本日代表電話 / 平日健診直通 の **2 番号併記**）。
- **年末年始**: 12/31〜1/4 は休止、1/5 8 時から再開。`時間判断.yaml` 内で「1/4 前の祝日 → 年末年始 TTS / 12/30 当日 → 通常時間判定 + 年末年始 fallback」の段階分岐あり。**12/30 が稼働判定の特異日**。director 判断で date_of_call_classifier 内に集約するか、後段 CMR で分けるかは元資料の段階分岐を尊重。
- **歯科・口腔外科は時間帯ルーティングが複雑**:
  - 8:30 前 → 「8:30 以降にかけ直し」案内 → 終話
  - 8:30〜12:30 → 歯科転送（`API_DRJOY_1` → `歯科転送` サブシナリオ）
  - 12:30〜13:30（昼休み） → 「13:30 以降にかけ直し」案内 → 終話
  - 13:30 以降 → 歯科転送
  - **時間外（時間判断 NG）→ 歯科夕診日チェック（隔週火木交互に実施）** → 16〜17 時は切断、17〜19 時は「歯科口腔外科のお問い合わせは代表電話へ」案内 → 切断。隔週火木交互の判定ロジックは元資料では「サブシナリオ呼び出し（カレンダー Script）」扱い。**Script モジュールで実装、director 判断で内部仕様定義**。
- **健診センターも時間帯ルーティング**:
  - 11 時前 → 「本日予約は代表 0 7 2-993-8501 / それ以外は平日 11〜16 時に直通 0 7 2-993-6505」案内 → 終話
  - 11 時以降 → 健診転送（`API_DRJOY_2` → `健診転送` サブシナリオ）
- **病児保育診察 → 代表電話案内**（「病児保育診察は代表電話へ」固定 TTS）。
- **「分岐なし診療科」リスト**（診療科.yaml `01_00_分岐なし診療科` エンティティ参照、内科 / 外科 / 整形外科 / 産婦人科 / 眼科 / 脳神経外科 / 循環器 / 消化器 / 耳鼻咽喉科 等 60 科以上）→ **全て代表電話案内に集約**。八尾は AI で受け付ける診療科は無く、ほぼ全件代表案内が前提。
- **検査名リスト**（CT / MRI / 大腸ファイバー等 30 種以上）→ こちらも代表電話案内。
- **当日への予約変更は不可**: 予約希望日.yaml の `当日は不可` 箱 → Y_N サブフロー → 「明日以降の予約変更でよろしいですか?」Yes / No 確認。No なら終話、Yes なら再聴取。
- **予約変更時は「予約日（現在）」と「予約希望日」を 2 回聴取**（予約日.yaml + 予約希望日.yaml）。予約キャンセル時は「予約日（現在）」のみ。
- **DTMF 受付**: 用件聴取（1=変更 / 2=キャンセル / 3=確認 / 4=歯科口腔外科）と Yes/No 確認、予約日聴取（シャープ/アスタ + 4 桁数字 → 過去日なら +1 年確定）。
- **終話直前に「終話電話番号確認」サブフロー**: 入電番号取得済み（=固定/携帯入電）なら「順番に折り返し対応」と案内して終話。**入電番号未取得（非通知 等）の場合のみ折り返し連絡先を再聴取**。湘南藤沢のような「携帯/固定で終話 TTS 分岐」ではなく「入電番号取得可否で分岐」。Gen3 では context_match_router で `入電番号取得` flag を見て分岐 → 未取得時のみ追加 hearing 1 段。
- **聞き取れず終話の TTS**: 「うまく聞き取ることができませんでした」→ 終話電話番号確認 → 終話。3 回失敗時は専用 termination ブロックに集約。
- **予約キャンセル特定診療科 TTS**: `予約キャンセルの流れ（特定診療科）` 箱に「ご予約のキャンセルを承りました。必要に応じて翌診療日までに職員より折り返しご連絡をさせていただく場合がございますのでご了承ください。また、1 か月以内にご連絡いただけましたら予約変更が可能です」固定 TTS あり。**予約キャンセル時の標準終話文言として使用**。
- **復唱（echo_back）対象**: 用件（4 種）/ 診療科（歯科口腔外科のみ）/ 予約日 / 予約希望日。氏名・生年月日・電話番号・診察券はサブフロー内で完結。
- **「もしもし」「もう一度」フォールバック**: 用件聴取で「もしもし」発話 → 1 回目「はい、聞こえております」/ 2 回目以上「はい、もしもしと聞こえております」と回数別 TTS。「もう一度」発話 → 「再度ご案内いたします」→ 用件聴取に戻る。Gen3 では prompter SKILL_A の類義語例に統合（director 判断、独立ブロック不要）。

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `API_DRJOY` の 3 箱は Gen3 では opening 冒頭に吸収。明示ブロック化不要。
- 「セリフ: none」かつ振り分けのみの「箱」（`用件の確認 / _1`、`用件を判断 / _1 / _2`、`用件セット / _1〜_4`、`完了フラグ`、`診察券有り / 診察券無し`、`時間外接続`、`電話切断へ`、`flag1〜3`、`代表案内分岐`、`分岐なし診療科セット / 検査名セット`、`歯科入電時間分岐`、`入電時間分岐`、`健診入電時間分岐`、`聴取内容チェック`、`日付チェック_1〜3`、`月チェック / 日チェック / 月の置換 / 日の置換`、`数字を日付に / 0 桁詰め`、`シャープトーン（アスタ）`、`予約日セット（DTMF）`、`日付比較`、`プラス 1 年後で確定 / そのまま確定`、`生年月日の判定 / 生年月日の取得` 等多数）は Gen3 で独立ブロック不要 → 前段 next 書き換え or context_match_router の一部として吸収。
- `判定: エンティティ含む` で参照される値（健診センター / 歯科口腔外科 / 病児保育診察 / 分岐なし診療科 / 検査名 / 肯定 / 否定 / もしもし / もう一度 / DTMF 等）→ entity_classification.md / entity_stt_patch.json / entity_openai_synonyms.md にすべて自動分類済み。director は転記不要。
- `用件聞き取り.yaml` に **2 段階の確認構造**あり: 1 回目の聞き取り（`用件の確認`）→ 用件確定後 echo_back（`予約変更へ進む` 等で「ご予約の変更でよろしいですか?」）→ Yes なら確定 / No なら再聴取（`否定` → `用件の確認へ` 戻し）。**Gen3 では用件 hearing → 用件別 echo_back hearing → 各用件フローに分岐するのが素直**。
- 用件聞き取り.yaml の `予約変更へ進む_1`（「ご予約の確認でよろしいですか?」）と `新規予約へ進む` `新規予約へ進む_1` の TTS は **歯科 / 健診向けの専用復唱**。歯科は「歯科・口腔外科でよろしいですか? よろしければ「はい」、歯科口腔外科ではない場合は「いいえ」とお答えください」と非対称な文言。**director は echo_back 文言を 4 用件で個別に書く**。
- 診療科.yaml の `flag2` 経路（歯科口腔外科）は診療科再聴取の確認文言「歯科・口腔外科でよろしいですか?」を含む二段階確認。Gen3 で診療科 hearing → 歯科 echo_back → CMR 分岐の構造に。**用件聞き取り.yaml で歯科を選んだ後と、診療科.yaml で歯科を選んだ後の合流先が同じ（歯科転送）**であることに注意（用件分岐先と診療科分岐先がクロス）。
- **`歯科夕診日チェック（隔週火木交互に実施）` は元資料で「サブシナリオ呼び出し」扱い** → Gen3 では Script モジュールが必要（隔週判定）。元資料に内部ロジックの定義がないため、**director 判断で「直近の火曜/木曜が偶数週か奇数週か」のロジックを定義する必要あり**。仕様不明の場合は確認レポートで BLOCKER を上げる。
- **`祝日チェック` も「サブシナリオ呼び出し」扱い** → カレンダー Script モジュールが必要（docs/brekeke/モジュール選定ガイド_v2.md §3.8 参照）。
- 予約日.yaml と予約希望日.yaml の **DTMF 4 桁機能**（シャープ/アスタ + 4 桁 → 過去日なら +1 年で確定）→ Gen3 で SKILL_C 自由テキスト + 後段 datetime 正規化で吸収するか、別途 dtmf 並走させるかは **director 判断**。
- `終話電話番号確認.yaml` は **入電番号取得済みなら「順番に折り返し対応」案内のみで終話、未取得なら折り返し連絡先を聴取** という条件付きフロー。湘南藤沢の「携帯/固定で終話 TTS 分岐」とは別物。**`入電番号取得確認` 箱は context（システム提供の発信者番号取得 flag）参照で分岐**するため、Gen3 では context_match_router + hearing 1 段の構成。
- フリーの内容確認 / フリーの変更理由 サブフローは「他に予約を変更（する内容はありますか）」「変更/キャンセルの理由をお話ください」と **用件分岐で TTS が変わる**（変更時は「他に予約を変更」、キャンセル時は「他に予約をキャンセル」）。Gen3 では用件 context を hearing の prompt 内で `<% var %>` で参照するか、用件別に hearing ブロックを 2 本立てるかは **director 判断**。
- `Y_N.yaml` は用途汎用の「明日以降の予約変更でよろしいですか?」確認用 Yes/No サブフロー → Gen3 では当日希望時の Yes/No 専用 hearing として 1 ブロック展開（subflow 化するか hearing 直書きかは director 判断、shared subflow なら 1 本で複数箇所から参照可）。
- 用件 `予約以外`（お見舞い / アクセス / くすり / 質問 / 処方箋 / 書類 / 症状 / 診断書 等）→ 「予約専用ダイヤル、ご予約以外は代表電話へおかけなおしください」固定 TTS で終話 → **`END_予約以外の用件` 専用 termination ブロック必要**。代表案内（5 回繰り返し）には流さない。
- ContextMatchRouter.reference_module で **subflow を参照する場合は scaffold が付ける `jump_{step 名}` prefix を使う**（subflow ブロックの step 名そのもの、例: `電話番号聴取` ではダメ）。湘南藤沢徳洲会で取り違えが発生したので、八尾でも同じミスをしないよう注意。
- **CMR conditions は必ず最後に `match: "other"` + next 明示**（catch-all）。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字（①②③…）等の環境依存文字は禁止。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。`<? $診療科の読み ?>` 等の Gen1 構文は Gen3 の `<% var %>` に変換。
