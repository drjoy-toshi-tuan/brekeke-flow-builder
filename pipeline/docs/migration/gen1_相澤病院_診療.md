### 移管方針
- 入電フローは「入電時間判断 → 救命救急センター誘導 → 用件聴取（診察予約/変更/キャンセル/予約確認/健診案内）→ GW・年末年始の特殊ガイダンス分岐 → 診療科聴取 → 紹介状有無 → 患者氏名 → 生年月日 → 連絡先電話番号 → 折り返し案内 → 終話」をベースに段階移行
- 用件分類は「診察予約 / 変更 / キャンセル / 予約確認 / 人間ドック・健診（代表案内）」の5値。subflow『用件』内で **DTMF 1/2/3** の分岐も同居しているため Gen3 では OpenAI hearing + DTMF 並行受付として分岐後 ContextMatchRouter に集約

### 施設固有の特殊ルール
- **GW 特別ガイダンス**: 4/29～5/6 に該当する入電は「ゴールデンウィークのため、5月7日以降の、2営業日以内に折返し」と案内。`GW日付分岐` を date_of_call_classifier で実装し用件確定後に分岐（**年末年始は acceptance_times 集約で吸収だが、GW は折返し日が変わるため独立 date_of_call_classifier で実装する施設特例**）
- **新規予約は通院中診療科のみ**: 用件が「診察予約」の場合、紹介状確認の前に「現在、通院中の診療科のご予約ですか?」を確認し、Yes は受付、No は紹介状有無確認に進む（subflow『用件』内『新規予約?』）
- **紹介状なし時の選定療養費案内**: 「8,000円をご負担いただいております」を必ず読み上げ。金額は施設指定なので prompter で改変禁止
- **救命救急センター誘導**: 全通話の最上流で 0263-36-9999 を案内。announcement 後に「お話を続ける/番号控えて切る」の入力待ちで分岐（commubo の『救命救急センター誘導』はクッション役）
- **診療科の多段絞り込み**: 「呼吸器」「脳神経」「循環器」「腫瘍」「精神」「放射線」「小児」「スポーツ」「放射線治療科(陽子線/トモ)」の **9系統が外科/内科または下位科への二択追い質問** を要求。ブロック数を抑えるため、Gen3 では「分岐あり診療科」hearing → CMR で系統判定 → 系統別 hearing サブフロー（_呼吸器/_脳神経/...）の二段構成にする
- **対応外診療科の振り分け**: 救急科 → 0263-36-9999 / 健康センター(ドック・健診) → 0263-34-6360 / 放射線科・内視鏡・PET → 0263-33-8600 / 心療内科・美容整形 → 「お近くの医療機関を」アナウンス後切断 / 麻酔科・皮膚科・精神神経科 → 再診のみ案内
- **代表電話への案内番号は 0263-33-8685**（外来予約センター、月～金 9-16時）。15回繰り返し読み上げで終話する仕様（リダイヤル猶予）。Gen3 では announcement に「番号は0263-33-8685です。繰り返します。…」を最大15回ループの context_match_router で実装する想定だが、scaffold デフォルトの 3回読み上げ＋無言切断で代替してよいか **承認待ち（要 CS 確認）**
- **転送先**: 月～金 9-16時の稼働時間内のみ Blind Transfer (0263-33-8685)。それ以外は番号案内のみで終話
- **携帯電話番号は別途追加聴取**（緊急連絡用）。Gen3 では subflow_phone を 2 回直列で呼び、2回目は「持っていない」を許容
- **STT 辞書は超大規模（777エントリ）**: 診療科 entity_classification.md を `additional_words` に展開。診療科テンプレ流用は不可（施設固有の科名・誤認識バリアントが多すぎる）→ basic_info.stt で `additional_words` 直書きを許容

### director が見落としやすいポイント
- **GW分岐とacceptance_times の二重**: GW は date_of_call_classifier で別途分岐させ、acceptance_times の通常時間外とは独立して扱う（GW は折返し日が変わるため）
- **『用件を判断_2』(変更/キャンセル+新規無し予約変更のみ)**: 通院中診療科以外で「診察予約」と回答した場合に「新規のご予約は承っておりません」と案内→『理由へ』に流すフォールバックがある
- **『健診⇒受診希望か確認』分岐**: 用件を「人間ドック・健診」と回答した場合「健診を受診後に、届いた紹介状をお持ちで、当院での診察をご希望ですか?」と確認し、Yes は通常フロー継続、No は健診センター(0263-34-6360) 番号案内へ
- **『予約確認』は予約センターへ案内**: 「予約確認」用件は AI で完結させず 0263-33-8685 番号案内のみで終話。reservation_check 終話パターンとして termination_patterns に登録要
- **『紹介状あり』は『それでは、紹介状の封筒に記載されている診療科』アナウンスを挟む**: 紹介状なしのフローと TTS 分岐させる必要あり。announcement 文言を 2系統用意
- **『放射線治療科』の二段確認**: 「放射線治療科ですね？陽子線治療センターとトモセラピーセンター、どちらでしょうか？」と二択でさらに絞り込む
- **case 数が多い echo_back**: 「<診療科の読み>でよろしいですか?」は分岐あり/なしどちらも復唱必須。動的TTS構文は `<% var %>` で揃える
- **生年月日サブフロー**: 0129更新版を採用（最新版）。古い `4_14 削除 予約日聞取り.yaml` `4_14 削除 理由聞き取り.yaml` は移管対象外（ファイル名通り削除予定の旧版）
