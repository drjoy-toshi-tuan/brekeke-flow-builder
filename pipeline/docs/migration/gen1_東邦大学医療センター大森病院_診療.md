### 移管方針
- Gen1 commubo の 13 yaml をベースに Gen3 標準ブロック構成へ変換。基本 12 項目の director デフォルトに従う。
- 用件は **4 分岐**: 「初診（要紹介状）」「予約変更」「予約キャンセル」「セカンドオピニオン」。`hearing_yoken_common` 流用。
- **携帯/固定の出し分けは Gen3 仕様で削除**: Gen1 は終話 TTS を「○○(携帯)」と「○○」の 2 系統で 8 パターン用意していたが、Gen3 では SMS 送信は IVR プロパティ側 + Dr.JOY 側で着番から判定する想定なので、終話 TTS は 1 本に統一する（director 判断: 念のため customer_doc が無いため浜口確認）。
- **GW 期間判定（5/1〜5/7 の日時分岐 + GW挨拶/GW診療/GW変更）は全削除**。Gen3 では date_of_call_classifier + acceptance_times の標準処理に統一。
- 個人情報サブフロー（氏名/生年月日/診察券/電話番号）は **必ず Jump to Flow で外出し**。インライン展開禁止。

### 施設固有の特殊ルール
- **紹介状必須**: 初診で「紹介状なし」と答えたら `END_紹介状なし` 終話 TTS（HP 案内）へ。Yes/No サブフロー後の分岐で実装。
- **3 診療日制約**: 初診で受診歴なし（=新規予約）の場合「3 診療日以内は AI 受付不可」と案内 → 外来予約センター誘導。
- **セカンドオピニオン**: 「必要書類お手元にあるか」Yes/No 分岐 → No なら受付不可で終話、Yes なら氏名・生年月日・電話番号・診療科聴取 → `セカンドオピニオン` 終話。郵送案内。
- **診療科分岐（初診）の特殊診療科 7 種は終話扱い**: リハビリ / リプロ泌尿器 / がんセンター / 緩和ケア / 臨床遺伝診療部 / 放射線 / 総合感染科 → それぞれ専用 TTS で代表番号誘導後に終話。
- **診療科分岐（初診以外）の特殊診療科 10 種は受付不可**: リハ / リプロ婦人科 / がんセンター / 緩和ケア / リプロ泌尿器 / 放射線 / 小児循 / 産科 / 臨床遺伝診療部 / 総合診療感染症科 → `代表案内（初診以外）` TTS で終話。
- **キャンセル詳細**: 「該当診療科を全てキャンセル（=1）」「今回の予約日のみキャンセル今後希望（=2）」の 2 分岐 → 2 のみ「希望日決まっているか」を追加聴取（決まっていれば変更フロー、未定なら連絡先聴取後終話）。
- **紹介元医療機関名 + 電話番号** の連続聴取（初診のみ）。`紹介元医療機関名.yaml` は機関名 → 同一サブフロー内で続けて電話番号も聴く構造。Gen3 では 2 ステップ hearing で。
- **代表番号 2 種**: 外来予約センター直通 `03-5763-5323`（聴取失敗/初診・セカンド時）と病院代表 `03-3762-4151`（聴取失敗/変更・キャンセル時）。半角/全角ハイフン混在しているので IVR プロパティで `03-5763-5323` 半角ハイフンに正規化。

### director が見落としやすいポイント
- **エンティティ「分からない」「肯定」「否定」が yaml ごとに微妙に値違い**（main/用件/紹介元/予約希望日/連絡先で分岐）。Gen3 では `hearing_yesno_common` `hearing_unknown` テンプレに集約 → 個別差分は `additional_words` で補う（曖昧時は OpenAI フォールバック）。
- **DTMF 対応**: 用件選択（1〜4）、予約希望日（4 桁数字）、キャンセル詳細（1〜2）が DTMF 受け付け。Gen3 標準 hearing で DTMF 同時受付フラグ ON。
- **「もしもし」連発カウント**: Gen1 は countもしもし で 2 回目以上の TTS を分けているが、Gen3 では retry_count + retry TTS の標準処理に統一可能（director 判断: 1 回目「はい、聞こえております」/2 回目以降「もしもし、と聞こえております」のニュアンス差は properties で吸収）。
- **連絡先聴取の「入電番号で OK?」分岐**: 入電番号取得済かチェック → 取得済なら復唱→Yes/No、未取得なら直接聴取。phone_number サブフローの標準仕様で対応可。
- **稼働前アナウンス（「4月21日零時より」）は無視**。Gen1 移管時の旧文言。
- **転送（聴取失敗の転送①/②、フラグ4/フラグ7設定）は削除**。Gen3 では call_transfer モジュールに統一、転送先は IVR プロパティ。
- **モジュール名の丸数字禁止**: 「聴取失敗の転送①/②」をそのまま流用しないこと。`jump_transfer_failed_first` 等 ASCII 化、または「聴取失敗時転送1/2」へ。CMR.reference_module は subflow 参照時 `jump_{step名}` 形式。CMR conditions 末尾に `match: "other"` + next 明示。
