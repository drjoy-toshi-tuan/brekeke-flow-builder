### シナリオ概要

患者または代理人が千葉徳洲会病院の予約専用AI電話に架電し、当日翌日除外チェック → 個人情報聴取 → 用件（予約/変更/キャンセル/確認）に応じた詳細聴取を行い、折り返し受付を自動化する。予約不可診療科は予約センター（047-774-0489）、健康診断は健診センター（047-774-0385）へ誘導する。

---

### 設計方針

- **フロー構成**: multiflow（メイン + サブフロー群）
  - 参考シナリオ（診療1）は Jump to Flow サブフロー方式だったが、**診療2 では scaffold の決定論インライン展開（氏名/生年月日/電話番号/診察券番号 = `slot` / `card_number` 型）に切り替える**。Jump to Flow を使わない新規構築として設計する。
  - RAGサブフロー（最後の質問）は参考シナリオと同様にパターン2（全終話前）で存続。

- **入電者種別**: 患者本人または代理人。区別はしない。
  - PatientName / Phone / DateOfBirth / MedicalCard は scaffold インライン展開を使用（Jump to Flow 禁止方針）。

- **用件分岐方式**: Pattern B（後段 ContextMatchRouter）
  - 用件確認（4択: 予約/変更/キャンセル/確認）の OpenAI 出力値を保存し、後段の診療科チェーン合流点・smsFlag 分岐で ContextMatchRouter が参照する。
  - 用件確認直後ではなく各ルート（予約/変更/キャンセル/確認）にそれぞれ遷移するため、分岐自体は OpenAI 出力に直接対応する。

- **受付時間判定**: acceptance_times をデフォルト配置（具体スケジュールは Dr.JOY 側で設定）。

---

### 施設固有の特殊ルール

1. **個人情報聴取順序が逆**: 氏名 → 生年月日 → 電話番号 を **用件確認より前** に聴取する（通常の「用件→個人情報」とは逆）。診療1 から引き継ぐ最重要ルール。

2. **生年月日は復唱禁止**: 他施設デフォルトと異なる。scaffold で復唱フラグを off にすること。

3. **電話番号のみ復唱あり**: 3項目の中で唯一の復唱許可項目。

4. **新規ルートは紹介状必須**: 紹介状なし → 即終話（END_紹介状無し）。紹介状ありのみ病名 → 診療科聴取に進む。

5. **予約不可診療科（6科）**: 眼科 / 放射線治療科 / 麻酔科 / 緩和ケア科 / 小児科 / 皮膚科 → 予約センター案内。**婦人科は予約可（予約センター送りにしてはいけない）**。

6. **健診キーワード検出が2か所**: 診療科聴取（予約/変更/キャンセルルート）と確認内容聴取（確認ルート）の両方で健診キーワードを検出し、健診センター案内に分岐する。

7. **smsFlag の分岐**: 新規/再診=1、変更/キャンセル=2、確認=3。ContextMatchRouter で classification 値を参照。

8. **当日翌日ガード**: 冒頭（opening 直後）で Yes/No 判定。「はい」→ 予約センター電話番号を読み上げて終話。

9. **連絡方法聴取は携帯電話のみ**: additionalPhoneNumber が携帯判定（MOBILE）の場合のみ聴取。固定/OTHER はスキップ。

### director が見落としやすいポイント

- **診療科チェーンは2セット必要**: 予約用（「受診を希望される診療科」）と変更/キャンセル用（「予約票に記載されている診療科」）でTTS文言が異なる。同じ ContextMatchRouter ロジックでも別ブロックとして配置すること。

- **診察券番号の聴取タイミング**: 変更/キャンセル/確認は用件確認直後。再診は受診歴確認後。新規（紹介状あり）は不要。計4ルートで出現タイミングが異なる点を整理して設計書に落とすこと。

- **予約日チェーンの分岐**: 変更 → 予約日 → 希望時期、キャンセル → 予約日 → 理由、と予約日後の遷移先が用件で異なる。ContextMatchRouter で classification を参照する合流構造が必要。

- **確認ルートに診療科聴取はない**: 確認ルートは「診察券番号 → 確認内容」のみ。診療科は聴取しない（customer_doc に明記）。健診キーワードのみ別途ガード。

- **BLOCKER 化すべき不明点**:
  - office_id（未確認）
  - 受付時間（曜日・時間帯の具体値）
  - 「診療2」が診療1と同一フローの再ビルドなのか、別電話番号・別用途フローなのかの意図確認（フロー名・050番号が未確定）
  - 診療1との共存・置き換え関係（同じ施設に2フロー存在する理由）
