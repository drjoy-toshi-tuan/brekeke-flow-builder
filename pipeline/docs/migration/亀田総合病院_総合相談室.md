### シナリオ概要

総合相談室（一次受けは MSW・PSW・看護師の専門職）の直通番号 04-7099-1223 への受電を AI が一次応答し、発信元（患者本人・家族／連携医療機関／行政）と用件を判定して、相談室（内線 7177）・大代表への転送、伝言・折返し受付、定型案内、予約センターへのかけ直し案内を行う電話。

### 設計方針

- **大原則（顧客意向 2026-06-10 浜口確認）**: 総合相談室は「**相談室の業務に関係する電話のみ受け付け、相談室に直接関係しない電話（特に予約関連）は受け付けずに弾く**」。弾き先は 予約関連→予約センターかけ直し案内終話、大代表の用件→大代表転送。分類プロンプト（prompter）は予約系発話を広めに検知して確実に弾く方向でチューニングする。伝言・折返しで受けるのも相談室業務の範囲のみ（予約系の伝言は受けず予約センター案内へ）。
- **1flow 推奨**（director 判断）。全パスが「聴取→転送/折返し受付/案内→終話」の単線収束構造。
- 入電者種別は **患者本人・家族／連携医療機関／行政の 3 分類**（患者本人だけではない）。患者氏名確認（E1/F_id）は PatientName サブフロー充当候補。担当者氏名のヒアリング（D1/D2/L_q/G_q）は患者名ではないため hearing で扱い、**context 保存して折返し時にスタッフが使う**。
- 用件分岐方式: 選択肢列挙型の単発分岐で **Pattern A（OpenAI 直後分岐）で足りる見込み**。director 判断。
- **取次の扱い（2026-06-10 浜口方針確定）**: 担当者名→内線/PHS の対応表は存在しないため、**「担当者につなぐ」（E2/F_connect/L_connect/G_connect）の個別動的転送は今回実装しない**。担当者特定可パスは担当者名を context 記録した上で**伝言・折返しパスへ合流**（折返し時にスタッフが担当者へ繋ぐ運用）。
- 固定転送先は call_transfer で維持: 相談室 7177（特定不可系の受け皿）・大代表 04-7092-2211（誤着信）。転送実装方式（内線/050）はプロパティ。
- **予約センター（2026-06-10 浜口方針確定）**: 番号未受領のため転送しない。t_resv（予約の件・誤着信）・F1「受診に関すること」（旧 F3）とも**「予約センターへおかけ直しください」ガイダンス → 終話**。案内文言に番号を含めるかは番号受領後にプロパティ差し替え（TODO）。
- **伝言・折返しパス新設（2026-06-10 浜口方針確定、提案サマリ 用件B=受電の33%）**: ①用件分岐の catch-all（その他・判定不能）の受け皿、②担当者特定可パスの終端、の両方から合流。構成は 伝言・用件内容 hearing（自由発話）→（氏名未聴取なら氏名）→連絡先確認（Phone サブフロー、固定/携帯の扱いは director 判断）→「スタッフから折返し」終話。詳細ブロック構成は director 判断。
- **定型案内パス新設（2026-06-10 浜口方針確定、提案サマリ 用件C=3%）**: 場所・受付時間・番号案内等の定型問い合わせを用件分岐で検知 → announcement（案内文言はコンテンツ未受領のためプロパティ placeholder、TODO）→ 終話。検知カテゴリの粒度は director 判断。
- 元資料に**営業時間外の扱いが無い**。Date of Call Classifier / 営業時間判定は勝手に追加せず、要否を確認レポートで顧客確認。

### 施設固有の特殊ルール

- 最上流で発信元 3 分類 → 各系統とも「担当者はわかりますか」聴取に収束する構造。
- **用件 3 分岐（相談／大代表／予約）は患者本人・家族系統のみ**。大代表は誤着信扱いで即転送。連携医療機関・行政は用件分岐なしで直接「担当者はわかりますか」。
- 患者系の「総合相談室への相談」→ 入院患者／外来患者／新規の相談の 3 分岐（C0）。
- 担当者特定可: 患者氏名を確認 → 担当者名 context 記録 → 伝言・折返しへ（個別転送しない、上記方針）。※患者氏名確認は写真では入院側のみだが drawio は外来側（F_id）にも対称配置（drawio 側の拡張）。
- 担当者特定不可: 入院→相談室 7177／外来→F1／連携医療機関・行政→相談室 7177。
- **F1 の位置づけと配線（2026-06-10 浜口確定・顧客意向裏付け）**: F1「相談予約ですか／受診に関することですか」は「行き先不明の入電を、相談室業務（相談予約→7177）か予約関連（受診→予約センターかけ直し案内終話＝弾く）かに仕分けるフィルタ」。**new_q（新規の相談）→ F1 と 外来・担当者特定不可 → F1 の両方から合流**させる（顧客イメージと drawio の両解釈を包含、矛盾なし）。

### director が見落としやすいポイント

#### 解消済み（2026-06-10 浜口確定 — 再 BLOCKER 化しない。詳細は「設計方針」参照）

- TTS 文言不在 → **director ドラフト起草で確定**（確認レポートに「全文言ドラフト・顧客確定前提」明記）。
- F3/t_resv 予約センター → **かけ直し案内終話で確定**（番号読み上げは番号受領後プロパティ差し替え＝TODO）。
- 伝言・折返し（用件B）／定型案内（用件C）パス欠落 → **新設で確定**（定型案内コンテンツはプロパティ placeholder＝TODO、分岐位置の最終形は確認レポートで顧客確認）。
- 内線表未受領 → **担当者個別取次を今回実装しないことで確定**（担当者名は context 記録のみ）。
- status / sms_flag → **TODO 降格**（本フローに SMS 要素なし→sms_flag は既定 0、status はテンプレで仮置きし確認レポートで運用確認。BLOCKER にしない）。
- new_q「新規の相談か」の遷移先・F1 帰属の両解釈 → **両合流配線で確定**（new_q→F1 と 外来・特定不可→F1。顧客意向「相談室業務のみ受け、予約関連は弾く」が裏付け。詳細は「施設固有の特殊ルール」参照）。確認レポートには「両合流で配線した」旨を記載し病院に最終確認。

#### 残 BLOCKER / 要確認（確認レポートに必ず載せる）

- **残 BLOCKER なし**（2026-06-10 時点。以下は構築を止めない要確認/TODO）。
- 非通知・国際入電ガード・営業時間外動作の記載なし → 図に忠実に「入れずに」構築し、要否を確認レポートで顧客確認。
- 発信元 3 分類・用件分岐の catch-all → 伝言・折返し受け皿へ（上記方針）。ただし**予約関連の発話はどの分岐で出ても予約センター案内へ弾く**（顧客意向）。発信元分類の無言/判定不能の扱いは director 判断＋確認レポート記載。
