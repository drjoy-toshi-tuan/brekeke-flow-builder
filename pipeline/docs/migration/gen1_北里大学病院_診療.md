### 移管方針
- Gen1「箱」→ Gen3 9 ブロック型へ変換。元資料は main.yaml + サブ yaml 16 本。
- opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「お電話ありがとうございます。北里大学病院の、予約専用、AI 電話です」。
- **用件分岐は Pattern A**（用件聞き取り OpenAI 直後に 4 分岐: 診察予約 / 予約変更 / 予約キャンセル / 予約確認 + 折り返し対応）。echo_back（診察予約復唱 / 予約変更復唱 / 予約キャンセル復唱 / 予約確認復唱）を各分岐直後に配置。
- **「初診 / 再診」の段階分岐は Pattern B（後段 ContextMatchRouter）必須**。Gen1 は新規予約後に「現在通院中の診療科ですか？」（再診/新規切替）を挟むため、用件 echo_back の後で context_match_router を組む。
- 診療科聞き取りは元資料が **新規用（新規診療科.yaml）と再診/変更用（5_28 診療科（再診変更用）.yaml）の二系統** に分かれる。Gen3 では同一 hearing ブロックに統合し、用件 context で出口振り分け（director 判断）。
- 診療科ハイラルキ分岐（呼吸器内科/外科・消化器内科/外科・脳神経内科/外科・産科/婦人科・甲状腺内科/外科・心臓内科/外科）は **OpenAI 出力 → ContextMatchRouter で「次の質問が必要かどうか」を判定 → 必要なら追加 hearing ブロックで内科/外科を聞き直す** 構造に再構成。
- 入電時間判断.yaml（祝日チェック + 12/27〜1/4 年末年始 + 平日 8:30〜17:00 + 7 つの祝日例外）→ date_of_call_classifier ブロック。**addCurrentDate: Yes 自動付与対象**。
- 予約日（当日チェック）.yaml は **当日予約 NG ガード**専用 → date_of_call_classifier 直後に当日チェック script ブロックで実装。当日該当 → 代表電話案内へ termination 直結。
- サブフロー（患者氏名 / 生年月日 / 電話番号 / 診察券）は Jump to Flow 経由でサンプル JSON 完コピ。
- 変更/キャンセル理由（変更_キャンセル理由.yaml）と確認内容（確認内容の聞き取り.yaml）は **どちらも自由テキスト hearing**（output_format=text、SKILL_C 自由テキスト）。発話あり/5 秒無言で打ち切り、上限 2 回再質問。

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `API_DRJOY` の 3 箱は Gen3 では opening 冒頭処理に吸収。明示ブロック化不要。
- 「セリフ: none」かつ振り分けのみの「箱」（`用件の確認 / _1 / _2 / _3`、`用件を判断 / _1 / _2 / _3 / （リハ用）`、`着信番号判別 / _1`、`変更/キャンセル-着信番号判別`、`確認-着信番号判別`、`新規-着信番号判別`、`完了フラグ`、`連絡先flag / _1`、`連絡先エラーflag / _1`、`携帯/固定の分岐`、`診察券無し / 有り`、`次へ_2 / _3 / _4`、`婦人科判別`、`SMSフラグ（折り返し用）`、`生年月日へ` 等多数）は Gen3 で独立ブロック不要 → 前段の next 書き換え or context_match_router の一部として吸収。
- **用件「予約変更」「予約キャンセル」が複数経路で出現**する点に注意。`用件を判断`（メイン）/ `用件の確認`（メイン）/ `用件を判断`（5_28 診療科.yaml）/ `用件を判断`（変更_キャンセル理由.yaml）/ `用件の確認`（確認内容の聞き取り.yaml）でそれぞれ条件分岐。**Gen3 では用件 context を一度確定したら以降は context_match_router で参照する**ことで重複聴取を避ける。
- **用件「折り返し（病院から着信があった方）」は最初の用件聞き取りでのみ取得可**。DTMF 4 / エンティティ「折り返し」（電話の/病院から/着信があった等）に該当 → END_折り返し → 連絡先電話番号聞き取り（折り返し用）→ 折り返し予約センター転送。Gen3 では DTMF 並走 + OpenAI 用件分類で吸収（SKILL_C / SKILL_A 併用、director 判断）。
- **「予約以外」分岐**は用件聞き取りで「入院 / 面会 / 予約じゃない」発話 → 緊急救急復唱_1（「予約以外の用件でよろしいですか？」）→ Yes なら 用件を確認_1（外来調剤 / 緊急救急 / 代表番号 / 入院会計 / 内視鏡受付 / 核医学受付 / 文書窓口 / 外来会計 / 栄養相談 / 歯科 / 遺伝診療部 / 放射線診断科 の 12 個に分岐 → 各代表電話案内）。**「予約以外」用件は代表電話案内でクローズ**、ボイスボットでは予約しない。Gen3 では context「予約以外」でフローを別レーンに切る設計が要る。
- **`判定: エンティティ含む` で参照されるエンティティ値** は `entity_classification.md` / `entity_stt_patch.json` / `entity_openai_synonyms.md` に全件自動分類済み。director は転記不要、scaffold/properties が自動取り込み。
- **新規診療科.yaml の出口記号 ①〜⑦/⑩** は導出ロジックが複雑（紹介状 × 診療科 × 診察券 × 依頼書のクロス分岐）。Gen3 では context_match_router 1 個でクロス分岐を集約 + 各出口に termination_pattern を置く。
- **5_28 診療科（再診変更用）.yaml は 2025/5/28 リリースの再診/変更/キャンセル専用**で、新規診療科とは別エンティティ（00_00_01_呼吸器 等の階層エンティティ）を使う。Gen3 では片寄せ統合か、再診時のみサブフロー分岐かは director 判断（**現状 2 サブシナリオあるのは Gen1 制約による分割**で、Gen3 の用件 context があれば 1 hearing で済む可能性が高い）。
- **婦人科 予約希望日確認.yaml は新規予約「婦人科」確定後の追加質問**（3 日以内必要 Yes/No）であって、診療科聞き取りの一部ではない。Gen3 では婦人科 context 確定後の追加 hearing ブロックとして配置。
- **リハ種別.yaml は「言語リハビリ」かどうかの Yes/No 単問**で、リハビリテーション科確定後の追加分岐。Yes → 予約変更の流れ_1（折り返し連絡 1 週間程度）/ No → 通常リハ（再診/変更用と同等）。**リハ用件は新規予約 AI 不可**で、再診/変更経由のみ受付。
- **`着信番号判別` の「国内番号」は携帯/固定全部含む**（3 つの値 050-1707-0320 / 050-1707-4509 / 国内番号 で同じ「導入文言」へ流す）。050-* は AI 受付電話番号自体、国内番号は通知ありの一般通話、非通知のみ別レーン。
- **連絡先電話番号聞き取り yaml が 4 種並列**（連絡先電話番号聞き取り.yaml / _1.yaml / _2.yaml / 連絡先電話番号聴取.yaml）→ 用件・タイミングで使い分け。Gen3 ではすべて同一 PhoneNumber subflow に集約可能（director 判断、PatientName と同じ考え方）。
- **`電話番号区切り.yaml` は読み上げ用補助サブシナリオ**（電話番号 11 桁を「090-1234-5678」のように区切って TTS）→ Gen3 では TTS 動的値構文 `<% var %>` の readable 形式で吸収（director 判断、別ブロック化不要）。
- **`連絡先エラーflag` の二重使い回し**: 通常の電話番号取得失敗（連絡先エラーflag → 患者氏名聞き取り_1 へ）と、海外終話/婦人科転送失敗の派生（連絡先エラーflag_1 → 折り返しのため個人情報聴取へ）の 2 種。director は両方の経路で **同じ「連絡先取得失敗」サブフロー再呼び出し**になる点を context で見分ける。
- **「新規 → 用件確定後に「現在通院中の診療科ですか？」を聞いて再診/新規を分け直す」**フロー（用件聞き取り.yaml の `セットした用件の分岐_1`）は **用件 OpenAI 出力直後の二段階確認**として独自実装。Gen3 では SKILL_B Yes/No で「再診ですか？」を 1 hearing 追加。
- **終話パターンの分散**: END_切断 / END_切断_1 / 海外終話案内 / 年末年始終話へ / 折り返しの連絡終話_1 / 聞き取れないので終話 / 通話失敗（→ 転送）/ 非通知電話番号不完了アナウンス / 代表電話案内 / 各代表電話案内（12 系統）→ Gen3 では termination_patterns に集約、TTS は携帯（SMS 言及あり）/ 固定（言及なし）の director デフォルト 9 を適用。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。`<? $clinical_department ?>` `<? $予約日の読み ?>` `<? $診療科の読み ?>` のような Gen1 変数記法は Gen3 では `<% var %>` に変換。
- 患者氏名は PatientName サブフロー 1 枠に充当（patient 1 名のみ、入電者 = 受診者前提の設計）。
