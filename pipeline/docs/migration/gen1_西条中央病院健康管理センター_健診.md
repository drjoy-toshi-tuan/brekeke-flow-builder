### 移管方針
- Gen1「箱」構造 → Gen3 scenario_flow 9 ブロック型へ変換。
  - `API_DRJOY_CONTEXT_SETTINGS` / `API_DRJOY` / `挨拶` → opening + 冒頭_アナウンス（2 ブロック 1 セット）。冒頭文言「お電話ありがとうございます。西条中央病院の、健康管理センター専用、AI電話です。」
  - `日付判断` / `時間判断_1` / `時間判断_2` / `4/1　10時半まで` の時期判定箱 → acceptance_times + Script モジュール（2/28〜3/2 の AI 休止期間 + 4/1 10:30 以降の電話番号変更告知）。Gen3 では time-of-call 系前段に集約、addCurrentDate: Yes 自動付与
  - `氏名聞き取り.yaml` → PatientName サブフロー 1 枠（受診者本人前提、TTS 内も「受診される方のお名前」が中心）
  - `生年月日.yaml` / `連絡先.yaml` / `電話番号区切り.yaml` → subflow ブロック Gen3 標準サンプル JSON 完コピ
  - `用件.yaml` → hearing（generate_by_OpenAI、4 分岐: 新規 / 変更 / キャンセル / 問い合わせ）。直後 echo_back 4 本 → Pattern A、DTMF 1〜4 並列対応
  - `種類.yaml` → 新規ルート専用 hearing 4 分岐（人間ドック / 協会けんぽ / 定期健診 / その他の健診）+ 各 echo_back。**ただし `セットした種類の分岐` で「定期健診」が `用件の復唱（人間ドック）_1` に流れる箱マッピングのバグ**が元資料にある → Gen3 では正しく定期健診 → 「定期健診のご予約、でよろしいですか？」echo_back を作る
  - `変更.yaml` → 変更ルート専用 hearing 2 分岐（受診内容 / 日程変更）+ 各 echo_back。さらに `ドックメニュー` `健診メニュー` `バリウム検査` `内視鏡検査` 等のサブエンティティが定義されているが**用件分岐内では使われない流用残骸**
  - `予約日.yaml` → 変更/キャンセル ルートで使用。datetime hearing
  - `詳細聞き取り.yaml` → 自由文 hearing（新規 / 受診変更 / 受診内容変更 / 問い合わせ用に 4 種の入口 TTS あり、output_format: text）。新規時は「胃の検査やオプション等ご希望の内容」、変更時は「オプション追加や変更等」、問い合わせ時は「お問い合わせの内容」、キャンセルは経由しない
  - 携帯電話番号チェック → phonetype 分岐 1 系統（`締めあいさつ` のみ、文言「3 営業日以内に、担当者から折り返し電話、もしくは、SMS にてご連絡いたします」共通）
  - `終話` → 「お電話ありがとうございました。それでは失礼いたします。」 → END_切断
- 用件別の合流先（main.yaml `用件の分岐` / `用件の分岐_1` / `用件の分岐_2`）:
  - **新規**: 種類 → 詳細聞き取り → 完了フラグ → 用件の分岐_1 → 「健康診断の予約の申し込みを受付いたしました」 → 締めあいさつ → 終話
  - **変更**: 予約日 → 用件の分岐_2（変更 → 次へ_3 → 変更_1 → 詳細聞き取り）→ 完了フラグ → 「予約変更の申し込みを受付いたしました」 → 締めあいさつ → 終話
  - **キャンセル**: 予約日 → 用件の分岐_2 デフォルト分岐 → 完了フラグ → 「予約キャンセルの申し込みを受付いたしました」 → 締めあいさつ → 終話（**詳細聞き取りスキップ**）
  - **問い合わせ**: 詳細聞き取り → 完了フラグ → 「お問い合わせを受付いたしました」 → 締めあいさつ → 終話
- **個人情報聴取は用件の前**: main.yaml `接続詞`（"初めに、"）→ 氏名聞き取り → 生年月日 → 連絡先 → 用件 → 用件分岐 …。Gen3 でもこの順序を保持

### 施設固有の特殊ルール
- **受付時間**: 平日（月〜金、祝日除く）8:00〜20:00。`日付判断` 経路は時期判定 + 時刻判定の組み合わせ。Gen3 では acceptance_times に集約（director デフォルト）
- **2/28〜3/2 の AI 受付休止期間**: 「令和8年度予約準備のため、AI 電話による予約受付は 3 月 1 日 まで休止しております。申し訳ございませんが、予約のお問い合わせは 3 月 2 日 10 時以降 におかけ直しください。」→ Gen3 では Script モジュール + 時期判定で実装（2026-04-30 移管完了タイミングでは過去日 → director デフォルトでは 2027 年度想定として保持）
- **4/1 10:30 以降は電話番号変更アナウンス**: 「4 月 1 日より電話番号が変わりました。050-5527-2746 へおかけ直しください。」→ Gen3 では時期判定 termination として残す（**新番号は 050-5527-2746、AI 用 050 番号**）
- **代表電話番号**: 0897-47-3625（健康管理センター）。Brekeke Blind Transfer は無し。`失敗アナウンス` + `電話番号` の音声案内 + `終話` で完結 → call_transfer ブロック不要、termination + 番号読み上げで完結
- **個人情報先行**: 用件分岐の前に氏名 → 生年月日 → 連絡先 を聴取する独自順序。Gen3 でも維持（飯塚・倉敷型の「用件先行 → 個人情報後段」とは逆）
- **健診種類 4 分岐は新規ルート専用**: 人間ドック / 協会けんぽ / 定期健診 / その他の健診。DTMF 1〜4 併用。**他の用件（変更/キャンセル/問い合わせ）では通らない**
- **変更内容 2 分岐**: 「受診内容」「日程変更」のみ。受診内容・日程変更どちらでも詳細聞き取りに合流（`次へ_4` 経由）。受診内容には「オプション追加や変更等、変更を希望されている内容をお話ください」、日程変更には予約日聴取後の同じ詳細聞き取りで吸収
- **用件「問い合わせ」**: 個人情報聴取後 → 詳細聞き取り（自由文「お問い合わせの内容をお話ください」）→ 締めあいさつ → 終話。問い合わせ専用の分岐 yaml は無く `詳細聞き取り.yaml` の入口箱 `問い合わせ` を使う
- **「もしもし」検知**: 用件入口で `countもしもし` 計数（1 回目「はい、聞こえております」、2 回目以上「はい、『もしもし』と聞こえております」→ 用件再聴取）
- **3 回聞き取れず**: 用件 / 種類 / 変更内容 各 yaml で 3 回失敗で `END_用件取得できず` → 代表電話案内 termination（営業時間内に 0897-47-3625 へかけ直し）

### director が見落としやすいポイント
- `API_DRJOY_CONTEXT_SETTINGS` / `API_DRJOY` 系は Gen3 では opening 冒頭の標準処理に吸収 → 明示ブロック化不要
- 「次へ」「次へ_1〜_5」「接続詞」（"初めに、" "では、" "では次に、" "それでは、" "続いて、"）は短い繋ぎ TTS → 次段ブロック先頭 TTS にマージ
- セリフ none 振り分けのみの箱（用件の分岐 / _1 / _2、完了フラグ、携帯電話番号チェック、4/1　10時半まで、時間判断 / _1 / _2、日付判断、時間判断（テスト用））は独立ブロック不要
- `セットした種類の分岐` で「定期健診」 → `用件の復唱（人間ドック）_1` に誤接続している箱マッピングバグ → Gen3 では正しく「定期健診のご予約、でよろしいですか？」echo_back を新設してマッピングし直す
- `変更.yaml` 内 `ドックメニュー` `健診メニュー` `バリウム検査` `内視鏡検査` 等のサブエンティティ定義は流用残骸 → 取り込まない
- `4/1　10時半まで` 箱の「ブレケケ移行のアナウンス」（"4 月 1 日より電話番号が変わりました。050-5527-2746 へおかけ直しください"）は **Brekeke 移行用の旧 → 新番号案内**。施設の本番電話番号は 0897-47-3625（旧）と 050-5527-2746（新）の 2 つ並存中
- 「<? $予約日の読み ?>」「<? $連絡先電話番号（ハイフン区切あり） ?>」等の Gen1 動的タグは Gen3 では `<% var %>` 構文へ
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字や全角スラッシュ「/」「：」等の環境依存文字は禁止
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**（scaffold prefix）を使う
- **CMR conditions は最後に `match: "other"` + next 明示**
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
