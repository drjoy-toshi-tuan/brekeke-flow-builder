### 移管方針
- Gen1「箱」構造 → Gen3 9 ブロック型へ変換。元資料は main.yaml + サブ yaml 14 本（用件聴取 / 時間判断 / 受診歴確認 / 紹介状聴取 / 紹介元医療機関 / 紹介元電話番号 / 前回の症状確認 / 予約日聴取 / 患者氏名聴取 / 生年月日聴取 / 生年月日復唱なし / 診察券番号 / 診察券復唱なし / 診療科聞き取り / 診療科復唱なし / 電話番号聴取 / 電話番号聴取復唱なし / 変更／キャンセル理由聞き取り）。
- opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「こちらは多摩総合医療センターの、予約専用、AI 電話です」、続けて END_事前アナウンス（3 営業日案内 + 紹介状/診察券準備依頼）を冒頭に統合。
- 用件分岐は **Pattern A** が妥当（用件聴取 OpenAI 直後に 6 分岐: 診察=初診 / 再診 / 予約変更 / キャンセル / 確認 / その他）。各分岐に echo_back（用件の復唱）を直配置。
- 診療科 → 用件再分岐 → 受診歴 → 紹介状 → 紹介元 のように **後段 ContextMatchRouter（Pattern B）も併用必須**。`用件の分岐_1 / _2 / _3` で「診療科 × 用件」のクロス分岐があり、初診不可診療科・変更/キャンセル不可診療科・再診不可診療科で別 NG メッセージを流すため、director は B パターンで段階分岐するか判断。
- サブフロー（患者氏名 / 生年月日 / 電話番号 / 診察券）は Jump to Flow 経由。`生年月日復唱なし` `診察券復唱なし` `電話番号聴取復唱なし` `診療科復唱なし` の「復唱なし」版は **2 回目以降の聞き直し用フォールバック**で、Gen3 では同一サブフローを再呼び出しする運用に集約可能（director 判断）。
- 時間判断.yaml（`祝日チェック` サブシナリオ呼び出し + 年末年始 + 平日 8:30〜17:00 判定）→ date_of_call_classifier ブロック。**addCurrentDate: Yes 自動付与対象**。
- entity_classification.md 参照: STT 辞書は分岐あり/なし診療科ごとに 7 グループ + use_template 9 種、OpenAI 正規化 103 件。診療科の no_result パターンと併用。

### 施設固有の特殊ルール
- **受付時間**: 日曜祝日・年末年始除く、月〜土 8:30〜17:00（土曜の特例なし、平日と同じ）。受付時間外アナウンスに「WEB フォームでも初診予約・確認・キャンセル受付可」の案内を含める。
- **年末年始**: 12/28 までは祝日チェック、12/29〜1/4 は専用「年末年始コメント」（1/5 8:30 以降に掛け直し依頼）。
- **代表電話 2 系統**:
  - 予約センター 042-323-9200（予約系問い合わせ・初診不可診療科の案内・連絡先聞き取り失敗時）
  - 代表電話 042-323-5111（予約以外・救急診療科）
- **3 回失敗時**: 用件聴取・受診歴・紹介状の各サブで `count 聞き直し` → 3 回不可で `END_用件聴取できず` 系。Gen3 では termination_patterns に集約。
- **紹介受診重点医療機関**: 初診で紹介状/健診結果票なし → `紹介状なし` 案内で「地域の医療機関にご相談」して切断。**初診フローでは紹介状有無確認が必須前段**。
- **紹介元医療機関 + 紹介元電話番号**: 初診で紹介状ありの場合のみ追加聞き取り（リウマチ外科 / リウマチ膠原病内科の `次へ_1` ルート経由）。「分からない」発話で skip 可。
- **診療科 × 用件マトリクス**（director 注意、main.yaml `診療科の分岐 / _1 / _2` を参照）:
  - **初診で受付不可**: ゲノム診療科 / 腫瘍内科 / 血液内科 / 緩和ケア内科 → 「初診予約は医療連携室経由」案内（血液内科のみ「医療連携へ」FAX 案内）
  - **初診で AI 完全不可**: リハビリテーション科 / 看護外来 / 救急診療科 / 救命・集中治療科 / 女性専用外来 / 小児科
  - **変更/キャンセルで特殊**: 精神神経科 / 産婦人科（妊婦健診）→ 「後ほどご連絡」、女性専用外来 / 救急診療科は受付不可
  - **再診で AI 不可**: 救急診療科 / 救命・集中治療科 / 腫瘍内科 / 緩和ケア内科 / 女性専用外来 / 小児科
- **救急用件の SMS/折り返し時間判定**（`現在の時間判断` 箱）: 8:30 まで or 15:30 まで → 当日折り返し / SMS、それ以降 → 翌営業日以降。Gen3 では time_of_day_classifier 的な ContextMatchRouter で実装。
- **DTMF（4 桁）受付**: 予約日聴取で「シャープ・アスタリスク + 4 桁数字」を受け付ける特殊仕様あり（過去日なら +1 年で確定）。Gen3 でどう扱うかは director 判断（SKILL_C 自由テキスト + DTMF 並走）。
- **携帯電話番号チェック**: 予約完了直前に 4 箇所（初診 / キャンセル / 確認 / 変更）で「携帯電話取得あり」フラグ判定 → 取得有無で `予約 / 予約_1` の分岐。Gen3 では context_match_router 1 個に集約可能。
- **復唱（echo_back）対象**: 用件のみ（6 種）。氏名・生年月日・電話・診察券はサブフロー内で完結、メインに echo_back を出さない。

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `API_DRJOY` の 3 箱は Gen3 では opening 冒頭処理に吸収。明示ブロック化不要。
- 「セリフ: none」かつ振り分けのみの「箱」（`用件の分岐 / _1 / _2 / _3`、`携帯電話番号チェック / _1〜_3`、`受診歴分岐`、`診療科の分岐 / _1 / _2`、`完了フラグ / _1`、`受診歴 CP / _1`、`調査内容聞き完了 / _1`、`紹介状 OK`、`予約日 OK`、`次へ_11`、`現在の時間の取得` 等多数）は Gen3 で独立ブロック不要 → 前段の next 書き換え or context_match_router 一部として吸収。
- `判定: エンティティ含む` で参照されるエンティティ値 → entity_classification.md / entity_stt_patch.json / entity_openai_synonyms.md にすべて自動分類済み。director は転記不要、scaffold/properties が自動取り込み。
- 用件聴取.yaml に `count もしもし`（4 回以上で打ち切り）と通常の `count 上記以外` の二系統 retry あり。Gen3 では用件 hearing ブロックの retry_count を共通化し、もしもし対応は OpenAI プロンプト側で吸収すれば良い（director 判断）。
- `受診歴分岐` で `受診歴有 → 次へ_5（診察券聴取）` / `受診歴無 → 電話番号聴取復唱なし` という分岐があり、**受診歴無の場合は診察券をスキップして電話番号聴取に直行**する。Gen3 でも同じ分岐構造を維持。
- 終話直前の `日時確認終話前アナウンス変更`（3 営業日案内）は **冒頭 END_事前アナウンスとほぼ同文の重複**。冒頭で既に流しているため、終話側は短縮（「ご連絡いたします」のみ）にするか director 判断。
- 紹介状あり初診ルートは `次へ_1（紹介元医療機関）→ 次へ_7（紹介元電話番号）→ 次へ_11（次へ_2 = 患者氏名）` と長い。リウマチ外科 / リウマチ膠原病内科のみこのルート（`診療科の分岐` で `次へ_1` に流れる）。一般初診は `次へ_2` に直行で紹介元情報の聞き取りなし → director は **同じ「初診」用件でも診療科で枝分かれ** することを忘れない。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。`<? $予約日の読み ?>` のような Gen1 変数記法は Gen3 では `<% var %>` に変換。
