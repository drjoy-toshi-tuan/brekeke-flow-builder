### 移管方針
- 健昌会は「予約変更 / キャンセル / 問合せ」の 3 用件のみ受付（**新規予約は無し**）。冒頭で必ず代表電話誘導の前提アナウンス（挨拶_1 = 「新規のご予約については、代表電話におかけ直しください」）を流してから施設選択へ
- 主流れ: opening + 挨拶_1（新規誘導告知）→ 入電時間判断（acceptance_times）→ 受付時間内へ → 問合せ先施設名 subflow（DTMF 4 択） → 用件聞き取り → 受診票有無確認 → 用件分岐 → 各ルート → 氏名 / 生年月日 / 連絡先 → 完了フラグ → 用件×年末年始×携帯 の 8 系統終話
- 用件 3 分岐: 1=変更 / 2=キャンセル / 3=問合せ。**Pattern A（OpenAI 直後分岐）+ 用件の分岐_1（年末年始フラグ込み）→ 携帯分岐**で完了 TTS を 4 系統選択（変更携帯 / 変更携帯以外 / 問合せ携帯 / 問合せ携帯以外）×（通常 / 年末年始）で計 8 種
- 変更ルート: 現在の予約日 → 変更希望日 → 希望時間帯 → オプション追加変更有無 → 内容聴取 → 連絡先 → 完了
- キャンセルルート: 現在の予約日 → **個人情報聴取スキップ** → 連絡先 → 予約キャンセル TTS → 終話（変更とは異なり氏名 / 生年月日聴取無し、連絡先のみ）
- 問合せルート: 内容聴取 → 連絡先 → 完了
- 用件分岐_3 (生年月日聴取後): 「問合せ→予約日確定有無確認」「それ以外→現在の予約日聞き取り」の 2 分岐 → 問合せ時のみ予約日が無いケースを救うサブフロー追加

### 施設固有の特殊ルール
- **新規予約は受付対象外**: 冒頭で必ず「新規のご予約については、代表電話におかけ直しください」を告知。Gen3 でも opening 直後 announcement として固定配置、用件 OpenAI プロンプトでも「新規予約」を返さない / 返ってきたら代表案内へ誘導するハードルール
- **施設選択 4 択（うめだ / 近畿 / 淀川 / 福島）**: 問合せ先施設名 subflow で DTMF 1=うめだ / 2=近畿 / 3=淀川 / 4=福島。施設名カスタムエンティティの値（うめだ=「うめ」、近畿=「キンキ」「金ピ」「現金」「禁忌」「印字」、淀川=「ヨドカワ」「ヨドガワ」「エガワ」「窓側」、福島=「ふくしま」「四番」「福芦マーク」）は STT 誤認識補正の蓄積 → SKILL_A の正規化 prompt に同義語例として残す
- **施設別代表番号 3 系統**: 福島 06-6441-6848 / 淀川 06-6303-7281 / 近畿 06-6365-1655 を 3 回繰り返す END_3回失敗 TTS。**うめだの番号は元資料 END_3回失敗：うめだ箱が見当たらず → director が customer_doc で確認**（main.yaml の `（現在省略）施設名で分岐` 箱で 4 施設すべて `年末年始分岐` 直結 = 同じ `3回失敗：代表案内へ` を共有する設計だが、END_3回失敗：うめだ TTS が無いと最終的にどの番号を読むか曖昧）
- **受付時間 (END_3回失敗 TTS より)**: 月-金 8:00〜16:30、土日祝・年末年始休
- **年末年始対応**: 1月8日（木）8 時以降に再開予定の TTS が `３回失敗：代表案内へ（年末年始）` に固定文言で埋め込み → Gen3 では holidays 配列で吸収、再開日 TTS は customer_doc 側で日付パラメータ化（年次更新を意識）
- **代表案内のナビダイヤル指定**: 「代表電話までおかけ直しいただき、ナビダイヤルの 5 番へお進みください」（年末年始版もナビダイヤル 5 番案内を保持）
- **携帯チェックで完了 TTS 切替**: 携帯あり → 「ショートメールにてご連絡」、携帯以外 → 「お電話にてご連絡」。連絡先 subflow 戻り値 phonetype を context_match_router で分岐
- **年末年始フラグ × 用件 × 携帯 の 3 軸完了 TTS**: 用件分岐_1 で年末年始 → 携帯分岐（年末年始）→ 携帯分岐（年末年始）_1 と中継、合計 8 系統の完了 TTS（変更受付 4 種、その他お問い合わせ 4 種）。**Gen3 では context_match_router 2 段（year_end_flag → mobile_flag）+ 用件 3 分岐で 12 通り**を整理し、文言は 4 種（携帯×年末年始の 2x2）に圧縮
- **キャンセルは個人情報聴取スキップ**: 蘇生会 / 飯塚と同等。main.yaml `用件の分岐` の `キャンセル` パスは `連絡先電話番号聞き取り` 直結 → 氏名 / 生年月日聴取をスキップ
- **オプション追加変更有無 (Yes/No)**: 変更ルート専用。Yes → END_変更あり / No → END_変更無し → 共通の内容聞き取りに合流
- **予約日確定有無確認 (Yes/No)**: 問合せルート専用。「ご予約の日時は、施設より連絡があり、確定していますでしょうか？」→ 確定 / 未確定の context を保存して終話 TTS には影響しない（情報収集のみ）

### director が見落としやすいポイント
- main.yaml の `（現在省略）施設名で分岐` 箱は **4 施設すべて `年末年始分岐` 直結**（差分なし）= 「将来的に施設別分岐を入れる予定だが現在は省略」の名残 → Gen3 では削除して直接 `３回失敗：代表案内へ` 系列に接続
- `用件の分岐` 箱（連絡先電話番号聞き取り 直前のキャンセル/その他振分け）は実質 dead code（条件分岐の両方が `連絡先電話番号聞き取り` 直結）→ 削除可
- `年末年始セット` 箱は flag セット用の中継箱でセリフ none → context save に置換
- 受付時間外用の挨拶や TTS が main.yaml に直接無い → 入電時間判断 yaml が `END_受付時間外` を返した時の遷移先は **`受付時間外へ → 終話`** で番号案内なし。director デフォルトで時間外 termination 文言を補う
- `挨拶_1` の TTS 「新規のご予約については、代表電話におかけ直しください。」は **opening の一部として必須**（新規排除の重要告知）→ 削らない
- 受診票有無確認 subflow が「無し」で `かけ直し依頼→終話`（「健康診断受診のお知らせが到着後、もう一度お電話ください」）→ 受診票なし時の独自フォールバック、Gen3 でも独立 termination として保持
- `エンティティ定義` の カスタム「施設名」「もしもし」「肯定」「否定」「決まっている / 決まっていない」「肯定（エンティティ）」「肯定否定」等は subflow 内に閉じる → メインフローへ持ち上げ不要
- Gen1 動的タグ `<? $現在の予約日 ?>` `<? $変更希望日 ?>` `<? $入電番号 ?>` 等は subflow echo_back に集約済み → `<% var %>` 構文への置換チェックのみ
- 大阪健昌会の正式表記は customer_doc 上「健昌会」または「大阪健昌会」のどちらが優先か director が確認（main.yaml TTS は「大阪健昌会」）
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字禁止
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**
- **CMR conditions は必ず最後に `match: "other"` + next 明示**
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
