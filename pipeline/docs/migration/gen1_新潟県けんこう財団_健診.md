## フローの全体構成

### 冒頭チェーン（Gen3標準）

```
冒頭_wait(Custom$wait) → saveContextModel2DB → incoming-classifier
  ├── 非通知 → 非通知終話チェーン
  ├── 携帯 → acceptance_times → script_営業日判定 → ...
  ├── 固定 → acceptance_times → script_営業日判定 → ...
  ├── 海外 → 海外終話チェーン
  └── その他 → acceptance_times → script_営業日判定 → ...
```

### script_営業日判定（冒頭スクリプト）

acceptance_timesの `true` の後に配置するScriptモジュール。
休日リスト + 曜日チェックを行い、結果を返す。

```javascript
var now = new Date();
var y = now.getFullYear();
var m = ("0" + (now.getMonth() + 1)).slice(-2);
var d = ("0" + now.getDate()).slice(-2);
var dateStr = y + "/" + m + "/" + d;
var dow = now.getDay(); // 0=日, 6=土

// 休日リスト（年度ごとに更新。年末年始・GW・お盆等の特別休業含む）
var holidays = [
    "2026/01/01", "2026/01/02", "2026/01/03", "2026/01/04", "2026/01/05", "2026/01/06",
    "2026/01/12", "2026/02/11", "2026/02/23", "2026/03/20",
    "2026/04/29", "2026/05/03", "2026/05/04", "2026/05/05", "2026/05/06",
    "2026/07/20", "2026/08/11", "2026/08/13", "2026/08/14", "2026/08/15",
    "2026/09/21", "2026/09/22", "2026/09/23",
    "2026/10/12", "2026/11/03", "2026/11/23",
    "2026/12/28", "2026/12/29", "2026/12/30", "2026/12/31"
];

if (holidays.indexOf(dateStr) >= 0 || dow === 0 || dow === 6) {
    $runner.setResult("休日");
} else {
    $runner.setResult("営業日");
}
```

分岐先:
- `^営業日$` → 冒頭アナウンス → メインフロー（用件聴取_時間内）
- `^休日$` → 休日アナウンス → メインフロー（予約・変更・キャンセルのみ受付。転送不可）

**注意**: acceptance_times が `false`（時間外）の場合は script を通らず直接時間外フローへ。
script_営業日判定は acceptance_times `true`（営業時間帯）の日に休日かどうかを追加判定するもの。

**Gen1の年末年始/連休中分岐は廃止**。Gen1では`年末年始応用分岐`（12/25まで→通常、12/26→休業開始当日、12/27〜1/6→連休中、1/7→休業再開当日）や連休中専用アナウンス・用件聴取が存在したが、Gen3ではscript_営業日判定のholidays配列に年末年始・GW・お盆等の日付を入れるだけで対応する。「連休中」という概念は「休日」に統合される。

## 用件聴取と5分岐

Gen1では「時間内」「時間外」「連休中」の3パターンの用件聴取があったが、Gen3では**1つのDTMF+OpenAI聴取に統合**。
「連休中」分岐は廃止し、script_営業日判定の `^休日$` に統合。
ただし、営業日/休日・時間内/時間外の状態は後段で参照される（転送可否、終話文言の使い分け）。

用件5分岐:

1. **健診の予約**（DTMF 1）→ 受診票有無確認 → 施設選択 → 登録番号 → 希望日聴取 → 希望時間聴取 → 氏名 → 生年月日 → 電話番号 → SMS送信判定 → 終話
2. **予約日時の変更**（DTMF 2）→ 予約日聴取（現在の予約日）→ 希望日聴取 → 希望時間聴取 → 氏名 → 生年月日 → 電話番号 → SMS送信判定 → 終話
3. **予約のキャンセル**（DTMF 3）→ 予約日聴取 → 氏名 → 生年月日 → 電話番号 → SMS送信判定 → 終話
4. **企業ご担当者様からの問い合わせ**（DTMF 4）→ 内容聴取 → 企業名・担当者名聴取 → 企業連絡先電話番号聴取 → 終話
5. **その他お問い合わせ**（DTMF 5）→ 営業時間内なら有人転送（025-250-0215）、営業時間外なら内容聴取 → 終話

## 施設固有の聴取項目（メインフロー内で実装）

### 受診票有無の確認

- TTS: 「お手元に健診のお知らせ、受診票はお持ちですか？」
- DTMF: 1=はい、2=いいえ
- STT: DTMF_AmiVoice
- 「はい」→ 施設選択 → 登録番号聴取 → 希望日聴取（施設分岐あり）
- 「いいえ」→ SMS送信判定（受診票なし→WEB予約案内 or SMS送信用携帯番号聴取）

### 施設選択（DTMF）

- TTS: 「受診を希望される施設をお選びください。1は東新潟、2は西新潟、3は長岡です」
- DTMF: 1=東新潟、2=西新潟、3=長岡
- max_dtmf_length: 1

### 登録番号（DTMF）

- TTS: 「受診票に記載されている登録番号を入力してください」
- DTMF: 数字入力（max_dtmf_length: 10程度）
- termdtmf: "#"

### 予約日聴取（現在の予約日 / キャンセル希望日）

- STT: DTMF_AmiVoice → OpenAI日付正規化
- 復唱確認あり（Re-confirmation node data）

### 希望日聴取

- 2パターン:
  - 施設分岐あり: 施設名聴取後の希望日（施設ごとに休業日が異なる可能性）
  - 分岐なし: 変更時の希望日
- 当日予約判定: script_当日予約判定（10:30以前→本日予約可、10:30以降→本日不可）

### 希望時間聴取

- STT: DTMF_AmiVoice → OpenAI時間正規化
- 復唱確認あり

### 内容聴取（フリーテキスト）

- STT: DTMF_AmiVoice（企業問合せ・その他問合せ用）
- OpenAI: フリーテキスト正規化

### 企業名・担当者名聴取

- STT: AmiVoice_STT（DTMFでは入力不能 -- 例外的にAmiVoice使用）
- OpenAI: フリーテキスト正規化

### 企業連絡先電話番号聴取

- STT: DTMF_AmiVoice → OpenAI電話番号正規化

## SMS送信判定フロー

- incoming-classifierで携帯判定:
  - 携帯 → 着信番号をSMS送信先に設定 → SMS送信案内TTS → 終話
  - 固定/その他 → SMS送信用携帯番号聴取（「SMSを送るための携帯番号をお知らせください」）→ SMS送信案内TTS → 終話
  - 受診票なし＋携帯なし → WEB予約案内TTS（「ホームページのWEB予約からお申込みください」）→ 終話

## 転送フロー（その他問い合わせ・営業時間内のみ）

- TTS: 「オペレーターへ転送いたします。少々お待ちください。」
- call-transfer → 025-250-0215
- 転送成功 → 完了フラグ → 切断
- 転送失敗 → TTS「ただいま電話が混み合っておりますので、再度おかけなおしください」→ 切断

## 折り返し確認（聴取失敗時）

Gen1ではリトライ上限到達後に「折り返し確認」サブシナリオを呼び出していた。
Gen3では:

- リトライ上限 → 用件_失敗TTS（「うまく聞き取れませんでした」）
- → 折り返し確認TTS（「担当者からの折り返しをご希望しますか？ 1=はい、2=いいえ」）
- → DTMF_AmiVoice:
  - 1=折り返し希望 → completionFlag(折り返し希望) → 終話
  - 2=折り返し不要 → completionFlag(折り返し不要) → 終話
  - 失敗 → completionFlag(聴取失敗) → 終話

## 当日予約判定スクリプト（予約ルート内）

予約希望日が「本日」の場合、10:30以前なら受付可、10:30以降なら不可。

```javascript
var now = new Date();
var hour = now.getHours();
var min = now.getMinutes();
var timeNum = hour * 100 + min;
if (timeNum < 1030) {
    $runner.setResult("受付可");
} else {
    $runner.setResult("受付不可");
}
```

- `^受付可$` → 希望時間聴取へ
- `^受付不可$` → TTS「本日の予約受付は締め切りました。別の日をお選びください」→ 希望日再聴取

## DTMF化の方針

以下の全STTモジュールを `DTMF AmiVoice STT Input` に置き換える:

- 用件確認（DTMF 1-5）
- 受診票有無（DTMF 1=はい/2=いいえ）
- 施設選択（DTMF 1=東新潟/2=西新潟/3=長岡）
- 登録番号（DTMF 数字入力）
- 予約日（音声入力、DTMF補助）
- 希望日（音声入力、DTMF補助）
- 希望時間（音声入力、DTMF補助）
- 内容聴取（音声入力、DTMF補助）
- 復唱確認（DTMF 1=はい/2=いいえ）
- 折り返し確認（DTMF 1=はい/2=いいえ）
- 企業連絡先電話番号（DTMF数字入力）

**例外（AmiVoice STTのまま）**:
- 企業名・担当者名聴取 -- DTMF入力不可能なフリーテキスト

## RAGサブフロー

Gen1→Gen3移管のためパターン2（全終話前に「最後にご質問はありますか？」）が既定だが、本案件では **不要** と判断。

Gen1にFAQ機能はなく、「その他問い合わせ」は有人転送で対応。RAGサブフローなし（`rag_subflow.pattern: none`）。

ただし施設の方針により後日追加の可能性はある。

## 特記事項

1. 休日リストの年次更新が必要（Script内のholidays配列）
2. 転送先電話番号 025-250-0215 は本番環境のみ。デモ環境では未設定
3. 10:30当日予約締切は予約ルート内でのみ使用
4. 「受診票なし」の患者にはWEB予約案内（SMS送信ルート）を提示
5. 企業問合せルートは個人向けと異なる聴取項目（企業名・担当者名・企業連絡先）
6. 3施設（東新潟・西新潟・長岡）は施設選択DTMF で分岐。施設ごとの休業日対応は希望日聴取の中で処理
7. Gen1 の「年末年始応用分岐」「連休中」パターンは**全て廃止**。script_営業日判定の holidays 配列で一元管理。年末年始・GW・お盆の特別アナウンス（「営業日に担当者よりご連絡いたします」等）は不要 — 通常の休日終話文言で統一
