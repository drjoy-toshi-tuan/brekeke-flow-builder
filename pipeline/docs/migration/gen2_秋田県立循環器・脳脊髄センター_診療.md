### 移管方針

- Gen2 yaml 構造（共通/分類別/終了）をそのまま Gen3 の scenario_flow に写像。用件分類後に診療科→(分岐A)→個人情報の 2 本合流型。
- **Pattern B（後段 ContextMatchRouter）を採用**。用件 classification（変更/キャンセル/予約日時確認）で「診療科2/検査/現在予約日/変更希望日/残薬/キャンセル理由」の有無が変わり、複数 context を後段で再判定する必要があるため。
- 分岐が 2 段ある: (1) 用件別_前半（予約日時確認のみ診療科2/検査/現在予約日をスキップ）、(2) 用件別_後半（変更=変更希望日+残薬、キャンセル=理由、確認=直接個人情報）。
- 氏名/生年月日/診察券/電話の個人情報はサブフロー必須（メインにインライン展開禁止）。RAG サブフローは customer_doc に記載なし、不要。
- 救急科判定は診療科1と診療科2の両方に配置が必要（customer_doc L366-370, L394-398, L421-425）。脳単独時の脳神経外科/脳神経内科切り分けも同じく両方に必要。
- 電話種別判定（携帯 070/080/090/060 vs 固定 018 系10桁）で終話1/終話2 に分岐、さらに classification で終話1(変更/キャンセル/確認)・終話2(変更/キャンセル/確認)の 6 分岐。終話6種類を termination_patterns で定義。
- 非通知 status=9、救急科 status=2、終話到達 status=1/smsFlag=1、それ以外 status=0。smsFlag は救急科・非通知では 0 のまま（customer_doc L201-203）で、終話時のみ 1。
- Date of Call Classifier: customer_doc に「時間外」「受付時間」の記載なし → **不要**。acceptance_times 分岐は作らない。
- 用件確認の「肯定」単語変換（「いいです」「肺」）、新規/予約/診察の単語変換（customer_doc L328-338）は classification プロンプトで吸収（director 判断）。

### 施設固有の特殊ルール

- **診療科リスト16件**（customer_doc L225-240 および function enum L627-645）。customer_doc 本文は「整形外科」含む15件だが、function の enum には本文非掲載の「血液内科」も含まれる（director 判断: enum 優先で16件採用）。
- **改訂履歴 2026/03/25**: 「当日受付変更」「診療科追加（整形外科）」（customer_doc L5-8）→ 診療科リストに整形外科を必ず含める。
- **脳/脳神経の単独発話時は脳神経外科/脳神経内科へ再ヒアリング**（customer_doc L369-370, L424-425）。診療科1と診療科2の両方で実施。予約日時確認の場合は「脳」のみ記載で「脳神経」への言及なし（L424）だが、director 判断: 揃えて両方拾う。
- **救急科はエラー切断**（status=2, endpoint=電話案内, customer_doc L367）。代表電話 018-833-0115 を3回繰り返す。
- **終話は6パターン**（携帯×変更/キャンセル/確認、固定×変更/キャンセル/確認、customer_doc L487-503）。終話1=ショートメール URL、終話2=電話連絡。**終話2 のキャンセル・予約日時確認の文言が「予約変更の申し込み」と誤記**されている（customer_doc L499-503）が、これは元資料の誤記であり director は文言そのまま転記（校閲 reviewer で指摘される前提）。
- **固定電話プレフィックス**: 018/0182/0183/0184/0185/0186/0187（秋田県外線, customer_doc L388, L415, L432）。その他10桁は固定扱い。携帯は 090/080/070/**060** 起点（customer_doc L387 は 060 含む、060 は現行携帯帯域）。
- **classification 用件判定の特殊単語変換**（customer_doc L328-338）:
  - 「印刷/定期/新薬/再診/最新/禁忌」→ 診察
  - 「薬/お薬」→ 予約
  - 「電気/でんき」→ 新規
  - 「いいです/肺」→ 肯定
  (当施設の用件は 変更/キャンセル/予約日時確認 の3択のため、この変換は用件3択内での補完として generate_by_OpenAI プロンプトに記載必要)
- **予約日と予約希望日は別物**（customer_doc L343-349）。reservationDate は DATE 型、DesiredreservationDate は TEXT 型（フリーワード）。10月以降で1-9月日付は翌年換算。

### director が見落としやすいポイント

- **function スキーマに `clinicalDepartment.enum` が16件**（customer_doc L627-645）で、本文の診療科リスト15件より多い（「血液内科」が enum にのみ存在）。function スキーマ全件転記のこと。
- **「015」「0184」等の特殊な入電番号プレフィックス**（customer_doc L388）が複数列挙されている点。固定電話判定で見落としやすい。
- **用件3択の dtmf**: `<dtmf digit="1"/>` を冒頭アナウンスではなく用件聴取 TTS に入れる（customer_doc L361）。キャンセル理由 TTS にも dtmf あり（L456）。
- **smsFlag の L194 に誤記**「status= '0' を保持」（実際は smsFlag の説明だが status と書かれている）→ 意図は「smsFlag='0' を保持」。director は smsFlag 仕様として正しく解釈。
- **患者名ヒアリングの苗字のみ判定**（customer_doc L459-461）: 「下のお名前もお伺いしてもよろしいですか？」は一度だけ、復唱なし。
- **診察券番号は3桁以上**（customer_doc L67-71, L968）、「C」→「4」置換、「わからない」で次へ。
- **連絡先電話番号の復唱は必須**（customer_doc L479, L483, L24-25）、手動聴取時も復唱する（Gen2 yaml L1031-1045）。
