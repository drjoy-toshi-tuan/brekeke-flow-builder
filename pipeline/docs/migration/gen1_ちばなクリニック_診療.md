### 移管方針
- 主流れ: opening → API_DRJOY → 時間外接続（受付時間内 / 時間外 / 年末年始の 3 分岐）→ 挨拶（時間内）→ 初めに（紹介状有無 Yes/No → 用件を診察予約に）→ 患者氏名 → 連絡先 → 生年月日 → 診察券番号 → 用件聞き取り（4 分岐: 予約変更 / キャンセル / 予約確認 / 二次健診）→ 用件別合流 → 完了アナウンス → termination
- **opening 冒頭で「本日の手術 / 入院予約は 0570-09-1301 へかけ直し / 二次健診で紹介状なしも 0570-09-1301 の新規予約へ」を必ず案内** → そのまま個人情報聴取に進む（別フロー離脱はしない、案内のみ）
- 用件分類は **4 分岐**: 予約変更 / 予約キャンセル / 予約確認 / 二次健診（新規）。診察予約（=初診/再診）は本フロー対象外で冒頭案内のみ
- 予約変更 / キャンセル: 予約日聞き取り → 「本日の予約に当日手術 Y/N」確認 → 当日 Yes は完了フラグ即離脱、No は当日チェック → 「現在の予約に当日検査 Y/N」確認 → 完了フラグ → 終話アナウンス
- 予約確認: 用件を判断_1 で WEB 確認案内（SMS で URL 送付）→ 終話
- 二次健診: 用件を判断_1 で紹介状有無 → 紹介状あり: 紹介元医療機関名聴取 → それでは → 診療科聞き取り → 予約希望日 1/2 まとめて聴取 → 完了フラグ → 終話。紹介状なし: ナビダイヤル案内（0570-09-1301）→ 終話

### 施設固有の特殊ルール
- **個人情報聴取が用件聴取の前段に来る**: 中頭は用件 → 診察券 → 診療科 → 個人情報の順だが、ちばなは **氏名 → 連絡先 → 生年月日 → 診察券 → 用件** の順。Gen3 設計書もこの順番を保持（director 注意）
- **時間外/年末年始は 3 分岐 termination**: `時間外接続.yaml` で受付時間内 / 時間外 / 年末年始を判定 → 時間内: 挨拶（時間内）、時間外: 挨拶（時間外）+ 受付時間外案内、年末年始: 挨拶（年末年始）→ 即終話。Gen3 では acceptance_times + Script（年末年始判定）+ 3 系統 termination
- **当日手術 / 当日検査 Y/N の二段確認**: 予約日聞き取り後、「本日の予約に当日手術はありますか？」Y/N、続けて当日チェック後に「当日検査（胃カメラ/大腸カメラ/CT/MRI）はありますか？」Y/N。Yes/No 両方とも完了フラグへ進む（情報保存のみ）
- **紹介状有無は二次健診ルート専用**: 用件を判断_1 で「二次健診」分岐時のみ呼び出し → 紹介状なし → ナビダイヤル 0570-09-1301 案内 → 終話。中頭の「冒頭で全用件に紹介状確認」とは異なり、ちばなは二次健診のみ
- **代表電話の使い分け**:
  - **098-939-1301**: ちばなクリニック直通代表（聞き取り NG 時の最終フォールバック、3 回繰り返し読み上げ）
  - **0570-09-1301**: ちばなクリニックナビダイヤル（当日手術・入院予約 / 二次健診で紹介状なし向け）
- **診療科は分岐なし 36 科の単一 enum**: 中頭と異なり分岐あり/なしの 2 段構成は無し。`診療科聞き取り.yaml` の 01_00_分岐なし診療科に 36 科目（外科 / 救急科 / 形成外科 / 血液腫瘍内科 / 血管外科 / 呼吸器外科 / 内科 / 産科 / 耳鼻いんこう科 / 循環器内科 / 小児科 / 消化器外科 / 内科 / 心臓血管外科 / 腎臓内科 / 整形外科 / 総合内科 / 内科 / 乳腺科 / 脳神経外科 / 泌尿器科 / 皮膚科 / 婦人科 / 放射線科 / 放射線治療科 / 麻酔科 ほか）。Gen3 では診療科 hearing 1 段で吸収
- **腎臓内科・膠原病/リウマチ科の特殊エンティティ**: main.yaml カスタムエンティティで「腎臓内科（じんぞうないか / 腎臓 / 透析）」「膠原病（リウマチ科）」を独立定義 → STT 辞書 + OpenAI 同義語に転記
- **WEB 確認案内（予約確認用）**: 「お電話終了後、ショートメールをお送りしますので、記載された URL にアクセスいただきご確認をお願いします」→ 終話。中頭と異なり 携帯チェックの分岐は無し（常に SMS 送付前提、携帯なしの場合は通話完了 termination で別案内）

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` と `通話開始チェックポイント` / `API_DRJOY` は opening 標準処理で吸収 → 明示ブロック化不要
- 「次へ」「次へ_1〜_3」「それでは」「本日の」「本日の_1」「現在の」「現在の_1」（短い繋ぎ TTS「ありがとうございます。続いて、」「本日の予約に」等）は次段ブロック先頭 TTS にマージ
- 「セリフ: none」かつ振り分けのみの箱（`用件を判断` / `_1`、`完了フラグ`、`電話切断へ`、`時間外テスト用外し`、`受付時間外へ`、`年末年始終話へ`、`終話へ_1`、`当日チェック`、`有` / `有_1`、`無` / `無_1`、`診療科NG`、`紹介状NG`、`紹介元医療機関NG`、`希望日NG`、`用件NG`）は独立ブロック不要 → 前段 next 書き換えか CMR の一部として吸収
- `時間外テスト用外し` 箱は元資料時点でテスト用の名残 → 削除して直接 `時間外接続` へ
- `当日手術 Y/N` / `当日検査 Y/N` は **専用サブフロー** で Yes/No 確認 → Gen3 では Yes/No hearing 2 本連続配置（共通 subflow 化はしない、scaffold が固定プロンプト埋め込み）
- 「予約希望日 1，2 まとめて聴取」サブフローは二次健診ルート専用 → datetime hearing 2 段（第 1 希望 / 第 2 希望）、addCurrentDate: Yes scaffold 自動付与
- 「紹介元医療機関名聴取」は二次健診ルート専用 → 自由文 hearing（output_format: text）、医療機関名がわからない場合は「医療機関名がわかる状態でおかけ直しください」termination 離脱
- `失敗案内` は最終フォールバック termination（聞き取り 3 回失敗時）→ 098-939-1301 を 3 回繰り返し読み上げ。文言短縮しない
- 患者氏名サブフローは **PatientName 1 枠**に充当
- 中頭にあった「予約種別確認」（当日手術/入院/健診紹介状）の Yes/No ゲートは ちばなには無い → 冒頭の `初めに` 案内 TTS でアナウンスのみ（離脱判定は紹介状有無 hearing で）
- 「<? $連絡先電話番号（ハイフン区切あり） ?>」「<? $入電番号（ハイフン区切りあり） ?>」等の Gen1 動的タグは Gen3 では `<% var %>` 構文へ
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字等の環境依存文字は禁止
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`** を使う
- **CMR conditions は必ず最後に `match: "other"` + next 明示**
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
