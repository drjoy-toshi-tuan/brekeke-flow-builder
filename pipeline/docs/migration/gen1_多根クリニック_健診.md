### 施設固有の特殊ルール
- **代表電話番号**: 多根クリニック健診の代表案内番号（聴取失敗/代表案内 box）。文言は「受付時間は、日曜、祝日、ゴールデンウィーク、年末年始を除く、月曜日から金曜日の9時から16時、土曜日の9時から11時」。**番号本体は box 内に明記なし** → director判断: customer_doc から番号を引く必要あり（`新エンジンへ案内` の 050-1807-8250 とは別の本来番号）
- **新エンジンへ案内**: `新エンジンへ案内` 箱は「10月1日8時より新しい番号 050-1807-8250 に変更」告知 → **過去の一時運用告知**。現時点では除外して director 判断、念のため customer_doc に番号変更の事実だけ残す
- **時間外文言が異常**: `聴取` 箱（時間外案内）の受付時間表記が「月〜金 朝5時から零時、土曜 朝5時から午前11時」と聴取失敗時（`聴取失敗/代表案内` の 9-16 / 9-11）と異なる → **Gen1 の文言バグ**。Gen3 では `聴取失敗/代表案内` 側の 9-16 / 9-11 を正とする
- **受診結果ルートで平日/土日切替**: `日にち判断` で平日（lessEqual 月〜金）→ 「本日15時から17時の間に、医師より順次お電話します」/ 土日 → 「午後休診のため、翌営業日の15時以降に医師より順次お電話します」。Gen3 では date_of_call_classifier ブロック相当 + announcement 2 本で実装
- **キャンセル時のみ携帯/固定で文言切替**: 携帯 → 「キャンセル完了後にSMSを送信いたします」、固定 → 「3営業日以内に、担当者から順次折り返し電話にてご連絡いたします」。それ以外（変更/紹介状/受診結果/問い合わせ/新規）は `クローズ` 共通（SMS リンク送信文言）
- **「本人 / 本人以外」分岐は問い合わせのみ**: 新規予約は本人前提（PatientName 1 枠）、問い合わせのみ企業/個人分岐 → 企業名聴取②（director判断: 受診者氏名が必要なら問い合わせルート用に hearing 追加するか customer_doc 確認）
- **diagnosis_subject 聴取は無し**: 入電者 = 受診者前提（問い合わせの企業名は別系統）

### director が見落としやすいポイント
- 冒頭の `API_DRJOY_CONTEXT_SETTINGS` / `API_DRJOY` は Gen3 標準処理に吸収 → 明示ブロック化不要
- 「次へ」/「次へ_1〜_8」（「続いて、」「では次に、」「それでは、」「最後に、」）は短い繋ぎ TTS → 次段ブロック先頭 TTS にマージ
- セリフ none 振り分けのみの箱（`用件の分岐` / `_1〜_4` / `企業/個人分岐` / `携帯電話番号分岐` / `日にち判断` / `完了フラグ` / `_1` / `_2` / `生年月日NG`）は独立ブロック不要 → CMR or save2db に吸収
- 聞き取り NG 系（`用件聞き取れず` / `当日確認聞き取れず` / `予約日聞き取れず` / `種類聞き取れず` / `変更聞き取れず` / `希望日聞き取りできず` / `内容聞き取りできず` / `氏名聞き取りできず` / `生年月日聞き取りできず` / `電話番号聞き取りできず`）は **すべて `聴取失敗/代表案内` に集約** → Gen3 では各 hearing/subflow の error_jump 先 termination に統一
- `END_予約日聴取` 終了箱は **「予約日聴取」サブシナリオを再呼出する自己参照** の名残（誤参照） → そのまま無視（Gen3 では削除）
- `END_予約/紹介状/受診結果の質問/問い合わせ` 終了箱と `予約/紹介状/受診結果の質問/問い合わせ` 共通箱は **同名違い**（END_側はサブフロー直結用、共通箱は通常呼出）→ Gen3 では同一 termination に集約
- `クローズ` 文言は「を受付いたしました。〜」と冒頭の主語が省略されている → 直前の announcement（`変更` / `キャンセル` / `初診` 等）と組み合わせて完成させる構造 → Gen3 では各用件 announcement + クローズ文言を 1 つの termination に統合
- 当日予約カット箱は **存在しない**（多根クリニックは当日予約も受付可能）→ 倉敷・関越と異なる
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字禁止
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**
- **CMR conditions は必ず最後に `match: "other"` + next 明示**
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
- Gen1 動的タグ `<? $var ?>` は `<% var %>` 構文へ
