### 移管方針
- Gen1「箱」→ Gen3 9 ブロック型へ変換。opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「お電話ありがとうございます。井上病院健診センターの、予約専用AI電話です」。
- **用件分岐は二軸マトリクス**（企業 or 個人 × 健診予約 / 予約変更 / 予約キャンセル / その他問い合わせ）。Gen3 では Pattern A（用件聴取直後 4 分岐）+ Pattern B（前段に企業_個人 hearing 1 本）を組み合わせる。
- 主流れ: opening → API_DRJOY → 入電時間判断（平日 9〜16 内 / 16〜17 / 12/26〜1/4 / 通常時間外の 4 系統）→ 受電前説明 announcement → 企業/個人確認（DTMF 1=企業 / 2=個人）→ 用件聴取（DTMF 1=新規 / 2=変更 / 3=キャンセル / 4=その他）→ 用件 CMR → 各レーン → 患者氏名/生年月日/連絡先聴取 → 完了フラグ（キャンセルかどうかで終話 TTS 分岐）→ 直通番号案内 termination。
- **入電時間判断 yaml** は date_of_call_classifier ブロック（addCurrentDate: Yes 自動付与）。`日付分岐_1` で 12/29〜1/3 を年末年始留保案内、それ以外を平日/曜日/時間で時間外切断。**16〜17 時は時間外扱い**（受付終了は 15:30 案内だが、内部判定は 16 時で切る）。
- 企業/個人確認サブフロー（DTMF 1/2 の Yes/No 系 hearing）→ 個人レーンは `用件聴取` → 用件分岐_1〜_5 の入れ子。企業レーンは Detail 聴取（自由テキスト）→ 受診者名（部署名）聴取 → 連絡先 で完結。**企業健診はフリーテキスト 1 本にまとめ**、複雑な分岐は director 判断で簡素化。

### 施設固有の特殊ルール
- **受付時間**: 月〜土 8:30〜15:30、日曜・祝日休、年末年始 12/28〜1/4 留保受付（翌営業日の 1/5 から順次折り返し）。**16 時以降は時間外扱い**（運用ガイダンスは 15:30 までと案内）。
- **直通番号**: `0120-86-9370`（フリーダイヤル、聞き取れず終話 / 時間外 / 完了 全部ここで案内）。**「電話番号は0120-86-9370です。繰り返します。電話番号は0120-86-9370です」と 2 回読み上げ**。
- **代表電話案内 TTS**: 「お電話が遠いようで、回答が確認できませんでした。恐れ入りますが、日曜・祝日を除く、月曜日から土曜日の、8時30分から15時30分に、井上病院健診センターの直通番号へおかけなおしください」→ 直通番号案内（DTMF 入力待ち = `直通番号案内` 箱で「電話番号が分かる場合は、そのままお電話をお切りください。分からない場合はご案内いたしますのでメモをご準備ください」）→ 直通電話 TTS（番号 2 回読み上げ）→ 電話切断へ → 終話。**この 4 段の announcement チェーンは聞き取り NG 12 種すべての合流先**。
- **時間外案内**: 「こちらのお電話では受付後に担当者より内容確認を行い、翌営業日以降に折り返し電話、もしくはSMSにてご連絡いたします」→ そのまま企業/個人確認に進行（時間外でもフロー継続）。
- **年末年始 announcement**: 「12/28から1/4までは年末年始のため、お問い合わせ内容の受付はいたしますが、翌営業日の1月5日より、順次折り返しのご連絡をさせていただきます」→ END_年末年始 として終話。
- **用件 4 分岐**: 1=新規健診予約 / 2=予約変更 / 3=予約キャンセル / 4=その他問い合わせ。**新規予約 + 個人受診** の場合のみ「WEBサイト誘導サブフロー」（携帯番号入電 → SMS で予約 URL 送信 or 直接 WEB 案内、`smsflag2` 送信）に分岐し、AI 電話側で氏名・生年月日まで聞かない。**新規予約 + 企業健診** は Detail 聴取（フリーテキスト）→ 種類分岐 → 企業名 → 受診者名 → 連絡先 で完結。
- **キャンセル前確認**: `タイトル` 箱「予約日が本日を含む一週間以内の場合、キャンセルにつきましてはお受けすることができませんので、お電話を切り井上病院健診センターの直通番号へおかけなおしください。一週間以内ではない場合はこのまま受付いたします」アナウンス必須（announcement ブロック）→ 現在の予約日聴取 → 用件分岐_3。
- **キャンセル終話文言の分岐**: `キャンセル終話内容分岐` で **「予定なし」発話** → 「キャンセルの申し込みを受付いたしました」（短）/ それ以外 → 「キャンセルの申し込みを受付いたしました。再受診のご予約を行う場合は再度AI電話へご連絡ください」（再受診案内付き）の 2 種。再受診予定サブフロー（DTMF 1/2 受付）の出力 context で切替。
- **変更時の二段聴取**: 変更該当者氏名サブフロー → 用件分岐_5 → 個人なら 受診者名（部署名）聴取、企業なら 企業名聴取_1 → 受診者名（部署名）聴取。**変更該当者は患者本人と限らない**ので PatientName とは別の自由テキスト hearing。
- **DTMF 受付**: 企業/個人確認（1/2）+ 用件聴取（1/2/3/4）+ 再受診予定（1/2）+ Yes/No 確認複数。
- **「もしもし」検知 / リトライ 3 回 / count上記以外 5 回再質問** は用件聴取・企業/個人確認・Detail聴取で共通実装。

### director が見落としやすいポイント
- 冒頭 5 箱（API_DRJOY_CONTEXT_SETTINGS / 挨拶 / API_DRJOY / 入電時間判断 / 受電前説明）は Gen3 で opening + date_of_call_classifier + 受電前説明 announcement に整理。
- セリフ none 振り分けのみの「箱」（用件の分岐 / _1〜_7、種類分岐 / _1〜_4、完了フラグ、通話完了、電話切断へ、各種CP（聞き取れず） 12 種、連絡先電話番号聞き取りへ、入電時間判断へ）は独立ブロック不要 → 前段の next 書き換え or context_match_router の一部として吸収。
- 「次へ」「次へ_1」「タイトル_1」「企業名聴取へ」「企業名聴取へ_1」の TTS（「続いて、」「ありがとうございます。」「かしこまりました。では、」「かしこまりました。続いて、」）は **次段サブフローの冒頭 hearing TTS にマージ**。独立 announcement ブロック禁止。
- **用件分岐の入れ子（_1〜_7）**: 元 yaml は「企業 × 個人 × 用件 4 種」のマトリクスを CMR 連鎖で表現。Gen3 では用件 1 値（4択）+ 企業/個人 1 値（2択）の 2 つの context を確定させた後、context_match_router 1 枚で 8 通りの分岐先 next を直接書く方が読みやすい（director 判断で展開）。
- **新規 + 個人** ルートの WEBサイト誘導サブフローは **携帯番号取得後に SMS 送信 → 終話**。氏名/生年月日を聞かないので、main.yaml の patient subflow チェーンには合流しない（独立 termination）。
- **PatientName 1 枠** = 受診者本人前提（個人受診時）。企業健診時は「受診者名（部署名）」を fr 自由テキストで聴取（PatientName subflow ではなく hearing 1 枠）。**変更時の `変更該当者氏名` も PatientName 1 枠ではなく自由テキスト hearing**（家族/同僚の代理可能性あり）。
- 用件聴取.yaml の **エンティティ「用件」値定義に同義語が大量**（新規=健康診断/健診/検診/特定健診/人間ドック/雇い入れ/就職/進学/事業主/生活習慣病/全身/千四百/予約を取る/取り直したい 等 25 個、変更=延期/時間/日程/日付/判子/平行/並行/別の/変えたい/変えて/変える/変更/予約の変更 13 個）。OpenAI 正規化辞書で吸収。
- 用件聴取.yaml の **DTMF 4 = 問い合わせ**（用件の復唱（問い合わせ）「その他お問い合わせ、ですね」）→ Detail聴取（自由テキスト）に直結。新規/変更/キャンセルとは独立した 4 番目の経路。
- 連絡先聴取.yaml の `市外局番5桁/4桁/3桁/2桁` 区切り処理は Gen3 phonetype subflow で吸収 → ノート不要。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**（全角括弧 `（）`・全角スペース・全角コロン `：`・特殊文字 `ＣＰ` 禁止）。例: `用件聞き取れず ＣＰ` → `END_yoken_failed`、`受診者名（部署名）聴取` → `subject_name_hearing`。
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**。CMR conditions 末尾は必ず `match: "other"` + next 明示。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。Gen1 動的タグ `<? $... ?>` → Gen3 `<% var %>`（本フローは静的 TTS のみ）。
