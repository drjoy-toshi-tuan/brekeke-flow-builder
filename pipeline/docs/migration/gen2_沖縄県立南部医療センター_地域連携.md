### 移管方針

- 地域連携室専用 AI 電話。入電者は**他医療機関の連携担当者**であり、患者本人ではない。氏名・電話番号・所属を聴取するフィールドはすべて医療機関側担当者の情報として扱う（clinicName / callerName / additionalPhoneNumber）。患者情報（patientName / patientDateOfBirth）は用件確認後に別枠で聴取する点に注意。
- **冒頭ゲート**: opening ブロック直後に「緊急対応の有無」yes/no 分岐を置く。generate_by_OpenAI（Yes_No 判定）→ yes なら代表電話案内 termination へ直行、no なら本フローへ進む。この分岐は classification より前に来るので、冒頭_アナウンス内で問いかけを完結させること。
- **Pattern 判定**: 緊急対応ゲート直後 → 共通聴取（医療機関名→入電者氏名→入電者電話番号）→ 用件4分岐、の素直な直列構造なので Pattern A（OpenAI 直後分岐）で組める。ただし「情報提供依頼」ルート内の整形外科除外は subflow 内で Yes_No 再分岐が必要なため、用件ブロック内で Pattern A の二段目を組む。
- **Date of Call Classifier**: 用件別受付時間制限は資料上明示されていない（代表電話の受付時間は月〜土 8:30〜12:00 だが、AI 電話自体の受付時間分岐は記載なし）。director は元資料の受付時間記載を再確認し、必要なら冒頭に date_of_call_classifier を置く。記載がなければ省略して良い。
- **サブフロー要否**: 患者氏名→PatientName サブフロー 1 枠に充当。生年月日→DateOfBirth サブフロー。入電者電話番号→PhoneNumber サブフロー（10/11 桁判定ロジックあり）。医療機関名は 260+ 施設の正規化マッピングありだが hearing + OpenAI 正規化で吸収、RAG は不要。

### 施設固有の特殊ルール

- **緊急対応 yes → 即終話**: emergencyResponse=「有り」を格納し、代表電話（098-888-0123、月〜土 8:30〜12:00）を 3 回読み上げて終話。status=2、smsFlag=0。
- **整形外科 AI 不可**: 用件「情報提供依頼」選択後の診療科聴取で「整形外科」「整形」「せけげか」等が入力された場合、AI 対応不可として代表電話案内で終話（status=2、smsFlag=0）。診療科 hearing 直後に OpenAI 分岐（整形外科判定）を入れる必要あり。
- **「その他お問い合わせ」ターン制御**: details フィールドの自由聴取。ユーザーが**「以上です。」を発話するまで次ステップへ遷移しない**。途中で割り込み・遮断禁止。hearing ブロックのプロンプトに明記させる（prompter 作業）。
- **echo_back 原則なし**: 本番プロンプト（2026/02/09 版）では**全項目 復唱禁止**。ただしデモ版プロンプトには「連絡先電話番号のみ復唱」の記載あり → **本番版を採用**、echo_back は全項目オフ。
- **共通聴取の順序固定**: 医療機関名 → 入電者氏名 → 入電者電話番号 → 用件、の順で固定。用件分岐後に各ルート固有の聴取が続く（用件ごとに聴取項目が異なる、下記参照）。
- **連携医療機関リスト**: 260+ 施設の正規化マッピングが元資料 §2-1 に列挙されている。director が元資料から全件抽出して clinicName の所属リストとして YAML に落とす。

### director が見落としやすいポイント

- **入電者 ≠ 患者**: callerName（入電者＝医療機関担当者）と patientName（患者本人）を**別フィールドで分けて聴取**する。入退院支援室ルートではさらに contactName（当院側担当者名）も聴取する 3 者構造。用件別 function スキーマでこの 3 フィールドを混同しないこと。
- **冒頭ゲートの配置ミス厳禁**: 緊急対応 yes/no は**必ず opening 直後**に置く。共通聴取（医療機関名等）の後に配置すると、緊急時に無駄な聴取が発生して設計崩壊。opening ブロックの TTS 文言に「本件は、緊急対応が必要でしょうか。はい、いいえでお答えください。」を含めて Yes_No 判定 OpenAI へ直結。
- **用件 4 分岐ごとに聴取項目が異なる**:
  - 外来入院申込み: 患者氏名 → 患者生年月日 → **FAX 送信時期（fax）** → 終話①
  - 情報提供依頼: **診療科（整形外科判定付き）** → 患者氏名 → 患者生年月日 → **返信期限（responseDeadline）** → 終話②
  - 入退院支援室: **担当者名（contactName）** → 患者氏名 → 患者生年月日 → 終話②
  - その他お問い合わせ: **問い合わせ内容自由聴取（details、「以上です」待ち）** → 終話②
- **終話文言は ① と ② で異なる**: ① 外来入院申込み専用（「確認事項がある場合のみ折り返し」）、② 情報提供依頼／入退院支援室／その他共通（「5 診療日以内にスタッフから連絡」）。termination ブロックを 2 本用意する。
- **status/smsFlag の遷移**: 初期 status=0/smsFlag=0。各用件完了時に status=1/smsFlag=1 へ昇格。緊急対応・整形外科案内・聴取失敗は status=2/smsFlag=0。その他問い合わせの「AI 応答不可」パターンのみ status=0（§1-2【A】）。
- **SSML 扱い**: 元資料の `<speak type="telephone">` `<break time="300ms"/>` はプロンプト転記時に削除禁止と記載されているが、Gen3 側は SSML 禁止（CLAUDE.md 準拠）。TTS 文言への転記時は SSML タグを除去し、`{tts_g:...}` 形式で電話番号読み上げを自然文に変換すること。prompter ではなく gen_properties 側の責務。
- **再質問ルール**: 本番版は「同一項目に最大 2 回まで」（§7.3、聴取失敗アナウンスへ）、デモ版は「最大 1 回」。**本番版を採用**（2 回リトライ）。
