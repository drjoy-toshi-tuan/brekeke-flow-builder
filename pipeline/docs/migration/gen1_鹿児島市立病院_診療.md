### 施設固有の特殊ルール
- **再診は AI 電話で受付不可**: 初診確認 No → `再診：代表案内`「この電話は、初診予約専用電話となっております。再診の方は代表電話におかけ直しください。」→ 終話。**再診は完全に終話のみ**（代表番号読み上げなし、文言内で「代表電話」と言うだけ）
- **紹介状必須**: 紹介状 No → `紹介状無しアナウンス`「紹介状が必須のため近隣の医療機関へ受診をお願いします。紹介状をおとりになられましたら、再度ご連絡をお願いいたします。」→ 終話
- **代表電話番号 099-230-7000**: 聴取失敗時の `失敗` box / `新生児内科：代表案内` で読み上げ。「代表電話に初診紹介予約についてとお伝えください」と必ず付記
- **医療機関直通ダイヤル 099-230-7115**: `連携医療機関からの入電` announcement で 3 回読み上げ（強調）。`紹介元番号判別` で連携医療機関と判定された入電に対して即切断
- **新生児内科は AI 電話で対応不可**: 診療科 = 新生児内科 → `新生児内科：代表案内`「申し訳ございません。代表電話へおかけ直しください。電話番号は 099-230-7000 です」→ 終話
- **精神科は入院患者様のみ**: 診療科 = 精神科 → `精神科` announcement「当院の精神科は、入院患者様のみのご対応とさせていただいております。近隣の医療機関へ外来受診をお願いします」→ 終話
- **「折返し」は用件の独立分岐**: 「先ほど」「電話が」「担当の」等の発話を「折返し」用件として捕捉 → 問い合わせ自由文 hearing → 折返し announcement「土日祝日を除く、3診療日以内に担当者から折り返しご連絡いたします。担当者から折り返しする電話番号は発信専用ダイヤルとなっておりますので…」→ 終話
- **発信専用ダイヤル明記**: 全用件の終話 announcement で「担当者から折り返しする電話番号は発信専用ダイヤルとなっておりますので、担当者からの折り返し電話をお待ちください」を必ず読み上げる（病院側の運用上の重要要素）
- **携帯/固定で文言切替なし**: 倉敷・蘇生会と異なり phonetype 分岐なし → 連絡先 subflow 戻り値は context 保存のみ、終話文言は共通

### director が見落としやすいポイント
- 冒頭の `API_DRJOY_CONTEXT_SETTINGS` / `API_DRJOY` / `紹介元番号判別_1` / `挨拶` の 4 箱連鎖は Gen3 では opening + 紹介元番号判別 1 ブロックに圧縮
- `社内テスト用分岐 / _1` の識別子（渡部 / 新坂 / 鹿児島① / 鹿児島②）は **本番移管時は削除**。customer_doc に「テスト識別子」として注記のみ残す
- `挨拶` と `冒頭` の 2 つの opening 文言が main にあり、`挨拶` は `社内テスト用分岐` 経由、`冒頭` は `社内テスト用分岐_1` 経由 → 一般入電 = 挨拶（初診専用 AI）、医療機関入電 = 冒頭（一般 AI）の使い分け → Gen3 では incoming-classifier の sip from で振り分けて 2 種の opening 文言を準備
- 「次へ」相当の繋ぎ箱は本シナリオには無し（直接 announcement に文言が埋まっている） → 短い「ありがとうございます。」「折返しの際にご確認いたします。」は次段ブロック先頭 TTS にマージ
- セリフ none 振り分けのみの箱（`用件を判断` / `_1〜_3` / `診療科分岐` / `紹介元番号判別` / `紹介元番号判別_1` / `紹介元病院名取得` / `年末年始終話へ`）は独立ブロック不要 → CMR or save2db に吸収
- 聞き取り NG 系（`用件NG` / `紹介状NG` / `連絡先番号NG`）は **すべて `失敗` box（代表案内 099-230-7000）に集約** → Gen3 では各 hearing/subflow の error_jump 先 termination に統一
- `クローズ` 文言は「を受付いたしました。土日祝日を除く、3診療日以内に〜」と冒頭の主語が省略されている → 直前の announcement（`初診` / `変更` / `確認` 等）と組み合わせて完成させる構造 → Gen3 では各用件 announcement + クローズ文言を 1 つの termination に統合
- `折返しに / _1〜_6`（「折返しの際にご確認いたします。」）は **聴取できない場合に「折返し時に再確認」で逃がす保険的中継** → Gen3 では削除して直接 next を成功側に向ける（保険的 announcement は冗長）
- `年末年始終話へ` 箱は **セリフ none で素通り** → Gen3 では入電時間判断 Script 内で年末年始判定し、該当時は時間外 termination へ
- `①` 箱（「医療機関の方は…」announcement）は名前が環境依存記号 ① だが、この一文は本フローの opening 後段で全入電に流れる重要文言 → ASCII モジュール名（例: `intro_announcement`）にリネーム
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字（`①` 等）は scaffold で `jump_xxx` 等にリネーム
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**
- **CMR conditions は必ず最後に `match: "other"` + next 明示**
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
- Gen1 動的タグ `<? $var ?>` は `<% var %>` 構文へ
- **diagnosis_subject 聴取は無し**: 受診者 = 入電者前提、PatientName 1 枠で完結
