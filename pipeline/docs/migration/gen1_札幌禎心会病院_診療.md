### 移管方針
- Gen1 commubo の単独中規模シナリオ。営業時間判定→当日受診確認→用件分類(2段階)→紹介状有無→診療科聞き取り→患者情報聴取(氏名/生年月日/連絡先電話)→折返し案内、という典型的な「予約専用ダイヤル」型。
- basic_info は main.yaml デフォルト(tts_g/AmiVoice/openai_assistant)を尊重。受付時間は平日9-17、土9-12、日祝・年末年始休。代表電話 011-712-1131。

### 施設固有の特殊ルール

**1. 営業時間判定が冒頭に必須(時間判断.yaml)**
- API_DRJOY_CONTEXT_SETTINGS → 挨拶 → API_DRJOY → 時間判断 という流れ。時間判断は **(a) 12/30〜1/4 → 年末年始アナウンス → END_切断 / (b) 平日9-17/土9-12 → 通常フロー / (c) それ以外 → 時間外アナウンス → END_切断** の3分岐。
- 年末年始アナウンス文言: 「申し訳ございませんが、ただいまの時間は、年末年始のため受付時間外となっております。受付開始は2026年1月5日月曜日の朝9時からとなっております。」(年度依存。director判断: テンプレ化のため「年末年始のため受付時間外…」程度に簡略化推奨、復帰日付は properties で年次更新)
- 時間外文言: 「受付時間は、日曜祝日、年末年始を除く、月曜日から金曜日の9時から17時、土曜日の9時から12時となっております。」

**2. 当日予約は受付不可**
- 用件聞き取り前に「本日の予約に関する内容ですか？」と聞き、Yes なら「当日のご予約に関しては、こちらのAI電話ではお受けできません」→代表案内→切断。
- No のときのみ用件聞き取りへ進む。

**3. 2段階用件分類(独特)**
- **第1段階(用件聞き取り.yaml)**: 「診察予約/予約変更/予約キャンセル/予約確認」の4択 + 復唱確認。新患確認の分岐は Gen3 では情報提供書サブシナリオ側に統合される。
- **第2段階(情報提供書.yaml)**: 「予約 / 情報提供書」の2択。情報提供書を選ぶと診療科フリー(医療機関宛フリーワード)へ。
- 用件を判断箱(main.yaml): 予約確認→確認内容聞き取り、再診/予約変更/キャンセル/新規予約→紹介状有無聞き取りへ分岐。**4分岐** のため CMR で実装、末尾は `match: "other"` で代表案内へフォールバック。

**4. 紹介状有無→選定療養費アナウンス**
- 新規予約のみ紹介状有無を聞く。**紹介状なし** の場合: 「初診時選定療養費として、1,100円を全額自己負担いただいております」アナウンス必須。

**5. 診療科 (27科 + 代表案内3科)**
- 通常診療科(分岐なし): てんかん外来/ペインクリニック外科/めまい難聴外来/緩和ケア外来/形成外科/健診センター/歯科口腔外科/腫瘍内科/循環器内科/消化器外科/消化器内科/心臓血管外科/神経内科/整形外科/脊椎脊髄末梢神経外科/糖尿病内科/頭頸科頭頸部腫瘍センター/乳腺外科/脳神経外科/泌尿器科/皮膚科/膝関節専門外来/婦人科/物忘れ外来/放射線治療科 (25項目, 残2は entity_classification.md 参照)
- **代表案内行き(AI不可)**: がんセカンドオピニオン外来 / 救急 / 血管内留置デバイス外来。マッチ時「<診療科>はAI電話では受付できません」→代表案内→切断。
- 5回不一致 + 2回再質問でも取れない場合は END_診療科聞き取りNG → 代表案内。

**6. 予約日聞き取り(予約変更/キャンセル/予約確認時のみ)**
- 「予約票に記載されている予約日を『4月1日』のように」聴取。
- 西暦不要(月日のみ)、過去日付なら自動で+1年。DTMF 4桁(月2桁+日2桁)対応。director判断: Gen3 hearing_datetime テンプレ + addCurrentDate=Yes で対応(年補完を Brekeke 側へ寄せる)。

**7. 診察券有無判定**
- 新規予約以外(予約変更/キャンセル/予約確認): 診察券番号聞き取り → 有り扱い。「わからない/持ってない」と発話で無し扱い。
- 新規予約: 診察券無し扱いで氏名→生年月日へ進む。

**8. 連絡先電話番号(必須・最後の聴取)**
- 入電番号が携帯ならそれを連絡先候補として復唱確認。
- 入電固定 or 復唱否定なら別番号聴取(0始まり市外局番 or 携帯番号)。10/11桁チェック、DTMF対応。
- 固定で取得した場合は **追加で「緊急時連絡先として携帯番号」聴取**(任意、「持っていない」で skip 可)。
- 完了文言の分岐: 携帯あり→「3診療日以内に折り返し電話 or SMS」、固定のみ→「3診療日以内に折り返し電話」。

### director が見落としやすいポイント

- **用件分類が2段階**: 1段目で「予約 vs 情報提供書」を分けず、第1段で4択(予約系のみ)→ 別途情報提供書ルートを並走させている設計。Gen3 では incoming-classifier 直後に「予約系/情報提供書」の context_match_router を1枚追加して整理することを推奨(director判断)。
- **新患確認(初診ですか?)**: 用件聞き取り内の `新患確認` は Gen3 では分岐としては保持するが、新規予約に統合可(初診=新規予約の同義)。
- **diagnosis_subject(誰の予約か)聴取は無し**: 患者氏名=受診者前提、入電者氏名や続柄は聞かない。PatientName 1枠で完結。
- **生年月日聴取は患者本人**: 「患者さんの生年月日」と明記されているため hearing_dob は subject="patient"。
- **代表案内の電話番号は2回読み上げ**: 「電話番号は011-712-1131です。繰り返します。電話番号は011-712-1131です。」
- **モジュール名は ASCII 必須**(丸数字禁止)、CMR.reference_module の subflow 参照は `jump_{step名}`、CMR conditions 末尾は `match: "other"` 必須。
