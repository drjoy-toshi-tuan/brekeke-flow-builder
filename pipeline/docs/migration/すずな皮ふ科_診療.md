# 既存診療フロー修正 — 疑義照会ルート追加

## 修正概要

既存「すずな皮ふ科 診療」フロー（用件 4 択：お問い合わせ／予約／予約変更／予約キャンセル）に **疑義照会ルート（用件⑤）** を新規追加する。入電者は薬局担当者で、Dr.JOY 電話帳登録 4 薬局を `incoming-category-classifier` で自動判定し、ヒット時は `Phone2Name` で薬局フリガナを動的差込、未ヒット時は薬局名 hearing にフォールバックする。

## 修正サマリー

- 影響ブロック数: **9 ブロック**
- 変更ブロック: **2 件**（用件 hearing / openAI_用件 分岐ラベル拡張）
- 追加ブロック: **7 件**（薬局判定 opening / 薬局名hearing / 担当者名 hearing / 患者名 hearing / 患者生年月日DTMF hearing / 疑義内容 hearing / 終話_疑義照会 termination）
- 削除ブロック: 0 件

## 追加ブロック詳細（新規 7 ブロック）

### ブロック「薬局自動判定」(opening タイプ — 疑義照会ルートのエントリポイント)

- **修正種別**: 追加
- **構成モジュール**:
  - `電話帳分類` ※type=`drjoy^Incoming$incoming-category-classifier`（**incoming-classifier ではない**、`-category-` 付き）
    - リスト名は人間が Dr.JOY 電話帳で事前登録（薬局名 4 件: クオール薬局 / ミツヤ薬局 / ミネ薬局 / 日本調剤）
    - next: FOUND → `Phone2Name`, NO_RESULT/TIMEOUT/ERROR → `薬局名` hearing block（フォールバック）
  - `Phone2Name` ※type=`drjoy^External Integration$Phone2Name`
    - `FOUND_KATAKANA_NAME_DEFAULT_TMP: "{recipient_name}さんですね。"` — raw.md L223 に従う
    - next: → `担当者名` hearing block
- **連結**: openAI_用件 の `疑義照会` 出力 → `電話帳分類`
- **fixer への指示**:
  - 4 薬局のリテラル名を TTS に埋め込む方式は禁止（電話帳メンテで TTS 群を直すのを避ける）
  - 必ず `Phone2Name` の `recipient_name` プレースホルダ経由で動的差込
  - 電話帳登録は人間作業として確認レポートに上げる

### ブロック「薬局名」(hearing — 電話番号登録なし時のフォールバック)

- **入電時の流入元**:

## director が見落としやすいポイント（再掲）

1. **`incoming-category-classifier` ≠ `incoming-classifier`** — 前回 director が誤選択（feedback_incoming_category_classifier）。薬局判定は `-category-` 付きの方
2. **`Phone2Name` の `recipient_name` プレースホルダ**を使う 1 本構成。薬局名リテラルを 4 本並べる方式は NG
3. **患者本人由来ではない** ので PatientName / DateOfBirth サブフローは使わず hearing として直接 main に配置（小山記念病院_薬剤部と同型）
4. **患者生年月日は DTMF 必須** — stt_type=DTMF_AmiVoice 系
5. **status=0 禁止** — 終話は status=2
6. **SMS 送信なし** — 疑義照会は薬局相手のため sms_flag=-1
7. **profile_words 4 薬局は全件転記** — クオール薬局 / ミツヤ薬局 / ミネ薬局（ミネドラッグ）/ 日本調剤、ヨミ付き
8. **echo_back は全 hearing で false** — raw.md アナウンス一覧の「復唱有無」全項目「無し」（連絡先電話番号復唱もこのフローでは不要）
