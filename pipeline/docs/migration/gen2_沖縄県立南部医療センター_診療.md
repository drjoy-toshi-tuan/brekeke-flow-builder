### 移管方針
- 用件分類は Pattern A（用件用 generate_by_OpenAI 直後に ContextMatchRouter で 新規 / 変更 / キャンセル / 確認 の 4 分岐 + catch-all `^*$`）。
- 年齢判定は用件より前段：**小児科か否か Yes/No** を OpenAI 判定し、`age=小児/成人` をセット。以降の診療科候補がこの分岐で変わるため、診療科ヒアリングは小児側／成人側で別モジュールに分ける（subflow 化せず scenario_flow 内で並列に持つ）。
- 「薬服用中か → Yes なら残薬数確認 → 無しなら AI 不可」という **二段 Yes/No ゲート** を変更・キャンセル共通で配置。残薬数は自由テキスト（OpenAI 正規化なしでよい）。
- PatientName / DateOfBirth / PhoneNumber / MedicalCardNumber は共通サブフロー（Jump to Flow）で接続。RAG サブフロー・Date of Call Classifier・FAQ ループは本シナリオ不要（最後の問い合わせは `question` 1 ターンの単純聴取で「なし/あり」判定のみ、RAG 化しない）。
- 非通知着信は incoming-classifier 側で分岐 → 非通知専用 termination（status=7, smsFlag=0）へ直行。

### 施設固有の特殊ルール
- **echo_back 対象は 用件 / 生年月日 / 予約日 の 3 項目のみ**。患者名は「絶対に復唱しない」と元資料で明示（L.80 付近）。診療科も原則復唱禁止、ただし **小児歯科口腔外科は復唱時のみ「歯科口腔外科」と言い換える**（小児歯科口腔外科 とは絶対に復唱しない）。
- **病理診断科は AI 予約不可**（成人のみリストあり）→ 専用案内 TTS → 電話案内（status=2, smsFlag=0）。
- **紹介状無し → AI 不可**（新規のみ）。文言に選定療養費 7,000 円の説明が含まれる固定文のため原文厳守。
- **残薬無し → AI 不可**(変更・キャンセル）。代表 098-888-0123 案内付き。
- **smsFlag は用件で分ける**：新規=1 / 変更=2 / キャンセル=3 / 確認=4。キャンセルのみ終話文言が別（「予約キャンセルのお申し込みを受付いたしました」）。
- **非通知着信専用 status=7**（一般的な status=2 ではない）。非通知用アナウンスは冒頭アナウンスを兼ねた 1 段で流して即終話。
- **診察/予約/新規/小児/眼科の救済語**（元資料 L.728-737）: 「印刷/定期/新薬/再診/最新/禁忌」→「診察」、「薬/お薬」→「予約」、「電気/でんき」→「新規」、診療科聴取時「そう二」→「小児」、「癌化」→「眼科」。OpenAI プロンプト側で吸収する前提。
- **数字正規化**（電話番号・診察券番号）: `C/c/Ｃ/ｃ/for/ふぉー/フォー/しー/しい/し → 4`、`ごー/ごお/ご → 5`。
- **「以上です／以上」ルール**: 単独発話は無視して同質問を繰り返す。末尾付きなら切り落として採用。最後の問い合わせ確認への返答としての「以上」のみ終話合図。
- 予約日の年補完: 10 月以降に 1〜9 月の日付を言われたら翌年扱い。

### director が見落としやすいポイント
- 04/01 更新版で **「復唱後否定の再聴取（最大 2 回）」「再質問アナウンス A/B/C」** の詳細仕様が追加されている（L.109-239）。Gen3 ではこの細部はサブフロー Retry 2 回 + 復唱用 retry ブロックに吸収され、固定文言は properties 側で扱う。director は scenario_flow に再質問用の独立ブロックを増やさないこと。
- **年齢判定が最初（用件より前）** である点に注意。用件 OpenAI の params.prompt は age 前提で書かれるので、年齢判定ブロックは必ず用件前に置く。
- **「変更用件の薬残数あり側」と「キャンセル用件の薬残数あり側」は合流しない**。両者とも「診療科→予約日→理由→診察券→氏名→DOB→電話→最後の問い合わせ」は共通だが、**終話先が変更=通常終話 / キャンセル=終話キャンセル** と異なる。ContextMatchRouter で classification を見て終話を分岐させる（情報収集は 1 本化、終話のみ分岐）。
- **新規（小児/成人）と変更/キャンセル/確認では都合が悪い日の位置が違う**: 新規=紹介状 Yes 直後、変更=理由の後、キャンセル=理由の後、確認=無し。director が「情報収集ブロックを共通化」しすぎて位置を統一しないよう注意。
- **成人の新規のみ「医療機関名(facility)」聴取あり**（紹介状 Yes 直後、都合が悪い日の前）。小児の新規ルートには medicalInstitution 聴取が無いため共通化不可。
- **「予約日の確認」用件では `reason` に確認事項を格納**（TTS は「次回の予約日時を教えて欲しい、など」）。理由欄を使い回す変則運用なので function 定義時に clinicalDepartment/reason の enum/text 区別を誤らないよう注意。
- 代表電話 `098-888-0123` は複数箇所で使用（病理診断科案内・残薬無し案内・RAG 未ヒット案内）。全箇所 `<speak type="telephone" breakc="300ms">` の SSML を Gen3 では削除し、{tts_g:...} の平文読みに揃える（Gen3 は SSML 禁止）。
- 04/01 版に残っていた **「function は 1 通話 1 回のみ」「endpoint=通話完了は終話アナウンス出力後限定、それ以外は途中切断に上書き」** は Gen3 では termination ブロック側で担保されるためプロンプト禁則に書かない（director 判断で落として可）。
