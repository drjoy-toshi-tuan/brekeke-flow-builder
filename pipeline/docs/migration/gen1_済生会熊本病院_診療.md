### 移管方針
- Gen1 commubo → Gen3 ブロック構造へ素直に移植。基本は director デフォルト 12 項目に従う。
- 用件は「予約確認 / 予約変更 / 予約キャンセル / その他お問い合わせ」の **4 値分類**。DTMF (1=確認 / 2=変更 / 3=キャンセル / 4=その他) も併用。
- 個人情報サブフローは **氏名 → 診察券（7桁）→ 生年月日 → 連絡先電話番号** の順。氏名は PatientName サブフロー 1 枠に充当（patient_name のみ、入電者氏名は無し）。

### 施設固有の特殊ルール

**1. GW 期間ガイダンス（Gen1 入電判断.yaml に常設）**
- 5月2日〜5月6日は専用「GW時間外ガイダンス」を流す: `ただいまの時間は受付時間外となります。ゴールデンウィークの休業期間は、5月2日から5月6日までです。5月7日8時30分より、受付を開始いたします。…`
- date_of_call_classifier 系で **GW分岐 + 平日8:30〜17:00** の二層判定が必要。**director 判断**: Gen3 標準 date_of_call_classifier に GW 専用分岐を入れるか、年末年始 Dr.JOY 側ロジックと同様に時期固定文言を別 announcement で出すか確認のこと。年単位で日付が変わるためロジック化推奨。

**2. 受付時間**
- 平日 8:30〜17:00。土日祝は時間外。

**3. 診察券 7 桁 + DTMF/シャープ/アスタ対応**
- 発話 (sys_number) と DTMF の両受け。「シャープ」「トーン（アスタ）」発話時は再聴取扱い（リセット）。
- **わからない** で END_わからない へ抜けてスキップ可（診察券無しでも進行）。
- 桁数チェック: 7桁以内=採用 / 8桁以上=再聴取 / 0桁=count再質問。

**4. 連絡先電話番号の特殊フロー**
- 入電番号取得済み → 「今おかけいただいている <番号> でよろしいですか？」確認 → 肯定なら入電番号採用、否定なら別番号聴取。
- 入電番号未取得（非通知含む後段）の場合は最初から「ご連絡先の携帯番号、または0から始まる市外局番を…」聴取。
- 携帯チェック: 入電番号 or 聴取番号が固定電話だった場合、追加で「緊急時連絡用に携帯電話番号」を別途聴取。「持っていない」発話で完了扱い。
- **director 判断**: Gen3 では携帯追加聴取は通常省略している施設が多い。済生会熊本は明示的に二段階聴取しているので **そのまま再現するか、簡略化するかは要確認**。

**5. 非通知時のアナウンス**
- 「電話番号が非通知設定の場合、AI電話では受付できません。先頭に数字の、1、8、6、を付けておかけなおしください。」 → 終話。
- 標準の incoming-classifier 非通知分岐に合流させる。

**6. 失敗時の代表案内（番号案内）**
- 2 回失敗集まれ → 入電番号有無分岐 → 「土日祝日を除く翌日までに、当院よりお電話いたします」 or 「8時30分から17時に、予約確認センターへお掛け直し下さい。電話番号は、096‐351‐8051です。」を3回繰り返してから終話。
- **director 判断**: Gen3 では call_transfer または termination + announcement で代替。代表番号 **096-351-8051** の読み上げを TTS で組み込む際は `<% var %>` 構文不要（リテラル）。

**7. 用件分岐後の挙動差**
- 「予約確認」「その他」→ そのまま `確認内容の聴取` へ（別途キャンセル理由などは聴かない）。
- 「予約変更」「予約キャンセル」→ **現在の予約日**（予約日.yaml）を聴取してから → `用件を判断_2` で再分岐: 変更なら **希望日**、キャンセルなら **確認内容の聴取**（キャンセル理由）。
- 完了時のセリフ差: 変更=「予約変更の申し込みを受付いたしました」/ 確認=「予約確認の申し込みを受付いたしました」/ キャンセル=「予約キャンセルの申し込みを受付いたしました」+「確認事項がある場合は、担当者よりお電話させていただきます…」/ その他=「その他お問い合わせ、の申し込みを受付いたしました」。
- 共通の締め文言: 「土日祝日を除く翌日までに、担当者から折り返し電話、もしくはショートメールにてご連絡いたします。」

**8. 終話文言**
- 通常: 「お電話ありがとうございました。それでは失礼いたします。」(END_切断)
- 時間外/GW/非通知: 同文言だが END_切断_1 系統。Gen3 では termination 1 本に集約してよい。

### director が見落としやすいポイント
- **GW 期間判定** は他施設にあまり無い特殊ロジック。年末年始と同レベルの優先度で扱う。Dr.JOY 側で吸収するか Gen3 ロジックに入れるか決める。
- **キャンセルのみ追加文言**「確認事項がある場合は、担当者よりお電話させていただきますので、ご了承ください。」を終話前に挿入する固有文言。
- **代表電話番号 096-351-8051** の読み上げ。
- **ASCII モジュール名必須**（丸数字・括弧・全角スペース禁止）/ CMR.reference_module の subflow 参照は `jump_{step名}` 形式 / CMR conditions 末尾は必ず `match: "other"`。
- 電話番号確認サブフローの **「持っていない」** エンティティ（hearing_phone_number テンプレ流用）は Gen3 でも携帯任意化ロジックで必要。
- 用件「その他」が確認系と同じ後段（確認内容の聴取）に合流する点。Gen3 で 4 値 enum 後の CMR 設計時に注意。
