### 移管方針
- Gen1「箱」構造 → Gen3 9 ブロック型に変換。`挨拶` → opening + 冒頭_アナウンス（2 ブロック 1 セット必須）。挨拶 TTS は「お電話ありがとうございます。西宮渡辺病院の、予約専用 AI 電話です」。
- `時間外判別` サブシナリオ（現在の曜日と時間 → 祝日チェック → GW分岐 → 平日/土曜の時間分岐）→ date_of_call_classifier ブロック（addCurrentDate: Yes 自動付与対象）。
- 用件分岐は **Pattern A**（用件 OpenAI 直後に 4 分岐: 診察予約 / 変更 / キャンセル / 確認）。各分岐に echo_back（診察予約復唱 / 予約変更復唱 / 予約キャンセル復唱 / 予約確認復唱）を直配置。
- 用件確定後に **「現在通院中の診療科のご予約ですか？」（タイトル_1） → 再診/新規セットの追加分岐**あり（diagnosis 用件のみ「本日のご受診をご希望でしょうか？」（タイトル）→ 当日予約チェックも挟まる）。Pattern B（後段 ContextMatchRouter）で実装妥当。
- 紹介状の有無確認サブフローは **新規予約ルートのみ実行**（`用件を判断` 箱で `新規 → 紹介状の有無確認`、それ以外 → 直接診療科聞き取り）。
- サブフロー（患者氏名 / 生年月日 / 電話番号 / 診察券）は Jump to Flow 経由。`電話番号区切り.yaml` は `連絡先携帯聞き取り.yaml` 配下から呼ばれる読み上げ整形用 → Gen3 サブフローテンプレ内蔵で完結、director は触らない。
- 終話直前で **携帯電話番号取得有無の分岐**（用件 4 種それぞれに `携帯電話番号チェック / _1〜_3`）→ 「電話、もしくは SMS でご連絡」vs「電話にてご連絡」の 2 種類の終話 TTS を出し分け。Gen3 では context_match_router 1 個に集約可能（director 判断）。

### 施設固有の特殊ルール
- **受付時間**: 日曜祝日除く、月〜金 8:30〜16:30、土曜 8:30〜11:30。GW 期間は **5/6 が診療日となる年がある**（`GW分岐` 箱）→ date_of_call_classifier の祝日カレンダーで個別ハンドリング（director 判断、時刻判定は通常の平日ロジックを再利用）。
- **代表電話**: 0798-74-2630（当日予約案内 / 聞き取れず終話 / 健康診断案内すべてで 3 回繰り返し含める）。
- **当日予約は AI 不可**: 診察予約フローのみ「本日のご受診をご希望でしょうか？」（タイトル箱）→ 肯定なら `END_当日予約` → 「当日のご予約に関しては、こちらの AI 電話では受付できません」案内で代表電話に誘導して切断。
- **健康診断は AI 不可**: `END_健康診断` 経由で「健康診断についてのお問合せは、11 時から 16 時の間に代表番号へ」案内。用件聞き取り中の `健康診断` エンティティと診療科聞き取り中の `健康診断`（条件4 = 健康診断）両方から到達。
- **緊急/救急**: 診療科聞き取りで「救急 / 緊急」エンティティ → `END_緊急救急` → 「大変申し訳ございません。この電話では受付しておりません。代表電話 0798-74-2630 へご連絡ください」案内で切断。
- **3 回失敗時**: 用件・診療科・紹介状・予約日・氏名・確認 各サブで `count 聞き直し`（5 以上で escalate）→ 3 回不可で `END_聞き取れず` → 「お電話が遠いようで」案内で代表電話誘導。Gen3 では termination_patterns に集約。
- **DTMF（4 桁）受付**: 予約日聴取で「シャープ / アスタ + 月 2 桁 + 日 2 桁」を受け付ける。比較日より前なら +1 年で確定する仕様（`日付比較 → プラス1年後で確定 / そのまま確定`）。Gen3 でどう扱うかは director 判断（SKILL_C 自由テキスト + DTMF 並走 / または OpenAI プロンプトで吸収）。
- **用件確定後の通院歴分岐**（タイトル_1 → セットした用件の分岐_1）: 診察予約用件のみ「現在、通院中の診療科のご予約ですか？」を聞き、肯定 → 再診セット / 否定 → 新規セット。新規のみ紹介状の有無確認に進む。**他用件（変更 / キャンセル / 確認）はこの分岐をスキップ**して直接診療科聞き取りへ。
- **復唱（echo_back）対象**: 用件のみ（4 種）+ 診療科 + 予約日 + 紹介状有無は内蔵。氏名・生年月日・電話・診察券はサブフロー内で完結、メインに echo_back を出さない。

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` → `挨拶` → `API_DRJOY` → `時間外判別` の 4 箱は Gen3 では opening 冒頭処理に吸収。`API_DRJOY_CONTEXT_SETTINGS` / `API_DRJOY` は明示ブロック化不要。
- 「セリフ: none」かつ振り分けのみの「箱」が大量（`次へ` / `次へ_1` / `新規チェック` / `キャンセルチェック` / `携帯電話番号チェック / _1〜_3` / `用件の確認` / `用件の確認_1` / `用件を判断` / `用件を判断_1` / `完了フラグ` / `当日チェックNG` / `診察券有り` / `受付時間内` / `変更_1` / `キャンセル_1` / `電話切断へ` 等多数）→ Gen3 で独立ブロック不要、前段の next 書き換え or context_match_router の一部として吸収。
- `次へ`（セリフ「では次に、」）/ `生年月日聞き取り_1`（セリフ「続いて」）/ `連絡先電話番号聞き取りへ`（セリフ「最後に、ご連絡先のお電話番号をお伺いします」）/ `新規予約・予約変更・キャンセル・定期受診・確認へ`（セリフ「それでは、」）/ `用件再設定へ`（セリフ「失礼しました、」）等の **短い繋ぎ TTS は次段 hearing/subflow ブロックの導入文言にマージ**するか、announcement ブロックで個別に配置するかは director 判断。
- `予約変更の流れ` / `予約キャンセルの流れ（特定診療科）` / `再診の流れ` / `予約確認の流れ` の 4 箱と、それぞれの `_1`（携帯あり版）の **計 8 箱は実質同じ「受付完了 + 折り返し連絡」案内の出し分け**だけ。Gen3 では「電話折り返し」「電話 or SMS 折り返し」の 2 文言に集約し、context_match_router で 4 用件 × 携帯有無の 8 通りを分岐させる構成（director 判断）。
- `用件聞き取り.yaml` 内に **「もしもし」エンティティと count もしもし系 retry はなし**（北里大・多摩総合と異なる）。OpenAI プロンプトに「もしもし」誤検知排除を入れる必要は薄い。
- `紹介状の有無確認.yaml` の到達は **新規予約ルートのみ**（`用件を判断` 箱で `入力: 新規` から `紹介状の有無確認` へ。`用件を判断_1` 箱の `入力: 新規` ルートは `携帯電話番号チェック_1` 経由なので別系統）→ Gen3 では「新規」かつ「診察予約用件」の組合せでのみ subflow 呼び出し。
- 用件聞き取り中の `肯定エンティティ` / `肯定（エンティティ）` / `否定エンティティ` / `否定（エンティティ）` / `肯定否定` 等の **微妙な命名違いが多数 同居**しているのは Gen1 の慣習。Gen3 では SKILL_B（Yes_No 判定）で 1 本化。
- `予約日の聞き取り.yaml` は「日にち先行発話 / 月先行発話 / 数字のみ / DTMF」の 4 系統 + 0 桁詰め + 日付比較で **複雑だが Gen3 では date OpenAI（YYYY-MM-DD 出力）+ DTMF 並走で代替**。元 yaml の `生年月日の判定`（大正/昭和/平成/令和の元号判定）は **予約日では不要、生年月日聞き取りサブフロー側のみで使う**点に注意（director 混同しやすい）。
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字（①②③…⑩）等の環境依存文字は禁止。
- **ContextMatchRouter.reference_module は subflow を参照する場合 `jump_{step名}`（scaffold が付ける prefix）を使う**。subflow ブロックの step 名そのもの（例: `電話番号聴取`）ではダメ。
- **CMR conditions は必ず最後に `match: "other"` + next 明示**。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。`<? $診療科の読み ?>` `<? $予約日の読み ?>` 等の Gen1 変数記法は Gen3 では `<% var %>` に変換。
- 患者氏名は PatientName サブフロー 1 枠に充当（patient 1 名のみ、入電者 = 受診者前提）。
- entity_classification.md / entity_stt_patch.json / entity_openai_synonyms.md にエンティティは自動分類済み。director は転記不要、scaffold/properties が自動取り込み。
