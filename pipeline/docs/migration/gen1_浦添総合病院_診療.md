### 移管方針
- 主流れ: opening → API_DRJOY → 用件分類 (4種+症状確認) → CMR (用件別) → 診療科 subflow → 患者氏名 subflow → 生年月日 subflow → 連絡先 subflow → 用件別完了アナウンス → termination
- 用件は **4分類**: `予約変更` `予約キャンセル` `予約確認` `予約（診察=初診/再診）`。発話＋DTMF 1/2/3/4 両対応
- 用件分類直後に **症状確認 hearing** (Yes/No) を全 4 用件の前段に挟む。Yes なら当日翌診療日変更希望として代表案内 (AI受付外) へ即離脱
- 診療科 subflow は分岐なし／復唱のみ／分岐あり (呼吸器・心臓・術前・肝臓/外科・膵臓/外科) の5系統＋復唱確認。診療科聴取は `予約変更/キャンセル/確認用` と `初診/再診用` で冒頭TTSが異なるが内部処理は共通
- 初診予約のみ **紹介状有無 subflow → 紹介状なしなら予約センター案内** で AI 受付対象外として離脱
- 連絡先聴取は API_DRJOY 入電番号取得済みならスキップ → 入電番号確認分岐で聞き取りせず登録
- 完了アナウンスは用件×携帯フラグで 3 種使い分け（予約/変更/キャンセル＝3診療日以内に折り返し or SMS、予約確認かつ携帯＝SMS、予約確認かつ携帯以外＝折り返し）

### 施設固有の特殊ルール
- **症状確認は全用件共通の前段ゲート**: 用件分類後・診療科聴取前に「現在症状がおありで、当日または翌診療日のご予約が必要でしょうか」を Yes/No hearing → Yes は AI 非対応として代表案内へ離脱、No のみ正規ルート進行
- **当日予約確認:5/16 サブフロー**: 予約変更/キャンセル/確認の場合のみ診療科聴取後ではなく**用件分類直後**に「本日のご予約の方ですか」を hearing → Yes は当日変更希望として代表案内、No なら通常ルート。`再診予約`の場合はこのサブフローを通らず直接診療科聴取
- **診察(=予約)用件の通院確認**: 4=予約 が選ばれたら「通院中の患者さんですか」を hearing → Yes=再診ルート / No=初診ルート。**初診のみ紹介状 subflow** → 紹介状なしは予約センター案内終話。再診は直接診療科聴取
- **内科単独発話**: 診療科聴取で「内科」のみ発話された場合、`内科発話後フリー聴取` subflow で「もう少し具体的に診療科をお話しください」と再ヒアリング
- **AI非対応診療科**: 救急外来 / 歯科口腔外科 (変更・キャンセルのみ) / 心臓リハビリ外来 → それぞれ専用の代表案内へ。救急外来は`代表電話 098-878-0231`、その他は予約センター
- **代表電話の使い分け**: ①予約系不明・聞き取れず → 予約センター `0120-979-706`、②救急外来・歯科口腔外科 → 病院代表 `098-878-0231`、③完了時の折り返し連絡先 → 予約センター `098-851-5171`（折り返し電話発信元）、④携帯番号取得済み完了 → SMS で `050-1721-8524` から発信
- **電話番号案内の繰り返し**: `案内回数カウント` で 3 回読み上げて終話。文言を縮めない
- **時間外/年末年始アナウンス**: acceptance_times 側責任。「月-金 8:30-17:30、土 8:30-12:30、日祝・年末年始休み」「年末年始は 1/5 (月) 8:30〜」を保持

### director が見落としやすいポイント
- main.yaml の `現在の時間取得 / 時間判別 / 時間外コメント / 年末年始アナウンス` は dead code 扱い (acceptance_times に集約) → ノートに反映しない
- `テストのためルート変更 / TESTルート / 再テスト` 等は移管対象外
- 用件聴取 yaml の `用件の復唱（再診）` `用件の復唱（キャンセル）_1` は END_戻る で main に戻ってからの分岐再判定。Gen3 では CMR で OpenAI 出力 enum (再診予約/初診予約) で分岐させる
- 予約日聞き取り subflow は和暦判定 (大正/昭和/平成/令和) を全枚挙しているが、Gen3 では generate_by_OpenAI の output_format=datetime で吸収 → director判断 で 単純な日付 hearing に
- 用件正規化 4 値: `予約変更` `予約キャンセル` `予約確認` `予約（=診察=初診/再診統合）` を OpenAI で返させる。「予約確認」「予約以外」「予約とる」等の派生は類義語例で集約
- 連絡先聴取の `市外局番5桁/4桁/3桁/2桁` 区切り処理は Gen3 phonetype subflow で吸収 → ノート不要
- `END_戻る：内科発話` `END_戻る：歯科口腔外科` `END_戻る：心臓リハビリ外来` `END_戻る：救急外来` は診療科 subflow の戻り値 context で main 側 CMR 分岐するパターン → SUBFLOW_RETURN_CONTEXTS 設計
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字（①②③⑩等）等の環境依存文字は禁止。
- **ContextMatchRouter.reference_module は subflow を参照する場合 `jump_{step名}`**（scaffold が付ける prefix）を使う。
- **CMR conditions は必ず最後に `match: "other"` + next 明示**。
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止。
