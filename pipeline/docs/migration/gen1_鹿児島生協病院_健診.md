### 移管方針
- Gen1「箱」構造 + サブシナリオ 14 本（用件聴取 / 1名予約か複数予約か / 健診種類聴取 / 受診希望時期 / 予約日聴取 / 胃カメラ検査確認 / 問合せ① / 問合せ② / 氏名聴取 / 生年月日聴取 / 職場名聴取 / 連絡先聴取 / 電話番号区切り / 時間判断）→ Gen3 scenario_flow 9 ブロック型に変換
- 主な対応:
  - `API_DRJOY_CONTEXT_SETTINGS / API_DRJOY / 挨拶` → opening + 冒頭_アナウンス（2 ブロック 1 セット）。冒頭文言「お電話ありがとうございます。鹿児島生協病院の、健診専用、AI電話です。」
  - `時間判断` サブシナリオ（祝日チェック + GW 期間 + 通常日付判断）→ Gen3 では opening 直後の acceptance_times（曜日・時刻）+ Script モジュール（祝日 / GW チェック）に分解。GW 5/2-5/6 期間は専用アナウンス、通常時間外は標準受付時間外文言
  - `1名予約か複数予約か` サブフロー（はい=1名 / いいえ=複数名）→ Gen3 では用件聴取の前段に Yes/No hearing 1 段配置。複数名時は氏名聴取後に職場名聴取が分岐するためフラグ保持
  - `用件聴取.yaml` → hearing（generate_by_OpenAI、4 分岐: 健診 / 変更 / キャンセル / 問い合わせ）。各分岐直後に echo_back（用件の復唱）4 本配置 → Pattern A
  - `健診種類聴取.yaml` → 新規ルート専用 hearing 3 分岐（採用時健診・定期健診 / 協会けんぽ / 人間ドックその他）+ DTMF 1〜3 並列。**「採用時健診・定期健診」のみ「所定用紙有無」の追加 Yes/No hearing が連続**（用紙無しは案内のみ、有りは「当院所定用紙のみ対応」announcement）
  - `予約日聴取 / 受診希望時期` → datetime hearing。addCurrentDate: Yes 自動付与（年補完を Brekeke 側）
  - `胃カメラ検査確認` → 変更ルート途中の Yes/No hearing
  - `氏名聴取 / 生年月日聴取 / 連絡先聴取（電話番号区切り含）` → subflow ブロック Gen3 標準サンプル JSON 完コピ。**氏名は受診者氏名 1 枠 PatientName** に充当（複数名時も代表 1 名のみ）
  - `問合せ①` → 新規ルートの「内容/コース」聴取（自由文 hearing）、`問合せ②` → 問い合わせルートの自由文聴取（output_format: text）
  - `職場名聴取` → 複数名予約 or 企業健診時のみ呼ばれる自由文 hearing。氏名後の `企業/複数名 分岐`（職場 / 複数名 → 職場名聴取、それ以外 → 生年月日）で振り分け
- 用件別の合流先:
  - **健診（新規）**: 健診種類 → 谷山生協番号案内 → 受診希望時期 → 氏名 → 企業/複数名分岐（職場名 or skip）→ 生年月日 → 連絡先 → 完了フラグ → 終話挨拶_1（受診結果2週間案内付き）→ 終話
  - **変更**: 予約日聴取 → 胃カメラ検査確認 → 受診希望時期 → 氏名 → 生年月日 → 連絡先 → 終話挨拶 → 終話
  - **キャンセル**: 予約日聴取 → 氏名 → 生年月日 → 連絡先 → 終話挨拶 → 終話（変更と異なり胃カメラ確認なし、希望時期なし）
  - **問い合わせ**: 問合せ① → 問合せ② → 氏名 → 生年月日 → 連絡先 → 終話挨拶 → 終話

### 施設固有の特殊ルール
- **受付時間**: 月〜金 8:30〜16:30、土 8:30〜12:00、日祝・年末年始休 + GW（5/2-5/6）休診。年末年始は 12/28 16時以降〜1/3 終日アナウンス（main.yaml `12/28 16時以降〜1月3日終日` ＋ `12/28時間判断`）
- **GW 専用アナウンス**: 「5月6日まで休診となっております。受付再開は5月7日からとなっております。」5/2 12時以降〜5/6 までを GW 終話扱い（時間判断.yaml `GW 日付分岐` / `5/2 時間分岐` / `GW`）
- **代表電話番号**: 099-267-1455（聴取失敗 / 時間外 / 年末年始時の案内）。3 回以上アナウンス後カット（`番号案内` → `回数カウント` ループ）
- **健診種類 3 分岐 + 所定用紙確認**: 採用時定期 / 協会けんぽ / 人間ドックその他。**採用時定期のみ「所定用紙お持ちですか？」追加聴取**、用紙有りは「当院所定の用紙での対応となり、手書きでの対応はできません」announcement。協会けんぽ・人間ドックは復唱のみで通過
- **複数名予約時のみ職場名聴取**: 1名予約 = skip、複数名 = 職場名聴取（企業/複数名 分岐の `職場` または `複数名` 一致時 → 次へ_8 → 職場名聴取）
- **新規ルートの `谷山生協番号案内` 箱**: 健診種類聴取後に「（鹿児島生協病院ではなく）谷山生協クリニックの健診をご希望の場合は…」相当の案内が入る経路あり（main.yaml `谷山生協番号案内` → 受診時期聴取へ）。**未稼働の【015】谷山生協クリニックを意識した分岐の名残** → director判断: 現状ではセリフ none で素通りしている可能性 → customer_doc を再確認、稼働してなければ単なる pass-through として削除
- **「もしもし」検知**: 用件聴取・健診種類聴取の入口で `countもしもし` 計数。1 回目「はい、聞こえております」、2 回目以上「はい、『もしもし』と聞こえております」→ 用件再聴取
- **携帯チェックで締め文言切替なし**: 倉敷・蘇生会と異なり、終話挨拶は「3営業日以内に担当者から折り返し電話、もしくは、ショートメールにてご連絡いたします」共通文言（携帯/固定で分岐しない）。新規時のみ「健診結果票は受診日から2週間程度かかります」追加（終話挨拶_1）

### director が見落としやすいポイント
- 冒頭の `API_DRJOY_CONTEXT_SETTINGS` / `API_DRJOY` は Gen3 では opening 冒頭の標準処理に吸収 → 明示ブロック化不要
- 「次へ」/「次へ_1〜_11」は短い繋ぎ TTS（「それでは、」「続いて、」「最後に、」）→ 次段ブロック先頭 TTS にマージ（director デフォルト）
- セリフ none 振り分けのみの箱（`用件の分岐` / `用件の分岐_1` / `用件の分岐_2` / `企業/複数名 分岐` / `複数名判別` / `完了フラグ` / `通話完了` / `現在の時間取得` / `12/28 16時以降〜1月3日終日`）は Gen3 では独立ブロック不要 → CMR or save2db に吸収
- 「CP 〜」相当のリトライ失敗箱（`用件聞き取れずCP` / `健診種類聞き取れず` / `予約日NG` / `会社名NG` / `胃カメラ聴取NG` / `希望日NG` / `氏名NG` / `生年月日NG` / `連絡先NG` / `予約人数聞き取れず` / `問合せ①聞き取れず` / `問合せ①聞き取れず_1`）は **すべて `3回失敗：終話へ` に集約** → Gen3 では各 hearing/subflow の error_jump 先 termination に統一（代表案内 + 受付時間案内 + 番号読み上げ + 切断）
- `胃調査CP` 箱は名称に反し「胃検査聴取後の合流地点」→ 受診希望時期へ繋ぐだけの pass-through。Gen3 では削除、胃カメラ確認の next を直接希望時期に向ける
- **モジュール名・termination 名は ASCII または安全な日本語のみ**。丸数字（①②③…⑩）等の環境依存文字は禁止
- **ContextMatchRouter.reference_module は subflow 参照時 `jump_{step名}`**（scaffold prefix）を使う
- **CMR conditions は必ず最後に `match: "other"` + next 明示**
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止
- Gen1 動的タグ `<? $予約日の読み ?>` `<? $連絡先電話番号（ハイフン区切あり） ?>` `<? $入電番号（ハイフン区切りあり） ?>` 等は `<% var %>` 構文へ
- **diagnosis_subject 聴取は無し**: 受診者 = 入電者前提（複数名時のみ職場名追加）。PatientName 1 枠で完結
