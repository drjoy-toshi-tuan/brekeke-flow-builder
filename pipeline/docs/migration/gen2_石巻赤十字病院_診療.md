### 移管方針
- シナリオ構造: 用件聴取（変更/キャンセル/確認）の3分岐。変更・キャンセルは共通フロー、確認のみ診療科聴取をスキップして内容確認に進む。共通情報聴取（診察券→患者名→生年月日→電話番号→最後の問い合わせ）は共通サブフロー化推奨。
- 分岐判定は **Pattern A**（用件聴取直後の OpenAI 分類 → ContextMatchRouter で3分岐）。予約日確認・診療科対象外判定はそれぞれ generate_by_OpenAI 直後の Yes/No または enum 分岐で Pattern A。
- Date of Call Classifier: 不要（予約日は「3日以上先か」のYes/No判定のみで、具体的な日付取得は行わない）。
- サブフロー化: 診察券番号 / 患者名 / 生年月日 / 連絡先電話番号 / RAG なし。標準サブフロー（Jump to Flow）を使用。内容確認はシンプルな自由テキスト保存のため scenario_flow 内 hearing ブロック。
- 終端: 「終話」(status=1, smsFlag=1) / 「代表案内」(status=2, smsFlag=0 維持) / 「聴取失敗案内」(AI補完経路) の3種を termination_patterns で定義。
- status/smsFlag のデフォルト '0' は opening ブロックの初期化 script で代入。

### 施設固有の特殊ルール
- **代表案内トリガー条件が2つ**:
  1. 予約日が「土日祝除き3日以上先」ではない（=いいえ）→ 代表案内（変更/キャンセル/確認すべてで適用）
  2. 診療科1が「慢性疼痛内科 / 放射線診断科 / 産科 / 救急科」→ 代表案内（変更/キャンセルのみ。確認では診療科ヒアリングしないため適用外）
- **診療科2のスキップ条件**: 診療科1が「わからない」の場合は診療科2を聞かず診察券番号へ直行。
- **確認フローの診療科非ヒアリング**: 用件=確認のときは診療科1/診療科2を一切聞かない。代わりに「内容確認」（自由テキスト、FAQ照合せずそのまま格納）。
- **復唱ルールの項目別分離**:
  - 患者名: 復唱禁止（「○○ですね」を言わない）。訂正時も「修正いたしました」のみ。
  - 生年月日: 通常は標準復唱。訂正発話時は即時上書きして「生年月日はYYYY年M月D日ですね。」と一度だけ確認。
  - 診察券番号: digits 分解フォーマット復唱（Gen3 では IVRプロパティで桁分割）。
  - 電話番号: telephone フォーマット復唱。10桁未満は必ず復唱確認。
- **名前カタカナ判定エラー経路**: 患者名がカタカナ化できない場合、専用エラーTTS「ゆっくり話すとAIが聞き取れません。普通のスピードではっきりと発話してください。」で再聴取ループ。
- **携帯電話番号の自動提示分岐**: 入電番号が 090/080/070/060 始まりなら「今おかけいただいている{Incoming_phone_number}でよろしいですか？」で確認、それ以外や非通知は直接聴取。incoming-classifier の phone_type で分岐判定。
- **相対日付換算ルール**: 10月以降に1月～9月の日付を言われた場合は翌年。ただし本シナリオでは「予約日」は日付確定せず Yes/No のみなので generate_by_OpenAI レベルでは影響小。「予約希望日」は入電者が自発的に発話した場合のみ記録（AIから問い返し禁止）。
- **END_聴取失敗案内の文言**: 元資料に「西部総合病院」「西部さくらクリニック」「048-854-1114」という他院の文言が記載されている（要件書の記載ミスの可能性）。**director で確認推奨**。石巻赤十字の代表案内番号は `0225217226`。

### director が見落としやすいポイント
- 元資料冒頭「0406 1415」は改訂タイムスタンプ。最新版の厳守ルール（アドリブ禁止、予約希望日をAIから質問禁止、連絡先番号確認完了後に他質問を挟まず最後の問い合わせへ直行）が後半セクション（5-1. 予約確認の「追加の厳守ルール」）に追記されている点に注意。
- function スキーマの enum `clinicalDepartment` に「一般外科」「登録なし」が含まれるが、プロンプト本文の診療科リストは「外科」。director は enum をマスターとして採用し、TTS 側は「外科」で整える。
- 診療科の enum は 36項目、キーワード辞書が存在（例: ふっと=糖尿病内科、あいこん=呼吸器内科、とうほくだいしんげ=心臓血管外科 等）。generate_by_OpenAI の分類プロンプトにキーワード辞書を埋める必要あり。原文の「各診療科のキーワード」セクション全件を director が customer_doc から直接読み込むこと。
- 用件分類で「診察」キーワード（印刷/定期/新薬/再診/最新/禁忌）と「予約」キーワード（薬/お薬/通訳）の特殊マッピングあり（5章冒頭）。用件分類プロンプトに含める。
- status=2（代表案内）時 smsFlag は `'0' 維持`（`'1'` にしない）。status=1 のときのみ smsFlag=1。
- endpoint enum に「時間外」「冒頭切断」が含まれるが、本フロー内で明示的な営業時間判定や冒頭切断ハンドリングの記載はない → director で Brekeke acceptance_times 側のデフォルト処理に委ねる方針を明記。
- 「最後の問い合わせ」は必ず聴取（連絡先電話番号確認直後にジャンプ強制）、結果は question フィールドに格納。折り返し時間要望は「ご要望を承りました。」で一律回答（元資料 1-10）。
