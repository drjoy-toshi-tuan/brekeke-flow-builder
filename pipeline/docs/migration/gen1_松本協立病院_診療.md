### 移管方針
- Gen1「箱」構造 → Gen3 scenario_flow 9 ブロック型に変換。主な対応:
  - `挨拶` → opening + 冒頭_アナウンス（2ブロック1セット必須）
  - `時間判断`（祝日チェック / 日付分岐 / 曜日分岐 / 時間分岐）→ date_of_call_classifier ブロック（addCurrentDate: Yes 自動付与）。年末年始（12/30〜1/4）を含む 4 系統分岐は Pattern B（後段 ContextMatchRouter）が妥当
  - `用件聞き取り`（サブ yaml） → hearing ブロック (generate_by_OpenAI)。**8 分岐**（再診/初診/予約変更/予約キャンセル/予約確認/検査変更/検査キャンセル/もう一度・もしもし）あり、echo_back（用件確認）必須
  - 用件分岐は分岐数が多く、各分岐後に `携帯電話分岐` が挟まる二段構造のため **Pattern B（後段 ContextMatchRouter）** で組む
  - 氏名 / 生年月日 / 電話番号 / 診察券番号 / 紹介状有無 / 当日予約確認 / 医師確認 / 受診希望確認 / 小児科対象確認 / 小児科リハビリ確認 / 検査名 / 希望日 / 予約日 / 後日予約希望 / 再予約希望 / 一般内科案内→不明点確認 / 問い合わせ内容 / 医師名聴取 → それぞれ subflow ブロックで Jump to Flow（feedback_subflow_connection_rule）
  - 患者氏名は PatientName サブフロー 1 枠に充当（feedback_patientname_subflow_allocation）
  - 診療科聞き取りは 2 系統（通常版 / 当日・発熱外来あり版）あり、director がサブフロー 2 本として用意

### 施設固有の特殊ルール
- **受付時間**: 祝日除く月〜金 9:00〜16:30。土日祝・年末年始（12/30〜1/4）は受付不可。time_of_call の文言・分岐は `時間判断.yaml` と `時間外アナウンス` 箱に従う
- **代表電話**: 0263-35-5288（予約センター）。`END_2回失敗→予約センター案内` で 3 回繰り返し読み上げ
- **転送先電話番号 7 種**（API_DRJOY_1〜7 と転送失敗_1〜6 の組合せ。番号は元資料転送箱を参照、director がそのまま転記）:
  - 予約センター 0263-35-5288 / 小児科 0263-35-9852 / 歯科 0263-35-8241 / 透析 0263-35-9726 / 4階リハビリ（成人）0263-35-5316 / 2階リハビリ（小児）0263-35-9728 / 健診 0263-35-0479
- **携帯電話分岐**: 用件確定後に毎回「携帯か固定か」を判定し、案内文言（折り返し電話 / SMS あり vs 折り返し電話のみ）を切替。固定/携帯で 5 分岐 × 2 = 10 通りの最終アナウンスがある
- **発熱外来**: 当日予約確認で当日と判明し、診療科が発熱外来の場合のみ「予約取らず直接来院」案内→終話。受付時間も別（月〜金 8:30〜16:30、土 8:30〜11:30）
- **精神科初診不可**: 新規予約で精神科を選択された場合は「新規受付不可」アナウンスして終話
- **紹介状必須の診療科ルート**: 紹介状有無サブフロー → 「下肢静脈瘤 / 心臓血管外科 / 紹介状必須診療科 / その他」の 4 分岐（紹介状必須：分岐 箱）
- **当日予約**: 再診・新規予約で当日希望と判明したら、診療科聞き取り（当日・発熱外来あり版）→ 小児科対象確認 → 予約センター転送（API_DRJOY_1）。AI では当日予約完結させずに転送する設計
- **3 回失敗時**: `END_2回失敗→予約センター案内` で予約センター番号案内して終話。termination_patterns に含める
- **DTMF 受付**: 用件聞き取り内で 1=予約取得 / 2=変更 / 3=キャンセル / 4=確認 を受付（director 判断: SKILL_A プロンプト側で吸収するか CMR で明示するか）

### director が見落としやすいポイント
- `最初の箱: API_DRJOY_CONTEXT_SETTINGS` と `API_DRJOY` 箱（× 8 個）は Gen3 では opening 冒頭処理に吸収 / 転送前のコンテキスト書き込みに統合 → 独立ブロック化不要
- 「セリフ: none」かつ振り分けのみの「箱」（`次へ`/`次へ_1〜_4`/`それでは`/`完了フラグ`/`時間内へ`/`時間外へ`/`現在の予約日聞き取りへ`/`新規/再診分岐へ` 等）は Gen3 では独立ブロック不要 → 前段 next を直接書き換えるか CMR の一部として吸収
- 短い繋ぎ TTS（「では次に、」「続いて、」「最後に、」「それでは、」）は次段 hearing/subflow ブロックの導入文言にマージするのが自然（director 判断、ただし全削除は不自然）
- `判定: エンティティ含む` の意味は「カスタム/システムエンティティに合致する surface を入力に含む」。Gen3 では generate_by_OpenAI の正規化結果で吸収するため **エンティティ値の全件転記不要**（entity_classification.md 参照、すべて R1〜R4 で分類済）
- `判定: lessEqual` / `数値以上` / `数値` は Gen1 の比較演算。Gen3 の date_of_call_classifier / context_match_router 条件式に置換
- `代表案内（電話番号NG）` `予約センター転送：聞き取り失敗` `予約センター転送へ（聞き取り失敗）_1〜_2` `2階リハビリ室へ転送（聴取失敗）` の各「聞き取り失敗→転送」箱は **転送先別に集約**して termination/call_transfer ブロックに整理
- 用件聞き取り.yaml の `count上記以外_1〜_4` `count再質問_1〜_4` は **3 回リトライ + 失敗時転送**の標準パターン → Gen3 では各 hearing ブロックの retry/error_branch で表現
- `API_DRJOY_CONTEXT_SETTINGS` 等のサブシナリオ呼び出しのうち実体（参照ファイル）が抽出ディレクトリに無いものは **Gen3 標準の API モジュール / Script モジュール** で代替（`祝日チェック` `転送(0263-35-XXXX へ)` 含む）。docs/brekeke/モジュール選定ガイド_v2.md §3.8 参照
- TTS は `{tts_g:...}` 小文字のみ、SSML 禁止（feedback_tts_ai_no_ssml）
- group_name URL長制限（19字目安）に注意。「松本協立病院_診療」は OK
