### 移管方針
- **冒頭は施設依存の Pattern A 分岐**: 着信元が `011-611-7766` (代表電話) の場合のみ短縮アナウンス「初めに、」、それ以外は通常アナウンス。Date of Call Classifier ではなく **incoming-classifier 的な phone_type 判定**として opening 直後に振り分ける。yaml の `内部_冒頭分岐` 参照
- **聴取順序が Gen2 標準と異なる**: 受診者名 → 生年月日 → 連絡先電話番号 → 用件 の順。**用件 (classification) 聴取は連絡先確定後**。これは customer_doc に「連絡先電話番号未完了時の全処理ブロック」として最優先ルールで明示されている
- **連絡先聴取の Pattern A 分岐 (3パターン)**: phone_type で `携帯 (060/070/080/090開始)` → incoming番号確認TTS → 肯定で確定/否定で直接聴取、`代表電話/固定/非通知` → 直接聴取TTS。携帯のみ「今おかけいただいている番号でよろしいですか？」分岐あり。**050 は yaml では携帯扱いに含まれていない**ので注意 (customer_doc 1-10 では 050 も含むが yaml 上は 060-090 のみ)
- **用件4分岐 (Pattern A)**: 予約 / 変更 / キャンセル / 確認(問い合わせ)。各分岐後に **classification 値を更新値で確定**。Pattern B (後段 ContextMatchRouter) ではなく OpenAI 直後分岐
- **subflow 化**: 受診者名 / 生年月日 / 電話番号 はそれぞれ専用サブフロー (PatientName / DateOfBirth / PhoneNumber)。RAG/FAQ サブフローは customer_doc 1-2 に言及あるが yaml に FAQ ブロック実体無し → **RAG サブフローは載せない**、reason 自由テキスト蓄積のみ
- **status 値マッピング**: `1` = 全項目完了 (END_終話)、`2` = 連絡先3回失敗 (END_聴取失敗3回)、`0` = それ以外。enum は文字列 "0"/"1"/"2"
- **smsFlag**: `status==1 → 1`、それ以外 → 0。終話直前の script ブロックで決定論的に算出

### 施設固有の特殊ルール
- **代表電話 (011-611-7766) からの入電**: 冒頭アナウンスを「初めに、」のみに短縮 (院内転送/折り返し想定)
- **電話番号3回失敗時のアナウンス**: 「ガイダンスの『1番の1』を選択してください」「月曜日から土曜日の8時半から17時」を含む長文。施設代表番号は `011-611-7766` (リテラル埋込)
- **救済語マッピング (course/classification)**: customer_doc 【3. 予約関連】に列挙。「検針/返信/天候診断/返金控寸断/検診/妊娠の予約」→「健診」、「人間ロック/ドック/ロック/人間独」→「人間ドック」、「薬/お薬」→「予約」、「取り直したい/別の/変えたい/変える/変えて」→「変更」、「やめる/取り消し/取消/やめたい」→「キャンセル」。**ただし「乳がん検診/子宮がん検診/子宮頸がん検診」は健診から除外**
- **コース選択肢は3つ固定**: 「人間ドック」「生活習慣病健診」「定期健診」。各間に `<break time="5000ms"/>` 指定あるが Gen3 では SSML 禁止、TTS では break タグを除去して読み上げ間を `tts_g` 表現で代替
- **受診歴確認は予約のみ**: 変更/キャンセル/確認では聴取しない。Medical_history は Yes/No → 「あり」/「なし」変換
- **企業名は予約のみ**: 「特にない」のような回答も受け付ける自由テキスト
- **予約日 vs 予約希望日の区別**: customer_doc に「予約日と予約希望日は別なので、予約日のヒアリングがなかった場合に予約希望日の中身を予約日に入れてはいけない」と明示。reservationDate (現予約) と Preferred_date (希望日) を混同しない
- **予約希望時期の制約 TTS**: 予約時のみ「本日から3週間以降の予約を受け付けております」を含める
- **変更項目分岐**: 「オプション変更」「日程変更」「それ以外の変更」の3択。日程変更のみ Preferred_date へ、他2つは reason に内容を聴取
- **生年月日復唱は1回のみ・収集時のみ**: 訂正時はもう1回だけ復唱、それ以降ルート変更/要約/終話で再復唱禁止
- **連絡先電話番号復唱は必須**: 入力後「電話番号は、〜 でよろしいでしょうか。」を必ず実施。否定なら再聴取へ戻る
- **氏名は全カタカナ変換**: 入力が「やまだ たろう」でも「ヤマダ タロウ」へ。読み判定不能 (例: 小山=コヤマ/オヤマ) のときは1回だけ再案内、それでも不一致なら聞き取れた表記で保存
- **複数氏名対応**: 最初の1名のみ patientName、2人目以降は reason に「カタカナ氏名 yyyy-MM-dd」1人1行で格納
