## Gen2→Gen3 移管ノート

### フロー概要

- n8n なし。標準的な Gen2 診療フロー。
- 受付制限: 予約変更は前日 16:00 まで。冒頭アナウンスで案内するが、Gen2 では時間チェックなし。
  - **移管方針**: director に判断を委ねる。時間チェックが必要な場合は Date of Call Classifier を追加。
- 用件分類: 新規 / 再診 / 変更 / キャンセル / 確認 / 問い合わせ

### 冒頭アナウンス

「お電話ありがとうございます。斗南病院の予約関連専用のAI電話です。ご予約について、緊急の診療が必要な場合は代表電話にお掛け直しくださいませ。また、予約変更の受付は、前日の16:00までとなっております。16:00以降の場合は代表電話にお掛け直しください。」

### フロー構造

```
冒頭アナウンス → 用件確認
  ├── 新規 → 紹介状確認 → 初診確認 → 診療科 → 診療科2 → 症状 → 予約希望日 → 氏名 → 生年月日 → 電話番号 → 終話（予約）
  ├── 再診 → 診療科 → 診療科2 → 予約希望日 → 氏名 → 生年月日 → 電話番号 → 終話（予約）
  ├── 変更 → 診療科 → 現予約日 → 新予約希望日 → 氏名 → 生年月日 → 電話番号 → 終話（日程変更）
  ├── キャンセル → 診療科 → 現予約日 → 氏名 → 生年月日 → 電話番号 → 終話（キャンセル）
  ├── 確認 → 診療科 → 現予約日 → 氏名 → 生年月日 → 電話番号 → 終話（受診結果確認）
  └── 問い合わせ → RAG / 代表電話案内
```

### function スキーマ（主要フィールド）

- classification: "新規"|"再診"|"変更"|"キャンセル"|"確認"|"問い合わせ"|""
- patientName: 患者名（カタカナ）
- medicalCardNumber: 診察券番号
- clinicalDepartment, clinicalDepartment2: 診療科
- reservationDate: 現在の予約日（変更/キャンセル/確認用）
- DesiredreservationDate: 希望予約日（新規/再診/変更用）
- reason: 受診理由/症状
- history: 初診確認
- institution: 紹介元医療機関名
- institutionPhoneNumber: 紹介元電話番号
- introduction: 紹介状の有無（"有"|"無"|""）
- doctor: 医師指定
- symptoms: 病名症状
- remainingDrugs: 残薬確認
- sex: 性別（"男"|"女"|""）
- additionalPhoneNumber: 連絡先電話番号
- otherhospital: 他院確認
- details: 確認内容
- question: 最後の問い合わせ
- checkout: チェックアウト
- endpoint: 通話完了|冒頭切断|途中切断|時間外|電話転送
- status: "0"|"1"|"2"|"3"
- smsFlag: "0"|"1"|"2"|"3"

### 特記事項

1. **1回聴取禁止 / 再ヒアリング禁止**: Gen2 では一度聴取した内容は再確認しない方針。Gen3 の generate_by_OpenAI モジュールでこのルールをプロンプトに含める。
2. **訂正対応**: 同項目が2回訂正された場合は「折り返し時に確認」としてスキップ。
3. **紹介状なし**: 新規の場合、紹介状なしなら終話（受付不可）アナウンス。
4. **終話アナウンス**: 「かしこまりました。このお電話でご予約の確定ではございません。3営業日以内に折り返しの電話またはショートメールでの確認をさせて頂き確定となります...」
