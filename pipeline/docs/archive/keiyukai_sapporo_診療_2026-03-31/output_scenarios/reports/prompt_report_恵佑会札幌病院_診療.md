## 注意事項

1. **TTS文言はIVRプロパティで上書きされる**: Context セクションのTTS文言は設計書セクション5から取得。実運用時はIVRプロパティの値が適用されるため、プロパティ側の文言と大きく異なる場合はContextの更新が必要
2. **紹介状の第3分岐（病院名等）**: 設計書では「病院名・クリニック名等はそのまま格納」とあるが、フローJSONのnext配列は有り/無しの2分岐+NO_RESULTのみ。病院名等の回答はNO_RESULTとなりリトライに入る。設計書とフロー構造に不整合がある可能性あり（generatorの設計判断を尊重しプロンプトはnext配列に合わせた）
3. **予約希望日はフリーワード格納**: Gen2プロンプト仕様「予約希望日はフリーワードで言われたまま文字起こしする。日付形式で格納しない」に従い、SKILL_C（日付変換型）ではなく SKILL_E（自由テキスト型）を採用。contextDisplayTypeもTEXT
4. **受付不可診療科のキーワード**: Gen2プロンプトのセクション1-0から全18診療科のキーワードを転記。「歯科」「歯科口腔外科」は受付対象のため除外条件を明記
5. **申し込み方法のSTEP3「に」→電話**: 「に」は非常に短い語句だが、Gen2プロンプトで明示的に指定されている（「に」= 2 = 電話）ため採用。部分一致で誤判定リスクがあるが、STEP2の数字モード判定が最優先のためDTMF入力時は問題なし
