## 2. フロー全体図

### 冒頭チェーン（必須構成）

```
冒頭(wait 2000ms)
  → コンテキスト設定(saveContextModel2DB)
    → 着信分類(incoming-classifier)
      ├── 非通知 → 非通知_アナウンス(TTS) → 完了フラグ_非通知(saveCompletionFlag2db, status=2, smsFlag=-1) → 切断_非通知(Disconnect)
      ├── 海外   → 非通知_アナウンス(TTS)に合流（非通知と同じ終話パス）
      └── 固定/携帯/その他 → 受付時間判定(acceptance_times)
            ├── 時間外/ERROR → 時間外_アナウンス(TTS) → 完了フラグ_時間外(saveCompletionFlag2db, status=6, smsFlag=-1) → 切断_時間外(Disconnect)
            ├── TIMEOUT → 冒頭_アナウンス(TTS)（時間内扱い）
            └── 受付可 → 冒頭_アナウンス(TTS)
```

## 8. 特記事項・制約

- **歯科口腔外科のみ対応**: 本施設は歯科口腔外科の予約専用であり、それ以外の17診療科の予約希望は全て「受付不可」として代表案内に遷移する
- **申し込み方法の2段階分岐**: 冒頭で「ネット申し込み（SMS送信）」と「電話申し込み」を分岐する施設固有構造。ネット申し込みルートはSMS送信後にstatus=8で終話する
- **status=8（SMS案内）**: 標準のstatus一覧（0,1,2,3,6）に含まれない独自ステータス。Dr.JOY側の状態編集画面で追加設定が必要
- **smsFlag=0 と smsFlag=-1 の使い分け**: Gen2プロンプトでは受付不可診療科のsmsFlag=0。Gen3標準では-1=送信なしだが、Gen2の挙動を維持。Dr.JOY側でsmsFlag=0の動作を確認推奨
- **smsFlag=2（キャンセル専用）**: キャンセル用件のみsmsFlag=2。予約・変更・その他はsmsFlag=1。終話分岐後にsaveCompletionFlag2dbで上書き
- **smsFlag=3（SMS送信案内）**: ネット申し込みルートでSMS送信後のフラグ
- **初診/再診の聴取順序の違い**: 初診ルートは診察券番号を聴取しない（紹介状→医師名→氏名→生年月日→電話番号）。再診ルートは診察券番号を聴取する（予約希望日→診察券→氏名→生年月日→電話番号）
- **予約希望日はフリーワード格納**: 日付形式に変換しない。Gen2プロンプトで「フリーワードで言われたまま文字起こしする」と指定
- **予約日は日付形式で格納**: `YYYY-MM-DD 00:00` 形式
- **Gen2の誤認識ワード対応**: Gen2プロンプトに多数の誤認識ワード→正解ワード変換が定義されている。Gen3ではOpenAIプロンプト内でこれらの変換ルールを実装する
- **FAQサブフローは未配置**: Gen2プロンプトにFAQ対応の記述があるが、現時点ではRAGサブフローを配置しない。必要に応じて追加
