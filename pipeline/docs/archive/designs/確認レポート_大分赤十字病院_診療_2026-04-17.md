### 抽出した要件サマリー

**フロー目的**: 外来予約（変更/キャンセル/確認）の受付を自動化。新規予約は代表電話案内。

**用件分類（classification）**:
- `1` または「変更」→ 予約変更
- `2` または「キャンセル」→ 予約キャンセル
- `3` または「確認」→ 予約確認
- 「新規」→ 代表電話案内後に終話（classification には保存しない想定だが、Gen3 ではルーティング用に enum 値として扱う）

**分岐構造**:
- 変更/キャンセル: 診療科 → 予約日（当日ならEND_当日案内）→ 希望日or理由 → 診察券番号 → 氏名 → 生年月日 → 連絡先電話番号 → 用件別終話
- 確認: 内容確認 → FAQ（RAGサブフロー）→ 診療科 → 診察券番号 → 氏名 → 生年月日 → 連絡先電話番号 → 用件別終話

**診療科（全23科 + 登録なし）**:
内分泌糖尿病内科/外科/整形外科/脳神経外科/皮膚科/腎泌尿器外科/産婦人科/眼科/耳鼻咽喉科/放射線科/歯科口腔外科/リウマチ科/循環器内科/呼吸器内科/消化器内科/精神神経科/腎臓内科/脳神経内科/肝臓胆のう膵臓内科/呼吸器外科/乳腺外科/心臓血管外科/緩和ケア科

**診療科の曖昧入力**: Gen2 では「げか」「こきゅうき」「のう」「ないか」「泌尿器」「人」の曖昧発話を個別ボックスで絞り込んでいたが、Gen3 では OpenAI プロンプト内の正規化ルール + 類義語辞書（profile_words）で1回の聴取に統合。該当できないものは `no_result_default: "登録なし"` で保存。

**個人情報聴取**: CLAUDE.md Rule 9 に準拠して 4サブフロー（氏名/生年月日/電話番号/診察券番号）を完全コピー。
- 生年月日・電話番号は復唱あり
- 氏名・診察券番号は復唱なし

**RAGサブフロー（パターン3）**: Gen2→Gen3 移管のデフォルトに従い、以下2箇所に配置
1. `内容確認` の後（Gen2 の「確認_FAQ判定 → 確認_FAQ回答」を置換）
2. 終話分岐の直前（「最後にご質問は？」の汎用RAG）

**終話パターン**:
- 正常終話（変更/キャンセル/確認 × 3種）: status=1, smsFlag=1
- 代表案内系（新規/当日）: status=2, smsFlag=-1
- 異常系（非通知/時間外/聴取失敗）: テンプレートデフォルト

## 6. 特記事項

- **診療科の曖昧入力処理**: Gen2 では6種類の曖昧発話（げか/こきゅうき/のう/ないか/泌尿器/人）に対して個別確認ステップがあったが、Gen3 では OpenAI プロンプト内の正規化ルール + profile_words（全74語の類義語辞書）で1回の聴取に統合。挙動差異が許容できない場合は、各曖昧発話に対応する hearing ブロックの追加を検討
- **FAQ判定**: Gen2 の `確認_FAQ判定` → `確認_FAQ回答` は Gen3 の RAG サブフロー（パターン3の前半）に置換
- **Gen2 の `DesiredreservationDate`（PascalCase）**: Gen3 では命名規則に合わせ `desiredReservationDate`（camelCase）に統一
- **Gen2 の `checkout`, `endpoint`**: Gen3 では context_fields に含めない（Dr.JOY 側で通話ログから判別）
- **Gen2 の `institution`, `introduction`, `disease`, `symptoms`, `question`, `clinicalDepartment2`**: Gen2 本文では未使用のため Gen3 では削除。必要なら後で追加可能
