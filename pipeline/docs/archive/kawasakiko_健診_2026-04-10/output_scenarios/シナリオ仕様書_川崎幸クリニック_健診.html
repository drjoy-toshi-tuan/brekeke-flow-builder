🔍 シナリオの目的

川崎幸クリニック健診予約専用ダイヤルのAI電話受付。 受診者名・生年月日・連絡先電話番号・診察券番号を聴取した後、 用件4択（予約/変更/キャンセル/問い合わせ）に分岐。 予約ルートは更に健診種類3択（川崎市/健保組合/その他）に分岐し種類別に聴取。 全ルート終話前にFAQ対応（RAGサブフロー）を実施。 終話TTSは用件 x 電話種別（携帯/固定）で分岐し、川崎市予約のみ特定健診受診券案内を追加。

─── 終話ルーティング実装方針 ───
1. ContextMatchRouter(classification)
   ├─ キャンセル → 直接 END_キャンセル チェーン
   ├─ 予約 → ContextMatchRouter(OpenAI_予約_健診種類)
   │    ├─ 川崎市 → ContextMatchRouter(phoneType) → 携帯/固定
   │    └─ 健保組合/その他/default → ContextMatchRouter(phoneType) → 携帯/固定
   ├─ 変更 → ContextMatchRouter(phoneType) → 携帯/固定
   └─ 問い合わせ/default → ContextMatchRouter(phoneType) → 携帯/固定
※ phoneTypeは電話番号聴取サブフロー内のsaveContext2DBで保存済み

─── リトライ失敗時 ───
用件リトライ超過 → 完了フラグ_聴取失敗(status=3,smsFlag=-1) → END_聴取失敗(TTS) → 切断
健診種類リトライ超過 → その他_希望検査へスキップ（Gen2デフォルト: その他にフォールバック）
その他聴取項目リトライ超過 → 次ステップへスキップ（Gen2準拠: 不明確でも次へ）
💡 冒頭チェーン: wait → saveContextModel2DB → incoming-classifier → acceptance_times → 冒頭_アナウンス 個人情報は用件確認の前に聴取（Gen2順序踏襲）。全サブフロー return 型。 RAGはパターン2: 全ルート終話前に「そのほかに何かお問い合わせはございますか？」 共通_人数受診時期は川崎市/健保組合の2ルートで共用（同一TTS・同一保存先・同一Retry先）。 変更_予約日とキャンセル_予約日は別モジュール（Retry先が異なるため）。
