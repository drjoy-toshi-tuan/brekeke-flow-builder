# Golden Templates (Legacy) — 廃止済み

## 廃止日
2026-04-15

## 廃止理由
ブロック構造アーキテクチャ（scaffold_generator v2）の導入により、設計書はゴールデンテンプレートを「見本として参照する」方式から、**入力資料からブロックを組み立てる**方式に移行しました。

- 旧フロー: 入力資料 → golden template のパターン選択 → テンプレートを参考に設計書を書く
- 新フロー: 入力資料 → `docs/brekeke/モジュール選定ガイド_v2.md` に従って 8 ブロック型に分解 → `scenario_flow` で記述

テンプレート自体もブロック構造化前の古いフォーマット（`scenario_flow` セクション無し、旧 `output_format` 値：`label` / `free_text` / `date_text` / `number` など）が残っており、現行パイプラインと不整合でした。一部テンプレートには「個人情報聴取 wrapper」パターンなど実態と異なる記述もあり、director が模倣すると静かなバグの原因となっていました。

## 現行の情報源

- **`docs/specs/設計書テンプレート.yaml`**: YAML フォーマット定義
- **`docs/brekeke/モジュール選定ガイド_v2.md`**: 8 ブロック型と output_format の選定ルール
- **`.claude/agents/director.md`**: director の挙動定義
- **`CLAUDE.md`**: プロジェクト共通ルール

## 残っているファイル（参考のみ）

| ファイル | 元の用途 |
|---|---|
| 本人確認型.yaml | 受付（個人情報聴取メイン） |
| 予約管理型.yaml | 予約・変更・キャンセル |
| 健診受付型.yaml | 健診・人間ドック |
| 時間外案内型.yaml | 時間外案内のみ |
| 着信番号判定型.yaml | 電話番号聴取（サブフロー） |

これらは**参照禁止**です。フォーマット・`output_format` 値・個人情報サブフロー構造が現行と乖離しており、模倣するとバグを再現します。
