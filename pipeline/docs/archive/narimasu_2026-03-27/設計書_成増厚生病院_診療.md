## 3. フローの目的

成増厚生病院の予約専用AI電話として、患者（または代理人）からの電話を受け、用件（初診予約・再診予約・変更・キャンセル・確認）・診療科・個人情報を聴取して折り返し受付を自動化する。

特殊対応として以下を含む:
- 当日予約 → 代表電話（03-3939-1191）案内・終話（checkpoint=2, smsFlag=0）
- アルコールの相談 → 専用番号（050-1725-6865）案内・終話（checkpoint=7, smsFlag=0）
- 児童思春期外来×初診 → ホームページ案内・終話（checkpoint=2, smsFlag=0）

## 4. シナリオフロー図

### 4.1 メインフロー（冒頭〜用件分岐）

```
冒頭(wait 2000ms)
  → saveContextModel2DB（コンテキスト設定）
  → incoming-classifier（着信電話番号分類）
      ├─ 非通知 → 非通知_アナウンス(TTS) → 完了フラグ_非通知(saveCompletionFlag2db) → 切断
      └─ 通常   → 受付時間判定(acceptance_times)
                    ├─ 時間外 → 時間外_アナウンス(TTS) → 完了フラグ_時間外(saveCompletionFlag2db) → 切断
                    └─ 受付可 → 冒頭_アナウンス(TTS)
                                  → 入力_用件_区分(STT)
                                      → OpenAI_用件_区分（当日判定＋用件分類）
                                          ├─ 当日予約    → END_代表案内(TTS) → 完了フラグ_代表案内 → 切断
                                          ├─ 初診予約    → 確認_受診歴(TTS) → 入力_確認_受診歴(STT) → OpenAI_確認_受診歴
                                          │                   ├─ 初診 → 予約_診療科（初診パス）
                                          │                   └─ 再診 → 予約_診療科（再診パス）
                                          ├─ 変更        → 予約_診療科（変更パス）
                                          ├─ キャンセル  → 予約_診療科（キャンセルパス）
                                          └─ 確認        → 予約_診療科（確認パス）
```

### 4.2 診療科聴取・特殊分岐

```
予約_診療科(TTS)
  → 入力_予約_診療科(STT)
      → OpenAI_予約_診療科
          ├─ アルコール     → END_アルコール案内(TTS) → 完了フラグ_アルコール → 切断
          ├─ 児童思春期     → OpenAI_児童思春期_初診判定

```

## 7. ステップ詳細

### 7.1 冒頭アナウンス

**TTSアナウンス**:
> お電話ありがとうございます。成増厚生病院の、予約専用、AI電話です。本日のご予約と歯科受診、発熱外来に関するお電話は代表電話におかけ直しください。

**入力方式**: なし（TTSのみ）

**次**: 入力_用件_区分

---

### 7.2 用件確認

**TTSアナウンス**:
> ご用件を、次の4つのうちの、いずれかでお話ください。「1、診察の予約を取る」「2、予約を変更する」「3、予約をキャンセルする」「4、予約を確認する」、それでは、お話ください。

**入力方式**: DTMF + 音声（DTMF最大1桁）

**OpenAI正規化ルール**:
- 出力値: `初診予約` / `変更` / `キャンセル` / `確認` / `当日予約` / `NO_RESULT`
- DTMF `1` または「予約を取る」「診察」等 → `初診予約`
- DTMF `2` または「変更」 → `変更`
- DTMF `3` または「キャンセル」 → `キャンセル`
- DTMF `4` または「確認」 → `確認`
- 「今日」「本日」「きょう」「ほんじつ」等の当日キーワードを含む → `当日予約`
- それ以外 → `NO_RESULT`

**保存先**: classification（ただしOpenAI判定後に個別パスで上書き）

**次**:
- `当日予約` → END_代表案内
- `初診予約` → 確認_受診歴
- `変更` → 予約_診療科（classificationに"変更"を保存）
- `キャンセル` → 予約_診療科（classificationに"キャンセル"を保存）
- `確認` → 予約_診療科（classificationに"確認"を保存）
- `NO_RESULT` → リトライ_用件_区分

**リトライ失敗時（No more）**: 終話_失敗（進行不可）

---

### 7.3 受診歴確認

**TTSアナウンス**:
> 診察のご予約ですね？当院でのご受診は初めてですか？

**入力方式**: 音声のみ

**OpenAI正規化ルール**:
- 出力値: `初診` / `再診` / `NO_RESULT`
- 「はい」「初めて」「初診」等の肯定 → `初診`
- 「いいえ」「以前」「再診」等の否定 → `再診`
- それ以外 → `NO_RESULT`

**保存先**: classification（"初診" または "再診"）

**次**:
- `初診` → 予約_診療科（初診パス）
- `再診` → 予約_診療科（再診パス）
- `NO_RESULT` → リトライ_確認_受診歴

**リトライ失敗時（No more）**: 終話_失敗

---

### 7.4 診療科聴取

**TTSアナウンス**:
> それでは、受診を希望される診療科を次の6つの中からお選びください。「1、アルコールの相談」「2、精神科」「3、内科」「4、児童・思春期外来」「5、老年期こころの外来」「6、整形外科」それでは、お話しください。

**入力方式**: DTMF + 音声（DTMF最大1桁）

**OpenAI正規化ルール**:
- 出力値: `アルコール相談` / `精神科` / `内科` / `児童思春期外来` / `老年期こころの外来` / `整形外科` / `NO_RESULT`
- DTMF `1` または「アルコール」「相談」 → `アルコール相談`
- DTMF `2` または「精神科」「せいしんか」「製品化」「耐震化」 → `精神科`
- DTMF `3` または「内科」 → `内科`
- DTMF `4` または「児童」「思春期」「じどう」「ししゅんき」 → `児童思春期外来`
- DTMF `5` または「老年期」「こころの外来」「ろうねんき」 → `老年期こころの外来`
- DTMF `6` または「整形外科」 → `整形外科`
- それ以外 → `NO_RESULT`

**保存先**: clinicalDepartment

**次**:
- `アルコール相談` → END_アルコール案内
- `児童思春期外来` かつ 用件=初診 → END_HP案内（OpenAI_児童思春期_初診判定で判定）
- `児童思春期外来` かつ 用件≠初診 → 用件別フローへ継続
- その他 → 用件別フローへ継続

**リトライ失敗時（No more）**: 終話_失敗

## 11. 特記事項・制約

1. **当日予約判定**: 用件聴取のOpenAI分岐で「今日」「本日」等のキーワードを検出した場合、用件番号に関わらず END_代表案内 に遷移する。冒頭アナウンスにも代表電話案内を含める。

2. **アルコール相談の終話分岐**: 診療科選択でアルコール相談が選ばれた場合（dtmf=1 または音声で「アルコール」等）、その時点で END_アルコール案内 に遷移し、個人情報聴取は一切行わない。checkpoint=7、smsFlag=0。

3. **児童思春期外来×初診の判定**: 診療科=児童思春期外来 かつ classification=初診 の場合に END_HP案内 へ遷移。judgment は OpenAI モジュール `OpenAI_児童思春期_初診判定` で classification の値を参照して判定する（saveContext2DB から取得）。checkpoint=2、smsFlag=0。

4. **本人/代理分岐**: 本人の場合は agentcheck への保存不要。代理の場合は agentcheck="代理" を saveContext2DB で保存後、入電者氏名サブフローへ遷移する。

5. **smsFlagの設計**: 正常終話では smsFlag=1（キャンセルのみ=2）、特殊終話（代表案内・HP案内・アルコール案内・進行不可）では smsFlag=0。saveCompletionFlag2db で checkpoint と smsFlag を同時設定する。

6. **診察券番号のスキップ**: 「わからない」と発話した場合は空欄のまま次の聴取項目へ進む。診察券番号サブフロー内で「わからない」を検出しスキップ処理を実装する。

7. **生年月日のスキップ**: 3回の試行（初回＋2回聴き直し）で特定できない場合は空欄のまま電話番号聴取へ進む。空欄でも正常終了扱い（checkpoint=1）。

8. **フロー名**: Brekekeのグループ名+フロー名の日本語部分合計10文字未満制限に注意。「な_成増厚生病院」（8文字）＋フロー名を調整すること。
