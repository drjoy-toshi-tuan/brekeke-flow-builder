# OpenAI プロンプトリファレンス

> 出典: Technical リンク集_Tips.xlsx「プロンプト」シート
>
> **スコープ注意**: プロンプトの実際の値はIVRプロパティで管理する。
> AIが生成するフローJSONにプロンプト本文を埋め込む必要はない。
> このファイルは人間がIVRプロパティを設定する際の参考資料。

---

## 基本ルール（全プロンプト共通）

```
# 絶対順守事項
出力結果をシステムで後続処理するため、前後に一切の情報を出力してはならない。

## 絶対禁止事項
- "\n"（改行コード）を出力してはいけない
- 謝辞・謝罪・説明・語尾を出力してはいけない
- 認識できない場合は NO_RESULT を出力する
```

- IVRプロパティにプロンプトを入れる場合は **1行のみ**
- プロンプト先頭に「今日は〇月〇日です」という日付が自動追加される

---

## 氏名（patientName）

**用途**: STTで取得した発話から患者氏名を抽出する

**出力形式**: カタカナフルネーム（例: `ヤマダタロウ`）または `NO_RESULT`

**ポイント**:
- 漢字・ひらがなで発話された場合もカタカナに変換する
- 「名前は〇〇です」等の語句は除去してフルネームのみ出力
- 認識不可の場合は `NO_RESULT` を出力（リトライへ遷移させる）

---

## 電話番号（phoneNumber）

**用途**: STTで取得した電話番号を正規化する

**出力形式**: 数字10桁または11桁のみ（ハイフンなし）または `NO_RESULT`

**ポイント**:
- ハイフン・スペース・「の」等を除去して数字のみ出力
- 10桁・11桁に正規化できない場合は `NO_RESULT`
- AmiVoice 出力にハイフンが含まれる場合があるため正規化が必須
- SMS送信には正しい番号が `additionalphonenumber` に入っている必要がある

---

## 用件（classification）

**用途**: 「予約・変更・キャンセル・その他」等を分類する

**出力形式**: 定義された選択肢のいずれか（例: `予約`, `変更`, `キャンセル`）または `NO_RESULT`

**ポイント**:
- 振り分けられない入力は必ず `NO_RESULT` にする（これがないと切断される）
- `OpenAI_用件_区分.prompt` として IVRプロパティに設定
- 各用件に対応する next 遷移先を `generate_by_OpenAI` モジュールで定義

---

## 受診歴（medicalHistory）

**用途**: 「初診/再診」を判定する

**出力形式**: `初診` または `再診` または `NO_RESULT`

**ポイント**:
- 「初めて」→ `初診`、「以前来たことがある」→ `再診` 等の表現を吸収する
- 不明確な発話は `NO_RESULT` で返す

---

## 生年月日（dateOfBirth）

**用途**: 発話内容から生年月日を YYYYMMDD 形式に変換する

**出力形式**: `YYYYMMDD`（例: `19800315`）または `NO_RESULT`

**ポイント**:
- 和暦（昭和・平成・令和）は西暦に変換する
- 「8年3月15日」等の年号省略は文脈から補完する
- `DOB Re-confirmation` モジュール（`drjoy^TS Custom Module$DOB Re-confirmation`）と組み合わせて使用

---

## function call スキーマ（共通）

OpenAI モジュールでの function call 形式:
```json
{
  "name": "dialogue_completed",
  "description": "聴取した内容を保存します",
  "parameters": {
    "type": "object",
    "properties": {
      "result": {
        "type": "string",
        "description": "抽出・分類した結果"
      }
    },
    "required": ["result"]
  }
}
```

## OpenAI 分岐モジュールのフロー設計パターン

```
[STT: 入力_用件_区分]
    ↓ ^.+$
[OpenAI_用件_区分]  ← プロンプトはIVRプロパティで定義
    ├─ ^予約$    → [患者_氏名]
    ├─ ^変更$    → [変更_項目]
    ├─ ^キャンセル$ → [キャンセル_確認]
    ├─ ^TIMEOUT$  → [リトライ_用件_区分]
    ├─ ^ERROR$   → [リトライ_用件_区分]
    └─ ^NO_RESULT$ → [リトライ_用件_区分]
```

- STT は `^.+$` の1本受けのみ
- **分岐条件（`^予約$` 等）は OpenAI モジュールの next に定義する**
- STT の next に個別パターンを書かない
