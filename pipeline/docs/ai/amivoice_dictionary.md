# AmiVoice 辞書登録仕様

## 概要

STTモジュールの音声認識精度を高めるために、`profile_words` パラメータに辞書単語を直接埋め込む。

---

## profile_words フォーマット

```
表記 よみがな
表記 よみがな
```

- 1行1エントリ。**半角スペース**で表記とよみがなを区切る
- 同一単語を複数のよみがなで登録可（AmiVoiceが候補として使用）
- 空行はグループ区切りとして使用可
- 改行コードは `\n`（JSONエスケープ済みの文字列として params に埋め込む）

### 記述例

```
昭和 しょうわ
昭和 しょーわ
平成 へいせい
平成 へいせー
令和 れいわ
東病棟 ひがしびょうとう
東病棟 ひがし
ICU病棟 アイシーユー
ICU病棟 あいしーゆー
```

## 辞書ファイル一覧

辞書ソースは `docs/reference/0_Amivoice辞書-*.zip` に格納されたDOCXファイル。

| 辞書ファイル | 内容 | 対象STTモジュール |
|---|---|---|
| `Brekeke：氏名.docx` | 全国頻出苗字 | `入力_患者_氏名` |
| `Brekeke：和暦辞書.docx` | 明治/大正/昭和/平成/令和 | `入力_患者_生年月日`、`入力_*_日付` |
| `Brekeke：month辞書.docx` | 1月〜12月の読み方 | `入力_患者_生年月日`、`入力_*_日付` |
| `Brekeke：day辞書.docx` | 1日〜31日の読み方 | `入力_患者_生年月日`、`入力_*_日付` |
| `Brekeke：hour辞書.docx` | 0時〜23時の読み方 | `入力_*_時間` |
| `Brekeke_曜日辞書.docx` | 月〜日曜日の読み方 | `入力_*_曜日` |
| `Brekeke_肯定否定（基本）辞書.docx` | はいそうです、大丈夫etc. | `入力_復唱_*` |
| `Brekeke：診療シナリオ_地域連携連携シナリオ.docx` | 診療科・外来名 | `入力_*_診療科`、`入力_*_外来` |
| `Brekeke：健診辞書_健診シナリオ.docx` | 健診・検診項目 | `入力_*_健診`、`入力_*_検診` |
| `Brekeke：処方薬剤辞書_薬剤部シナリオ.docx` | 薬剤・処方関連 | `入力_用件_薬`、`入力_*_薬剤` |
| `Brekeke：全統合版_重複排除済.docx` | 全カテゴリ統合 | 汎用STT（用途が特定できない場合） |
| `用件_大河原さん.docx` | 用件区分（施設固有） | `入力_用件_区分`（施設ごとに要確認） |

> イントネーション設定CSV（`イントネーション設定_20250827.csv`）はAmiVoiceサーバーへの辞書登録用。`profile_words` とは別の仕組み。

## STTモジュール種別ごとの辞書設定ルール

### 1. 患者氏名入力

```
profile_words: "サトウ さとう\nスズキ すずき\nタカハシ たかはし\n..."
```
→ `Brekeke：氏名.docx` の内容を使用

### 2. 生年月日入力（和暦）

```
profile_words: "昭和 しょうわ\n昭和 しょーわ\n平成 へいせい\n令和 れいわ\n1月 いちがつ\n..."
```
→ `Brekeke：和暦辞書.docx` + `Brekeke：month辞書.docx` + `Brekeke：day辞書.docx` を結合

### 3. 診療科入力

```
profile_words: "アレルギー科 あれるぎーか\nアレルギー膠原病内科 あれるぎーこうげんびょうないか\n胃カメラ いかめら\n..."
```
→ `Brekeke：診療シナリオ_地域連携連携シナリオ.docx` の内容を使用（施設ごとの診療科リストに合わせて調整）

### 4. 復唱確認入力

```
profile_words: "はいそうです はいそうです\nはいそうです はいそーです\nはい はい\n大丈夫です だいじょうぶです\n..."
```
→ `Brekeke_肯定否定（基本）辞書.docx` の内容を使用

### 5. 用件区分入力

```
profile_words: "面会 めんかい\n面会 めんがい\n呼び出し よびだし\n予約 よやく\n..."
```
→ 施設固有の用件定義に合わせて設定。`用件_大河原さん.docx` を参考に施設ごとにカスタマイズ

### 6. 病棟・フロア入力

施設固有の病棟名・フロア名を設定。テンプレートなし。設計書の病棟一覧を元に生成。

```
profile_words: "東病棟 ひがし\n東病棟 ひがしびょうとう\nICU ICU\nICU あいしーゆー\n..."
```

### 7. 時間入力

```
profile_words: "..."
```
→ `Brekeke：hour辞書.docx` の内容を使用

### 8. 曜日入力

```
profile_words: "月曜 げつよう\n月曜日 げつようび\n火曜 かよう\n..."
```
→ `Brekeke_曜日辞書.docx` の内容を使用

## generator での運用ルール

### 辞書設定が必要なSTTモジュール

以下の入力種別に対応するSTTモジュールは `profile_words` を設定すること。

| 入力種別キーワード | 使用辞書 |
|---|---|
| `氏名` | 氏名辞書 |
| `生年月日`、`日付` | 和暦辞書 + month辞書 + day辞書 |
| `時間` | hour辞書 |
| `曜日` | 曜日辞書 |
| `診療科`、`外来` | 診療科辞書 |
| `健診`、`検診` | 健診辞書 |
| `薬`、`薬剤` | 処方薬剤辞書 |
| `復唱` | 肯定否定辞書 |
| `用件` | 施設固有（設計書の用件リストを参照） |
| `病棟`、`フロア`、`階` | 施設固有（設計書の病棟リストを参照） |

### 辞書内容の取得方法

1. **サンプルbivrから参照可能なもの**: `output/profile_words_inspect.json` に49モジュール分のサンプルあり
2. **DOCXファイル参照が必要なもの**: DOCXはバイナリのため直接読めない。`profile_words_inspect.json` のサンプルを流用する
3. **施設固有辞書**: 設計書の該当項目（診療科リスト・病棟名等）から生成する

### profile_words が空白でよいケース

- 用途が特定できない汎用STTモジュール（フリーテキスト入力等）
- 設計書に語彙リストの記載がないケース

---

## IVRプロパティ経由での上書き

`profile_words` はIVRプロパティ（`モジュール名.profile_words=...`）でも上書き可能。
施設固有の語彙が多い場合はIVRプロパティで管理する方がメンテナンスしやすい。
