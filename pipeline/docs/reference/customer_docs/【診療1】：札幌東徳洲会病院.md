**1-4. 着信日時・電話番号の取得**  
	通話開始時に着信日時と着信元電話番号を取得する。  
	着信元電話番号の先頭の0は省略してはいけない  
	「今日」「明日」「明後日」のような相対的日付表現は着信日時から自動換算する。  
	予約で過去日付と思われる日付が指定された場合は、今日の日付より未来の日付かどうかを確認し、未来の日付のみ受け付ける。

\- フロー内の質問をされた場合は、聴取した内容から該当する回答をする。

\- 折り返しの電話してほしい時間を伝えられた場合は「ご要望を承りました。」と回答する。

**1-11.生年月日**  
発話内容を”YYYYMMDD”として判断する（和暦の場合は西暦へ変換して判別する）  
生年月日の認識が計2回できなかった場合は’patientDateOfBirth’ をブランクでセットし、エラー内容はアナウンスせず、次の質問へ進む。  
以下の条件で発話内容を判別し、必要に応じ補正する。  
\# 生年月日の情報が欠けている場合は、下記に従い判断し条件に応じて再ヒアリングする。  
　再ヒアリング時にエラーの理由は説明せず「恐れ入りますが再度、」とアナウンスしヒアリングする。  
（補足：再ヒアリングでは、選択肢を明示して短く聞くこと。例：「‘03年’は令和ですか？平成ですか？」）  
　\#\# 年（Year）の条件：  
　　- まず数値と表記を正規化する（全角→半角、「年」を付与、誤変換語を補正）  
　　- 元号語が誤変換・誤記のときは、文脈と「○○年」から適切な元号に置換する  
　　　- 例：「町は46年」→「昭和46年」  
　　- 西暦／元号の取り違えがあり得るため、「○○年」の数値部分を基軸に解釈する。  
　　- 未来の年や Year(NOW()) − 110 より古い年は無効。該当時は再ヒアリングする。  
　　　- 例：Year(NOW())=2025 のとき、1914年以下は再ヒアリング  
　　\#\#\# 年の3桁目（百の位）が’0’の場合：  
　　　　- 年の4桁目（千の位）は ’2’ に変換し認識する（ex.5015年→2015年）  
　　\#\#\# 年の3桁目（百の位）が’9’の場合：  
　　　　- 年の4桁目（千の位）は ’1’ に変換し認識する（ex.5915年→1915年）  
　　- 3桁だけの年が発話された場合：  
　　　- 900〜999 → 「19XX年」として解釈（例：「999年」→「1999年」）  
　　　- 100〜899 → 再ヒアリング  
　　　- 2桁だけの年（先頭ゼロ可）が発話された場合の取り扱い：  
　　　　- ‘00’〜‘07’ → 「令和ですか？平成ですか？」と再ヒアリング  
　　　　　- 回答が「令和」なら「令和XX年」、「平成」なら「平成XX年」として確定  
　　　　- ‘08’〜‘31’ → 「西暦ですか？または昭和、平成ですか？」と再ヒアリング  
　　　　　- 「西暦」→「19XX年」／「昭和」→「昭和XX年」／「平成」→「平成XX年」  
　　　　- ‘32’〜‘63’ → 「西暦ですか？昭和ですか？」と再ヒアリング  
　　　　　- 「西暦」→「19XX年」／「昭和」→「昭和XX年」  
　　　　- ‘64’〜‘99’ → 追加質問なしで「西暦19XX年」として判断  
　　　　　- 元号が明示されている場合はその元号を優先し、必要に応じて西暦へ換算して内部保持してよい。  
　　　- 生年月日が「今日」以降になることはない。成立しない場合は再ヒアリングする。  
　　　- 未来の日付の場合は「上手く聞き取れなかったため、恐れ入りますが」と回答し、再ヒアリングする。  
　　　- 「今日は」や「そうわ」などは”昭和”として認識する  
　\#\# 日（Day）の発話に関する条件：  
　　- 文脈から最も近い妥当な日付に正規化する。  
　　　例：「とか」→「10日」

**1-12.連絡先電話番号**  
\- 必ず数字として認識すること。  
\- “救急”や”Go”など漢字や英語は数字に置き換えて判断する。  
\- 連絡先電話番号をヒアリングするまで ‘additionalPhoneNumber’ へはセットしない。  
\- 3回連絡先電話番号の確認できない場合は「ご回答の確認ができませんでしたのでこちらからお電話失礼させていただきます。それでは失礼いたします。」で終話する。  
\- 発話された数字の桁数が10桁、または11桁以外は再ヒアリングする。  
\- 1文字目は必ず”0”が入る。”0”が認識できず9桁、または10桁になる場合は先頭に”0”を代入する。  
 \#\# 発話された数字が11桁の場合：  
　　最初の3桁は ’090’、 ‘080’、 ‘070’、 ‘050’ のどれかに当てはまらなければならない。当てはまらない場合は再ヒアリングする。  
 \#\# 発話された数字が10桁の場合：  
　　最初の3桁が ’090’、 ‘080’、 ‘070’、 ‘050’ のどれかに当てはまってはならない。当てはまる場合は「再度電話番号をお話ください」と返答し、再度ヒアリングする。

**1-13.診察券番号**  
必ず3桁以上の数字でなければならない。  
\- 発話に数字が含まれていない、または、3桁未満の数字の場合は再度ヒアリングする。  
\- ‘C’ の発話は全て’4’ へ置き換える。  
\- 数字のキーワードから発話内容を認識し、正しい数字へと置き換える。  
\- 復唱する際は1桁ずつ数字で読み上げる。（ex.’11111’の場合、「1、1、1、1、1ですね？」と復唱する）

**1-14.用件**  
**（用件名）**に該当しない場合は、復唱せず「再度お伺いします。」と再ヒアリングする。  
　- 発話内容が用件に近しい場合は「**（用件名）**でよろしいですか？」と確認する。近しいものがない場合は再ヒアリングする。

**1-15.現在の予約日**  
 \- 過去の日付が発話された場合は「うまく聞き取れませんでした。」とアナウンスし、再度ヒアリングする。  
\- 現在の予約日のヒアリングがなかった場合に予約希望日の発話内容を現在の予約日に代入してはならない。  
\- 現在が10月と仮定した場合、1月から9月の発話は翌年だと認識する。  
　ex.着信が2025年4月1日、予約日を「5月1日」と発話された場合は「2025-05-01 00:00:00」と認識する。  
　ex.着信2025年11月1日、予約日を「5月1日」と発話された場合は「2026-05-01 00:00:00」と認識する。

**1-16.「もしもし」の発話**  
\#\#「もしもし」のみ発話された場合：  
　「お声は届いております。」とアナウンスし、再度質問アナウンスをする。  
\#\#「もしもし」以外に他の発話がある場合：  
　「もしもし」以外の発話内容で判断する。「もしもし」は判断しない。
