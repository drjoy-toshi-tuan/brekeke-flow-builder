## ② 顧客作成フローイメージ（写真手起こし）

```
入院患者さまですか／外来患者さまですか     （横に）新規の相談か
├─ 入院患者
│   └─ 担当者はわかりますか
│       ├─ 担当者わかる → 患者さまの氏名 → 担当者につなぐ
│       └─ 担当者わからない → 相談室（7177）へ
├─ 外来患者
│   └─ 担当者はわかりますか
│       ├─ 担当者わかる → 担当者につなぐ
│       └─ 担当者わからない → （※接続が写真上曖昧）
└─ 新規の相談か
    └─ 相談予約ですか／受診に関することですか
        ├─ 受診に関することは予約センターへ
        └─ 相談予約であれば 7177 へ
```

- 写真上の曖昧点: 「相談予約ですか／受診に関することですか」ブロックが「新規の相談」の下にあるのか「外来・担当者わからない」の続きなのか、線の接続が読み取れない。
  drawio v1.0 は「外来・特定不可 → F1」と配線し、new_q（新規の相談）は出口未確定のまま「※病院に確認」と注記している。**両解釈が並存**。
- 写真では「患者さまの氏名」は入院側のみ。drawio は外来側（F_id）にも対称に追加している（drawio 側の拡張）。
- 写真には drawio 上流（発信元3分類・ご用件3分類・大代表/予約センター転送）は存在しない（上流は Dr.JOY 提案由来）。

## ③ 提案サマリからのシナリオ関連事実（詳細は 亀田_提案内容サマリ.md）

- 受電 月間約1,700件: 担当者への転送52% / 折返し・かけ直し依頼19% / 伝言預かり→テキスト伝達14% / 定型案内3% / 書類・FAX受領連絡3% / 人対応（専門判断）残り。

  **→ drawio v1.0 には B（伝言・折返し）と C（定型案内）に相当するパスが存在しない**（転送系のみ）。
- 転送方式は A（内線転送・PBX連携）/ B（050外線転送）が未確定（情シス確認待ち）。F3 の「かけ直し or 外線転送？」も同根の未確定。
