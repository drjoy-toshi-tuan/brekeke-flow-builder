# 提案内容サマリ（亀田総合病院 総合相談室 AI電話導入）

原典: `亀田総合病院_AI電話導入提案資料.pdf`（全19ページ, 2026-04-08, Dr.JOY AI電話事業部）

## 1. 現状課題（p.3-4）

- 専門職（MSW・PSW・看護師）が一次受けの「電話番」になり、本来業務（退院調整・患者相談・ケアマネ連携）の時間が削られている。
- 構造的に伝達ミス（聞き間違い・手書きメモ漏れ）、担当者不在時の折返しループが発生。
- **月間約1,700件の受電内訳**:

| カテゴリ | 件数/日 | 比率 | AI対応 |
|----------|--------|------|--------|
| 担当者への転送・通話 | 約42件 | 52% | AI一次対応→転送 |
| 折返し・かけ直し依頼 | 約15件 | 19% | 用件・連絡先を自動取得→通知 |
| 伝言預かり→テキスト伝達 | 約10件 | 14% | 音声認識でテキスト化→通知 |
| 定型案内・番号案内 | 約2件 | 3% | 完全自動対応 |
| 書類・FAX受領連絡 | 約2件 | 3% | テキスト化→通知 |
| チーム内引継ぎ・新規相談等 | 約9件 | — | 人対応（専門判断） |

→ **約89%（約1,455件/月）が省力化見込み**。

## 2. 実現できること（p.5-6）

既存の総合相談室番号はそのまま、受電業務を自動化。Dr.JOY IVR が用件を自動判定し、
- 用件A: 担当者へ転送（院内PBX→内線・PHS）
- 用件B: 伝言・折返し（音声→テキスト変換→Dr.JOYアプリ通知）
- 用件C: 定型案内（自動音声で完結）

## 3. 構成3案（p.7-9）

| 構成 | 概要 | 補足 |
|------|------|------|
| 通常のAI電話構成（p.7） | クラウドIVR + WebRTC(無料Web通話) / 050外線転送(有料) | 標準形 |
| **内線転送型構成（p.8）★** | ONU + VoIPゲートウェイ(Netcommunity OG) + 院内PBX連携。緊急時迂回ルート(待機系)あり | SIP Trunk / PRI・BRI |
| 外線転送型構成（p.9） | 050番号経由で院内外線へ外線転送 | PBX変更不要 |

## 4. 構成決定のための3確認（p.10）

1. **転送の即時性** — 52%の転送は「今すぐ話したい」が必要か（即時 vs 折返しで十分）
2. **院内PBX設定変更** — 情報システム部門への調整は可能か → 可能なら方式①(内線転送)推奨／難しければ方式②(050)で先行導入し後日移行
3. **発信元区分** — 院内外（外部施設 vs 患者・家族）で対応を変える必要があるか

## 5. 方式比較（p.12-13）

| | 方式A 内線転送（PBX連携）★推奨 | 方式B 050外線転送 |
|--|------------------------------|-------------------|
| 追加通話コスト | ゼロ | +約13.2万円/月 |
| 月次追加コスト | +約10万円（設備費差分のみ） | +約13.2万円（通話費） |
| PBX設定変更 | 必要（情シス調整） | 不要（スピード導入可） |
| 将来の代表電話展開 | ◎ | △ |

- 亀田の転送対象は推定 **882件/月**（42件/日×21日）。損益分岐 **667件** を超えるため方式A推奨。
- ⚠️ 全件折返し(B-2①)は月額ゼロに見えても、コールバック工数がMSW人件費換算で最大60万円・常勤2.5名分の"隠れコスト"。

## 6. 料金（p.14・16・18, 税別）

- **AI電話稼働費用（月額・1日平均外来患者数別）**: ～200人 35万 / 201-300人 40万 / 301-400人 45万 / 401-700人 50万 / 701-1,000人 55万 / 1,001人～ 60万
- **初期費用**: シナリオ初期設定・AI学習・院内説明会・患者周知支援 計100万 / シナリオ追加 25万 + 維持 3万/本
- **従量**: AI稼働 20円/分、SMS 15円/通(70字)、転送 固定30円/分・携帯40円/分
- p.16「事例公開前提プラン」は各費用を割引（例: 月額が半額水準、追加シナリオ0円 等）
- **今回 特別プラン（初号案件）（p.18）**: 初期 ¥1,000,000 / 月額基本料 **¥250,000〜** / 内線転送対応 **含む** / 初号案件専任対応。※初号案件導入決定が条件、月額は構成・転送件数で変動。

## 7. ロードマップ（p.17）

- **PHASE 1（今回）**: 総合相談室 — 月1,700件自動化、AI電話基盤を院内に構築
- **PHASE 2（半年〜1年後）**: 他部門横展開（医療連携室・地域連携室など）— 同一基盤で追加コスト最小
- **PHASE 3（将来）**: 病院代表電話 — 内線転送基盤で全院展開、年間637,000人規模
