# IVR フロー layout 座標の自動計算仕様

> 実フロー（HCクリニック厚木）の座標分析に基づく配置ルール
> Claude Code の generator が JSON 出力時に適用する

---

## 1. 座標系の基本

- **x軸**: 右が正。分岐パスごとに横にずれる
- **y軸**: 下が正。フローの進行方向
- **単位**: Brekeke フローデザイナーのグリッド単位（概ね10px刻み）

---

## 3. 会話ステップ内の相対配置

1つの会話ステップ（TTS→STT→OpenAI→Retry→Save）の内部配置。
**TTSの座標を基準(0,0)** として、各モジュールの相対位置:

| モジュール | Δx | Δy | 備考 |
|---|---|---|---|
| **TTS** | 0 | 0 | 基準点 |
| **STT** | 0 | +110 | 直下 |
| **OpenAI** | 0 | +230 | STTの下 |
| **Retry** | -280 | +230 | OpenAIの左隣 |
| **Save** | -280 | +110 | Retryの上（STTと同じ高さ） |

### 追加Save（sub用）がある場合
| モジュール | Δx | Δy |
|---|---|---|
| 追加Save | +400 | +230 | OpenAIの右隣 |

---

## 4. ステップ間の垂直間隔

### メインパス（予約フロー: 用件→受診歴→...→終話）

連続するステップのTTS間の垂直間隔（Δy）:

| 間隔タイプ | Δy | 適用場面 |
|---|---|---|
| **標準** | 400 | 通常のステップ間 |
| **広め** | 500 | 複雑な分岐を持つステップの後（電話番号→その他問合せ等） |
| **狭め** | 350 | シンプルなステップ間 |

**推奨デフォルト: Δy = 400**

### 開始点
- メインパスの最初のステップ（用件）: **(x=20, y=180)**

---

## 5. 分岐パスの横方向配置

用件確認の結果で分岐するパスは、横にオフセットして配置する。

| パス | x基準 | 備考 |
|---|---|---|
| **予約パス** | x = -550〜20 | メインの左側 |
| **変更パス** | x = 920 | 中央右 |
| **キャンセルパス** | x = 1750 | 右 |
| **問い合わせパス** | x = 2390 | さらに右 |
| **遅刻パス** | x = 2810〜4830 | 最も右 |
| **雇用時健診パス** | x = -800 | 最も左 |

### 分岐パスの開始y座標
- 全ての分岐パスは、用件確認ステップの結果分岐なので **y = 500〜650** あたりから開始
- その後は各パス内でΔy = 400ずつ下に進む

## 8. 自動計算アルゴリズム（擬似コード）

```python
def calculate_layouts(steps, branches):
    """
    steps: メインパスのステップ定義リスト
    branches: 分岐パス定義辞書
    """
    
    # 1. 開始チェーン（固定座標をそのまま使う）
    assign_fixed_start_chain()
    
    # 2. メインパスの最初のステップ
    current_y = 180  # 用件の開始位置
    main_x = 20
    
    for step in main_path_steps:
        assign_step_chain(step, x=main_x, y=current_y)
        current_y += 400  # 標準間隔
    
    # 3. 分岐パス
    branch_x_offsets = {
        "予約": -550,
        "変更": 920,
        "キャンセル": 1750,
        "問合せ": 2390,
        "遅刻": 2810,
        "雇用時健診": -800,
    }
    
    for branch_name, branch_steps in branches.items():
        bx = branch_x_offsets[branch_name]
        by = 550  # 分岐開始y
        for step in branch_steps:
            assign_step_chain(step, x=bx, y=by)
            by += 400
    
    # 4. 共通後半ステップ（固定座標）
    assign_common_tail_steps()
    
    # 5. 終話群（階段配置）
    assign_ending_steps()


def assign_step_chain(step, x, y):
    """1つの会話ステップの5モジュールを配置"""
    step.tts.layout     = {"x": x,       "y": y}
    step.stt.layout     = {"x": x,       "y": y + 110}
    step.openai.layout  = {"x": x,       "y": y + 230}
    step.retry.layout   = {"x": x - 280, "y": y + 230}
    step.save.layout    = {"x": x - 280, "y": y + 110}
```

---

## 9. 注意事項

- **座標はフローデザイナーの表示にのみ影響** — IVR動作には一切影響しない
- 実フローでは手動調整が入っているため完全な規則性はないが、上記ルールで十分実用的な配置になる
- 複雑な案件（分岐が多い）では `branch_x_offsets` の間隔を広げる
- フローデザイナーで後から手動調整するのは容易なので、完璧でなくても構わない
