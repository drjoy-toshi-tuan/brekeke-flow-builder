## 4. シナリオフロー図

```
冒頭(wait)
  → コンテキスト設定(saveContextModel2DB)
  → 着信電話番号分類(incoming-classifier)
      ├─ 非通知 → 非通知_アナウンス(TTS) → 完了フラグ_非通知 → 切断
      ├─ 海外   → （必要に応じて）
      └─ 通常   → 営業時間チェック(acceptance_times)
                    ├─ 時間外 → 時間外_アナウンス(TTS) → 完了フラグ_時間外 → 切断
                    └─ 受付可 → 冒頭_アナウンス(TTS)
                                  └─ {ここから本フローの聴取ステップ}
                                        └─ 終話(TTS) → 完了フラグ → 切断
```

> 営業時間チェックが不要な場合は省略し、incoming-classifierから直接 冒頭_アナウンス へ遷移。

## 5. コンテキストフィールド一覧（saveContextModel2DB 用）

Dr.JOY 画面に表示される項目の定義。`contextDisplayType` と `itemDefault` の値はモジュール詳細設定ガイドのセクション6を参照。

| contextName | contextNameJp | contextDisplayType | rangeValues（CLASSIFICATION/DEPARTMENT/STATUSの場合） | itemDefault | 備考 |
|---|---|---|---|---|---|
| classification | 区分 | CLASSIFICATION | {例: 初診, 再診, その他} | true | 用件区分。フロー固有の値を設定 |
| patientName | 氏名 | TEXT | — | true | |
| medicalCardNumber | 診察券番号 | NUMBER | — | true | 不要なら削除 |
| patientDateOfBirth | 生年月日 | DATE_OF_BIRTH | — | true | 不要なら削除 |
| telephoneNumber | 電話番号（着信） | PHONE_NUMBER_CALL | — | true | 着信番号自動格納 |
| additionalPhoneNumber | 連絡先電話番号 | PHONE_NUMBER | — | true | 聴取する電話番号 |
| status | 状態 | STATUS | 未処理, 代表案内, 転送, 時間外 | true | 1=未処理 2=代表案内 3=転送 6=時間外 |
| dateOfCall | 受電日時 | DATE | — | true | |
| callId | 通話ID | TEXT | — | true | |
| {施設固有フィールド} | {表示名} | {TYPE} | {選択肢} | false | 施設固有のフィールドはitemDefault=false |

> **itemDefault について**:
> - `true` = Dr.JOY 標準フィールド（削除不可）
> - `false` = このフローで追加したフィールド（設計書に必ず記載すること）
